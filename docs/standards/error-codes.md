# 错误码体系

## 分段规则

| 区间 | 含义 |
|------|------|
| `0` | 成功 |
| `1xxxx` | 系统错误 |
| `2xxxx` | 认证授权 |
| `3xxxx` | 业务错误 |
| `4xxxx` | 参数错误 |
| `5xxxx` | 外部依赖（MinIO、存储等） |

## 已知治理问题

- 认证错误码常量为 `20001`–`20004`，但当前运行时异常实际返回 `40101`、`40102`、`40301`、`40302`。
- `40020` 同时用于原密码错误与品牌排序校验；`40021` 同时用于新密码策略与类目排序校验。
- `40000`、`40001` 当前承载多种字段级校验文案，后续如前端需要精确分支处理，建议拆分领域错误码。
- `30043` 当前运行时 HTTP 状态为 `400`，历史文档曾登记为 `409`；以当前代码为准。

## 实现

- Python 常量：`src/backend/app/core/error_codes.py`
- 异常：`src/backend/app/core/exceptions.py`

## AI 规则

新增业务错误 MUST 在本表与 `error_codes.py` 同步登记，禁止随意使用未文档化数字。
