## Why

`BUG-0071-login-page-theme-language-selector-misalignment` 已评审通过。Web 管理端登录页右上角同时展示主题选择模块与语言选择模块，但两个控件分别由不同布局容器控制：语言选择模块位于 `.form-panel` 面板右上角，主题选择模块位于 `.login-card` 内部顶部区域。两个控件不共享同一个工具区布局、右侧边界和响应式规则，导致登录页首屏出现可见错位。

现行 `web-client` 规格要求登录页展示语言切换占位，也要求登录页视觉对齐原型，但尚未约束新增主题选择模块与语言选择模块之间的统一布局关系。需要通过修复 Change 明确登录页辅助工具区对齐要求，并要求回归主题切换、语言按钮可访问性和登录流程。

## What Changes

- 修改 Web 管理端登录页规范：主题选择模块与语言选择模块必须进入统一的登录页工具区布局或共享等价对齐规则。
- 要求两个控件在桌面与窄屏视口下保持稳定的垂直对齐、控件高度、视觉重心、交互热区和间距。
- 要求修复不得遮挡登录标题、表单字段、安全提示或品牌面板内容。
- 要求回归主题切换、语言按钮可访问名称、登录表单提交、记住登录状态和登录后跳转。
- 明确本修复不改变认证 API、后端权限、用户模型、数据库结构或完整 i18n 能力。

## Capabilities

### New Capabilities

无。

### Modified Capabilities

- `web-client`: 修正管理端登录页辅助工具区布局规范，补充主题选择与语言选择的对齐和回归要求。

## Impact

- **web/admin:** 预计修改 Web 管理端登录页组件结构或登录页 CSS，使主题选择模块与语言选择模块共享工具区布局或等价对齐规则。
- **api:** 不涉及 API 契约变化；若实现阶段发现必须修改响应字段，必须补充 OpenAPI、Orval、接口文档和测试。
- **database:** 不涉及表结构或迁移。
- **miniapp:** 不涉及。
- **tests:** 需要补充或更新 Web 前端测试，覆盖主题选择与语言选择共存、语言按钮可访问性、主题切换不回退、登录流程不回退；建议补充登录页桌面/窄屏视觉证据。

## Rollback Plan

如修复后登录页工具区遮挡表单、主题切换不可用或登录流程出现严重回归，可回滚登录页组件结构与 CSS 到修复前版本，同时保留本 Change 文档并在验收记录中说明回滚原因。回滚后仍需重新评估 `BUG-0071` 的对齐问题，不得直接关闭该 BUG。
