## Context

BUG-0071 指出 Web 管理端登录页右上角工具区存在视觉对齐偏差：

- 语言选择模块由登录表单面板级样式定位在右上角。
- 主题选择模块由登录卡片内部行布局控制。
- 两个控件同属登录页辅助工具，却使用不同布局容器与坐标系。

该问题属于 design / ux 类缺陷。它不阻断登录、主题切换或语言按钮点击，但影响管理端入口页首屏一致性。

## Goals

- 登录页主题选择模块与语言选择模块使用统一工具区布局或等价对齐规则。
- 桌面与窄屏视口下，两个控件保持稳定对齐，不重叠、不裁切、不遮挡登录内容。
- 主题切换能力、语言按钮可访问名称和登录流程保持不回退。
- 修复范围限制在 Web 管理端登录页工具区。

## Non-Goals

- 不实现新的多语言切换能力。
- 不新增主题模式。
- 不改变后端认证 API、权限策略、用户模型或数据库结构。
- 不改变登录页品牌面板、登录表单字段、密码策略或登录后路由目标。

## Root Cause

直接原因是两个控件处在不同布局层级：

- `.language` 使用 `.form-panel` 级绝对定位。
- `.login-theme-row` 位于 `.login-card` 内部，受卡片宽度和居中布局影响。

根本原因是新增主题选择模块时，没有把登录页右上角辅助工具抽象为统一布局区域；现有测试也只验证控件是否存在，没有覆盖两者的布局关系。

## Proposed Design

1. 统一工具区布局：
   - 将主题选择模块与语言选择模块放入同一登录页工具区容器，或通过 CSS 保证两者共享同一右侧边界、垂直对齐规则和响应式策略。
   - 工具区应位于登录页右栏可见区域，不遮挡标题、表单、安全提示或品牌面板。
2. 保留现有视觉体系：
   - 登录页继续使用 CSS Port 策略与 `features/auth/styles/login-page.css`。
   - 语言按钮继续保持 `.language` 原型形态与文案「简体中文⌄」。
   - 主题选择器继续使用现有主题能力，不新增主题枚举。
3. 响应式处理：
   - 桌面端右栏工具区应保持水平排列或等价专业布局。
   - 窄屏下工具区可以换行或调整排列，但不得重叠、裁切、遮挡表单或引起异常页面滚动。
4. 回归边界：
   - 登录接口、认证状态、记住登录状态、主题持久化、登录成功跳转和语言按钮可访问名称必须保持不回退。

## Testing

- Web 单元测试 / Testing Library 覆盖登录页同时渲染主题选择模块与语言选择按钮。
- Web 单元测试覆盖语言按钮仍可通过 `aria-label="切换语言"` 定位。
- Web 单元测试覆盖主题选择控件仍可通过 `aria-label="主题"` 定位，并可触发主题模式变更。
- 登录表单既有测试继续覆盖必填校验、登录提交、记住登录状态和登录成功跳转。
- 如有本地浏览器验收条件，补充桌面与窄屏截图，证明两个控件对齐且不遮挡登录内容。

## Risks

- 将控件合并到同一工具区时，可能改变原型语言按钮位置或登录卡片顶部间距。
- 窄屏工具区若宽度不足，可能出现换行、遮挡登录标题或挤压表单。
- 主题选择器宽度随主题文案变化，可能影响语言按钮间距。

缓解方式：以 AC-BUG-001 至 AC-BUG-009 为回归标准，并在实现阶段补充桌面/窄屏布局证据。
