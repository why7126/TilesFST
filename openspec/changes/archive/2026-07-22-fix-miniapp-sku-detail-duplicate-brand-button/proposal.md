## Why

`BUG-0070-miniapp-sku-detail-duplicate-brand-button` 已评审通过。微信小程序 SKU 商品详情页内容区已经提供“查看品牌主页”入口，但底部固定操作栏仍显示品牌按钮，两个入口语义和跳转目标重复，造成底部操作区冗余。

现行 `miniapp-sku-detail-page` 规格仍要求“底部固定操作栏包含收藏、品牌和分享”，与已交付页面中的内容区品牌卡入口职责冲突。需要通过修复 Change 收敛品牌主页入口：保留内容区品牌卡 / “查看品牌主页”入口，删除底部品牌按钮。

## What Changes

- 修改 SKU 商品详情页品牌入口规范：品牌主页入口由内容区品牌卡承载，不再要求底部品牌按钮。
- 修改底部固定操作栏规范：底部操作栏保留收藏和分享，不再显示品牌按钮。
- 要求删除底部品牌按钮后，底部操作栏无空白占位、错位、异常间距或残留点击热区。
- 要求回归验证内容区“查看品牌主页”仍能跳转到正确品牌主页。
- 明确本修复不新增或调整 API 字段，不改变数据库结构，不新增交易类能力。

## Capabilities

### New Capabilities

无。

### Modified Capabilities

- `miniapp-sku-detail-page`: 修正 SKU 商品详情页品牌入口与底部操作栏规范，移除底部重复品牌按钮。

## Impact

- **miniapp:** 预计修改 SKU 商品详情页底部操作栏 WXML / TS / JS / WXSS，删除品牌按钮渲染与点击绑定，保留内容区品牌入口。
- **api:** 不涉及 API 契约变化；若实现阶段发现必须修改响应字段，必须补充 OpenAPI、Orval、接口文档和测试。
- **database:** 不涉及表结构或迁移。
- **web/admin:** 不涉及。
- **tests:** 需要补充或更新小程序静态测试，覆盖底部品牌按钮不存在、内容区品牌入口存在、底部操作区布局无残留。

## Rollback Plan

如修复后品牌主页入口不可达或底部操作栏出现严重回归，可回滚小程序 SKU 详情页底部操作栏相关代码到修复前版本，同时保留本 Change 文档并在验收记录中说明回滚原因。回滚后仍需重新评估品牌入口重复问题，不得直接关闭 `BUG-0070`。
