## Context

BUG-0070 指出 SKU 商品详情页存在两个品牌主页入口：

- 内容区品牌卡 / “查看品牌主页”入口。
- 底部固定操作栏品牌按钮。

两个入口职责相同，用户反馈要求删除底部品牌按钮。该问题属于 design / ux 类缺陷，根因是内容区品牌入口落地后，底部操作栏没有同步裁撤同类入口。

## Goals

- SKU 商品详情页只保留一个清晰的品牌主页入口。
- 内容区品牌卡 / “查看品牌主页”继续负责品牌主页跳转。
- 底部固定操作栏不再显示品牌按钮。
- 删除按钮后底部操作栏布局稳定，无空白、错位、异常间距或残留点击热区。

## Non-Goals

- 不新增品牌主页能力。
- 不修改品牌主页展示内容。
- 不新增或修改 SKU 详情 API 字段。
- 不新增购物车、购买、下单、支付、库存、优惠券、促销倒计时或询价承诺等 `REQ-0044` 范围外能力。

## Root Cause

直接原因是 SKU 商品详情页在内容区已有品牌主页入口后，底部固定操作栏仍渲染品牌按钮。

根本原因是规格和实现没有把“品牌主页入口唯一性”作为约束：规格一方面要求品牌卡承载品牌入口，另一方面仍要求底部操作栏包含品牌按钮，导致后续实现保留了重复入口。

## Proposed Design

1. 规格层收敛入口职责：
   - 品牌入口由内容区品牌卡 / “查看品牌主页”承载。
   - 底部固定操作栏不再承担品牌主页入口职责。
2. 实现层删除底部品牌按钮：
   - 删除底部操作栏品牌按钮节点。
   - 删除仅服务于底部品牌按钮的点击处理、埋点或样式。
   - 保留内容区品牌入口跳转逻辑和 fallback 逻辑。
3. 布局回归：
   - 底部操作栏删除按钮后重新分配剩余入口宽度或布局。
   - 确认安全区避让、44x44px 点击目标和主按钮层级不回退。

## Testing

- 小程序静态测试覆盖 SKU 详情页 WXML 中底部操作栏不再包含品牌按钮或底部品牌跳转绑定。
- 小程序静态测试覆盖内容区品牌卡 / “查看品牌主页”入口仍存在。
- 如有可运行小程序验收，补充微信开发者工具或真机截图：内容区入口存在，底部操作区无品牌按钮、无空白和错位。
- 若 API 未变化，明确不运行 Orval；若实现触及 API 契约，必须同步 OpenAPI / Orval / docs / tests。

## Risks

- 删除底部按钮后，底部操作栏布局可能出现空白或点击热区残留。
- 若内容区品牌入口 fallback 覆盖不足，删除底部按钮可能让部分 SKU 的品牌主页入口不可用。

缓解方式：以 AC-BUG-001 至 AC-BUG-008 为回归标准，优先验证内容区入口和底部操作栏布局。
