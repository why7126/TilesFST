## Why

菲尚特当前已有后台维护品牌、SKU、规格、类目与 Banner 的能力，但缺少面向终端客户的微信小程序首页。需要把后台已有商品资料转化为微信内的数字展厅入口，形成首页、搜索、商品详情、门店信息、分享、咨询和热销行为统计的首期闭环。

## What Changes

- 新增原生微信小程序首页能力，包含品牌导航、搜索入口、Banner、快捷找砖入口、新品推荐、热销推荐、品牌服务区和底部 TabBar。
- 新增小程序端门店信息页、搜索页、商品详情页的首期闭环要求。
- 新增首页/商品详情分享、门店咨询入口和热销行为统计要求。
- 首页数据复用现有店主端/后台公开数据源；必要时新增小程序首页聚合接口和行为事件接口。
- 明确不实现收藏、预约表单、到店询价规则、快捷入口后台配置、服务入口后台配置、复杂用户画像和收藏驱动热销算法。
- 小程序端只能访问后端授权接口和后端返回的安全图片 URL，不得直连未授权对象存储。

## Capabilities

### New Capabilities

- `miniapp-home`: 微信小程序首页、门店信息、搜索、商品详情、分享、咨询、热销行为统计和首期 UI/数据/安全验收。

### Modified Capabilities

- `product-usage-logging`: 扩展使用行为事件字典，支持小程序商品详情访问、首页/商品分享、首页/商品咨询等事件，用于热销推荐统计。

## Impact

- **Backend/API:** 可能新增 `GET /api/miniapp/home` 或等价聚合接口，以及行为事件接收/统计查询能力；所有 API contract 变化必须同步 OpenAPI、Orval、docs 和测试。
- **Miniapp:** 新增 `src/miniapp/` 下原生微信小程序首页与相关页面；不使用 Web 浏览器专属 API。
- **Database:** 若现有 usage events 不足以支撑热销统计，可能新增或扩展统计事件/聚合字段；必须同步 SQLite/MySQL schema、数据库文档和测试。
- **Storage/media:** 复用后端授权图片 URL 与对象存储适配层；小程序不得直连未授权 MinIO。
- **Admin/Web:** 不新增快捷入口/服务入口后台配置，不修改店主 Web 展示端；Banner 管理仅作为数据来源复用。
- **Testing:** 覆盖小程序首页核心流程、分享、咨询、热销事件、API 失败态、320-430pt 移动宽度和 375x812 / 390x844 视觉验收。
