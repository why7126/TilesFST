## Why

微信小程序已经具备首页、分类列表页、搜索组件和 SKU 详情页，但分类、搜索、品牌和推荐入口进入商品浏览时缺少统一的商品列表页能力。需要将 `REQ-0047` 的商品列表页通用组件正式纳入 OpenSpec，作为 sprint-008 后续实现、追溯和验收的事实源。

## What Changes

- 新增微信小程序商品列表页能力，覆盖商品列表容器、商品卡片、分类/搜索/品牌/推荐入口上下文、筛选排序、分页加载、空/错/无更多状态和商品列表埋点。
- 明确分类页二级分类跳转 `pages/product-list/index?categoryId={id}` 后由商品列表页承接，不在分类页展示商品卡、价格、筛选或排序。
- 明确搜索结果中的 SKU 列表复用商品列表容器与商品卡片，并保留关键词、筛选和排序上下文。
- 商品卡片整卡进入 SKU 详情页，复用既有 SKU 详情页公开数据与安全媒体 URL 边界。
- 保留小程序单端范围，明确不包含 Web 管理端、店主 Web、后台商品管理列表、商品数据模型新增、购物车、询价、在线下单、收藏、瀑布流增强、3D 或 AR 能力。

## Capabilities

### New Capabilities

- `miniapp-product-list-page`: 微信小程序商品列表页通用容器、商品卡片、入口上下文、筛选排序、分页加载、状态治理、埋点和原型验收。

### Modified Capabilities

- `miniapp-category-list-page`: 分类页二级分类跳转的目标页面由商品列表页正式承接，分类页仍不展示商品列表。
- `miniapp-sku-detail-page`: 商品列表卡片进入 SKU 详情页，沿用 SKU 详情页公开数据、安全媒体 URL 和不可公开状态边界。
- `product-usage-logging`: 扩展小程序商品列表页浏览、商品曝光/点击、筛选、排序、刷新、加载更多和失败事件。

## Impact

- **Backend/API:** 可能新增或调整小程序公开商品列表接口，覆盖分类、关键词、品牌、规格、价格区间、排序、分页、公开状态过滤和 facets；如新增或调整 API，必须同步 OpenAPI、Orval、docs 和测试。
- **Miniapp:** 新增 `pages/product-list/index` 或等价商品列表页，沉淀商品列表容器、商品卡片、筛选抽屉、排序、分页和状态治理。
- **Database:** 默认复用现有 SKU、品牌、分类、规格、媒体和 usage event 数据结构；若实现发现缺字段，必须在 design/tasks 中明确同步 SQLite/MySQL schema、迁移、数据库文档和测试。
- **Web/Admin:** 不影响；不得新增管理端商品列表组件、后台商品管理列表改造或店主 Web 商品列表。
- **Storage:** 不涉及上传或对象存储写入；商品图片仅使用后端公开安全 URL，不拼接原始 object key。
- **Testing:** 需要覆盖小程序商品列表流程、公开字段过滤、分页/筛选/排序、状态降级、埋点脱敏和原型视觉验收。
