## 背景

`POST /api/v1/usage-events` 采用事件字典白名单校验：后端依据 `event_name` 查找定义，检查必填字段和禁止字段，合法后写入 `usage_events`。该安全边界是正确的，但微信小程序页面和组件持续新增埋点时，没有建立与后端字典同步的测试门禁，导致合法小程序业务事件被后端视为未知事件。

## Root Cause

| 层面 | 说明 |
|---|---|
| 直接原因 | 小程序发送 `favorite_list_*`、`brand_detail_*`、`product_card_*`、`brand_card_*` 等事件，但后端 `EVENT_DEFINITIONS` 未覆盖全部合法事件 |
| 根本原因 | 小程序 `track()` 事件与后端事件字典之间缺少自动化一致性测试和新增事件流程约束 |
| 触发条件 | 用户进入收藏页、品牌详情页、SKU 详情页品牌入口、通用商品/品牌卡片组件等会触发对应事件 |
| 安全边界 | 未知事件和禁止字段仍必须拒绝；不能通过接受任意 event_name 来消除 400 |

## 修复方案

1. **事件字典补齐**
   - 梳理 `src/miniapp/**` 当前所有 `track()` 字面量事件。
   - 对动态事件名调用点建立明确样例，例如品牌详情页 `brand_products_load_more`。
   - 为收藏页、品牌详情页、商品卡片和品牌卡片事件补齐后端 `EVENT_DEFINITIONS`。
   - 每个事件定义必须包含 `category`、`required` 和 `forbidden`。

2. **字段契约统一**
   - 小程序 payload 必须携带后端 required 字段。
   - 对存在命名漂移的字段进行收敛，例如 `skuId` / `sku_id`、`requestId` / `request_id`、`errorCode` / `error_code` 等，按事件定义保持一致。
   - 事件 metadata 只保留业务分析必要字段，不写入手机号、客户姓名、Authorization header、Cookie、raw payload、raw object key、内部备注或 `.env` 内容。

3. **防漂移测试**
   - 增加后端 pytest，覆盖本 BUG 中列出的全部事件。
   - 增加一个静态或单元测试，提取小程序 `track()` 字面量事件并确认后端字典存在对应定义。
   - 对动态事件名调用点维护样例列表，纳入测试。
   - 保留负向测试：未知事件仍返回 400，禁止字段仍返回 400。

4. **可观测性与回归**
   - 合法事件写入 `usage_events` 后，应包含 `client_type=wechat_miniapp`、合理 `event_category`、`page_path` 和摘要。
   - 埋点失败仍不阻断小程序主流程。
   - 微信开发者工具 macOS / mp 2.01.2510290 / lib 3.16.2 环境下关键页面不再大量出现 usage-events 400。

## 方案取舍

| 方案 | 结论 | 原因 |
|---|---|---|
| 后端接受任意小程序 event_name | 不采用 | 会破坏事件字典治理，导致统计口径不可控，并可能写入敏感字段 |
| 前端静默停用未知事件上报 | 不采用为最终方案 | 可临时降噪，但会继续丢失统计，无法修复契约漂移 |
| 补齐事件字典 + 测试防漂移 | 采用 | 保留安全边界，同时恢复合法事件写入和可维护性 |

## 测试策略

- 后端 pytest：
  - 合法小程序事件 payload 返回 200。
  - `usage_events` 写入 `client_type=wechat_miniapp`。
  - 未知事件返回 400。
  - 禁止字段返回 400。
- 静态一致性测试：
  - 提取小程序 `track('literal_event')`。
  - 与后端事件字典比对。
  - 对动态事件名使用手工维护的样例列表。
- 小程序验证：
  - 收藏页、品牌详情页、SKU 详情页品牌入口、商品/品牌卡片组件关键交互不再刷 `POST /api/v1/usage-events 400`。

## 风险与缓解

| 风险 | 缓解 |
|---|---|
| 新增事件过多导致日志量上升 | 只接受已定义合法事件，并保留后续限流/采样治理空间 |
| 字段命名调整影响统计查询 | 在测试中固定 required 字段和摘要生成关键字段 |
| 放宽 forbidden 字段导致敏感数据入库 | 负向测试覆盖禁止字段，修复不得删除全局敏感字段集合 |
