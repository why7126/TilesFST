## Why

`REQ-0041` 已交付微信小程序首页首期能力，但新的首页原型要求把首页从“可用闭环”升级为深色品牌展厅体验，并增加全部产品瀑布流，提升客户连续浏览和销售展示效率。

`REQ-0043` 已评审并纳入 `sprint-008`，需要用 OpenSpec 明确首页视觉、信息架构、TabBar 安全降级和埋点边界，避免实现阶段把收藏、证书、品牌馆等后续能力误纳入本期。

## What Changes

- 更新小程序首页首屏规范：使用深色品牌视觉、品牌 Header、搜索入口、Banner、四入口快捷导航、新品推荐、热销推荐和全部产品瀑布流。
- 更新首页快捷入口：由按空间/规格/风格/颜色等筛选入口调整为“选瓷砖、品牌馆、新品榜、热销榜”，未实现目标必须安全降级。
- 增加全部产品瀑布流规范：支持分页加载、触底追加、去重、无更多、失败重试和模块级降级。
- 增加底部 TabBar 目标文案和安全降级规范：首页、分类、找砖、收藏、证书；未完成收藏/证书页不得白屏。
- 更新首页原型与范围控制：允许收藏视觉反馈或建设中提示，但禁止收藏持久化、证书聚合页、品牌馆独立页和榜单独立页进入本期。
- 更新产品使用行为事件规范：补充首页搜索、快捷入口、推荐商品、瀑布流加载/点击/失败、收藏视觉点击和证书 Tab 点击事件边界。
- 不写 `src/` 实现代码；实现阶段由 `/opsx-apply update-miniapp-home-style-optimization` 执行。

## Capabilities

### New Capabilities

无。

### Modified Capabilities

- `miniapp-home`：更新小程序首页首屏、Banner/快捷入口、原型范围控制，并新增全部产品瀑布流与 TabBar 安全降级要求。
- `product-usage-logging`：扩展小程序首页交互事件字典和敏感信息过滤要求。

## Impact

- **Miniapp:** 修改首页页面结构、样式、推荐卡片、全部产品瀑布流、底部 TabBar 文案和安全降级交互。
- **API:** 优先复用现有 `/api/v1/miniapp/products` 与首页/Banner 数据；若缺少 `has_more`，实现阶段可基于 `total/page/page_size` 推导或扩展响应字段，并同步 OpenAPI/Orval/docs/tests。
- **Backend:** 仅在现有公开商品分页或 usage event 字典不足时需要小范围扩展。
- **Database:** 默认不新增；若新增 usage event 名称无需结构变更。
- **Storage:** 不改变 MinIO 策略；图片仍使用后端受控 `/media/` 地址，不直连未授权对象存储。
- **Web/Admin:** 不影响运行时代码。
- **Tests:** 需要补充小程序静态/状态机测试、后端事件字典测试和必要的 API 分页/公开字段测试。
