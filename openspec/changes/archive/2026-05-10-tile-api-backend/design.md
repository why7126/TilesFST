## Context

本项目是瓷砖信息管理平台的 Python FastAPI 后端服务，为小程序前端、Web前台、Web后台提供统一的 REST API 接口。

技术约束：
- 使用 FastAPI 框架
- 使用 SQLite 存储业务数据
- 连接已部署的 MinIO 服务存储图片和视频
- 所有配置通过环境变量注入

## Goals / Non-Goals

**Goals:**
- 提供产品、类目、品牌、员工、文件、统计等模块的 API
- 实现员工登录认证和基于角色的权限控制
- 实现图片/视频上传和存储到 MinIO
- 实现产品全文搜索
- 实现数据统计

**Non-Goals:**
- 不部署 MinIO 服务
- 不处理订单、支付等交易流程

## Decisions

### 1. 架构设计：分层架构

```
┌─────────────────────────────────────────┐
│           API 层 (routers/)             │
│   FastAPI 路由、请求解析、响应格式化      │
├─────────────────────────────────────────┤
│          Service 层 (services/)         │
│       业务逻辑、规则校验、事务控制         │
├─────────────────────────────────────────┤
│        Repository 层 (repositories/)     │
│         SQLite 数据访问封装              │
├─────────────────────────────────────────┤
│         Storage 层 (storage/)            │
│         MinIO 对象存储封装               │
└─────────────────────────────────────────┘
```

### 2. 数据库设计

使用 SQLite，主要表结构：

- products: 商品表
- categories: 类目表
- brands: 品牌表
- employees: 员工表
- statistics: 统计表

### 3. 认证方案

**决策：JWT Token + 角色权限**

- 登录成功后返回 JWT Token
- Token 有效期 24 小时
- 基于角色（管理员/普通编辑）控制 API 访问权限

### 4. 文件存储方案

**决策：后端接收文件 → 转存 MinIO → 返回访问 URL**

- 允许的图片格式：jpg、jpeg、png、webp
- 允许的视频格式：mp4、mov、avi
- 单个文件大小限制：图片 10MB，视频 100MB
- 对象 key 规则：prefix + uuid + 后缀

### 5. 搜索方案

**决策：MySQL 全文索引 + Redis 缓存**

- 产品名称、型号、描述建立全文索引
- 搜索结果缓存到 Redis（5分钟过期）

## Risks / Trade-offs

| 风险 | 描述 | 缓解方案 |
|-----|------|----------|
| SQLite 并发 | 并发写入可能产生锁 | 使用 WAL 模式，控制事务粒度 |
| 大文件上传 | 上传大文件可能超时 | 调整超时配置，分片上传 |
| MinIO 连接 | MinIO 服务不可用 | 实现重试机制，记录日志 |
