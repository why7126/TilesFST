## Why

`BUG-0077-miniapp-category-secondary-name-truncated` 已评审通过。微信小程序分类页中，二级分类名称超过 4 个字时会被截断为 `...`，用户无法直接识别完整分类含义，影响分类浏览和进入对应商品列表的判断。

该缺陷关联已归档需求 `REQ-0045-category-list-page`。当前正式 spec 仍要求二级分类“仅展示单行名称”且“名称超长时单行省略”，与本次缺陷评审后的用户体验目标冲突，因此需要通过 fix Change 修正规范与实现验收边界。

## What Changes

- 修改 `miniapp-category-list-page` 中二级分类宫格的长名称展示要求：二级分类名称超过 4 个字时不得仅以 `...` 省略导致不可辨识。
- 要求 4 字以内、5-8 字、超过 8 字二级分类名称均有可验收展示策略。
- 要求长名称展示不遮挡相邻分类、商品列表、导航栏或页面操作区。
- 保持二级分类点击进入商品列表、点击热区、防重复跳转、分类 ID 和选中态正确。
- 明确本修复默认仅调整小程序分类页前端展示样式，不修改分类 API 字段、不修改数据库结构、不新增管理端或交易类能力。

## Capabilities

### New Capabilities

无。

### Modified Capabilities

- `miniapp-category-list-page`: 修正二级分类长名称展示契约，确保分类页长名称可辨识且布局稳定。

## Impact

- **miniapp:** 需要调整 `pages/category/index` 相关二级分类卡片文本展示样式或布局策略，并回归微信开发者工具和真机视口。
- **api:** 默认不涉及 API 变更；继续复用公开分类树数据中的 `id`、`name`、`sort`。若实现阶段修改字段或响应结构，必须同步 OpenAPI、Orval、接口文档和测试。
- **database:** 不涉及表结构、迁移或数据修复。
- **web/admin:** 不涉及管理端或 Web 展示端。
- **tests:** 需要补充或更新小程序静态测试，覆盖二级分类长名称样式策略、点击入口和页面边界回归；如能自动化断言样式类或快照，应覆盖 4 字以内、5-8 字、超过 8 字名称。

## Rollback Plan

如修复后造成分类页布局重叠、点击热区错乱、商品列表入口失效或多端显示回归，可回滚小程序分类页二级分类卡片样式到修复前版本，并保留本 Change 文档记录回滚原因。回滚后 `BUG-0077` 不得关闭，需重新评估长名称展示策略，例如改为双行展示、动态字号、弹层/tooltip 或其他小程序可用方案。
