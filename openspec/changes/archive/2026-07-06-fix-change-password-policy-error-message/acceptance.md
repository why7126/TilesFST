# Acceptance

## AC-001 具体策略失败提示

- [ ] 长度不足时显示“至少需要 N 位字符”或等价文案，N 与 effective 策略一致。
- [ ] 缺少大写字母时显示“需要包含大写字母”或等价文案。
- [ ] 缺少小写字母时显示“需要包含小写字母”或等价文案。
- [ ] 缺少数字时显示“需要包含数字”或等价文案。
- [ ] 缺少特殊字符时显示“需要包含特殊字符”或等价文案。
- [ ] 策略失败时不得只展示泛化“新密码不符合安全策略”。

## AC-002 既有错误无回归

- [ ] 弱密码继续展示“密码过于常见，请更换”或等价文案。
- [ ] 新密码与原密码相同继续展示“新密码不能与原密码相同”或等价文案。
- [ ] 原密码错误继续归属原密码字段。
- [ ] 限流和受保护账号不可改密继续展示可理解 message。

## AC-003 动态策略一致性

- [ ] 修改密码弹窗规则提示与当前 effective 密码策略一致。
- [ ] 系统设置调整密码最小长度或复杂度开关后，修改密码提示和后端校验不产生明显矛盾。

## AC-004 API / Orval / Docs

- [ ] 若 API 错误响应结构变化，OpenAPI 已更新。
- [ ] 若 OpenAPI 变化，Orval 已重新生成。
- [ ] `docs/03-api-index.md` 已说明修改密码策略失败详情。

## AC-005 安全边界

- [ ] 不放宽密码安全策略。
- [ ] 不绕过后端校验。
- [ ] 不在响应、日志、埋点、URL、本地存储或长期文档中写入明文密码。
- [ ] 不修改数据库 schema、MinIO、Docker Compose、小程序或店主端。
