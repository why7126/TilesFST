## Why

REQ-0038 已完成品牌证书管理页，但证书缩略图、状态 Badge、预览入口和上传文件卡片仍容易停留在页面内拼装。REQ-0055 要把这些展示与状态逻辑沉淀为管理端业务通用组件，降低后续证书列表、品牌详情快捷区和弹窗回显场景的重复实现与验收漂移。

## What Changes

- 在管理端品牌证书能力中补充通用业务组件契约，覆盖缩略图、证书摘要、有效期、有效状态、前台展示状态、预览入口和文件卡片。
- 将 `/admin/brand-certificates` 列表证书列、有效期列、有效状态列、前台展示列，以及新增/编辑弹窗文件展示区对齐到通用组件或通用展示方法。
- 明确组件只负责展示和回调，不下沉筛选、分页、权限、保存、删除、显示/隐藏确认、上传 API 调用或 toast。
- 保持既有 API、数据库、MinIO 单桶策略、上传限制、Orval 生成接口和小程序能力不变。

## Capabilities

### New Capabilities

- 无。

### Modified Capabilities

- `brand-certificate-management`: 补充品牌证书管理端通用组件、页面应用与组件化 UI 验收要求。

## Impact

- **Web/Admin:** 影响管理端品牌证书页与管理端品牌证书业务组件目录。
- **API:** 不新增或修改 API；不需要 Orval。
- **Database:** 不新增或修改 SQLite/MySQL 表结构、迁移或初始化数据。
- **Storage:** 不修改 MinIO Bucket、对象 Key 生成、上传前缀、文件大小限制或 MIME 白名单。
- **Miniapp:** 不影响微信小程序。
- **Tests:** 后续实现需补充组件单元测试、页面集成/回归测试和 1440x1024 管理端验收。
