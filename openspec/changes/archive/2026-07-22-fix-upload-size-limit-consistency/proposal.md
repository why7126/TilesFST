## Why

`BUG-0073-video-upload-23m-file-fails` 已评审通过。用户上传约 23M 文件失败，且反馈问题不止视频，图片也存在同类问题，文档预计也会受影响。

当前已确认：图片上传默认上限为 20MB；品牌证书 / PDF 上传入口存在 `CERTIFICATE_MAX_SIZE_MB = 20` 硬编码；视频上传默认上限为 500MB，若 23M 视频仍失败，需要继续排查入口误用、生产代理、对象存储写入、超时或 MIME Type 不匹配。该缺陷暴露公共上传链路在图片、视频、文档三类文件上的大小限制、配置入口、错误提示和测试边界不一致。

## What Changes

- 统一图片、视频、文档三类上传大小限制的规格来源：业务允许值、后端校验、系统设置、前端提示和部署代理配置必须一致。
- 将文档 / 证书类文件大小上限纳入可配置或明确规格治理，避免继续使用不可调整的硬编码 20MB。
- 保持上传必须经后端鉴权、MIME Type 校验、扩展名校验、对象 Key 校验和 MinIO / S3 兼容对象存储适配层。
- 统一超限、类型不支持、对象存储不可用等失败场景的可诊断错误提示。
- 补充图片、视频、文档约 23M 边界文件、超过上限文件和不支持类型的回归测试。
- 如新增或修改系统设置字段、错误响应或上传限制查询能力，同步 `.env.example`、OpenAPI、Orval、接口文档和测试。

## Capabilities

### New Capabilities

无。

### Modified Capabilities

- `object-storage`: 统一后端上传限制、错误映射、对象 Key 与受控读取约束，新增文档类上传大小配置一致性要求。
- `system-settings`: 扩展媒体设置字段范围，使图片、视频、文档限制可按统一规则读取和展示。
- `brand-certificate-management`: 修正品牌证书文件上传大小限制从硬编码 20MB 转为与文档类上传规格一致。

## Impact

- **api:** 可能修改管理端系统设置 media 分组字段，可能新增文档类上传限制字段或上传限制查询字段；若字段或 schema 变化，必须同步 OpenAPI、Orval、`docs/03-api-index.md` 和 API 测试。
- **backend:** 需要调整上传大小限制来源、文件/证书上传校验、错误提示和对象存储失败映射；不得绕过鉴权或对象存储适配层。
- **web/admin:** 需要使上传入口前端提示与后端 effective 限制一致；系统设置 media Tab 若新增文档类限制，需要同步表单和校验。
- **miniapp:** 若小程序涉及上传入口或受上传结果影响，需要回归媒体展示和错误提示；不新增小程序直传对象存储。
- **database:** 若新增 `media.max_file_size_mb` 或等价系统设置，需要同步 seed / repository / 文档 / 测试；无业务数据表结构变更预期。
- **object-storage:** 继续使用单 Bucket + 标准前缀策略，图片写入 `images/`，视频写入 `videos/`，文件写入 `files/`。
- **tests:** 需要 pytest 覆盖图片、视频、文档 23M 边界、超限、不支持类型和存储不可用；前端测试覆盖上传提示与失败状态。
- **docs:** 需要同步 `.env.example`、媒体 / 对象存储 / API 文档中上传限制说明。

## Rollback Plan

如修复后出现合法上传被误拒、超限文件被误放行、对象存储写入异常或系统设置回归，可回滚上传大小配置读取、证书文件限制改造和前端提示调整到修复前版本，并保留后端安全校验。回滚后 `BUG-0073` 不得关闭，需要重新确认是产品大小上限、配置迁移、代理配置还是存储适配导致。
