## Context

REQ-0049 是 `REQ-0047-product-list-common-component-application` 的子需求，范围聚焦微信小程序单个商品卡片组件。现有规范中，`miniapp-product-list-page`、`miniapp-search`、`miniapp-home` 和 `miniapp-sku-detail-page` 已分别描述商品卡片字段或入口，但缺少统一组件边界，导致不同列表页可能复制卡片 DOM、兜底文案、跳转参数和埋点逻辑。

原型位于 `issues/requirements/archive/REQ-0049-miniapp-product-card-component/prototype/miniapp/`，优先级为 HTML > PNG > context/interaction > acceptance > `rules/ui-design.md` > 既有 spec。当前需求为小程序端，不触发 Web UI Explore Gate。

## Goals / Non-Goals

**Goals:**

- 建立小程序商品卡片组件的字段展示、图片稳定、触控反馈、详情跳转、异常状态和埋点契约。
- 让分类商品列表、搜索结果 SKU 列表和首页推荐复用统一核心结构，同时允许列表页传入不同来源上下文和展示密度。
- 保持小程序深色企业轻奢风，价格和状态强调使用品牌金语义，卡片圆角接近直角。
- 明确实现阶段需要验证 320px 小屏、固定图片比例、字段缺失、图片失败、不可查看和防重复点击。

**Non-Goals:**

- 不实现列表查询、筛选、排序、分页、加载态和空状态。
- 不新增收藏、分享、加入询价、购物车、联系客服或在线下单入口。
- 不新增或修改后端 API、数据库、对象存储、Web 管理端或 Orval 生成物。
- 不建设跨端 React/小程序统一组件库。

## Decisions

### D1. 采用小程序本地可复用组件

实现阶段 SHOULD 在 `src/miniapp/components/` 下沉淀商品卡片组件，由列表页传入 SKU 数据、展示密度、来源上下文、位置序号和可选标签。卡片组件不得直接请求列表接口，也不得维护分页、筛选、排序或下拉刷新状态。

备选方案是继续在各页面局部实现卡片，但会导致字段兜底、图片占位和跳转规则漂移；本需求要求统一复用，因此不采用。

### D2. 展示密度由列表容器选择，核心字段语义不变

卡片支持单列大卡、双列紧凑卡和首页横向/推荐卡的布局差异，但主图、SKU 名称、品牌、规格、参考价格、状态和来源上下文的语义必须一致。列表容器可以决定密度和布局，不得改变核心字段含义。

### D3. 字段归一化在卡片边界内完成

卡片 SHOULD 在渲染前归一化单个 SKU 字段：价格缺失、非正价格或旧无价文案统一展示“暂无”，品牌缺失展示“品牌待确认”，规格缺失展示“规格待补充”，名称缺失展示“未命名商品”或后端兜底名称。卡片不得展示空字符串、`null`、`undefined`、接口字段名或异常金额。

### D4. 跳转和埋点参数由安全白名单组装

卡片点击进入 SKU 详情页时 MUST 携带 `skuId`，并可携带 `sourcePage`、`sourceModule`、`categoryId`、`brandId`、`keyword`、`listContext`、`index` 和 `requestId` 等白名单来源参数。不得透传 Authorization header、Cookie、手机号、raw payload、raw object key 或敏感配置。

### D5. 原型冲突决议

- HTML 原型展示双列紧凑卡和横向宽卡示例；acceptance 要求“单列或双列”复用。实现阶段允许多密度，但核心字段和异常状态必须一致。
- `interaction.md` 提到标签最多展示 3 个，且优先级为规格 > 材质 > 工艺 > 风格。该约束比 PRD 更具体，作为卡片辅助标签选择规则吸收。
- HTML 原型使用 `新品`、`热销`、`下架` 标签；REQ 范围不包含收藏/分享/询价/购物车操作。实现阶段可展示状态/运营标签，但卡片内不得新增 v1 范围外快捷操作。
- 原型价格示例为 `¥168/片`，REQ 明确必须使用“参考价格”语义。实现阶段价格文案必须带“参考价格”或等价语义，避免暗示在线成交价。

## Risks / Trade-offs

- [Risk] 页面已有局部卡片结构与统一组件字段不一致 → Mitigation: 实现阶段先盘点首页、搜索和商品列表现有卡片，再替换或适配调用点。
- [Risk] 320px 小屏中品牌、规格和价格挤压名称 → Mitigation: 名称最多两行，低优先级字段使用省略或隐藏策略。
- [Risk] 连续点击导致多个详情页打开 → Mitigation: 卡片内部维护短时点击锁或调用方传入跳转中状态。
- [Risk] 搜索来源参数可能包含敏感或过长内容 → Mitigation: 只透传白名单字段，并限制 `keyword`、`listContext` 等参数长度。

## Migration Plan

1. 在小程序端新增或整理商品卡片组件。
2. 将分类商品列表、搜索结果 SKU 列表、首页推荐商品区改为复用统一组件。
3. 补充字段兜底、图片失败、不可查看、防重复点击和来源参数测试。
4. 进行 320px 小屏、普通视口和小程序预览/真机证据验收。

Rollback 策略：如统一组件在某入口回归，可在实现分支中临时回退该入口调用，但不得保留长期分叉；需要通过同一 Change 修复复用组件或适配层。
