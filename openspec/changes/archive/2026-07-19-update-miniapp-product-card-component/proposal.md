## Why

微信小程序分类列表、搜索结果和首页推荐都需要展示商品卡片，但现有规范只在各页面局部描述字段和跳转，容易在实现阶段出现字段兜底、图片占位、触控反馈、来源参数和埋点语义不一致。REQ-0049 已进入 sprint-009，需要将商品卡片沉淀为小程序端可复用组件契约，作为后续商品列表能力的稳定基础。

## What Changes

- 明确小程序商品卡片组件的复用边界：列表容器负责查询、筛选、排序、分页、加载和空状态，卡片只处理单个 SKU 的展示、触控、跳转和卡片级异常。
- 增强商品卡片字段、图片比例、缺失兜底、不可查看状态、防重复点击、来源上下文和埋点要求。
- 约束分类商品列表、搜索结果 SKU 列表和首页推荐商品区复用同一商品卡片核心结构，不复制分叉实现。
- 对齐 REQ-0049 miniapp HTML 原型：深色企业轻奢风、固定比例图片区、品牌金价格、最多三枚辅助标签、无收藏/分享/询价/购物车快捷按钮。
- 不新增 API、数据库、对象存储、Web 管理端或交易能力。

## Capabilities

### New Capabilities

- 无。

### Modified Capabilities

- `miniapp-product-list-page`: 将商品卡片从页面局部展示规则强化为小程序商品列表可复用组件契约。
- `miniapp-search`: 搜索结果 SKU 卡片复用统一商品卡片核心结构和来源上下文。
- `miniapp-home`: 首页新品、热销和全部产品区域复用统一商品卡片核心结构，并移除卡片内非 v1 操作分叉。
- `miniapp-sku-detail-page`: SKU 详情页入口补充商品卡片来源参数、安全边界和不可查看兜底。

## Impact

- 影响端：微信小程序。
- 可能涉及源码：`src/miniapp/components/`、`src/miniapp/pages/**` 中商品列表、搜索、首页和 SKU 详情入口相关文件。
- API：默认不新增或修改后端接口；若实现发现缺字段，只能通过后续评审决定是否另建 API Change。
- 数据库：无结构变更。
- 对象存储：无上传或存储策略变更。
- Web / 管理端：无影响。
- Orval：默认不需要。
- 测试：需要补充小程序静态测试、组件渲染/点击测试或等价验收脚本，并保留 320px 小屏、图片失败、字段缺失、不可查看和连续点击场景证据。
