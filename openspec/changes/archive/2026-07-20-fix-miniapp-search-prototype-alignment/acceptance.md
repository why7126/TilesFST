# Acceptance - fix-miniapp-search-prototype-alignment

## 回归验收标准

- [ ] AC-FIX-001 搜索输入态顶部展示返回按钮和搜索框；搜索结果页顶部标题展示搜索关键词，结果页不展示搜索框。
- [ ] AC-FIX-002 独立搜索页复用 `components/search-entry`，或实现与通用组件一致的关键词、清空、提交、取消/返回、禁用态、`scope` 和 `sourcePage` 行为。
- [ ] AC-FIX-003 搜索首页展示最近搜索和热门搜索，不展示最近浏览；最近搜索支持单条删除和全部清空，清空按钮与搜索框右侧对齐。
- [ ] AC-FIX-004 联想态仅展示品牌与 SKU 两组，不展示最近搜索、普通关键词、类目、规格或证书。
- [ ] AC-FIX-005 搜索结果页展示综合、品牌、SKU、证书 Tab，综合默认激活且横向可滚动，不展示类目 Tab。
- [ ] AC-FIX-006 综合结果页渲染最多 1 条最佳匹配，并按品牌、SKU、证书顺序展示非 0 条分区；品牌/SKU/证书 Tab 内不展示分区标题和数量。
- [ ] AC-FIX-007 SKU 卡片展示主图、产品名称、SKU 编码、品牌、规格和参考价格，整卡点击进入 SKU 详情，不展示收藏、分享、购物车或询价快捷操作。
- [ ] AC-FIX-008 搜索结果页不展示快捷筛选、筛选按钮或筛选抽屉。
- [ ] AC-FIX-009 最佳匹配支持 SKU、品牌和证书：SKU 编码或名称直接命中优先，其次品牌名精确命中，最后证书名称或编号精确命中。
- [ ] AC-FIX-010 品牌、SKU、证书结果均以一行一个卡片式结构展示；SKU 复用 `components/product-card/`，品牌/证书保留自身跳转行为。
- [ ] AC-FIX-011 无结果页按 `05-search-empty.html` 展示当前关键词、搜索图标、调整建议列表和推荐搜索词，不展示联系商家、提交找砖、购物车、询价或在线下单入口。
- [ ] AC-FIX-012 搜索首页、联想态、加载态、结果态、筛选态、失败态和无结果态切换时无明显布局跳动、内容重叠或底部操作遮挡。
- [ ] AC-FIX-013 小程序静态或单元测试覆盖搜索通用组件应用、综合分区渲染、联想类型限制、最佳匹配优先级、结果页无筛选和无结果结构。
- [ ] AC-FIX-014 验收证据明确对照 REQ-0046 的 5 个 HTML/PNG 原型完成复核。

## 非目标

- 不新增管理端搜索配置中心。
- 不新增后台热门词、同义词或自然语言搜索管理能力。
- 不改变搜索 API 的公开状态过滤、安全校验或埋点敏感字段策略，除非实现阶段发现接口契约实际不满足本 Change。
