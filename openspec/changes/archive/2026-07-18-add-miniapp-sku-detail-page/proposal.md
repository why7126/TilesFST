## Why

微信小程序首页、分类、搜索、品牌页、收藏页和分享卡片都需要承接到单个瓷砖 SKU 的完整详情，但现有小程序能力只定义了基础商品详情闭环，尚不足以覆盖高清图片/视频浏览、收藏、分享、品牌入口、推荐和异常状态。

本 Change 将 `REQ-0044-miniapp-sku-detail-page` 转为可实现的 OpenSpec 范围，使 sprint-008 的 SKU 详情页可以在不引入购物车、在线下单、库存或询价承诺的前提下进入开发。

## What Changes

- 新增微信小程序 SKU 详情页能力，覆盖入口路由、详情字段、媒体混合轮播、图片全屏预览、视频播放、收藏/取消收藏、微信分享、品牌入口、同系列/同品牌推荐和异常状态。
- 定义小程序 SKU 详情公开接口边界，要求只返回公开字段、安全媒体 URL 和推荐数据，不暴露后台内部字段、未授权对象存储 key 或敏感配置。
- 定义收藏接口幂等、失败回滚和未登录授权降级行为。
- 定义 SKU 详情页视觉、性能、安全、非功能和不做项验收，延续小程序首页 v6 深色企业轻奢风。
- 扩展产品使用行为事件字典，补充 SKU 详情页浏览、媒体、收藏、分享、品牌入口、推荐和加载失败事件。
- 不新增管理端 SKU 编辑能力；如需新增 SKU 字段、媒体关系或公开状态字段，必须同步数据库、OpenAPI、Orval、docs 和测试。

## Capabilities

### New Capabilities

- `miniapp-sku-detail-page`: 微信小程序 SKU 详情页展示、媒体浏览、收藏分享、推荐承接、异常状态、公开 API 和对象存储安全 URL 边界。

### Modified Capabilities

- `product-usage-logging`: 补充小程序 SKU 详情页行为事件字典与脱敏要求，覆盖详情浏览、媒体交互、收藏、分享、品牌点击、推荐点击和加载失败。

## Impact

- 后端：可能新增或调整小程序 SKU 详情接口、收藏/取消收藏接口、公开字段过滤、推荐数据查询与 usage event 字典。
- 数据库：默认优先复用现有 SKU、品牌、类目、规格、图片/视频元数据；若新增收藏或公开状态存储，需同步 SQLite/MySQL schema、迁移和文档。
- 小程序：新增 SKU 详情页及从首页、分类、搜索、品牌页、收藏页、分享卡片进入详情的路由承接。
- 对象存储：图片、视频、分享图必须经后端返回安全 URL，不允许小程序直连未授权对象存储。
- API/Orval/docs/tests：若新增或调整接口 contract，必须同步 OpenAPI、Orval、接口文档、数据库文档和后端/小程序相关测试。
- Web/管理端：默认不改运行时代码；仅在后续发现管理端字段维护缺口时另行评估。
