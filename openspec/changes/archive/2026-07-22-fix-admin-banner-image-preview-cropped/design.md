# Design: 管理端 Banner 图片完整预览修复

## 背景

`BUG-0080-admin-banner-image-preview-cropped` 指出管理端 Banner 列表和 Banner 弹窗中图片均显示不全。缺陷包初步判断问题来自前端图片容器、`object-fit`、固定尺寸或父容器 overflow 策略。

## 根因假设

当前 Banner 列表缩略图与弹窗预览可能复用了“填满容器优先”的裁切型样式，例如固定宽高 + `object-cover` + `overflow: hidden`。该策略适合统一视觉尺寸，但不适合运营人员完整确认素材内容。对于横幅图、竖图、超宽图等比例差异明显的素材，关键信息可能被裁掉。

## 修复方案

1. 定位 `/admin/banners` 列表缩略图与 Banner 弹窗预览的实际组件和 CSS。
2. 将列表缩略图调整为“主体完整可识别”的展示策略，优先避免裁切关键内容。
3. 将弹窗预览调整为“素材完整可确认”的展示策略，上传后预览与编辑回显保持一致。
4. 使用管理端 Design System semantic token 处理预览背景、边框、占位和空白区域，避免裸 Hex。
5. 保持布局稳定：表格行高、列宽、分页、筛选、弹窗滚动、上传控件和底部保存按钮不得出现新的溢出或遮挡。

## 测试策略

- 前端组件或页面测试覆盖 Banner 列表缩略图与弹窗预览 class/状态。
- 使用横幅图、方图、竖图、超宽图进行人工或截图验收。
- 回归 Banner 新建、编辑、保存后回显、上线/下线、排序和跳转类型配置。
- 若现有测试基础难以断言视觉裁切，应记录人工验收截图或 Playwright 截图证据。

## 风险与缓解

| 风险 | 缓解 |
|---|---|
| 图片完整展示后留白增加 | 使用语义化背景、边框和稳定尺寸，让留白可接受 |
| 列表行高变化影响扫描效率 | 固定缩略图容器尺寸，避免图片加载改变行高 |
| 弹窗预览扩大导致按钮不可见 | 保持弹窗内容区滚动和 footer 固定策略 |
| 不同图片来源表现不一致 | 统一自定义上传图、品牌 Logo、SKU 主图的预览组件或样式 |
