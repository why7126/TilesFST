# Proposal: 修复管理端 Banner 图片预览裁切

## Why

来源：`BUG-0080-admin-banner-image-preview-cropped`。

Web 管理端 Banner 管理页面中，Banner 图片在列表缩略图和新建/编辑弹窗预览中均显示不全。该问题不会直接阻断 Banner 保存或影响 API/数据库，但会影响运营人员识别素材、编辑确认和上线前检查，可能造成错误素材上线或返工。

## What

- 修复 `/admin/banners` 列表中 Banner 图片缩略图的显示策略，确保图片主体完整可识别。
- 修复 Banner 新建/编辑弹窗中的图片预览显示策略，确保上传后预览和编辑回显完整一致。
- 覆盖横幅图、方图、竖图、超宽图，以及自定义上传图、品牌 Logo、SKU 主图等图片来源。
- 保持列表行高、弹窗滚动、上传控件、保存按钮和现有 Banner 配置流程稳定。

## Non-goals

- 不新增 Banner 类型、展示位置或跳转类型。
- 不修改 Banner API、数据库表结构、对象存储策略或 Orval 生成物。
- 不改变小程序或店主展示端真实投放图片的裁切策略；如修复阶段发现展示端也存在同源缺陷，应另行记录或扩展评审。

## Impact

| 领域 | 影响 |
|---|---|
| Web 管理端 | 影响 `/admin/banners` 列表与弹窗图片预览 UI |
| API | 不涉及 |
| 数据库 | 不涉及 |
| 小程序 | 不涉及 |
| 对象存储 | 不涉及 |
| 测试 | 需要补充或更新前端 UI/组件测试，必要时补截图验收 |

## Rollback Plan

如修复引入列表布局、弹窗滚动或上传回显回退，可回滚本 Change 涉及的前端样式/组件修改，恢复到当前 Banner 图片预览样式；回滚不需要数据库迁移、API 回滚或数据修复。
