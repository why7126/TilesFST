## Context

本项目是瓷砖信息管理平台的 Web 后台管理系统，目标是为公司内部员工提供产品管理、类目品牌管理、员工管理、数据统计等能力。

技术约束：
- 前端使用 React + TypeScript + Vite 开发
- 需要员工登录认证
- 调用后端 REST API 进行数据操作
- 图片/视频上传通过后端转存到 MinIO

## Goals / Non-Goals

**Goals:**
- 实现商品管理（增删改查、上下架）
- 实现图片/视频批量上传
- 实现类目管理
- 实现品牌管理
- 实现员工管理（管理员/普通编辑角色分级）
- 实现数据统计（浏览量、收藏量、分享量）

**Non-Goals:**
- 客户管理系统
- 订单管理系统
- 财务报表系统

## Decisions

### 1. 技术选型

**决策：React + TypeScript + Vite + Ant Design**

理由：
- 与 Web 前端保持技术栈一致
- Ant Design 提供成熟的企业级 UI 组件
- Vite 提供更快的开发体验

### 2. 状态管理

**决策：React Context + React Query**

理由：
- React Query 处理服务端状态（数据获取、缓存、失效）
- React Context 处理客户端状态（用户信息、权限）

### 3. 权限控制

**决策：基于角色的权限控制（RBAC）**

- 管理员：所有权限
- 普通编辑：商品管理、类目管理、品牌管理，无员工管理权限

### 4. 批量上传方案

**决策：前端选择文件 → 调用后端 API 上传 → 后端转存 MinIO**

理由：
- 前端不直接接触 MinIO 凭证
- 后端统一处理文件类型校验、大小限制
- 保持与小程序/Web前台上传一致性

## Risks / Trade-offs

| 风险 | 描述 | 缓解方案 |
|-----|------|----------|
| 批量上传大文件 | 10000 SKU，每个10张图，批量上传可能超时 | 分批上传，显示进度 |
| 并发编辑冲突 | 多人同时编辑同一商品 | 乐观锁或最后写入胜出 |
| 权限控制 | 普通编辑可能误操作员工管理 | 后端API级别权限校验 |
