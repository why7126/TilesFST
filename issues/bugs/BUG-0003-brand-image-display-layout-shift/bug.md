# 现象

瓷砖品牌管理页在图片上传和状态反馈两个关键交互上存在可见缺陷：

1. 在品牌新增或编辑弹窗中上传品牌图片/Logo 后，已上传图片不能在品牌列表页正常展示。
2. 再次打开品牌编辑弹窗时，已上传图片不能正常预览或回显，用户无法确认当前品牌是否已有图片。
3. 在品牌列表页执行状态变更操作后，页面顶部会临时新增一行 Tips，几秒后消失；Tips 出现与消失会推挤主体内容，造成整页上下波动。

# 复现步骤

1. 以 admin 或 employee 登录 Web 管理端。
2. 进入「瓷砖品牌」列表页。
3. 新增品牌，或编辑已有品牌。
4. 在品牌表单中上传品牌图片/Logo。
5. 保存品牌并返回列表页。
6. 观察品牌列表中的图片/Logo 展示。
7. 再次打开该品牌编辑弹窗，观察图片/Logo 是否正常回显。
8. 在品牌列表页对任一品牌执行启用或停用操作。
9. 观察页面顶部 Tips 出现和消失时，页面主体内容是否发生上下位移。

# 期望结果

- 上传成功后，品牌列表页 MUST 正常展示品牌图片/Logo。
- 编辑已有品牌时，弹窗 MUST 正常回显已上传图片/Logo，便于用户确认当前素材。
- 状态变更反馈 SHOULD 使用不影响页面布局的提示方式，例如固定位置 toast、inline 弱提示或预留空间提示。
- Tips 的出现与消失 MUST NOT 推挤品牌列表、筛选区、指标卡或页面其他主体内容。

# 实际结果

- 品牌图片/Logo 上传后，在品牌列表页不能正常显示。
- 打开品牌编辑弹窗时，已上传图片/Logo 不能正常预览或回显。
- 品牌状态变更后，页面顶部 Tips 通过临时插入一行内容展示，几秒后消失。
- Tips 出现和消失会改变页面内容纵向位置，造成明显上下波动。

# 影响范围

| 范围 | 影响 |
|---|---|
| Web 管理端品牌列表页 | 品牌图片/Logo 无法展示，状态变更反馈造成页面抖动 |
| Web 管理端品牌新增/编辑弹窗 | 已上传图片/Logo 无法预览或回显 |
| 媒体资产展示 | 上传成功后的素材可见性不可验证 |
| 管理端操作体验 | 状态变更提示破坏页面稳定性，影响连续维护品牌数据 |

# 严重等级说明

严重等级建议为 `high`。

理由：

- 品牌图片/Logo 是品牌主数据的关键展示字段；上传后无法在列表和编辑弹窗展示，会影响运营确认素材是否生效。
- 该问题发生在品牌管理核心流程中，影响新增、编辑和后续复查。
- 状态变更 Tips 导致页面上下波动，属于明显可感知的交互稳定性问题，影响管理端连续操作体验。
- 当前问题暂未明确阻塞品牌文本字段维护或状态变更接口调用，因此不定为 `critical` 或 `blocker`。
