# 临时规避方案

## 1. 操作习惯规避（推荐）

在正式修复前，管理端仍可完成用户冻结与 SKU 上下架；运营可采取：

1. **用户管理**：点击「冻结」「解冻」前二次核对目标行用户名与角色；误操作后可通过「解冻」恢复。
2. **SKU 管理**：上下架/恢复前核对 SKU 名称与当前状态 badge；误下架可通过「恢复/上架」回滚（须满足上架条件）。
3. **用户删除**：当前仍有 `window.confirm` 原生提示，删除前务必阅读浏览器弹窗文案；仅从未登录用户可删。
4. 批量维护时以**列表刷新后的状态 badge** 为准，勿连续快速点击相邻行操作按钮。

## 2. 无产品内技术规避

本缺陷为纯前端交互缺口，**不存在**配置开关、环境变量或 API 参数可启用 modal 确认。

以下路径**不能**补齐 DS 二次确认：

| 路径 | 说明 |
|---|---|
| 刷新页面 | 不改变点击即 API 行为 |
| 换浏览器 | 用户删除仍为 `window.confirm`；冻结/SKU 上下架仍无确认 |
| 仅走 API 文档 | 绕过 UI 不能作为运营规避 |

## 3. 已修复页面对照

若需确认「正确交互」参考：

- **瓷砖品牌** `/admin/brands`：启用/停用/删除均有 modal（REQ-0008）。
- **瓷砖类目** `/admin/tile-categories`：启用/停用/删除均有 modal（REQ-0007）。

运营培训时可说明：用户/SKU 页尚未对齐上述交互，修复前需格外谨慎。

## 4. 验收规避

修复前相关验收应标注：

- 用户冻结/解冻、SKU 上下架 **功能结果**（API、Toast、列表刷新）可单独验证通过；
- **二次确认 modal** 应标记为本 BUG 未通过项；
- 品牌/类目启停确认 **不应** 计入本 BUG 未通过（已交付）。

## 5. 规避有效期

- **有效期**：自 2026-06-27 起至 `fix-admin-list-status-action-confirm` 发布并验收通过。
- **修复后**：用户冻结/解冻、用户删除、SKU 上下架/恢复 MUST 使用 DS modal；本规避仅作修复前参考。

## 6. 风险说明

| 风险 | 说明 |
|---|---|
| 误操作 | 冻结用户导致无法登录；SKU 误下架影响前台可见性 |
| 功能 | **不阻断** API；操作可逆（冻结/上下架）或仍有原生 confirm（删除） |
| 安全/数据 | 无权限绕过；无数据损坏 |
| 关联 | 重置密码 confirm UI 不一致 → **BUG-0017**，与本 BUG 独立 |

severity **medium**，建议与 BUG-0015/BUG-0017 等同 Sprint 编排，但可独立 fix-* 发布。
