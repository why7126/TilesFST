# 验收标准

## AC-001 品牌列表展示 Logo

- [ ] 在 `/admin/brands` 品牌列表中，已上传 Logo 的品牌必须显示图片。
- [ ] 图片加载失败时必须展示稳定空态，不应出现破图图标或布局跳动。

## AC-002 品牌编辑弹窗回显 Logo

- [ ] 打开品牌编辑弹窗时，已上传 Logo 必须正常回显。
- [ ] 空 Logo 品牌仍显示标准占位态。

## AC-003 新上传 Logo 可见

- [ ] 在品牌编辑弹窗中重新上传 Logo 后，上传成功的图片必须立即更新预览。
- [ ] 保存品牌后关闭弹窗，再重新打开仍能看到最新 Logo。
- [ ] 刷新品牌列表后仍能看到最新 Logo。

## AC-004 MinIO 对象读取闭环

- [ ] 品牌 Logo 对象必须写入项目统一桶 `tile-info-platform`。
- [ ] 品牌记录中的 `logo_object_key` 必须能映射到实际存在的 MinIO 对象。
- [ ] 品牌接口返回的 Logo URL 必须能被浏览器加载。
- [ ] 不允许前端绕过后端授权直接访问未受控对象存储。

## AC-005 历史数据兼容

- [ ] 对象存储修复前已存在的品牌 Logo 数据应有明确兼容策略。
- [ ] 若需要重新上传或迁移，必须在修复说明中写明范围与操作。

## AC-006 回归品牌管理功能

- [ ] 品牌查询、分页、新增、编辑、启用、停用、删除不回归。
- [ ] `BUG-0004` 的上传进度反馈不回归。
- [ ] `BUG-0003` 的提示布局稳定性不回归。

## AC-007 测试覆盖

- [ ] 后端测试覆盖媒体 URL 或受控读取成功场景。
- [ ] 前端测试覆盖品牌列表 Logo 展示。
- [ ] 前端测试覆盖品牌编辑弹窗 Logo 回显。
- [ ] 如涉及 OpenAPI 响应字段变化，必须同步 Orval 并补充相关测试。

## AC-008 规范约束

- [ ] 修复必须遵守 `rules/object-storage.md` 的单桶策略。
- [ ] 修复必须遵守 `rules/media.md` 的后端授权上传与受控访问要求。
- [ ] Web UI 不得新增裸 Hex 或绕开 Design System 的样式。
