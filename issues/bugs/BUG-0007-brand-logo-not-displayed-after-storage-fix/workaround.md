# 临时规避方案

## 1. 可用规避

当前没有可靠的业务侧规避方案。

可以尝试以下临时检查手段辅助定位，但不应视为正式修复：

1. 重新上传某个品牌 Logo，确认对象是否进入 MinIO 桶。
2. 在品牌列表接口响应中检查 `logo_object_key` 与 `logo_url`。
3. 直接访问 `logo_url`，确认是否返回图片、404、401、500 或其他错误。
4. 若仅历史数据不显示，可尝试重新保存品牌 Logo 以生成新对象 key。

## 2. 不建议的规避

- 不建议前端直连 MinIO 未授权对象 URL。
- 不建议把桶设置为公开读来绕过受控访问策略。
- 不建议手工修改数据库中的 `logo_url` 或对象 key，除非后续修复方案明确迁移规则。

## 3. 后续修复方向

后续修复应保证：

- 上传后的对象 key 与品牌记录一致。
- 品牌接口返回浏览器可加载的 Logo URL。
- 后端媒体读取策略与 MinIO 存储策略一致。
- 历史品牌 Logo 数据有明确兼容或迁移策略。
