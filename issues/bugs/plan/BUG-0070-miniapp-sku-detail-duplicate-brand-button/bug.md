# 现象

微信小程序 SKU 商品详情页存在两个品牌主页入口：

- 页面内容区已经展示“查看品牌主页”入口。
- 页面底部操作区仍显示品牌按钮。

两个入口语义相同，预期跳转目标也相同，导致详情页底部操作区出现冗余按钮。用户明确反馈希望删除底部品牌按钮，保留内容区的“查看品牌主页”入口。

# 复现步骤

1. 使用微信开发者工具或真机打开小程序。
2. 进入任意 SKU 商品详情页。
3. 在页面内容区查看“查看品牌主页”入口。
4. 查看页面底部操作区。
5. 观察底部是否仍存在品牌按钮。

# 期望 vs 实际

期望：

- SKU 商品详情页只保留一个清晰的品牌主页入口。
- 内容区“查看品牌主页”入口保留，并继续跳转到正确品牌主页。
- 底部操作区不再显示重复的品牌按钮。

实际：

- 内容区已有“查看品牌主页”入口。
- 底部操作区仍显示品牌按钮，造成重复入口和操作区冗余。

# 影响范围

- 影响终端：微信小程序。
- 影响页面：SKU 商品详情页。
- 影响区域：底部操作区与内容区品牌入口。
- 影响用户体验：用户会看到两个语义重复的品牌主页入口，底部操作区信息密度增加，可能削弱其他关键操作入口的清晰度。
- 不预期影响：品牌主页本身、商品详情基础信息展示、商品图片/视频浏览、分享等其他详情页能力。

# 严重等级说明

严重等级建议为 `medium`。

该问题不阻断商品详情页浏览，也不影响品牌主页入口的基础可达性；但它属于已交付详情页交互与信息架构偏差，会造成重复入口和底部操作区冗余，影响小程序商品详情页的操作清晰度，需要在后续修复流程中移除底部品牌按钮并做布局回归。
