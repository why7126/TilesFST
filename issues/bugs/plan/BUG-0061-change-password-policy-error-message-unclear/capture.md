# 现象

Web 管理端修改密码时，系统提示“新密码不符合安全策略”，但没有告知用户具体不符合哪一条策略，也没有说明应如何修改密码，导致用户只能反复试错。

# 复现步骤

1. 登录 Web 管理端。
2. 打开“修改密码”弹窗。
3. 输入正确的原密码。
4. 输入一个不满足密码安全策略的新密码。
5. 提交修改密码表单。

# 期望 vs 实际

| | 说明 |
|---|---|
| **期望** | 提示应明确指出未满足的具体策略，例如长度不足、缺少大写字母、缺少数字、缺少特殊字符、与旧密码过于相似或属于常见弱密码，并给出可操作的修改建议。 |
| **实际** | 仅提示新密码不符合安全策略，用户无法判断失败原因，也不知道如何修改。 |

# 待澄清

- [ ] 当前密码安全策略的完整规则列表以哪份后端校验或产品规范为准。
- [ ] 前端是否应在提交前逐条展示规则状态，还是仅在服务端校验失败后展示失败项。
- [ ] 是否需要与创建用户、重置密码等密码输入场景统一错误提示格式。
