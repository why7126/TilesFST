# 回归验收标准

## AC-001 品牌列表分页必须与用户管理页分页保持一致

**Given** 管理员已登录 Web 管理端  
**When** 分别访问「瓷砖品牌」和「用户管理」列表页  
**Then** 两个页面底部分页区域的布局、按钮尺寸、边框、圆角、字号、激活态和每页显示控件必须保持一致。

## AC-002 品牌分页不得使用割裂的局部视觉结构

**Given** 品牌列表存在分页数据  
**When** 用户查看品牌列表底部分页  
**Then** 不应出现与用户管理页不一致的 `page-left` / `brand-pagination-right` 视觉结构。  
**And** 如保留跳页能力，跳页输入必须作为统一分页组件的可选区域呈现，不能破坏主分页布局一致性。

## AC-003 品牌 Logo 选择文件控件必须符合管理端表单风格

**Given** 管理员打开「新增品牌」或「编辑品牌」弹窗  
**When** 查看「品牌Logo」区域  
**Then** Logo 选择文件控件必须与弹窗内输入框、按钮和用户管理弹窗头像上传控件保持一致的视觉层级。  
**And** 文件选择入口必须使用明确按钮或可访问标签，不展示浏览器默认文件控件皮相。

## AC-004 Logo 空态、预览态和帮助文案必须一致

**Given** 品牌 Logo 尚未上传  
**When** 查看 Logo 区域  
**Then** 应展示统一的空态说明和上传入口。  

**Given** 品牌 Logo 已上传  
**When** 查看 Logo 区域  
**Then** 应展示 Logo 预览、文件更换入口和帮助文案，且不挤压弹窗布局。

## AC-005 修复不得影响品牌管理功能

**Given** 管理员在品牌管理页操作  
**When** 执行查询、分页、修改每页显示、新增品牌、编辑品牌、上传 Logo、保存品牌  
**Then** 原有功能必须保持可用。  
**And** 不得引入 API 请求参数、响应结构或数据库字段变化。

## AC-006 Design System 约束必须满足

**Given** 修复完成  
**When** 检查 Web UI 修改  
**Then** 新增或修改的 UI 代码必须使用已有管理端样式变量、语义 Token 或既有组件模式。  
**And** 不得新增裸 Hex、未登记的页面局部色值或与 `rules/ui-design.md` 冲突的圆角/字号/边框风格。

## AC-007 回归测试与验收记录必须补齐

**Given** 缺陷进入修复阶段  
**When** 完成 `fix-*` OpenSpec Change  
**Then** 应补充或更新品牌管理页相关测试，覆盖分页和 Logo 上传入口渲染。  
**And** 应在 Change 的 acceptance 或 trace 中记录与用户管理页分页、Logo 上传控件的对齐结论。
