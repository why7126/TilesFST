# Workaround - BUG-0066 搜索组件整体交互与原型差异较大

## 临时规避方案

暂无可完全规避的产品侧方案。当前搜索页基础搜索、联想、提交、结果展示和筛选主流程仍可作为内部联调用途，但不建议作为 REQ-0046 的正式原型验收版本。

## 可接受的临时使用边界

在修复前，可按以下边界临时使用：

- 内部开发和接口联调可以继续使用当前搜索页验证搜索 API、埋点和基础跳转。
- 产品验收时应明确标注“搜索原型视觉与交互尚未对齐”，不要将当前搜索页截图作为最终验收证据。
- 若需要演示搜索能力，应优先说明功能主链路可用，但首页、联想、结果分区、筛选抽屉和无结果空态仍待修复。

## 不建议的规避方式

- 不建议通过修改 REQ-0046 原型或降低 AC 来规避当前实现偏差。
- 不建议仅更新 `tasks.md` 或验收文字来掩盖页面结构差异。
- 不建议绕过 OpenSpec 流程直接修改 `src/miniapp/pages/search/*`。

## 修复前风险

| 风险 | 说明 |
|---|---|
| 验收误判 | 原型验收任务已勾选完成，但实际页面未完全匹配原型 |
| 体验不一致 | 搜索页与首页深色轻奢风及 5 个搜索状态原型存在落差 |
| 返工扩大 | 若后续继续基于当前结构扩展搜索能力，可能增加二次返工成本 |

## 推荐处理

完成 BUG 评审后创建 `fix-*` Change，集中修复搜索页原型结构、通用组件复用、综合分区、筛选价格区间、无结果空态和静态测试断言。
