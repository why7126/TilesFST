# 临时规避方案

## 1. 操作规避（有限）

在正式修复前，运营可尝试以下方式 **部分** 确认上传结果，但 **不能作为验收通过依据**：

1. **打开浏览器 DevTools → Network**，上传时筛选 `tile-videos`；若返回 **200** 且响应含 `object_key`，说明后端已接收，问题在弹窗反馈而非存储。
2. 上传完成后 **向上滚动** 至弹窗顶部，查看是否出现红色 `.admin-notice` 错误（如「仅支持 MP4 格式」）；若有则为失败而非「未回显」。
3. 在「商品视频」区块 **仔细查看「上传视频」按钮上方** 是否已出现 MP4 文件卡片（文件名 + KB）；卡片样式较图片缩略图不显著，易被忽略。
4. 使用 **较小 MP4 样例**（< 5MB）缩短无反馈等待时间，便于观察列表是否追加。
5. 若怀疑已上传但未看见，可填写必填项后 **保存 SKU**，再通过编辑入口重开（**不在本 BUG 验收范围**，仅作业务兜底）。

## 2. 功能规避

**无可靠功能规避。** 无法在弹窗内稳定、可感知地完成「上传 → 确认 → 继续添加」闭环时，SKU 视频素材维护体验不达标。

## 3. 验收规避

修复前，REQ-0006 **AC-035**（多视频 + 文件卡片 + **上传状态**）**不得标记通过**。

本 BUG 聚焦的 **即时回显** 验收项在 `acceptance.md` 中单独列出，修复前 MUST 判失败。

## 4. 风险说明

- 运营无法确认视频是否已加入待保存列表，可能重复上传或误以为未上传而放弃维护视频素材。
- 大文件上传长时间无 UI 反馈，易被误判为功能损坏。
- 错误仅显示在弹窗顶部时，底部操作区用户看不到失败原因。

## 5. 建议优先级

severity 为 **high**，建议在 SKU 弹窗相关 medium 视觉类缺陷之后、或与 REQ-0006 视频验收同批优先修复。
