# 现象

管理端用户管理中，一次性密码弹窗的“复制密码”功能未生效。无论是刚创建用户后展示初始密码，还是重置密码后展示新随机密码，点击复制后都无法完成有效复制。

# 复现步骤

1. 进入管理端用户管理页。
2. 创建一个新用户，等待创建成功后展示一次性初始密码弹窗。
3. 点击弹窗中的“复制密码”按钮，尝试粘贴到输入框或文本区域验证。
4. 对任意已有用户执行“重置密码”，确认后展示一次性随机密码弹窗。
5. 再次点击“复制密码”按钮，尝试粘贴验证。

# 期望 vs 实际

- 期望：创建用户或重置密码后，点击“复制密码”应将当前展示的一次性密码写入系统剪贴板，并给出成功反馈；后续粘贴内容应与弹窗中的密码一致。
- 实际：点击“复制密码”后未能复制有效密码，创建用户与重置密码两个场景均未生效。
