# 管理端一次性密码弹窗复制密码未生效

## 现象

管理端用户管理页中，一次性密码弹窗的「复制密码」功能点击后未能完成有效复制。该问题同时出现在以下两个入口：

- 创建新用户成功后展示的初始密码弹窗。
- 对已有用户执行「重置密码」后展示的新随机密码弹窗。

管理员点击「复制密码」后，再粘贴到输入框、文本区域或其他可编辑区域验证时，粘贴内容未与弹窗展示的一次性密码一致，或没有粘贴出有效密码。

## 复现步骤

1. 进入 Web 管理端用户管理页。
2. 点击「添加用户」，填写必要信息并创建新用户。
3. 等待创建成功后，确认页面展示一次性初始密码弹窗。
4. 点击弹窗中的「复制密码」按钮。
5. 在任意输入框或文本区域执行粘贴，验证剪贴板内容。
6. 回到用户管理列表，对任意允许操作的已有用户点击「重置密码」。
7. 在二次确认弹窗中确认重置，等待新随机密码弹窗展示。
8. 再次点击「复制密码」按钮并粘贴验证。

## 期望结果

- 创建用户成功后，点击「复制密码」应将当前弹窗展示的 `initial_password` 写入系统剪贴板。
- 重置密码成功后，点击「复制密码」应将当前弹窗展示的 `password` 写入系统剪贴板。
- 复制成功后应给出明确成功反馈，避免管理员误判。
- 如果浏览器剪贴板权限不可用、非安全上下文受限或写入失败，应给出可理解的失败提示或手动复制兜底方式。
- 粘贴验证时，粘贴内容应与弹窗展示的一次性密码完全一致。

## 实际结果

- 创建用户和重置密码两个场景中，点击「复制密码」后均未能完成有效复制。
- 用户反馈中未观察到可靠的复制成功结果。
- 当前记录尚未确认点击时是否存在 Toast、错误提示或浏览器权限提示。

| 影响角色 | 系统管理员 / 有用户管理权限的后台用户 |

| 数据库影响 | 暂未发现数据库结构或数据写入异常 |
| 小程序影响 | 不涉及 |

## 严重等级说明

严重等级评估为 `high`。

理由：

- 一次性密码关闭弹窗后不可再次查看，复制失败会直接影响管理员向用户安全交付密码。
- 创建用户和重置密码两个核心用户管理闭环均受影响。
- 该能力已在 `REQ-0005-user-management` 中作为验收要求存在，当前行为属于既有能力偏差。
- 虽然管理员仍可能手动选中输入框内容复制，但当前交互未提供可靠反馈或兜底提示，容易造成交付失败或重复重置。
