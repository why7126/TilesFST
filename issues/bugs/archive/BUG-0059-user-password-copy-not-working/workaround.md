# 临时规避方案

## 当前可用规避

在正式修复前，管理员可在一次性密码弹窗中手动复制密码：

1. 创建用户或重置密码后，不要关闭一次性密码弹窗。
2. 点击弹窗中的「随机密码」输入框。
3. 使用鼠标或键盘选中完整密码文本。
4. 使用系统复制快捷键复制：
   - macOS：`Command + C`
   - Windows / Linux：`Ctrl + C`
5. 粘贴到临时输入框中核对内容是否与弹窗展示一致。
6. 确认后再安全交付给用户。

## 注意事项

- 一次性密码弹窗关闭后不可再次查看。若未成功复制且已关闭弹窗，只能重新执行重置密码生成新密码。
- 不建议通过截图、聊天记录或不受控文档长期保存一次性密码。
- 管理员应在交付后提醒用户尽快修改密码。

## 不可规避场景

- 如果浏览器或系统阻止手动复制输入框内容，当前产品内无其他安全复制方式。
- 如果管理员关闭弹窗后才发现复制失败，当前无法恢复原一次性密码。

## 临时运营建议

- 在修复上线前，创建用户或重置密码时，管理员应先粘贴验证再关闭弹窗。
- 若必须批量创建用户，建议减少一次性批量操作，逐个确认复制结果，避免密码交付失败。
