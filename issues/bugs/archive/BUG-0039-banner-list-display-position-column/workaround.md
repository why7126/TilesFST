# 临时规避方案

## 1. 可用性规避

该缺陷不阻断 Banner 管理功能：

1. 展示位置仍以副标题形式显示在第一列，运营可阅读「首页顶部轮播」等文案。
2. 「展示端」列可区分 Web 首页 / 小程序 / 专题页。
3. 筛选关键词支持搜索位置相关文案（若后端 keyword 覆盖 position）。
4. 新增/编辑弹窗内可查看并修改「展示位置」字段。

## 2. 验收规避

正式修复前验收可标注：

- 列表第一列标题与展示位置叠放 **暂不作为阻塞项**（功能可用）。
- 与 `banner-management-list.png` 第一列结构 **暂按原型 pass**（本 BUG 修复后将偏离 PNG）。

## 3. 运营规避

扫读展示位置时：

1. 关注第一列标题下方的灰色小字（展示位置）。
2. 结合「展示端」列理解完整投放上下文。
3. 编辑单条 Banner 时在弹窗「展示位置」下拉确认精确值。

## 4. 风险说明

规避不能消除：

- 第一列信息密度高、列对齐与扫读效率差。
- 与「展示端」独立成列的设计不一致。

建议 `/bug-review` 通过后通过 `fix-*` Change 修复；可与 BUG-0040 合并同一 Banner UI fix change。
