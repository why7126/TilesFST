# 现象

生产环境微信小程序中，商品详情页品牌入口跳转目标错误。用户在商品详情页点击品牌卡片后，进入搜索页，而不是对应品牌详情页。

该入口位于商品详情页品牌展示区域，与用户反馈中的「商品参数的品牌卡片」同属 SKU 详情页品牌入口体验。当前表现会让用户无法从商品详情页直接进入品牌主页，品牌信息、品牌商品和品牌证书的浏览链路被打断。

# 复现步骤

1. 打开生产环境微信小程序。
2. 进入任意包含品牌信息的商品详情页。
3. 在商品详情页找到品牌卡片入口。
4. 点击品牌卡片。
5. 观察跳转后的页面。

# 期望 vs 实际

期望：

- 点击品牌卡片后进入对应品牌详情页。
- 品牌详情页收到正确的品牌 ID，并展示品牌名称、品牌图片、简介、商品 Tab 和证书 Tab。
- 不应把可访问品牌入口降级为搜索页。

实际：

- 点击品牌卡片后跳转至搜索页。
- 用户无法直接进入对应品牌详情页。
- 品牌入口的文案与实际跳转目标不一致，造成路径误导。

# 影响范围

- 影响端：微信小程序。
- 影响环境：生产环境已反馈可见。
- 影响页面：商品详情页。
- 影响模块：SKU 详情页品牌卡片入口。
- 影响用户：从商品详情页查看品牌主页、同品牌商品或品牌证书的用户。
- 潜在范围：所有由 SKU 详情接口返回品牌入口路径的商品；如果该路径统一指向搜索页，则所有有品牌的 SKU 都可能复现。

# 严重等级说明

严重等级暂定为 `medium`。

理由：

- 这是生产环境用户可见的核心商品详情页跳转错误。
- 问题影响品牌浏览链路和品牌页转化，但不阻断商品详情页主体浏览。
- 暂未发现数据写入、权限绕过、崩溃、白屏或支付类风险。
- 修复面预计集中在 SKU 详情品牌入口路径与回归测试，适合常规 BUG 修复流程；若确认生产全部 SKU 均受影响，可在评审时提升优先级。
