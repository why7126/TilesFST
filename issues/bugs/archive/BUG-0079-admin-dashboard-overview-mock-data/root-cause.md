# 直接原因

管理端首页数据概览区域仍从 Mock 常量、演示数据或临时 fallback 数据源读取指标，没有切换到真实后端接口或真实业务聚合结果。

# 根本原因

首页概览从原型或演示阶段进入可用页面后，缺少“Mock 数据退出”验收门禁。页面实现与真实数据接口、统计口径、加载状态和错误兜底之间没有形成闭环，导致 Mock 数据继续留在管理端首页。

当前根因基于用户反馈和 capture 信息推断，尚未进入源码定位阶段。后续修复前需要确认：

- 前端首页是否存在硬编码 `mock`、`stats`、`overview` 常量或开发 fallback。
- 后端是否已有可复用的首页统计接口。
- 若无后端接口，是否需要新增管理端 dashboard summary API。
- 现有列表页统计口径是否可复用，避免首页与列表页口径漂移。

# 触发条件

1. 用户登录 admin 管理端。
2. 打开管理端首页。
3. 首页数据概览组件渲染。
4. 组件读取 Mock 数据源或固定演示值，而不是调用真实接口。

# 缺陷分类

| 分类 | 结论 | 说明 |
|---|---|---|
| code | 是 | 前端数据源或接口接入实现未完成，导致页面使用 Mock |
| design | 部分相关 | 首页概览统计口径和空/错状态可能缺少明确设计约束 |
| api | 待确认 | 如果后端没有首页统计接口，需要在后续 OpenSpec Change 中补齐 API |
| db | 否 | 当前没有证据表明数据库结构错误 |
| permission | 待确认 | 真实统计接入时需要确认管理端权限边界 |
| data | 否 | 当前不是业务数据本身损坏，而是页面未读取真实数据 |

# 修复方向

1. 定位管理端首页数据概览组件的当前数据源。
2. 移除或限制 Mock 数据仅在开发故事/测试夹具中使用。
3. 接入真实后端接口或新增受鉴权保护的管理端统计接口。
4. 明确各指标统计口径，并与相关列表页、后端查询保持一致。
5. 补充加载、空数据、接口错误和权限场景的回归测试。
