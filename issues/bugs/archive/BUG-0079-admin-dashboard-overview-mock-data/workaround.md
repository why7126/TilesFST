# 临时规避方案

暂无可靠的页面内规避方案。管理端首页数据概览在修复前不能作为真实业务状态依据。

# 可临时采用的操作方式

1. 管理员需要查看真实数据时，优先进入对应管理端列表页或使用后端真实接口查询。
2. 对外沟通或内部运营判断中，避免引用首页概览卡片的数值。
3. 若必须临时上线展示首页，需在发布说明或验收记录中标注该概览数据不可信，直到修复完成。

# 风险说明

- 手工对账只能缓解误判风险，不能解决首页信息失真。
- 继续保留 Mock 数据会掩盖真实数据异常，也会降低管理端首页可信度。
- 不建议用“接口失败时展示 Mock”作为兜底策略；应展示错误状态、空状态或上次有效数据并明确标识。

# 解除条件

当首页数据概览全部切换为真实数据源，并通过 `acceptance.md` 中 AC-001 至 AC-007 后，本规避方案可撤销。
