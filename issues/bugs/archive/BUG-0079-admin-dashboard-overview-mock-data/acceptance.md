# 验收标准

## AC-001 数据源真实

管理端首页数据概览的所有指标均来自真实后端接口或真实业务聚合结果，不再从页面 Mock 常量、演示数组或固定 fallback 值读取。

## AC-002 统计口径一致

首页概览指标与对应业务表、后端查询或管理端列表页统计口径一致。至少覆盖品牌、商品/SKU、分类、媒体/证书、待处理事项等页面实际展示的指标项。

## AC-003 数据变更可反映

当测试数据发生新增、更新或删除后，首页概览在刷新或重新请求后能展示更新后的真实结果，不保持固定演示值。

## AC-004 加载与空状态明确

接口请求过程中展示明确 loading 状态；真实数据为空时展示空状态或数值 0，不使用 Mock 数据填充。

## AC-005 错误状态不误导

接口失败、鉴权失败或网络异常时展示错误状态或重试入口，不展示 Mock 数据作为成功结果。

## AC-006 权限边界正确

不同管理端权限账号只能看到其权限范围内允许展示的概览数据；无权限时不得泄露受限统计。

## AC-007 回归测试覆盖

补充或更新后端/API、前端页面或集成测试，覆盖真实数据成功返回、空数据、接口失败和至少一个指标口径校验。

# 验收证据要求

| 证据类型 | 要求 |
|---|---|
| 代码证据 | 指出管理端首页概览组件移除 Mock 数据源并接入真实接口的位置 |
| API 证据 | 若新增或调整接口，提供请求、响应、错误码和鉴权说明 |
| 数据证据 | 提供至少一组测试数据与首页指标的对应关系 |
| 测试证据 | 提供相关自动化测试命令与通过摘要 |
| UI 证据 | 提供首页 loading、empty、error 和真实数据展示的验收说明或截图 |
