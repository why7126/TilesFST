# 现象

admin 管理端首页的数据概览区域仍展示 Mock 数据，未接入真实业务数据。页面上的概览指标无法准确反映当前系统中的品牌、商品、分类、媒体、待办或其他业务对象状态。

该问题会让管理端首页成为静态演示视图，而不是可用于运营判断和管理决策的真实数据入口。用户进入管理端后看到的关键指标可能与数据库、后端接口或其他管理端列表页统计结果不一致。

# 复现步骤

1. 登录 admin 管理端。
2. 进入管理端首页。
3. 查看首页数据概览区域的各项指标。
4. 对比对应后端接口、数据库记录或管理端其他列表页统计结果。
5. 刷新页面或变更业务数据后再次观察首页概览指标是否随真实数据变化。

# 期望 vs 实际

期望：

- 首页数据概览通过后端接口或真实业务聚合结果展示。
- 各项指标与业务数据口径一致，并能随数据变化更新。
- 加载、空数据和接口失败时有明确状态，不使用 Mock 数据误导用户。
- 数据展示遵循管理端权限边界，不暴露越权统计。

实际：

- 首页数据概览仍使用 Mock 数据或固定演示值。
- 指标无法反映当前系统真实业务状态。
- 数据变化后首页概览不随真实数据更新，容易误导运营判断。

# 影响范围

- 影响端：Web 管理端。
- 影响环境：admin 管理端，具体本地、测试或生产环境待 `/bug-complete` 阶段确认。
- 影响页面：管理端首页。
- 影响模块：首页数据概览 / overview metrics。
- 影响用户：登录管理端查看经营概况、内容规模、待处理事项或系统状态的管理员。
- 潜在范围：若当前首页所有概览卡片共用同一 Mock 数据源，则所有首页概览指标均可能受影响；若只有部分卡片使用 Mock，需要逐项核对。

# 严重等级说明

严重等级暂定为 `medium`。

理由：

- 问题影响管理端首页核心信息可信度，可能误导运营与管理判断。
- 当前描述未表明存在登录阻断、数据写入失败、权限绕过、崩溃或白屏。
- 修复预计涉及前端首页数据源、后端统计接口或现有接口接入，需要补充接口与回归测试。
- 若后续确认生产环境关键运营指标全部失真，或影响管理决策闭环，可在评审阶段提升严重度。
