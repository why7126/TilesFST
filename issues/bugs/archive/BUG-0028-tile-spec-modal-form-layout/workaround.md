# 临时规避方案

## 1. 可用性规避

该缺陷不阻断规格主数据维护，当前可继续使用：

1. 新增/编辑弹窗仍可填写宽度、长度、厚度、排序、备注并保存。
2. 只读尺寸名称 preview 仍会随宽长变化实时更新（格式 `{w}×{l}mm` 正确）。
3. 备注框宽度不足不影响文本输入与 200 字限制；保存后数据完整写入后端。

## 2. 验收规避

在正式修复前，验收时应明确标注：

- 弹窗字段顺序与 `tile-size-management-modal.html` 不一致，**暂不作为 REQ-0009 AC-019、AC-046 通过项**。
- 备注文本框整行宽度与原型不符，**暂不作为弹窗 CSS Port 通过项**。
- 仅验证弹窗 CRUD 功能、宽长校验、保存成功后列表行为（另见 BUG-0029）是否可用。

## 3. 运营规避

内部管理员维护规格时可：

1. 先填写宽度、长度，再向下滚动查看只读尺寸名称（当前位于厚度/排序下方）。
2. 以宽/长字段标签中的 `(mm)` 理解单位；只读区 `{w}×{l}mm` 即为系统将写入的 `display_name`。
3. 备注内容正常填写即可，无需依赖整行宽度展示。

## 4. 风险说明

该规避方案只能保证功能可用，不能消除以下问题：

- 弹窗布局与 REQ-0009 原型不一致，影响 AC-046 并排验收。
- 字段顺序偏离可能导致运营误以为尺寸名称与厚度/排序存在逻辑关联。
- 缺少 AC-021 冲突提示时，重复宽长组合需依赖服务端 `TILE_SPEC_DUPLICATED` 错误反馈（体验较差）。

因此仍建议进入 `/bug-review`，评审通过后通过 `fix-*` OpenSpec Change 修复。
