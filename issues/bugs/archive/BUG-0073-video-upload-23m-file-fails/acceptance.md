# Acceptance - BUG-0073 上传 23M 文件失败，图片、视频和文档均受影响

## 回归验收标准

- [ ] AC-BUG-001 明确并记录图片、视频、文档三类上传的业务允许大小上限；前端提示、后端校验、系统设置和部署代理配置 MUST 保持一致。
- [ ] AC-BUG-002 若产品确认约 23M 图片应允许上传，则约 23M 的 JPG / PNG / WebP 图片 MUST 上传成功，并返回 `/media/{object_key}` 地址。
- [ ] AC-BUG-003 若产品确认图片上限仍为 20MB，则约 23M 图片 MUST 被明确拒绝，错误提示 MUST 包含当前限制说明，且前端上传前提示 MUST 与后端一致。
- [ ] AC-BUG-004 约 23M 视频文件在允许 MIME Type 且走视频上传入口时 MUST 上传成功；若失败，必须能定位为明确的类型、大小、代理、超时或对象存储错误。
- [ ] AC-BUG-005 若产品确认约 23M 文档 / PDF 应允许上传，则品牌证书或文档上传入口 MUST 支持该大小，并纳入统一配置；若不允许，必须返回清晰错误提示。
- [ ] AC-BUG-006 超过各类型上限的文件 MUST 返回统一结构错误响应，不得出现无提示失败、上传卡住、500 或暴露内部路径。
- [ ] AC-BUG-007 不支持的 MIME Type 或扩展名 MUST 被拒绝，且不得因为放宽大小限制而绕过类型校验。
- [ ] AC-BUG-008 上传成功的对象 MUST 使用标准对象前缀和安全对象 Key，并通过后端 `/media/{object_key}` 受控读取，不得前端直连对象存储写入。
- [ ] AC-BUG-009 后端测试 MUST 覆盖图片、视频、文档三类约 23M 边界文件，以及超过上限和不支持类型的失败场景。
- [ ] AC-BUG-010 若新增或修改 API 字段、错误码、系统设置或上传限制查询能力，MUST 同步 OpenAPI、Orval、接口文档、`.env.example` 和相关测试。

## 验收证据要求

| 类型 | 要求 |
|---|---|
| 配置证据 | 展示图片、视频、文档大小限制的配置来源和最终有效值 |
| 图片上传证据 | 覆盖约 23M 图片成功或清晰拒绝的截图 / Network 响应 |
| 视频上传证据 | 覆盖约 23M 视频上传成功，并可通过 `/media/{object_key}` 读取 |
| 文档上传证据 | 覆盖约 23M PDF / 文档成功或清晰拒绝的截图 / Network 响应 |
| 错误响应证据 | 覆盖超限、类型不支持、对象存储不可用时的错误码和提示 |
| 自动化测试 | pytest 覆盖后端上传限制、对象 Key、MinIO 适配层错误映射；前端测试覆盖错误提示和上传状态 |
| 部署证据 | Docker / Nginx / 生产代理请求体大小和超时配置与后端限制一致 |
