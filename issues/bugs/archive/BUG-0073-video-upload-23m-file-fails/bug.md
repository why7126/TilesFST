# 现象

上传约 23M 的文件失败。用户补充说明该问题不止发生在视频文件，图片也存在同样问题，预计文档类文件也会受到影响。

当前问题倾向不是单一媒体类型解析失败，而是公共上传链路在文件大小、请求体限制、前后端限制不一致、代理层限制、后端校验或对象存储写入等环节存在缺陷。

# 复现步骤

1. 进入支持文件上传的页面或入口，例如图片上传、视频上传或文档上传入口。
2. 分别选择约 23M 的图片、视频和文档文件。
3. 发起上传。
4. 观察页面提示、上传进度、Network 响应和后端日志。
5. 对比小文件是否可成功上传，确认失败是否与文件大小或上传链路配置相关。

# 期望结果

符合系统允许大小、MIME Type 和扩展名限制的图片、视频、文档文件应能正常上传，并通过后端鉴权与 MinIO 适配层完成存储。

如果文件超过限制或类型不支持，系统应返回明确错误码、限制说明和用户可理解的提示，不应表现为无提示失败、通用失败或上传卡住。

# 实际结果

约 23M 的文件上传失败，且用户反馈图片、视频均出现同类问题，文档也可能受影响。暂未确认失败发生在前端校验、后端请求体大小限制、媒体类型校验、临时文件处理、MinIO 写入、代理层请求体限制或超时环节。

# 影响范围

| 范围 | 影响 |
|---|---|
| 图片上传 | 可能影响品牌 Logo、Banner、瓷砖图片、证书图片等资料维护 |
| 视频上传 | 可能影响商品视频、品牌视频或详情页轮播视频资料维护 |
| 文档上传 | 可能影响证书、说明文档或后续文档类附件上传 |
| 管理端 | 若上传入口在管理端，会阻断企业内部资料维护 |
| 小程序/展示端 | 若依赖上传后的媒体资源，可能导致图片、视频或附件无法展示 |
| 对象存储 | 若对象写入失败或返回地址不可用，会影响后续读取和播放链路 |

# 严重等级说明

严重等级为 `high`。文件上传是媒体资料维护的核心链路，且问题可能覆盖图片、视频和文档等多种类型；23M 文件并不属于明显异常大文件，如果系统没有明确拒绝原因，会直接阻断业务资料维护和后续展示。
