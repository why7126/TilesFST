# 临时规避方案

## 1. 操作习惯规避（推荐）

在正式修复前，规格启停/删除功能仍可用；运营可采取：

1. 点击「启用」「停用」「删除」后，**务必阅读确认弹窗正文**，核对尺寸名称（如 `800×1200mm`）后再点主按钮。
2. 停用/删除前二次确认目标行状态与关联 SKU 数，避免误操作生产数据。
3. 若需参考「标准 confirm 交互」，可对照「瓷砖类目」页同类操作弹窗（含 × 关闭、语义化主按钮、停用后果说明）。

## 2. 无产品内技术规避

本缺陷为纯前端 confirm **形态/文案**不一致，**不存在**配置开关或环境变量可切换为类目页 modal 模板。

以下路径**不能**改为标准 DS confirm：

| 路径 | 说明 |
|---|---|
| 刷新页面 | 仍为简化 confirm 模板 |
| 换浏览器 | 结构/文案不变 |
| 仅走 API 文档 | 绕过 UI 不能作为运营规避 |

## 3. Golden Reference 对照

若需确认「正确交互」参考：

- **瓷砖类目** `/admin/tile-categories`：启用/停用/删除 confirm modal。
- **瓷砖品牌** `/admin/brands`：同等 confirm modal 模式。

培训时可说明：规格页 confirm 修复前与类目/品牌页视觉不一致，但仍有二次确认门槛。

## 4. 验收规避

修复前相关验收应标注：

- 规格启停/删除 **功能结果**（API 成功、Toast、列表刷新）可单独验证通过；
- **confirm UI 与类目/品牌 modal 一致** 应标记为本 BUG 未通过项（REQ-0009 AC-013 / AC-018）；
- BUG-0027/28/29 已交付项（分页、表单、刷新）**不应** 计入本 BUG 未通过。

## 5. 规避有效期

- **有效期**：自 2026-06-28 起至 `fix-tile-spec-status-confirm-ui` 发布并验收通过。
- **修复后**：规格页 confirm MUST 使用与类目页一致的 DS modal；本规避仅作修复前参考。

## 6. 风险说明

| 风险 | 说明 |
|---|---|
| 误操作 | 误点「确认」会执行启停/删除；删除不可恢复 |
| 功能 | **不阻断**；仍有二次确认门槛 |
| 安全/数据 | 无权限绕过；服务端仍二次校验删除条件 |
| 关联 | BUG-0027/28/29 职责独立；可同 Sprint 编排 |

severity **medium**，建议独立 `fix-*` change 发布，MUST NOT 回改已 archived `fix-tile-spec-admin-ui`。
