# BUG-0067 首页推荐模块查看更多和榜单入口误跳搜索页

## 现象

首页点击「新品推荐」「热销推荐」模块的「查看更多」时，当前全部进入搜索页；点击「新品榜」「热销榜」入口时也同样进入搜索页。

## 复现步骤

1. 打开首页。
2. 点击「新品推荐」模块的「查看更多」。
3. 返回首页，点击「热销推荐」模块的「查看更多」。
4. 返回首页，分别点击「新品榜」和「热销榜」。
5. 观察实际进入的页面。

## 期望 vs 实际

| 项 | 期望 | 实际 |
|---|---|---|
| 新品推荐查看更多 | 进入「新品推荐商品列表页」 | 进入搜索页 |
| 热销推荐查看更多 | 进入「热销推荐商品列表页」 | 进入搜索页 |
| 新品榜 | 进入「新品推荐商品列表页」或与新品推荐一致的商品列表入口 | 进入搜索页 |
| 热销榜 | 进入「热销推荐商品列表页」或与热销推荐一致的商品列表入口 | 进入搜索页 |

## 影响范围

- 影响端：待确认，用户描述为首页，疑似微信小程序首页。
- 影响页面：首页推荐模块、榜单入口、商品列表跳转链路。
- 影响类型：入口路由错误，用户无法通过首页推荐入口直达对应推荐商品列表。

## 附件

暂无。

## 待补充

- 确认影响端是微信小程序首页、Web 店主端首页，或两端均存在。
- 确认「新品推荐商品列表页」「热销推荐商品列表页」是否已有独立路由和筛选参数。
- 确认榜单入口是否与推荐模块查看更多完全复用同一路由策略。
