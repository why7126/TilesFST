# Workaround - BUG-0069 SKU 商品详情页轮播图视频不能显示和播放

## 临时规避方案

暂无推荐的代码外彻底规避方案。根因疑似在小程序详情接口的视频 URL 组装链路，可靠修复应通过 `fix-*` Change 统一调整接口字段读取与测试覆盖。

## 可接受的临时使用边界

在正式修复前，可按以下边界临时处理：

- 对外演示或验收时优先选择只包含图片素材的 SKU，避免把视频播放能力作为已完成证据。
- 若必须演示视频 SKU，应先通过接口响应确认 `media[].url` 是 `/media/...` 或完整可访问 URL。
- 管理端录入视频后，可临时检查数据库中 `tile_videos.object_key` 与 `file_name` 是否符合当前接口预期，但不建议以手工修改生产数据作为长期方案。
- 小程序详情页其余文本、图片、品牌、推荐和收藏能力可继续用于内部联调。

## 不建议的规避方式

- 不建议让小程序端直接拼接对象存储地址或绕过后端媒体访问层。
- 不建议把真实 MinIO 内部地址、密钥、未授权 object key 或临时本地路径写入小程序配置。
- 不建议仅修改测试种子继续使用 `file_name=/media/...` 掩盖字段语义问题。
- 不建议通过删除 SKU 视频来宣称详情页视频能力验收通过。

## 修复前风险

| 风险 | 说明 |
|---|---|
| 商品表达缺失 | SKU 视频素材无法展示，影响瓷砖纹理、铺贴效果或质感表达 |
| 验收误判 | 后端测试覆盖了理想 URL，但没有覆盖管理端真实保存路径 |
| 安全边界漂移 | 若临时绕过后端媒体 URL 规范，可能破坏对象存储访问边界 |
| 用户体验不完整 | 混合轮播计数显示有视频，但视频项不可播放时会造成明显体验落差 |

## 推荐处理

完成 BUG 评审后创建 `fix-*` Change，集中修复小程序 SKU 详情接口视频 URL 组装逻辑，补充管理端真实保存语义的回归测试，并在小程序端回归视频展示、播放、失败提示和页面隐藏暂停逻辑。
