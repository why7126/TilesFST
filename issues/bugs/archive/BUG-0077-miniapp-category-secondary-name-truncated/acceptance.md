# Acceptance - BUG-0077 微信小程序分类页二级分类名称超过 4 个字被省略

## 回归验收标准

- [ ] AC-BUG-001 微信小程序分类页中，4 字以内二级分类名称 MUST 正常显示，布局不得回退。
- [ ] AC-BUG-002 微信小程序分类页中，5-8 字二级分类名称 MUST 可被用户完整识别或以业务可接受方式清晰展示。
- [ ] AC-BUG-003 微信小程序分类页中，超过 8 字二级分类名称 MUST 不遮挡相邻分类、商品列表、导航栏或其他操作区。
- [ ] AC-BUG-004 二级分类长名称展示区域 MUST 不出现仅展示前 4 个字后省略为 `...` 且无法辨识完整含义的问题。
- [ ] AC-BUG-005 点击长名称二级分类后 MUST 进入对应分类商品列表，分类 ID、选中态和商品列表筛选结果必须正确。
- [ ] AC-BUG-006 长名称二级分类在微信开发者工具、iOS 真机、Android 真机和窄屏设备下 MUST 布局稳定。
- [ ] AC-BUG-007 修复不得引入一级分类、二级分类切换、商品列表加载、空态展示或返回导航回归。
- [ ] AC-BUG-008 若仅调整小程序前端展示样式且不改变接口契约，MUST 明确说明不需要 OpenAPI / Orval；若实际修改 API 字段或响应结构，MUST 同步 OpenAPI、Orval、接口文档和测试。

## 验收证据要求

| 类型 | 要求 |
|---|---|
| 小程序截图 | 覆盖 4 字以内、5-8 字、超过 8 字二级分类名称 |
| 真机证据 | 至少覆盖 iOS 或 Android 真机之一；如条件允许同时覆盖两端 |
| 交互证据 | 点击长名称二级分类后进入正确商品列表 |
| 布局证据 | 证明长名称不遮挡相邻分类、商品列表和页面操作区 |
| 回归证据 | 一级分类切换、二级分类选中态、商品列表加载和空态展示仍正常 |

## 非目标

- 本 BUG 不要求新增分类管理能力。
- 本 BUG 不要求调整分类 API 字段。
- 本 BUG 不要求修改数据库分类表结构。
- 本 BUG 不要求改变商品列表排序、筛选或分页规则。
- 本 BUG 不要求新增购物车、购买、下单、支付、询价或营销能力。
