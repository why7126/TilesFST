# 现象

微信小程序分类页中，二级分类名称超过 4 个字时无法完整显示，会被截断并显示为省略号 `...`。用户在分类页无法直接识别完整二级分类名称，影响分类浏览和后续进入对应商品列表的判断。

该问题发生在已交付的分类列表页能力范围内，关联需求为 `REQ-0045-category-list-page`。当前缺陷聚焦二级分类名称展示，不涉及新增分类能力。

# 复现步骤

1. 打开微信小程序。
2. 进入分类页。
3. 查看包含二级分类的分类区域。
4. 找到名称超过 4 个字的二级分类，例如 5 字、6 字或更长名称。
5. 观察二级分类名称在页面中的展示。
6. 点击该二级分类，确认是否能进入对应商品列表。

# 期望 vs 实际

期望：

- 二级分类名称超过 4 个字时，用户仍能在分类页识别完整或足够可辨识的分类名称。
- 页面可以通过换行、合理宽度、字号、布局调整或其他展示方式处理较长名称。
- 长名称展示不应遮挡相邻分类、商品列表、导航栏或其他操作区。
- 点击长名称二级分类后，仍应进入正确分类商品列表。

实际：

- 二级分类名称超过 4 个字后被截断为省略号 `...`。
- 用户无法从分类页直接判断该二级分类的完整含义。
- 暂未确认省略是否只影响展示，还是同时影响点击热区、选中态或商品列表入口识别。

# 影响范围

| 范围 | 影响 |
|---|---|
| 微信小程序分类页 | 二级分类长名称不可完整识别 |
| 二级分类浏览 | 用户难以区分名称前缀相近的分类 |
| 商品列表入口 | 用户可能误点或无法判断应进入哪个分类商品列表 |
| 多屏适配 | 窄屏设备、字体放大或较长分类名称下更容易暴露 |
| 已交付分类页验收 | 与分类页应清晰展示分类入口的体验目标存在偏差 |

不预期影响：

- 后端分类 API 数据结构。
- 分类层级本身。
- 商品列表接口。
- 管理端分类维护能力。

# 严重等级说明

严重等级建议为 `medium`。

该问题不阻断分类页打开，也不一定阻断二级分类点击；但它会直接影响小程序分类页核心浏览体验，使用户无法准确识别超过 4 个字的二级分类名称。分类页是商品浏览入口之一，长名称省略会降低用户选择效率，并可能造成分类入口误判，因此需要进入后续修复流程。

# 建议后续处理

1. `/bug-complete BUG-0077-miniapp-category-secondary-name-truncated` 补齐 root-cause、workaround、acceptance。
2. 在微信开发者工具和真机上补充截图证据，覆盖 4 字以内、5-8 字、超过 8 字分类名称。
3. 修复时优先检查分类页二级分类文本样式是否存在固定宽度、单行限制或 `text-overflow: ellipsis`。
4. 修复后需回归分类点击、选中态、商品列表入口和不同屏幕宽度下的布局稳定性。
