# Root Cause - BUG-0077 微信小程序分类页二级分类名称超过 4 个字被省略

## 直接原因

微信小程序分类页的二级分类名称展示区域对文本长度做了单行截断或固定宽度限制。当二级分类名称超过 4 个字时，文本无法获得足够展示空间，最终以省略号 `...` 呈现。

## 根本原因

根本原因初步判断为分类页二级分类展示组件的长文本适配规则不足：

- 二级分类名称展示更偏向短标签场景设计，未覆盖 5 字及以上中文分类名称。
- 样式可能使用了固定宽度、单行显示、`text-overflow: ellipsis` 或等价小程序样式策略。
- 分类页验收更关注分类层级、入口跳转和商品列表联动，缺少“长分类名称可辨识”的明确验收约束。
- 多屏宽、字体大小和真实分类数据下的视觉回归证据不足，导致长名称展示问题在交付后暴露。

由于本阶段不修改 `src/`，上述根因需在后续修复阶段结合小程序分类页样式与组件代码进一步确认。

## 触发条件

满足以下条件时可稳定触发：

1. 打开微信小程序分类页。
2. 当前一级分类下存在二级分类。
3. 二级分类名称超过 4 个中文字符。
4. 二级分类名称展示区域宽度不足，且文本被限制为单行或固定宽度。

## 影响判断

该问题影响已交付分类列表页的核心浏览体验。它不阻断分类页打开，也不必然阻断二级分类点击，但会让用户无法辨识超过 4 个字的二级分类名称，尤其在多个分类名称前缀相近时容易造成误判。修复应在保证布局稳定和点击入口正确的前提下，让长名称可读或至少足够可辨识。
