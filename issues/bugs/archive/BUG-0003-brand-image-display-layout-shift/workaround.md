# 临时规避方案

## 1. 图片显示规避

当前没有稳定的产品内规避方案可以让品牌 Logo 在列表页和编辑弹窗中正常展示。

在正式修复前，内部运营可临时采用以下方式降低影响：

1. 上传品牌 Logo 后，以品牌名称、简称、英文名称和排序作为主要确认信息。
2. 若必须确认图片内容，可在上传前保留本地文件记录，并在品牌备注或操作记录中标注对应文件名。
3. 暂不把品牌 Logo 展示作为品牌资料维护完成的唯一判断依据。

限制：

- 以上做法不能证明上传后的对象已可被系统访问。
- 不能保证前台、小程序或后续 SKU 关联场景能正确展示品牌 Logo。
- 不能替代媒体访问链路修复。

## 2. Tips 布局波动规避

当前页面仍可继续执行品牌状态变更，但操作时应注意：

1. 启用、停用、删除或保存后，等待顶部 Tips 自动消失再继续进行下一次精细点击操作。
2. 避免在 Tips 出现或消失的瞬间连续点击表格操作按钮，降低误点风险。
3. 验收时将页面上下波动单独标注为未通过项，不把状态变更接口本身误判为失败。

限制：

- 该规避只能减少误操作概率，不能消除页面位移。
- 对需要连续维护大量品牌的运营场景仍有明显体验影响。

## 3. 验收规避

在正式修复前，品牌管理相关验收应明确拆分：

- 品牌文本字段、排序、启停、删除等数据操作可单独验证。
- 品牌 Logo 上传后的列表展示和编辑回显应标记为本 BUG 未通过。
- 状态变更 Tips 不应作为通过项，需等待修复后重新验收页面稳定性。

## 4. 风险说明

本缺陷涉及媒体可访问性和管理端布局稳定性，临时规避不能消除以下风险：

- 上传成功但用户无法确认图片是否真正可展示。
- 后续前台展示、小程序展示或 SKU 关联品牌展示可能继承同一媒体 URL 问题。
- 状态提示继续造成页面位移，影响连续操作体验。

因此建议进入 `/bug-review`，评审通过后通过 `fix-*` OpenSpec Change 修复。
