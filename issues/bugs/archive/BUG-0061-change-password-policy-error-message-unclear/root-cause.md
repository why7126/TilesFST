# 根因分析

## 1. 结论

该缺陷属于 Web 管理端修改密码流程的错误反馈设计与实现缺陷。后端已具备可配置密码策略校验能力，但策略失败原因被折叠为单一 `policy` 结果；前端修改密码弹窗仍使用固定的旧规则提示与泛化错误文案，未能展示当前生效策略的具体失败项。

因此用户在输入不满足策略的新密码后，只能看到“新密码不符合安全策略”，无法知道应补足长度、大写字母、小写字母、数字、特殊字符，还是避开弱密码。

## 2. 直接原因

### 2.1 后端策略失败被合并为同一错误类型

`validate_password_policy()` 对以下策略失败均返回 `"policy"`：

- 长度小于 `policy.min_length` 或超过 `policy.max_length`。
- 缺少大写字母。
- 缺少小写字母。
- 缺少数字。
- 缺少特殊字符。

服务层收到 `"policy"` 后统一抛出 `PasswordPolicyError`，默认错误消息为“新密码不符合安全策略”。该消息没有携带失败项列表，也没有说明下一步如何修改。

### 2.2 前端本地规则与后端当前策略不一致

修改密码弹窗当前前端规则为：

- 8-32 位字符。
- 至少包含字母和数字。
- 不能与原密码相同。

但后端当前默认策略为：

- 最小长度 12。
- 必须包含大写字母。
- 必须包含小写字母。
- 必须包含数字。
- 必须包含特殊字符。

前端没有读取或同步当前有效密码策略，也没有展示大写、小写、特殊字符等规则状态。用户可能按前端提示输入了看似合规的密码，但仍被后端拒绝。

### 2.3 API 错误响应缺少结构化策略详情

统一错误响应当前可表达 `code`、`message`、`data`。修改密码策略失败使用错误码 `40021`，但没有返回类似 `violations`、`failed_rules` 或可展示的规则说明。

这使前端只能展示后端的泛化 `message`，无法根据服务端判定结果精准提示“缺少特殊字符”等具体原因。

## 3. 根本原因

密码策略从固定规则演进为系统设置可配置规则后，修改密码链路没有同步完成以下配套设计：

- 前端规则展示未从“静态提示”升级为“当前有效策略提示”。
- 后端校验未从“布尔/单码失败”升级为“可解释失败项”。
- API 契约未定义策略失败详情的结构化返回格式。
- 测试只覆盖错误码或泛化文案，未覆盖“具体失败原因必须可见”。

这不是单一文案问题，而是策略事实源、API 错误表达和前端可用性反馈之间的契约缺口。

## 4. 触发条件

满足以下任一条件时可稳定触发：

1. 用户输入长度不足的新密码并提交。
2. 用户输入缺少大写字母、小写字母、数字或特殊字符的新密码并提交。
3. 用户输入满足前端静态规则但不满足后端当前默认策略的新密码，例如包含字母和数字但缺少特殊字符。
4. 管理员调整系统安全策略后，前端展示规则未随有效策略变化。
5. 用户输入命中弱密码表的常见密码。

## 5. 分类

| 维度 | 分类 |
|---|---|
| 缺陷类型 | design / code / frontend-ui / api-contract |
| 影响层级 | Web 管理端、修改密码 API 错误表达 |
| 主要前端模块 | `src/web/src/features/admin/components/ChangePasswordModal.tsx` |
| 主要后端模块 | `src/backend/app/core/password_validation.py`、`src/backend/app/core/exceptions.py` |
| 接口层 | 需要补充策略失败详情或更具体 message |
| 数据库层 | 不涉及表结构变更 |
| 权限层 | 不涉及权限绕过 |
| 是否回归 | 更偏向配置化策略演进后的体验债务，不是单点近期回归 |

## 6. 修复方向建议

后续 `bug-opsx` / `opsx-apply` 可考虑：

1. 后端密码策略校验返回具体失败项，例如 `min_length`、`missing_uppercase`、`missing_lowercase`、`missing_digit`、`missing_special`、`same_as_old`、`weak`。
2. `PasswordPolicyError` 或等价异常携带可展示详情；若涉及 API 契约，需同步 OpenAPI、Orval、API 文档与测试。
3. 前端修改密码弹窗展示当前有效策略，至少覆盖长度、大小写、数字、特殊字符、不能与原密码相同、弱密码提示。
4. 前端错误提示应能展示一条或多条具体原因，并提供修改建议。
5. 补充后端测试断言策略失败详情，补充前端测试断言具体规则文案可见。
