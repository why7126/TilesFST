# 修改密码安全策略错误提示不清晰

## 1. 现象

Web 管理端用户在“修改密码”弹窗中提交不满足密码安全策略的新密码时，系统仅提示“新密码不符合安全策略”。该提示没有说明具体失败原因，也没有提供可操作的修改建议，导致用户只能反复试错。

典型不清晰场景包括：

- 新密码长度不足或超过上限。
- 新密码缺少大写字母、小写字母、数字或特殊字符。
- 新密码与原密码相同。
- 新密码命中常见弱密码。
- 当前系统安全策略被配置调整后，前端展示规则与后端实际校验规则不一致。

## 2. 复现步骤

1. 登录 Web 管理端。
2. 打开“修改密码”弹窗。
3. 输入正确的原密码。
4. 输入一个不满足密码安全策略的新密码，例如长度不足、缺少特殊字符或缺少必需字符类型的密码。
5. 提交修改密码表单。

## 3. 期望结果

系统应明确指出新密码未满足的具体策略，并给出可操作修改建议。

示例：

- “新密码至少需要 12 位字符。”
- “新密码需要包含大写字母。”
- “新密码需要包含小写字母。”
- “新密码需要包含数字。”
- “新密码需要包含特殊字符。”
- “新密码不能与原密码相同。”
- “该密码过于常见，请更换为更复杂的密码。”

如果密码策略支持后台配置，前端规则提示应与当前生效策略保持一致，避免用户按界面提示输入后仍被后端拒绝。

## 4. 实际结果

系统仅显示泛化错误提示“新密码不符合安全策略”，用户无法判断具体不符合哪一条规则，也无法确认下一次应如何修改。

## 5. 影响范围

| 范围 | 影响 |
|---|---|
| Web 管理端 | 影响“修改密码”弹窗的新密码错误提示体验。 |
| 后端 API | `POST /api/v1/admin/profile/password` 在策略失败时返回粗粒度错误信息。 |
| 管理员 / 员工账号 | 需要自助修改密码的内部用户会受到影响。 |
| 小程序 | 当前未发现直接影响。 |
| 数据库 | 当前缺陷不涉及表结构变化。 |
| 媒体 / MinIO | 当前缺陷不涉及上传或对象存储。 |

## 6. 严重等级说明

严重等级：`medium`。

理由：

- 该问题不会直接阻断所有用户登录，也不构成权限绕过或数据泄露。
- 该问题会显著增加修改密码失败后的试错成本，影响管理端账号安全配置的可用性。
- 当系统密码策略被配置为更严格规则时，前端提示与后端校验不一致会放大困惑，容易造成用户误判系统异常。
