# 现象

管理端 Shell 中的「界面主题」选择器当前固定显示在页面右上角内容区。用户期望该控件作为全局偏好设置入口放入左侧侧边栏，并位于底部用户头像/账号信息块上方。

当前布局会让主题选择器占用页面顶部操作区域，也使全局设置入口与侧边栏中的用户区域分离，破坏管理端 Shell 的信息层级一致性。

# 复现步骤

1. 以管理员身份登录 Web 管理端。
2. 进入任意管理端页面，例如「接口文档」。
3. 查看页面右上角是否出现「界面主题 / 系统默认」选择器。
4. 查看左侧侧边栏底部管理员头像与账号信息块上方是否存在主题选择入口。

# 期望结果

- 「界面主题」选择器 MUST 显示在左侧侧边栏内。
- 主题选择器 MUST 位于侧边栏底部用户头像/账号信息块上方。
- 页面右上角内容区 MUST 不再显示主题选择器。
- 控件视觉样式 MUST 延续管理端 Design System，并与侧边栏信息密度、间距、对齐方式保持一致。

# 实际结果

- 「界面主题」选择器显示在页面右上角内容区。
- 左侧侧边栏底部用户头像/账号信息块上方没有主题选择入口。
- 页面顶部操作区域被全局主题偏好控件占用，与期望的侧边栏布局不一致。

# 影响范围

- 影响端：Web 管理端。
- 影响页面：使用管理端 Shell 的页面。
- 影响对象：管理员、内部员工。
- 影响类型：全局偏好入口位置、页面顶部操作区布局、侧边栏信息层级。

# 严重等级说明

严重等级为 `medium`。

该问题不阻断登录、浏览或管理数据，也不影响 API、数据库或权限校验；但它影响管理端全局 Shell 的布局一致性与用户操作路径，且会在所有管理端页面暴露，因此按中等严重等级处理。
