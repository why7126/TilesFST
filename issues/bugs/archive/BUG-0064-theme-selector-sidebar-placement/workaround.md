# 临时规避方案

## 1. 操作规避

修复前，管理员仍可使用页面右上角的「界面主题」选择器切换主题。该控件当前位置不符合期望，但不阻断主题切换能力。

建议运营或验收人员临时按以下方式处理：

1. 进入任意管理端页面。
2. 在页面右上角找到「界面主题」选择器。
3. 按需选择「系统默认」或其他主题选项。
4. 验收时明确标注：主题切换能力可用，但入口位置未通过本 BUG 验收。

## 2. 无配置规避

当前没有产品内配置、环境变量或权限开关可将主题选择器移动到侧边栏。刷新页面、切换浏览器或更换账号不会改变控件位置。

## 3. 培训说明

修复前如需向内部员工说明，可临时告知：

- 主题选择入口暂时位于页面右上角；
- 后续会移动到左侧侧边栏用户头像上方；
- 主题切换结果本身不受该布局问题影响。

## 4. 规避有效期

- **有效期**：自 2026-07-11 起至本 BUG 对应 fix change 发布并验收通过。
- **修复后**：应以侧边栏用户头像上方的主题选择器为准，页面右上角不再保留该控件。

## 5. 风险说明

| 风险 | 说明 |
|---|---|
| 功能阻断 | 无；主题切换仍可操作 |
| 误操作 | 低；控件含明确标签和选项 |
| 布局一致性 | 中；所有管理端页面均暴露右上角位置偏差 |
| 培训成本 | 中；全局偏好入口位置与用户预期不一致 |
