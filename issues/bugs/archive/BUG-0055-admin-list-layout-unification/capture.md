# 现象

管理端多个已交付页面的列表布局和交互不统一，涉及瓷砖 SKU、瓷砖品牌、瓷砖类目、瓷砖规格、Banner 管理、用户管理、日志审计、接口文档等页面。

用户期望这些页面统一为同一套管理端列表页结构与交互规范：

1. 模块顺序统一为：标题模块、指标卡模块、筛选/搜索模块、列表模块。
2. 筛选/搜索模块交互与样式以瓷砖 SKU 页为准。
3. 全部页面不需要【查询】功能；多余查询按钮应移除。
4. 重置按钮尺寸保持一致。
5. 列表模块最后一列统一固定浮动，以接口文档页为准。
6. 分页功能统一最多展示 5 个可点击页码；当前只有一个页码时仍应保持分页控件一致性。

# 复现步骤

1. 打开 Web 管理端。
2. 依次进入「瓷砖 SKU」「瓷砖品牌」「瓷砖类目」「瓷砖规格」「Banner 管理」「用户管理」「日志审计」「接口文档」页面。
3. 对比各页面的标题区、指标卡区、筛选/搜索区和列表区顺序。
4. 对比筛选/搜索区是否存在多余【查询】按钮、重置按钮尺寸是否一致。
5. 横向滚动或查看表格末列，确认最后一列操作区是否按接口文档页样式固定浮动。
6. 查看分页控件，确认可点击页码最多是否为 5 个，且当前只有一个页码时展示是否一致。

# 期望 vs 实际

- 期望：上述管理端列表页统一采用「标题模块 → 指标卡模块 → 筛选/搜索模块 → 列表模块」顺序；筛选/搜索模块以瓷砖 SKU 页为交互和样式基线；无【查询】按钮；重置按钮尺寸一致；表格最后一列按接口文档页固定浮动；分页最多展示 5 个可点击页码。
- 实际：不同列表页在模块顺序、筛选按钮、重置按钮尺寸、末列固定浮动和分页页码展示上存在不一致，影响管理端操作习惯和视觉一致性。

# 影响范围

- Web 管理端：瓷砖 SKU、瓷砖品牌、瓷砖类目、瓷砖规格、Banner 管理、用户管理、日志审计、接口文档。
- 主要影响管理端列表页视觉一致性、筛选操作效率、表格操作列可达性与分页体验。
- 不涉及后端接口、数据库、MinIO、媒体上传、小程序。
