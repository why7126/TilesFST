# 临时规避方案

## 1. 可用性规避（有限）

在正式修复前，用户可尝试以下方式**部分**访问被裁切内容，但均不可靠，**不能作为验收通过依据**：

1. **最大化浏览器窗口**或提高视口高度（如外接显示器全屏），使表单总高度落入可视区。
2. **缩小浏览器缩放比例**（如 90% / 80%），间接减少内容占用像素高度。
3. 使用 DevTools **Responsive 模式**临时拉高视口（仅开发调试，非运营路径）。

上述方式在含多图/多视频时仍可能无法触达底部字段，且 footer 按钮可能不可点击。

## 2. 功能规避

**无可靠功能规避。** 若底部「保存草稿 / 创建SKU / 保存」不可见或不可点，则无法完成 SKU 创建/编辑闭环。

仅填写顶部基础字段无法替代完整业务操作（图片、视频、备注等位于表单中下部）。

## 3. 验收规避

在修复前，REQ-0006 弹窗相关 AC（尤其 **AC-022**、AC-024～AC-036、AC-055）**不得标记通过**。

验收报告 MUST 标注：BUG-0011 阻塞矮视口下弹窗可操作性验收。

## 4. 风险说明

- 运营人员在小屏或分屏办公环境下**无法维护 SKU 主数据**。
- 阻塞 `add-tile-sku-management` 产品验收与 Sprint 收尾。
- 临时放大窗口不能消除 accessibility / 可用性缺陷，必须进入修复流程。

## 5. 建议优先级

该缺陷 severity 为 **high**，建议在 BUG-0009/0010（medium UI 一致性）之前或同批 Sprint 内优先修复。
