# Root Cause - BUG-0070 小程序商品详情页底部品牌按钮与内容区查看品牌主页重复

## 直接原因

SKU 商品详情页在内容区已经提供“查看品牌主页”入口后，底部操作区仍保留了品牌按钮。两个入口语义一致，预期都指向品牌主页，导致同一页面出现重复品牌主页入口。

## 根本原因

根本原因是 SKU 商品详情页品牌入口的信息架构没有在迭代调整后统一收敛：

- 内容区品牌模块已经承担“查看品牌主页”的主入口职责。
- 底部操作区没有随内容区入口落地同步裁撤同类品牌按钮。
- 验收要点更关注品牌主页可达性，缺少“品牌主页入口唯一性”和“底部操作区不得保留重复入口”的明确约束。

因此，页面最终同时保留了内容区入口和底部操作入口，形成重复交互。

## 触发条件

满足以下条件时可稳定触发：

1. 打开微信小程序 SKU 商品详情页。
2. 当前 SKU 关联品牌信息，页面内容区展示“查看品牌主页”入口。
3. 页面底部操作区仍渲染品牌按钮。

## 影响判断

该问题影响已交付的 SKU 商品详情页操作区清晰度。它不阻断品牌主页访问，也不影响商品详情基础浏览；但重复入口会让用户误以为两个按钮承担不同职责，并增加底部操作区视觉负担。修复应删除底部品牌按钮，保留内容区“查看品牌主页”入口，并回归底部操作区布局稳定性。
