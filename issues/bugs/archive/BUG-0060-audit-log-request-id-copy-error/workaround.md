# BUG-0060 临时规避

## 当前可用规避

在修复完成前，管理员或排障人员可通过以下方式临时获取 `request_id`：

1. 在日志审计列表中将鼠标悬停到短展示的 `request_id` 上，查看浏览器 title 中的完整值。
2. 打开日志详情抽屉，在详情字段中查找完整 `request_id`。
3. 使用日志审计页的“路径 / request_id”筛选框，通过已知关键词或接口路径缩小范围后手动记录。
4. 如页面可选择文本，可手动选中 `request_id` 文本后使用系统复制快捷键。

## 局限

- 手动记录容易漏字符，不适合高频排障。
- 短展示值不足以替代完整 `request_id`。
- 若 title 或详情抽屉在特定样式/浏览器下不可复制，仍会影响排障效率。

## 建议修复前提示

如需要对外说明，可提示用户：

> 当前日志审计页复制按钮在部分浏览器环境下可能失败，请先通过日志详情查看完整 request_id 并手动复制。

## 是否可长期接受

不可长期接受。

`request_id` 复制是 REQ-0024 日志审计能力的明确验收点，临时规避只能用于修复前的低频排障场景。
