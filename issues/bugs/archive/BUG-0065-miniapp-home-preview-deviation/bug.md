# BUG-0065 微信小程序首页预览效果与 REQ-0041 原型和验收差异明显

## 现象

用户在微信开发者工具中预览微信小程序首页时，实际页面与 `REQ-0041-miniapp-home` 原型和验收标准差异明显。

当前预览截图中仅能看到：

- 顶部小程序导航标题“菲尚特瓷砖”。
- 页面内“门店信息”入口。
- 搜索框“搜索瓷砖名称、编号或规格”。
- 空商品提示“暂无公开商品，欢迎联系门店咨询。”。
- “品牌服务”标题。
- 底部 TabBar。

未按预期展示：

- 品牌 Logo 与门店名称组合导航。
- 首屏 Banner。
- “按空间、按规格、按风格、按颜色、全部分类”快捷入口。
- 新品推荐商品卡片。
- 热销推荐商品卡片。
- 完整品牌服务卡片。

## 复现

1. 使用微信开发者工具打开项目 `src/miniapp/`。
2. 进入首页 `pages/index/index`。
3. 在模拟器中预览首页首屏。
4. 对比 `issues/requirements/archive/REQ-0041-miniapp-home/prototype/miniapp/prototype.png` 与当前预览截图。

## 期望结果

首页首屏应与 REQ-0041 原型保持同等级布局和视觉表现，至少展示品牌导航、搜索入口、Banner、快捷找砖入口和一个推荐模块，并在有公开商品/Banner 数据时正常展示内容。

在 375x812、390x844 以及 320 到 430 pt 宽度范围内，页面不应出现大面积空白、内容缺失、控件重叠或底部 TabBar 遮挡。

## 实际结果

首页首屏未展示 Banner、快捷入口、新品推荐、热销推荐和完整服务区，空态占据大面积首屏区域，整体视觉气质、模块密度和内容完整性均明显偏离原型。

从当前文件结构观察，首页页面存在 `index.ts` 与 `index.js` 并存情况；`index.ts` 包含首页数据加载与交互逻辑，而 `index.js` 仍是空模板。该线索需在 `/bug-complete` 阶段继续确认根因。

## 影响范围

| 影响面 | 说明 |
|---|---|
| 微信小程序首页 | 首屏核心模块缺失，无法形成“品牌数字展厅”的入口体验 |
| 内容加载 | Banner、快捷入口、新品/热销推荐和服务卡片未正常出现在预览中 |
| 用户路径 | 搜索、快捷筛选、Banner 跳转、推荐商品详情、门店咨询等路径可用性需要复核 |
| 验收标准 | 可能不满足 REQ-0041 的 AC-002、AC-006、AC-007、AC-008、AC-009、AC-UI-001、AC-STATE-003 |
| 交付信心 | 当前预览效果不适合作为终端客户数字展厅首屏验收版本 |

## 严重等级说明

严重等级为 `high`。

理由：该缺陷发生在微信小程序首页首屏，直接影响终端客户进入小程序后的第一屏体验，并阻断 REQ-0041 的核心验收目标。虽然不涉及数据破坏或安全泄露，但对本期小程序首页交付质量和验收通过率影响较大，应优先进入后续 `/bug-complete` 与修复评审流程。
