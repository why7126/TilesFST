# 临时规避方案

## 1. 可用规避

修复前，不建议将当前微信小程序首页预览作为 REQ-0041 的验收版本。

如必须临时演示，可按以下方式降低误判风险：

1. 先说明当前小程序首页存在运行入口缺陷，预览不代表最终验收效果。
2. 使用 `issues/requirements/archive/REQ-0041-miniapp-home/prototype/miniapp/prototype.png` 作为视觉目标参考。
3. 同时展示后端 `GET /api/v1/miniapp/home` 聚合接口测试结果，说明有数据时接口可返回公开 Banner、推荐商品和服务区字段。
4. 对微信开发者工具预览截图单独标注为“BUG-0065 待修复状态”，不得作为验收通过证据。

## 2. 开发调试规避

开发人员本地排查时可临时确认：

- 微信开发者工具实际加载的是 `pages/index/index.js` 还是由 `.ts` 编译生成的运行脚本。
- `pages/index/index.js` 是否包含首页 `data`、`onLoad()` 和 `loadHome()`。
- 本地后端 `http://localhost:8000/api/v1/miniapp/home` 是否可访问。
- 本地数据中是否存在 `PUBLISHED` 商品与 `ONLINE` 且 `display_client` 命中小程序的 Banner。

这些操作只能用于定位和演示，不应替代正式 fix change。

## 3. 无配置规避

当前没有产品内配置、环境变量或后台开关可以直接恢复首页首屏完整展示。刷新微信开发者工具、切换 TabBar 或重新进入首页，不能从根本上修复空运行脚本问题。

## 4. 风险说明

| 风险 | 说明 |
|---|---|
| 验收误判 | 高；当前预览明显不符合原型，不应签收 |
| 用户体验 | 高；终端客户首屏看不到品牌展厅核心内容 |
| 数据判断 | 中；容易误以为只是后端无商品数据，但实际还存在运行入口脱节 |
| 修复绕行 | 中；手工复制代码到 `.js` 可临时验证，但必须通过正式 Change 和测试固化 |

## 5. 规避有效期

- **有效期**：自 2026-07-16 起至本 BUG 对应 fix change 发布并验收通过。
- **修复后**：以微信开发者工具真实预览和自动化回归结果为准，不再依赖原型图或接口测试作为首页完整展示的替代证据。
