# Workaround - BUG-0080 管理端 Banner 列表和弹窗中 Banner 图片显示不全

## 临时规避方案

在正式修复前，可以通过以下方式降低运营误判风险：

1. 上传或选择 Banner 图片前，先在本地图片查看器中确认完整素材内容和关键文字位置。
2. 若管理端列表或弹窗预览显示不全，优先以原始素材文件或可访问的图片 URL 为准进行二次确认。
3. 对包含关键文字、品牌 Logo 或活动信息的 Banner，尽量将关键信息放在图片中心安全区域，减少边缘裁切带来的误判。
4. 编辑已有 Banner 时，记录 Banner 标题、图片来源、原始文件名或素材库位置，便于修复后回归核对。
5. 上线前增加人工复核：至少同时检查管理端弹窗预览和实际展示端效果。

## 不建议的规避方式

| 方式 | 原因 |
|---|---|
| 反复压缩或强行改图比例 | 可能破坏正式运营素材质量，且无法解决管理端预览策略问题 |
| 直接修改生产数据图片字段 | 会绕过管理端上传和审计链路，增加数据不一致风险 |
| 仅依赖列表缩略图判断素材 | 当前列表缩略图正是受影响位置，不能作为唯一确认依据 |

## 运营提醒

该问题暂不阻断 Banner 的创建、编辑和保存，但会降低预览可信度。正式修复前，涉及重要活动、品牌露出、价格/促销文案的 Banner 应额外核对原始素材和展示端效果。

## 修复前验证建议

| 验证项 | 说明 |
|---|---|
| 原始图片 | 保存至少一张横幅图、一张方图、一张竖图和一张超宽图作为回归素材 |
| 受影响位置 | 分别记录列表缩略图和弹窗预览的表现 |
| 视口尺寸 | 记录桌面宽屏和较窄管理端视口下是否均复现 |
| 图片来源 | 标记图片来自自定义上传、品牌 Logo 还是 SKU 主图 |
