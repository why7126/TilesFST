# Root Cause - BUG-0080 管理端 Banner 列表和弹窗中 Banner 图片显示不全

## 直接原因

初步判断，Banner 列表缩略图与 Banner 弹窗图片预览使用了不适合“完整预览”的图片适配策略：图片容器可能采用固定宽高、裁切型 `object-fit`、父容器隐藏溢出或表格/弹窗局部高度限制，导致图片主体或边缘内容被裁掉。

当前缺陷尚未进入代码修复阶段，以上为基于用户现象和管理端图片预览常见实现风险的初步判断，后续 `/bug-opsx` 或修复阶段需要用具体 DOM/CSS、截图和不同图片比例验证。

## 根本原因

Banner 管理链路同时承载列表快速识别和弹窗编辑确认两个目标，但现有图片预览规则未明确区分：

- 列表缩略图应优先保证图片主体可识别。
- 弹窗预览应优先保证素材完整可确认。
- 不同比例图片需要统一、可预期的展示策略，避免为了填满容器而裁切关键内容。

因此，若当前实现复用了裁切型缩略图样式，或为保持行高/弹窗布局而强行隐藏溢出，就会在列表和弹窗两个位置同时出现“显示不全”的表现。

## 触发条件

| 条件 | 说明 |
|---|---|
| 图片比例与容器比例不一致 | 横幅图、方图、竖图或超宽图进入固定比例容器时容易被裁切 |
| 列表缩略图固定尺寸 | 表格列宽、行高或图片框高度固定，且图片以覆盖方式填充 |
| 弹窗预览容器受限 | 弹窗图片区域固定高度或被上传组件父容器裁切 |
| 上传后回显 | 新图上传后立即使用同一预览样式，问题会在编辑确认阶段暴露 |

## 分类

| 分类 | 判断 |
|---|---|
| code | 是，疑似前端图片预览组件或页面 CSS 适配问题 |
| design | 是，需要明确列表缩略图与弹窗预览的展示策略 |
| api | 否，当前未发现接口字段或响应契约异常 |
| db | 否，当前未发现数据持久化异常 |
| media | 部分相关，涉及图片展示但暂未发现上传或对象存储异常 |

## 需补充证据

| 证据 | 用途 |
|---|---|
| 问题截图 | 确认列表与弹窗具体裁切位置 |
| Banner 图片原始尺寸 | 判断是否由图片比例触发 |
| 浏览器视口尺寸 | 排除窄屏或弹窗滚动区域导致的偶发问题 |
| DOM/CSS 截图 | 确认 `object-fit`、容器尺寸、overflow 与表格行高 |
| 回显来源 | 区分自定义上传图、品牌 Logo、SKU 主图等来源是否一致受影响 |

## 风险判断

该缺陷主要影响管理端运营预览准确性，不直接修改 Banner 业务数据，也暂未发现影响前台展示端真实投放。但如果运营人员因预览不完整而误判素材，可能导致错误 Banner 上线、返工或漏检素材关键内容。
