# BUG-0080 管理端 Banner 列表和弹窗中 Banner 图片显示不全

## 现象

管理端 Banner 管理页面中，Banner 图片在列表和弹窗两个关键预览位置均显示不全：

1. Banner 列表中的图片缩略图无法完整呈现素材内容。
2. 新建或编辑 Banner 弹窗中的图片预览无法完整呈现当前图片。

该问题会导致运营人员在列表浏览、编辑确认或上传后回显时无法准确判断 Banner 素材是否正确。

## 复现步骤

1. 登录 Web 管理端。
2. 进入 Banner 管理页面。
3. 查看列表中已有 Banner 的图片缩略图。
4. 新建或编辑一条 Banner，打开 Banner 弹窗。
5. 查看弹窗中的 Banner 图片预览区域。
6. 使用不同图片比例复核，包括横幅图、方图、竖图和超宽图。

## 期望结果

- Banner 列表缩略图应完整呈现图片主体，不裁掉关键信息。
- Banner 弹窗中的图片预览应完整展示当前图片，便于运营确认上传或已选素材。
- 不同比例图片应使用一致、可预期的适配策略，避免明显拉伸、压扁或比例失真。
- 修复后列表行高、弹窗滚动、上传控件和保存按钮布局应保持稳定。

## 实际结果

- Banner 列表缩略图存在显示不全问题，图片主体或边缘内容可能被裁切。
- Banner 弹窗图片预览同样存在显示不全问题，影响新建、编辑和保存前确认。
- 当前缺少截图、具体 Banner ID、图片原始尺寸和浏览器视口信息，后续探索阶段需要补齐。

## 影响范围

| 范围 | 影响 |
|---|---|
| Web 管理端 | 运营人员无法准确确认 Banner 素材内容 |
| Banner 列表 | 缩略图无法完整预览，可能误判当前 Banner 图片 |
| Banner 弹窗 | 新建或编辑时无法完整确认上传或已选图片 |
| 展示端数据 | 暂未发现影响 API、数据库或实际投放数据，当前主要为管理端预览展示问题 |

## 严重等级说明

严重等级：`medium`。

理由：该问题不阻断 Banner 创建、编辑或保存，也暂未发现影响展示端实际投放数据；但它发生在运营人员高频使用的管理端 Banner 维护链路，会影响素材识别、编辑确认和上线前检查。如果素材被误判，可能间接导致错误 Banner 上线或运营返工，因此评估为中等严重度。

## 初步排查方向

| 方向 | 说明 |
|---|---|
| CSS 适配策略 | 检查图片是否使用 `object-cover`、固定高度或父容器 `overflow: hidden` 导致裁切 |
| 列表容器 | 检查缩略图列宽、行高、图片容器比例和表格布局限制 |
| 弹窗容器 | 检查弹窗预览区域高度、宽度、滚动区域和上传组件样式 |
| 图片来源 | 覆盖自定义上传图、品牌 Logo、SKU 主图等 Banner 图片来源 |
| 视口差异 | 覆盖桌面宽屏、较窄管理端视口和弹窗滚动场景 |

## 回归建议

- 覆盖已有 Banner 列表预览。
- 覆盖新建 Banner 上传图片后的预览。
- 覆盖编辑已有 Banner 的图片回显。
- 覆盖横幅图、方图、竖图、超宽图等不同比例图片。
- 确认修复不影响 Banner 保存、上线/下线、排序和跳转类型配置。
