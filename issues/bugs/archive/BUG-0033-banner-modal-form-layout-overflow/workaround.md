# 临时规避方案

## 1. 可用性规避（有限）

在正式修复前，用户可尝试以下方式**部分**访问被裁切内容，但均不可靠，**不能作为验收通过依据**：

1. **最大化浏览器窗口**或提高视口高度（如外接显示器全屏），使表单总高度落入 `92vh` 可视区。
2. **缩小浏览器缩放比例**（如 90% / 80%），间接减少内容占用像素高度。
3. 选择字段较少的 `jump_type`（如 `NO_JUMP`）以减少表单高度——仍可能在矮视口下触发 footer 裁切。

上述方式在 `SKU_DETAIL` 分支（含图片区 + SKU 搜索/下拉）时仍可能无法触达底部按钮。

## 2. 功能规避

**无可靠功能规避。** 若底部「取消 / 保存 Banner」不可见或不可点，则无法完成 Banner 创建/编辑闭环。

运营备注宽度不足**不阻断**文本输入与保存（数据可写入后端），但矮视口下 footer 不可达时整个保存流程被阻断。

## 3. 验收规避

在修复前，REQ-0016 弹窗相关 AC 中以下项 **不得标记通过**：

- **AC-024**（640px / 92vh / 内容可滚动）
- **AC-051**（四套 modal HTML/PNG 并排，含运营备注整行与 footer 可达）
- 涉及弹窗可操作性的矮视口验收

验收报告 MUST 标注：BUG-0033 阻塞矮视口下 Banner 弹窗可操作性验收。

## 4. 风险说明

- 运营人员在笔记本分屏或小屏环境下**无法维护 Banner**。
- 阻塞 REQ-0016 产品验收中与弹窗布局相关的 AC。
- 临时放大窗口不能消除 accessibility / 可用性缺陷，必须进入修复流程。

## 5. 建议优先级

该缺陷 severity 为 **high**，建议在 BUG-0031/0032（low/medium 文案类）之前或同批 Sprint 内与弹窗 scroll 一并优先修复；可与 BUG-0031～0035 合并为单一 `fix-banner-modal-ui` Change 降低回归成本。
