# Workaround - BUG-0074 生产环境主题偏好同步失败提示持续不消失

## 临时规避方案

在正式修复前，可采用以下用户侧规避方式：

1. 主题本机已生效时，可继续使用当前页面；该提示不代表本机主题切换失败。
2. 如果常驻提示遮挡或干扰操作，可刷新页面或重新进入管理端页面清除当前页面状态。
3. 如需账号偏好跨会话生效，可稍后重新切换一次主题，或在修复后再确认账号偏好保存状态。

## 运维 / 客服说明

如生产用户反馈该提示，可说明：

1. 当前本机主题切换已生效，问题集中在账号偏好同步与错误提示关闭体验。
2. 常驻提示不会修改瓷砖、品牌、Banner、媒体等业务数据。
3. 建议用户先刷新页面清除提示，并继续使用管理端。
4. 请收集浏览器 Network 中 `PATCH /api/v1/auth/me/theme` 的状态码、响应体、用户账号和发生时间，便于定位生产同步失败原因。

## 运维排查建议

优先排查以下项目：

| 项目 | 操作 |
|---|---|
| 登录态 | 确认用户访问生产管理端时 Token 未过期，主题保存请求包含 Authorization Header |
| 接口状态 | 在生产环境确认 `PATCH /api/v1/auth/me/theme` 是否返回 200 |
| 后端日志 | 按用户账号与时间检索认证、用户偏好更新和数据库异常日志 |
| 数据库字段 | 确认生产 `users.theme_mode` 字段存在且允许 `system`、`dark_flagship`、`comfort_dark`、`light` |
| 反向代理 | 确认 Web Nginx `/api/` 转发到后端服务，未被外层网关改写或拦截 |

## 风险

临时规避只能清除页面上的常驻提示，不能保证账号主题偏好已保存。若生产同步失败来自接口、鉴权或数据库迁移问题，用户刷新或重新登录后仍可能回到旧主题偏好。

不得通过关闭鉴权、跳过后端保存、手工修改真实用户数据或绕过 API 契约来规避该问题。
