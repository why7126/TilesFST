# 缺陷说明

## 1. 现象

在 Web 管理端编辑品牌弹窗中，用户点击「更换 Logo」并选择图片后，系统没有明显展示上传过程，Logo 预览也没有更新。用户无法判断图片是否已经开始上传、是否上传成功，或需要等待多久。

## 2. 复现步骤

1. 进入 Web 管理端品牌管理页。
2. 打开一个已有品牌的编辑弹窗。
3. 点击「更换 Logo」。
4. 在系统文件选择窗口中选择一张图片并打开。
5. 观察弹窗中的 Logo 预览区域、按钮状态和上传反馈。

## 3. 期望结果

- 选择图片后，系统应立即触发 Logo 上传。
- 上传过程中应显示进度条、百分比或等价进度反馈，使用户可以感知等待时长。
- 上传成功后，弹窗 Logo 预览应更新为新图片。
- 上传失败时，应展示明确错误信息，并保留重新选择/重试入口。

## 4. 实际结果

- 选择图片后没有明显上传反馈。
- Logo 预览没有更新。
- 缺少上传进度条或等待态，用户无法判断上传是否发生或是否仍在等待。

## 5. 影响范围

| 维度 | 影响 |
|---|---|
| Web 管理端 | 影响 `/admin/brands` 编辑品牌弹窗 |
| 品牌主数据 | 影响品牌 Logo 更换与运营确认 |
| 用户体验 | 用户无法感知上传等待时长，容易重复点击或误以为操作失败 |
| 后端 API | 待分析；当前缺陷表现主要在前端上传触发、进度反馈和预览更新链路 |
| 数据库 | 暂未发现直接影响 |
| 小程序 | 不涉及 |

## 6. 严重等级说明

严重等级暂定为 `medium`。

理由：

- 缺陷影响品牌 Logo 更换这一常用编辑流程，且用户缺少上传进度反馈。
- 当前未明确阻断品牌文本字段编辑、品牌保存或列表浏览。
- 若后续确认实际完全未调用上传接口或保存后 Logo 无法更新，可在评审阶段上调严重等级。
