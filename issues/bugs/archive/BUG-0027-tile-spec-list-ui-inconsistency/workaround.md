# 临时规避方案

## 1. 可用性规避

该缺陷不阻断规格管理功能，当前可继续使用：

1. 规格列表分页仍可通过 ‹ / › 按钮与每页条数下拉完成翻页。
2. 筛选、查询、重置、新增规格、编辑、启停、删除等操作均不受影响。
3. 尺寸名称列字号偏大不影响数据阅读与行内操作。

## 2. 验收规避

在正式修复前，验收时应明确标注：

- 规格列表底部分页与用户管理页分页存在视觉/结构差异，**暂不作为 REQ-0009 AC-042 通过项**。
- 尺寸名称列字号与同表其他列不一致，**暂不作为 UI 一致性通过项**。
- 仅验证规格列表查询、分页、CRUD、启停、条件删除功能本身是否可用。

## 3. 运营规避

内部管理员可继续按现有路径维护规格：

1. 进入「瓷砖规格」列表页。
2. 使用现有筛选与分页控件定位目标规格。
3. 通过「新增瓷砖规格」或行内「编辑」维护主数据。

## 4. 风险说明

该规避方案只能保证功能可用，不能消除以下问题：

- 管理端列表页视觉一致性不足（与用户管理、品牌、SKU 分页不对齐）。
- REQ-0009 列表原型并排验收（AC-045）可能因分页差异无法完全通过。
- 后续新增管理端列表页可能继续 invent 非标准分页 DOM。

因此仍建议进入 `/bug-review`，评审通过后通过 `fix-*` OpenSpec Change 修复。
