# Acceptance - BUG-0071 登录页右上角主题选择模块与语言选择模块没有对齐

## 回归验收标准

- [ ] AC-BUG-001 Web 管理端登录页 MUST 同时展示主题选择模块与语言选择模块。
- [ ] AC-BUG-002 主题选择模块与语言选择模块 MUST 在统一的登录页工具区布局中保持清晰的视觉对齐。
- [ ] AC-BUG-003 两个模块的垂直对齐、控件高度、视觉重心和交互热区 MUST 保持一致，不得出现上下错位。
- [ ] AC-BUG-004 桌面视口下，两个模块之间的水平间距 MUST 稳定，右上角工具区不得重叠、裁切或异常换行。
- [ ] AC-BUG-005 窄屏视口下，两个模块 MUST 保持可见且布局稳定，不得遮挡登录标题、表单字段或安全提示。
- [ ] AC-BUG-006 修复后主题切换能力 MUST 正常工作，切换主题后页面 `data-theme-mode` / `data-theme` 表现不回退。
- [ ] AC-BUG-007 修复后语言选择按钮 MUST 保持可点击和可访问名称，不得丢失 `aria-label`。
- [ ] AC-BUG-008 修复不得改变登录接口、认证流程、账号密码表单提交、记住登录状态或登录后跳转行为。
- [ ] AC-BUG-009 若仅调整 Web 登录页布局且不改变接口契约，MUST 明确说明不需要 OpenAPI / Orval；若实际修改 API 字段或响应结构，MUST 同步 OpenAPI、Orval、接口文档和测试。

## 验收证据要求

| 类型 | 要求 |
|---|---|
| 桌面截图 | 覆盖登录页右上角主题选择模块与语言选择模块对齐状态 |
| 窄屏截图 | 覆盖窄屏视口下工具区不重叠、不裁切、不遮挡表单 |
| 交互证据 | 主题选择切换后页面主题正常变化 |
| 可访问性证据 | 语言选择按钮仍可通过 `aria-label="切换语言"` 定位 |
| 回归证据 | 登录表单提交、记住登录状态、登录后跳转不受影响 |

## 非目标

- 本 BUG 不要求实现新的多语言切换能力。
- 本 BUG 不要求新增主题模式。
- 本 BUG 不要求改变登录页品牌面板、登录表单字段或认证策略。
- 本 BUG 不要求修改后端登录接口、用户模型或数据库结构。
