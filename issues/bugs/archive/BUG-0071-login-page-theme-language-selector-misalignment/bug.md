# 现象

Web 登录页右上角同时展示主题选择模块与语言选择模块，但两个模块没有在同一行或同一视觉基线上对齐，导致登录页工具区出现明显错位。

从现有实现看，语言选择按钮位于登录表单面板右上角，主题选择器位于登录卡片内部靠右区域。两个控件不在同一个布局容器中，页面渲染后容易出现水平位置、垂直位置或视觉重心不一致。

# 复现步骤

1. 打开 Web 管理端登录页。
2. 查看页面右上角的语言选择模块。
3. 查看登录卡片顶部附近的主题选择模块。
4. 对比两个模块的顶部位置、垂直居中效果、右侧边界和视觉基线。
5. 在桌面视口和较窄视口下重复观察工具区布局。

# 期望 vs 实际

期望：

- 登录页右上角主题选择模块与语言选择模块位于统一的工具区布局中。
- 两个模块在同一行内保持一致的垂直对齐、视觉高度和交互热区。
- 桌面和窄屏视口下，两个模块之间的间距稳定，不出现错位、重叠、裁切或异常换行。

实际：

- 语言选择模块固定在登录表单面板右上角。
- 主题选择模块位于登录卡片内部的顶部区域。
- 两个模块使用不同布局容器与坐标系，导致右上角工具区视觉不对齐。

# 影响范围

- 影响终端：Web。
- 影响页面：管理端登录页。
- 影响区域：登录页右上角工具区、登录卡片顶部区域。
- 影响用户体验：登录页首屏出现控件错位，降低界面精致度和一致性，尤其在用户首次进入管理端时较明显。
- 不预期影响：登录认证流程、账号密码提交、主题切换逻辑、语言切换按钮点击行为。

# 严重等级说明

严重等级建议为 `medium`。

该问题不阻断登录主流程，也不影响主题或语言控件的基础可达性；但它属于登录页首屏可见 UI 偏差，影响管理端入口页面的一致性与专业感，需要在常规缺陷修复流程中对工具区布局做回归。
