# Root Cause - BUG-0071 登录页右上角主题选择模块与语言选择模块没有对齐

## 直接原因

登录页右上角语言选择模块与主题选择模块分别放在不同布局容器中：

- 语言选择模块位于登录表单面板右上角，由面板级绝对定位控制。
- 主题选择模块位于登录卡片内部顶部区域，由卡片内部行布局控制。

两个控件不共享同一个工具区容器、右侧边界、垂直对齐规则和响应式策略，因此页面渲染后会出现视觉对齐偏差。

## 根本原因

根本原因是登录页右上角工具区的布局职责没有统一收敛：

- 语言选择模块沿用了登录页面板级绝对定位方案。
- 主题选择模块后续新增到登录卡片内部，没有接入同一工具区布局。
- 现有验收与单测只覆盖控件是否存在，没有覆盖两个控件在桌面和窄屏视口下的对齐关系。

因此，两个控件虽然都属于登录页辅助工具，但实际由不同层级布局独立控制，形成了可见错位。

## 触发条件

满足以下条件时可稳定触发：

1. 打开 Web 管理端登录页。
2. 登录页同时渲染语言选择模块与主题选择模块。
3. 对比两个控件在右上角区域的顶部位置、右侧边界或视觉基线。
4. 切换到桌面视口或窄屏视口重复观察。

## 影响判断

该问题影响 Web 管理端登录页首屏观感和工具区一致性。它不阻断登录流程，也不影响主题切换或语言按钮的基础可达性；但登录页是管理端入口页面，控件错位会降低界面专业感。修复应统一主题选择模块与语言选择模块的布局容器或对齐规则，并回归桌面和窄屏视口。
