# 临时规避方案

## 1. 可用性规避

该缺陷不阻断密码修改功能，当前可继续使用：

1. **阅读错误文案语义**：若提示含「新密码」字样（如「新密码过于常见，请更换」「新密码不符合安全策略」），应修改「新密码」字段而非原密码，即使提示显示在原密码下方。
2. **原密码错误单独识别**：仅当提示为「原密码不正确」时，才需检查原密码输入。
3. **多次尝试**：调整新密码后重新提交，API 与策略逻辑正常，最终可修改成功。

## 2. 操作规避

无配置、脚本或后端变更可规避字段错位；需等待前端 fix change 合并。

## 3. 风险说明

上述规避依赖用户自行理解文案与字段对应关系，无法消除：

- 误以为原密码有误而反复修改原密码。
- 增加改密失败次数，可能触发频率限制（42901）。
- 与 BUG-0025 叠加时，原密码字段「显示/隐藏」按钮可能错位，进一步降低可用性。

仍建议进入 `/bug-review BUG-0024 --approve`，通过 `fix-change-password-modal-errors` OpenSpec Change 修复字段映射。
