# 临时规避方案

## 1. 可用性规避

该缺陷不阻断 Banner 关联 SKU / 专题的保存，当前可继续使用：

1. 将「跳转类型」设为「SKU 详情」或「专题页」后，在上方搜索框输入 SKU 名称/编码或专题名称/编码。
2. 按 **Enter** 触发搜索，下方下拉框选项会刷新。
3. 在下方 `<select>` 中选择目标 SKU 或专题，完成其余字段后保存 Banner。

若默认下拉已含目标项，可跳过搜索框，直接在 `<select>` 中选择。

## 2. 验收规避

在正式修复前，验收 REQ-0016 弹窗时应明确标注：

- 「关联 SKU / 关联专题」双控件交互 **暂不作为 AC-031 / AC-036 UX 通过项**。
- 搜索须按 Enter、选择与搜索分离为已知 UI 缺陷，**不影响** keyword API 与保存 payload（`sku_id` / `topic_id`）功能验收。
- 编辑模式 select 空白 **暂不作为数据丢失**，可通过搜索 Enter 后重新选择同一目标。

## 3. 运营规避

内部管理员配置 Banner 时：

1. 记住「先搜后选」两步：搜索框输入 → Enter → 下拉选择。
2. 编辑已有 Banner 时，若关联目标显示空白，在搜索框输入已知 SKU/专题名称后 Enter，再在下拉中重新选中。
3. 新建 Banner 时优先从默认下拉前 20 条中选择，减少搜索步骤。

## 4. 风险说明

该规避方案只能保证功能可用，不能消除：

- 操作割裂、效率低，易误以为搜索框即选择框。
- 编辑模式无法直观确认当前关联目标，可能误改跳转链接。
- REQ-0016 弹窗 prototype 并排验收（SKU 详情 / 专题页 modal PNG）可能因双控件结构不通过。

因此仍建议进入 `/bug-review`，评审通过后通过 `fix-*` OpenSpec Change 修复。
