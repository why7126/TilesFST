# 临时规避方案

## 1. 可用性规避

该缺陷不阻断保存与资料更新，当前可继续使用：

1. **任选其一按钮保存**：页头与表单底部「保存修改」行为完全相同，点击任一即可触发 PATCH 与 inline save-tip。
2. **固定使用习惯入口**：建议始终使用表单底部按钮（与「重置」、成功提示同区），忽略页头重复 CTA。
3. **读屏用户**：两个按钮文案相同，辅助技术会列出两个「保存修改」；任选其一功能等价。

## 2. 操作规避

无额外配置或脚本规避；无需改后端、环境变量或数据库。

## 3. 风险说明

上述规避只能保证功能可用，不能消除：

- 页面上两个相同主按钮造成的视觉噪音与选择困惑。
- 页头 CTA 与表单底 save-tip 分离，成功反馈与触发按钮可能不在同一视觉区域（若使用页头按钮）。

仍建议进入 `/bug-review BUG-0023 --approve`，通过 `fix-profile-page-ux-refine` OpenSpec Change 收敛为单入口。
