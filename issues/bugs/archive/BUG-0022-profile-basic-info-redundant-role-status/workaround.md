# 临时规避方案

## 1. 可用性规避

该缺陷不阻断资料编辑、保存或权限查看，当前可继续使用：

1. **忽略表单内重复字段**：「所属角色」「账号状态」为只读，不参与 PATCH；用户可仅关注昵称、邮箱、手机、备注等可编辑项。
2. **以账号安全卡片为准**：右侧卡片含状态 badge 与角色，可作为权限/状态的唯一权威阅读区。
3. **身份条快速扫读**：身份条 meta 与 gold badge 亦含角色摘要，无需再读表单底部只读 input。

## 2. 操作规避

无必要操作步骤；表单重复字段不影响「保存修改」「重置」「更换头像」流程。

## 3. 风险说明

上述规避只能保证功能可用，不能消除：

- 表单视觉冗长、可编辑字段聚焦度下降。
- 新用户可能困惑「为何两处显示相同信息」。
- 与产品期望的信息架构（角色/状态单区展示）不符。

仍建议进入 `/bug-review BUG-0022 --approve`，通过 `fix-profile-page-ux-refine`（或等价 fix-* Change）正式修复并归档。
