# 临时规避方案

## 1. 可用性规避

该缺陷不阻断 SKU 管理功能，当前可继续使用：

1. SKU 列表分页仍可通过上一页、当前页、下一页与每页条数选择完成翻页。
2. 筛选、查询、重置、新增 SKU、编辑、上下架、删除等操作均不受影响。
3. 多余的「SKU 列表」标题行不影响表格数据阅读与行内操作。

## 2. 验收规避

在正式修复前，验收时应明确标注：

- SKU 列表底部分页与用户管理页分页存在视觉/结构差异，**暂不作为 AC-051 / AC-054 通过项**。
- 表格卡片内重复标题行与 REQ-0006 原型及用户管理页结构不一致，**暂不作为 UI 一致性通过项**。
- 仅验证 SKU 列表查询、分页、CRUD 功能本身是否可用。

## 3. 运营规避

内部管理员可继续按现有路径维护 SKU：

1. 进入「瓷砖 SKU」列表页。
2. 使用现有筛选与分页控件定位目标 SKU。
3. 通过「新增 SKU」或行内「编辑」维护商品资料。

## 4. 风险说明

该规避方案只能保证功能可用，不能消除以下问题：

- 管理端列表页视觉一致性不足（与用户管理、品牌管理分页不对齐）。
- REQ-0006 原型并排验收（AC-054）可能因分页/布局差异无法通过。
- 后续新增管理端列表页可能继续复制 `page-left` / `brand-pagination-right` 废弃模式。

因此仍建议进入 `/bug-review`，评审通过后通过 `fix-*` OpenSpec Change 修复。
