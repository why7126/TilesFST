# 临时规避方案

## 1. 操作习惯规避（推荐）

在正式修复前，四列表页仍可正常完成状态变更与 CRUD；运营可采取：

1. 执行启用/停用、冻结、上下架、删除等操作后，**等待顶部 Tips 自动消失（约 3.2 秒）** 再进行下一次精细点击。
2. 避免在 Tips 出现或消失瞬间连续点击表格行内操作按钮，降低误点相邻行的风险。
3. 连续批量维护时，以接口/数据结果（列表刷新、状态 badge 变化）为准，不依赖 Tips 位置判断操作是否成功。

**品牌页**：当前 fixed toast 已不推挤主体，可按正常节奏操作。

## 2. 无产品内技术规避

本缺陷为纯前端布局模式问题，**不存在**配置开关、环境变量或 API 参数可关闭文档流 Tips。

以下路径**不能**消除布局抖动：

| 路径 | 说明 |
|---|---|
| 刷新页面 | 仅清除当前 Tips，下次操作仍会抖动 |
| 改浏览器缩放 | 不改变 notice 文档流插入行为 |
| 跳过 Tips 触发 | 无法跳过；成功/失败反馈为既有 UX |

## 3. 验收规避

修复前相关验收应标注：

- 数据操作（启停、冻结、上下架、CRUD）可单独验证通过；
- **页面布局稳定性**（Tips 不推挤 hero/表格）应标记为本 BUG 未通过项；
- 品牌页 toast 稳定性若已满足 BUG-0003 AC-004，可单独注明「品牌页已通过，其余三页待 BUG-0015」。

## 4. 规避有效期

- **有效期**：自 2026-06-27 起至 `fix-admin-list-status-toast-layout` 发布并验收通过。
- **修复后**：四页 MUST 统一 fixed toast；本规避仅作修复前参考。

## 5. 风险说明

| 风险 | 说明 |
|---|---|
| 体验 | 用户/类目/SKU 连续操作时页面上下跳动，影响效率与操作准确性 |
| 功能 | **不阻断** API 调用、数据写入或权限校验 |
| 安全/数据 | 无 |
| 回归 | 品牌页若在未统一改造时误改，可能回退至文档流 notice |

severity **medium**，建议与 BUG-0016/BUG-0017 等同 Sprint 编排，但可独立 fix-* 发布。
