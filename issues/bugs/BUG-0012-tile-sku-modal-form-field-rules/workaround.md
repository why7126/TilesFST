# 临时规避方案

## 1. 表面工艺留空时的规避

在正式修复前，若运营希望跳过表面工艺校验完成创建/编辑：

1. **创建 SKU**：在「表面工艺」字段填写任意占位文本（如 `-` 或 `待补充`）后再提交。
2. **保存草稿**：仅填 SKU 名称即可保存草稿；后端会将空表面工艺写入 `"-"`（但后续「创建 SKU」完整校验仍会要求填写表面工艺）。

> 注意：即使填写 `-`，当前上架逻辑仍可能拒绝 `surface_finish == "-"` 的记录，需先完善工艺或使用草稿状态。

## 2. 参考价格未填时的规避

1. 新建或编辑时手动填写参考价格（可为 `0`）。
2. 若已保存为 `null`，编辑该 SKU 并补填价格后保存。

列表中对 `null` 价格显示「—」，不影响其他列操作。

## 3. 验收规避

在 BUG-0012 修复前，SKU 弹窗字段规则验收应标注：

- **表面工艺必填、参考价格选填** 为当前实现行为（对齐 REQ-0006 v4 旧文档），**不作为 UAT 新产品规则通过项**。
- 产品方期望的「表面工艺非必填、参考价格必填默认 0」待本 BUG fix 后验收。

## 4. 风险说明

规避方案不能保证与产品最新规则一致：

- 运营可能误填占位工艺，上架时仍可能被 `publish_sku` 拦截。
- 未填参考价格的 SKU 在列表长期显示「—」，与「默认 0 元」预期不符。
- 前后端规则不一致（草稿 vs 正式创建）增加培训成本。

因此仍建议 `/bug-review` 通过后通过 `fix-tile-sku-modal-form-field-rules` 正式修复。
