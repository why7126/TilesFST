# 原型说明

本 REQ 不交付新的小程序页面或组件视觉稿。`prototype.html` 仅作为 best-practice 文档结构、页面接入 checklist 和截图验收矩阵的信息架构原型，用于后续 `/req-review` 与 `/req-opsx` 确认文档内容边界。

## 原型目标

- 表达 best-practice 文档应包含的状态栏、胶囊、返回兜底、页面 offset 和截图矩阵章节。
- 表达新增页面接入时的 checklist 结构。
- 表达 DevTools / 真机 evidence 需要分层记录，不等同于实际小程序 UI 实现。

## 非目标

- 不作为小程序页面视觉稿。
- 不要求转换为 WXML / WXSS。
- 不验证真实设备安全区。
- 不替代 REQ-0052 的完整 evidence 模板。
