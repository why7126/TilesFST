# 微信小程序商品列表页改为双列商品卡片展示需求文档

## 1. 背景

微信小程序商品列表页当前承接分类、首页推荐、品牌或搜索等入口，已具备列表加载、商品卡片、搜索入口、筛选入口、排序和筛选抽屉等能力。实际产品体验上，商品列表页更适合作为轻量浏览页：用户进入后应直接看到商品卡片，而不是先面对搜索、筛选和排序控件。

本需求对 `REQ-0047-product-list-common-component-application` 的商品列表页展示策略进行收敛：商品列表页不再展示搜索功能和筛选功能，商品卡片信息与首页“热销推荐”保持一致，并采用一行 2 个的双列卡片布局。搜索页自身的关键词检索能力不在本需求内移除。

## 2. 目标用户

| 用户 | 核心诉求 |
|---|---|
| 装修客户 | 从分类、首页推荐或品牌入口进入后，快速扫读商品图、名称和价格，继续进入详情了解 |
| 设计师 | 在移动端以更高密度浏览商品候选，减少控件干扰 |
| 门店导购 | 向客户展示商品集合时，优先呈现商品卡片内容，降低解释成本 |
| 普通访客 | 不需要理解搜索和筛选规则，也能直接浏览当前入口下的商品 |

## 3. 范围

### 3.1 包含

- 微信小程序商品列表页 `pages/product-list/index` 的展示策略调整。
- 商品列表页不展示搜索框、搜索按钮、筛选按钮、筛选 chips 和筛选抽屉入口。
- 商品列表页不展示排序 tabs；列表默认按入口或后端既有默认顺序展示。
- 商品列表页商品卡片采用与首页“热销推荐”一致的信息结构和视觉密度。
- 商品列表页采用一行 2 个商品卡片的双列布局。
- 商品列表页继续支持入口上下文参数，例如 `categoryId`、`categoryName`、`categoryLevel`、`brandId`、`section`、`keyword` 和 `sourcePage`，用于初始结果查询和标题/空态展示。
- 商品列表页继续保留首屏加载、下拉刷新、上拉加载更多、无更多数据、空状态、错误状态和商品详情跳转。

### 3.2 不包含

- 微信小程序搜索页 `pages/search/index` 的搜索、筛选或结果页能力调整。
- Web 管理端、店主 Web 展示端或其他终端的商品列表页调整。
- 后台商品管理、上下架、库存、价格、批量操作等管理能力。
- 新增商品、分类、品牌或筛选项数据模型。
- 新增复杂推荐排序、人工置顶、个性化推荐或瀑布流算法。
- 商品卡片新增收藏、询价、购物车、在线下单或客服找砖能力。

## 4. 功能要求

### FR-001 商品列表页信息架构收敛

- 商品列表页 MUST 不展示搜索框、搜索按钮或跳转搜索页的搜索入口。
- 商品列表页 MUST 不展示筛选按钮、筛选 chips、筛选抽屉或筛选确认/重置入口。
- 商品列表页 MUST 不展示排序 tabs 或用户可切换排序入口。
- 商品列表页 SHOULD 仅保留页面标题、状态反馈、商品卡片列表和必要的加载/错误/空态操作。
- 若入口携带 `keyword`，页面 MAY 使用关键词生成标题或空态文案，但不得在页面内提供继续搜索或二次筛选控件。

### FR-002 双列商品卡片布局

- 商品列表页 MUST 以一行 2 个商品卡片展示列表内容。
- 双列布局 MUST 在常见小程序手机宽度下保持稳定间距，不因商品名称、价格或图片加载导致错位。
- 商品卡片 SHOULD 复用现有 `product-card` 组件的 `grid` 展示密度，避免重复维护一套卡片结构。
- 商品图片区域 MUST 使用稳定比例，图片加载前后不得引起明显布局跳动。
- 单数商品数量时，最后一张卡片 MUST 保持左侧自然排列，不拉伸为整行。

### FR-003 卡片信息对齐首页热销推荐

- 商品列表页卡片 MUST 参照首页“热销推荐”的商品卡片展示主图、商品名称、品牌或规格信息、参考价格和状态徽标。
- 商品名称 MUST 优先展示 SKU 名称或商品名称，并在双列卡片中限制行数，避免撑破卡片。
- 品牌、规格和价格缺失时 MUST 使用现有卡片组件的兜底文案或隐藏策略，不得出现空白异常字段。
- 商品主图加载失败时 MUST 展示统一占位图或占位背景，不展示破图。
- 整个商品卡片 MUST 可点击并进入 SKU 详情页，跳转参数与既有商品卡片能力保持一致。

### FR-004 入口上下文与默认查询

- 分类入口进入商品列表页时 MUST 继续支持 `categoryId`、`categoryName` 和 `categoryLevel` 初始查询。
- 首页“新品推荐”“热销推荐”查看更多入口进入商品列表页时 MUST 继续支持 `section=new` 或 `section=hot` 初始查询。
- 品牌入口进入商品列表页时 SHOULD 继续支持 `brandId` 初始查询。
- 搜索页或其他入口若传入 `keyword`，商品列表页 MUST 能基于该关键词展示初始结果，但不得提供页面内搜索框。
- 查询参数无效、分类/品牌下架或结果为空时 MUST 展示可恢复空态，不得白屏。

### FR-005 分页、刷新与状态

- 商品列表页首屏 MUST 请求第一页商品数据。
- 下拉刷新 MUST 清空旧分页游标并重新请求第一页。
- 上拉加载更多 MUST 防止重复请求。
- 接口返回无更多数据时 MUST 展示统一无更多状态。
- 首屏请求失败时 MUST 展示错误文案和重试入口。
- 加载更多失败时 MUST 保留已加载商品，并提供重试加载更多入口。

### FR-006 埋点与兼容

- 商品列表页浏览、商品曝光、商品点击、刷新、加载更多和加载失败埋点 SHOULD 保留。
- 与搜索、筛选、排序相关的商品列表页交互埋点 SHOULD 停止触发；搜索页自身埋点不受本需求影响。
- 商品列表页埋点公共参数 SHOULD 继续包含 `sourcePage`、`categoryId`、`categoryName`、`categoryLevel`、`brandId`、`keyword`、`section`、`page`、`pageSize`、`resultCount`、`skuId` 和 `requestId`。
- 本需求 SHOULD 尽量不引入后端 API 变更；如实现发现现有接口强依赖筛选/排序字段，后续 OpenSpec Change 中必须说明兼容策略。

## 5. UI 约束

- 视觉风格 MUST 与菲尚特小程序“工业石材 · 暗色旗舰风”保持一致。
- 商品列表页双列卡片 SHOULD 与首页“热销推荐”的网格密度、卡片信息层级和暗色金色强调风格一致。
- 页面首屏 SHOULD 优先露出商品卡片，避免由搜索、筛选、排序等控件占用主要浏览空间。
- 商品卡片点击区域 SHOULD 不小于 44px，适配小程序触控场景。
- 双列卡片中的商品名称、品牌/规格和价格 MUST 不互相遮挡，长文本应截断或限制行数。
- 加载态、空状态、错误态和无更多状态 MUST 与现有小程序列表页语义保持一致。
