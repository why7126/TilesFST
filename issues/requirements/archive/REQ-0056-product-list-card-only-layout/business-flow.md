# 业务流程

## 1. 主流程

```text
用户入口
  ├─ 分类页：categoryId/categoryName/categoryLevel
  ├─ 首页推荐：section=hot/new
  ├─ 品牌入口：brandId
  └─ 搜索页跳转：keyword
        |
        v
微信小程序商品列表页 pages/product-list/index
        |
        ├─ 读取入口上下文并生成标题/空态文案
        ├─ 请求第一页商品数据
        ├─ 渲染双列商品卡片（每行 2 个）
        ├─ 下拉刷新 / 上拉加载更多
        └─ 点击商品卡片进入 SKU 详情页
```

## 2. 页面职责收敛

```text
旧 REQ-0047 商品列表页职责
  商品浏览 + 搜索入口 + 筛选入口 + 排序入口 + 分页状态机

REQ-0056 商品列表页职责
  商品浏览 + 入口上下文 + 双列卡片 + 分页状态机

搜索 / 筛选 / 排序
  保留在搜索页或入口前置页面，不在商品列表页继续暴露控件
```

## 3. 与父需求差异

| 维度 | REQ-0047 已定义 | REQ-0056 收敛后 |
|---|---|---|
| 搜索入口 | 商品列表页可展示搜索入口 | 商品列表页不展示搜索入口 |
| 筛选入口 | 商品列表页可展示筛选按钮、筛选 chips、筛选抽屉 | 商品列表页不展示筛选能力 |
| 排序入口 | 商品列表页可展示排序 tabs | 商品列表页不展示用户可切换排序入口 |
| 卡片布局 | 单列大卡或双列卡片待确定 | 明确一行 2 个双列卡片 |
| 卡片信息 | 通用商品卡片字段 | 对齐首页热销推荐信息结构和视觉密度 |
| 保留能力 | 分页、刷新、空态、错误态、详情跳转 | 继续保留 |

## 4. 异常与状态流

```text
首屏加载
  ├─ 成功且有商品 -> 双列卡片
  ├─ 成功但无商品 -> 当前上下文空态
  └─ 失败 -> 错误文案 + 重新加载

上拉加载更多
  ├─ 成功且 hasMore -> 追加商品
  ├─ 成功且 noMore -> 已经到底了
  └─ 失败 -> 保留已有商品 + 点击重试
```

## 5. 设备验收提醒

基于 `docs/knowledge-base/retrospectives/sprint-008-retrospective.md` 的小程序复盘经验，本需求后续实现验收应单独确认 320 / 375 / 430 pt 视口下双列卡片不遮挡、不溢出，且自定义导航、页面标题、底部 TabBar 与列表内容不重叠。
