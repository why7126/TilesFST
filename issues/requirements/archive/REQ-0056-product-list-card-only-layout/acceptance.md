# 验收标准

## 功能 AC

- [ ] AC-001 商品列表页不展示搜索框、搜索按钮或跳转搜索页的搜索入口。
- [ ] AC-002 商品列表页不展示筛选按钮、筛选 chips、筛选抽屉或筛选确认/重置入口。
- [ ] AC-003 商品列表页不展示排序 tabs 或用户可切换排序入口。
- [ ] AC-004 商品列表页可接收 `categoryId`、`categoryName`、`categoryLevel`、`brandId`、`section`、`keyword` 和 `sourcePage` 作为入口上下文。
- [ ] AC-005 分类入口进入商品列表页后，页面按分类上下文展示初始商品结果。
- [ ] AC-006 首页“新品推荐”“热销推荐”查看更多入口进入商品列表页后，页面按 `section=new` 或 `section=hot` 展示初始商品结果。
- [ ] AC-007 搜索页或其他入口传入 `keyword` 时，商品列表页可展示关键词初始结果，但不提供页面内二次搜索控件。
- [ ] AC-008 商品列表页首屏请求第一页商品数据。
- [ ] AC-009 下拉刷新清空旧分页游标并重新请求第一页。
- [ ] AC-010 上拉加载更多防止重复请求。
- [ ] AC-011 接口返回无更多数据时展示统一无更多状态。
- [ ] AC-012 首屏请求失败时展示错误文案和重试入口。
- [ ] AC-013 加载更多失败时保留已加载商品，并提供重试加载更多入口。
- [ ] AC-014 查询参数无效、分类/品牌下架或结果为空时展示可恢复空态，不得白屏。
- [ ] AC-015 整个商品卡片可点击并进入 SKU 详情页，跳转参数与既有商品卡片能力一致。
- [ ] AC-016 商品列表页浏览、商品曝光、商品点击、刷新、加载更多和加载失败埋点保留。
- [ ] AC-017 商品列表页不触发搜索、筛选、排序相关交互埋点；搜索页自身埋点不受影响。

## UI AC

- [ ] AC-UI-001 商品列表页采用一行 2 个商品卡片的双列布局。
- [ ] AC-UI-002 双列布局在 320 / 375 / 430 pt 视口下保持稳定间距，商品卡片不互相遮挡、不横向溢出。
- [ ] AC-UI-003 商品卡片复用现有 `product-card` 组件的 `grid` 展示密度或等价视觉，不重复实现一套不一致卡片。
- [ ] AC-UI-004 商品卡片信息结构对齐首页“热销推荐”，展示主图、商品名称、品牌或规格信息、参考价格和状态徽标。
- [ ] AC-UI-005 商品图片区域使用稳定比例，图片加载前后无明显布局跳动。
- [ ] AC-UI-006 商品主图加载失败时展示统一占位图或占位背景，不展示破图。
- [ ] AC-UI-007 商品名称、品牌/规格和价格在双列卡片中不互相遮挡，长文本截断或限制行数。
- [ ] AC-UI-008 单数商品数量时，最后一张卡片保持左侧自然排列，不拉伸为整行。
- [ ] AC-UI-009 页面首屏主要空间用于商品卡片，不被搜索、筛选、排序控件占用。
- [ ] AC-UI-010 商品卡片点击区域不小于 44px。
- [ ] AC-UI-011 视觉风格与菲尚特小程序“工业石材 · 暗色旗舰风”保持一致。
- [ ] AC-UI-012 自定义导航、页面标题、底部 TabBar 与商品列表内容不重叠。

## 横切 AC（knowledge-base）

无横切 AC。本需求为微信小程序商品列表展示策略，不命中 `admin-list`、`admin-form`、`admin-modal`、`media-upload` 标签；管理端列表、表单、弹窗、上传相关 best-practices 不适用。

参考复盘：`docs/knowledge-base/retrospectives/sprint-008-retrospective.md`。后续实现验收需关注小程序设备/视口证据、运行入口 `.ts` / `.js` 同步和页面组件边界清晰。
