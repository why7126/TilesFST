# 用户故事

## US-001 分类入口直接浏览商品

作为装修客户，我希望从分类页进入商品列表后直接看到双列商品卡片，而不是先看到搜索和筛选控件，以便快速判断当前分类下是否有合适商品。

验收要点：

- 打开分类商品列表页时，首屏不展示搜索框、筛选入口或排序 tabs。
- 商品列表按一行 2 个卡片展示。
- 商品卡片可进入 SKU 详情页。

## US-002 首页推荐查看更多延续热销样式

作为普通访客，我希望从首页“热销推荐”点击查看更多后，看到与首页热销推荐一致的信息结构和卡片密度，以便在熟悉的视觉模式下继续浏览。

验收要点：

- `section=hot` 入口进入商品列表页后保留热销榜上下文标题。
- 商品卡片展示主图、名称、品牌/规格、参考价格和状态徽标。
- 列表卡片视觉密度与首页热销推荐保持一致。

## US-003 门店导购快速展示商品集合

作为门店导购，我希望商品列表页减少搜索、筛选、排序控件干扰，让客户第一眼看到更多商品，以便现场讲解时更顺畅。

验收要点：

- 首屏主要空间用于商品卡片。
- 长商品名、缺图、缺价格时不破坏双列布局。
- 下拉刷新和上拉加载更多仍可正常使用。

## US-004 搜索能力边界清晰

作为已经在搜索页完成搜索的用户，我希望商品列表页可以承接关键词结果，但不再出现二次搜索或筛选入口，以便页面职责清晰。

验收要点：

- 商品列表页可基于入口传入的 `keyword` 展示初始结果。
- 页面内不展示继续搜索、筛选抽屉或排序切换。
- 搜索页自身的搜索和筛选能力不受本需求影响。
