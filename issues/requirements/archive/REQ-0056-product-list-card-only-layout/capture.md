# 商品列表页改为双列商品卡片展示

商品列表页不需要搜索功能，也不需要筛选功能。页面主体应直接展示商品卡片，卡片信息展示参照首页的热销推荐样式，并采用一行 2 个的双列布局。

# 原始描述

商品列表页，不需要搜索功能，也不需要筛选功能，商品卡片信息展示参照首页的热销推荐（一行2个）

# 需求要点

- 商品列表页移除或不展示搜索入口。
- 商品列表页移除或不展示筛选入口。
- 商品卡片的信息结构、视觉层级和字段呈现参照首页热销推荐。
- 商品列表采用一行 2 个卡片的双列布局。
- 页面应保持商品列表浏览优先，避免搜索/筛选控件占用首屏空间。

# 待澄清

- [ ] 商品列表页具体指微信小程序商品列表页、Web 店主端商品列表页，还是两端都需要调整。
- [ ] 首页热销推荐卡片的字段是否作为唯一基准，还是允许在列表页补充分类、规格、价格等字段。
- [ ] 双列布局在小屏、长商品名、无图或加载态下的展示规则。
- [ ] 如现有搜索/筛选能力仍需保留，是否迁移到其他入口或完全移除。

# 建议验收要点

- [ ] 打开商品列表页时，不出现搜索框、搜索按钮、筛选按钮或筛选面板入口。
- [ ] 商品卡片与首页热销推荐在主要信息、视觉样式和点击区域上保持一致。
- [ ] 商品列表在目标端按一行 2 个卡片展示，长文本和图片缺失状态不破坏布局。
- [ ] 点击商品卡片可进入既有商品详情页，路径和参数保持现有能力一致。
