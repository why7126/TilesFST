# REQ-0064 微信小程序多页面支持微信分享

## 1. 需求背景

菲尚特微信小程序已经承载首页、商品详情、商品列表和品牌详情等核心浏览路径。装修客户、设计师与门店销售在选砖、讲解方案或推荐品牌时，需要把当前页面直接分享给微信朋友，或分享到微信朋友圈，帮助被分享用户直接进入对应商品、列表筛选结果或品牌主页。

当前小程序部分页面已有微信好友分享能力，但页面覆盖、朋友圈分享、路径参数保留和验收标准尚未形成统一契约。本需求将分享能力收敛为页面级标准，避免不同页面各自实现导致标题、路径、图片、埋点和直达兜底不一致。

## 2. 目标用户

| 用户 | 诉求 |
|---|---|
| 装修客户 | 将喜欢的商品、品牌或商品列表分享给家人朋友共同决策。 |
| 设计师 | 将指定 SKU、品牌主页或筛选后的商品列表分享给客户确认。 |
| 门店销售 | 在微信会话或朋友圈中转发小程序页面，提高商品和品牌触达效率。 |
| 被分享用户 | 点击分享卡片或朋友圈入口后直接进入正确页面，不丢失商品、品牌或筛选上下文。 |
| 开发 / 测试人员 | 有统一的页面分享矩阵、路径参数和验收规则，减少遗漏。 |

## 3. 需求目标

- 首页、商品详情页、商品列表页、品牌详情页 MUST 支持分享给微信朋友。
- 首页、商品详情页、商品列表页、品牌详情页 MUST 支持分享到微信朋友圈。
- 分享标题、路径、必要参数和分享图 MUST 可预测、可验收。
- 商品详情页和品牌详情页分享直达 MUST 保留对应 `skuId` 或 `brandId`。
- 商品列表页分享 SHOULD 保留当前列表上下文，至少覆盖分类、品牌、关键词和榜单入口。
- 分享行为 SHOULD 接入现有小程序埋点，不阻断用户分享。

## 4. 范围

### 4.1 本期包含

| 页面 | 微信朋友 | 微信朋友圈 | 参数保留重点 |
|---|---|---|---|
| 首页 | 支持 | 支持 | 首页路径与来源标识。 |
| 商品详情页 | 支持 | 支持 | `skuId`、`source=share`，分享图优先商品分享图或主图。 |
| 商品列表页 | 支持 | 支持 | `categoryId`、`categoryLevel`、`categoryName`、`brandId`、`keyword`、`section` 等当前列表上下文。 |
| 品牌详情页 | 支持 | 支持 | `brandId`、`source=share`，标题使用品牌名称。 |
| 分享埋点 | 支持 | 支持 | 区分页面、分享渠道和关键业务 ID。 |
| 异常兜底 | 支持 | 支持 | 缺失或无效参数时给出可返回或可重试状态，不白屏。 |

### 4.2 本期不包含

| 不包含 | 说明 |
|---|---|
| 分享海报 | 不生成商品海报、品牌海报或朋友圈图片海报。 |
| 后台配置 | 不新增管理端配置分享标题、分享图、默认文案或渠道策略。 |
| 裂变活动 | 不包含邀请奖励、分享裂变、优惠券、渠道分销或活动追踪。 |
| 短链系统 | 不新增短链、落地页、中转页或外部 H5 分享页。 |
| 微信群差异策略 | 微信好友和微信群沿用同一 `onShareAppMessage` 能力，不单独区分微信群文案。 |
| 新增 API / DB | 默认不新增接口、表或字段；优先使用现有页面数据。 |

## 5. 功能要求

### FR-001 首页分享

- 首页 MUST 通过微信小程序页面级分享配置支持分享给微信朋友。
- 首页 MUST 通过微信小程序页面级分享配置支持分享到微信朋友圈。
- 首页分享标题 SHOULD 使用门店 / 品牌名称；无动态数据时 MUST 使用稳定兜底标题。
- 首页分享路径 MUST 指向首页，并 SHOULD 带来源标识，便于后续统计分享入口。
- 首页分享不得破坏已有“添加到我的小程序”引导、原生胶囊避让和首页加载流程。

### FR-002 商品详情页分享

- 商品详情页 MUST 支持分享给微信朋友，并保留当前 SKU 的 `skuId`。
- 商品详情页 MUST 支持分享到微信朋友圈，并保留当前 SKU 的 `skuId`。
- 商品详情分享路径 MUST 指向当前 SKU 详情页，并携带 `source=share` 或等价来源参数。
- 分享标题 SHOULD 优先使用商品分享标题；无配置时使用 SKU 名称与品牌名称组合。
- 分享图 SHOULD 优先使用商品分享图；无配置时使用商品主图或现有占位图兜底。
- 被分享用户打开商品详情页时，页面 MUST 能正确加载商品信息；SKU 不存在、下架或参数无效时 MUST 展示可理解的错误 / 空状态。

### FR-003 商品列表页分享

- 商品列表页 MUST 支持分享给微信朋友。
- 商品列表页 MUST 支持分享到微信朋友圈。
- 商品列表分享标题 MUST 与当前列表语义一致，例如搜索词、分类名称、品牌商品、新品榜、热销榜或全部商品。
- 商品列表分享路径 SHOULD 保留当前上下文参数，包括但不限于 `categoryId`、`categoryLevel`、`categoryName`、`brandId`、`keyword`、`section`。
- 商品列表分享路径 MUST 对中文分类名、关键词等参数进行安全编码，避免分享后解析失败。
- 被分享用户进入商品列表页后，列表标题、筛选结果、空状态和加载失败提示 MUST 与分享前上下文一致。

### FR-004 品牌详情页分享

- 品牌详情页 MUST 支持分享给微信朋友，并保留当前 `brandId`。
- 品牌详情页 MUST 支持分享到微信朋友圈，并保留当前 `brandId`。
- 品牌详情分享标题 SHOULD 使用品牌名称；品牌数据未加载完成时 MUST 使用稳定兜底标题。
- 品牌详情分享路径 MUST 指向当前品牌详情页，并携带 `source=share` 或等价来源参数。
- 被分享用户打开品牌详情页时，页面 MUST 能正确展示品牌信息、品牌商品和证书入口；品牌参数无效时 MUST 给出可返回或可重试状态。

### FR-005 分享渠道区分

- 页面 SHOULD 能区分“分享给微信朋友”和“分享到朋友圈”的触发渠道，至少在埋点或后续验收中可识别。
- 微信朋友分享 SHOULD 使用小程序卡片所需字段，包括标题、路径和可用分享图。
- 朋友圈分享 SHOULD 使用朋友圈支持的字段，包括标题、query 或路径参数、分享图等平台允许字段。
- 若微信平台对朋友圈分享图、路径或 query 有限制，实现阶段 MUST 按微信小程序能力降级，并在验收记录中说明。

### FR-006 分享埋点

- 分享触发 SHOULD 记录现有小程序行为埋点。
- 埋点 SHOULD 包含页面类型、分享渠道、页面路径和关键业务 ID，例如 `skuId`、`brandId`、`categoryId`、`keyword`。
- 埋点失败 MUST NOT 阻断分享行为，也不得向用户暴露技术错误。
- 若当前埋点服务不可用，分享仍 MUST 正常返回微信平台要求的分享对象。

### FR-007 分享直达与返回兜底

- 从微信朋友或朋友圈直达页面时，页面 MUST 能在无上一页栈的情况下正常展示返回或首页兜底能力。
- 商品详情、商品列表和品牌详情直达时，页面加载态、错误态、空状态和返回行为 MUST 稳定。
- 分享直达不得与自定义导航栏右侧微信原生胶囊区域重叠或冲突。
- 页面不得自绘模拟微信分享按钮、关闭按钮或系统胶囊。

### FR-008 分享配置一致性

- 四个目标页面 SHOULD 使用一致的分享字段命名、路径构建和参数编码策略。
- 分享标题不应超过微信卡片和朋友圈常见展示长度；过长标题 SHOULD 以业务语义优先截断。
- 分享图片必须使用小程序可访问的安全 URL 或本地兜底资源，不得直连未授权对象存储地址。
- 若分享图来源于商品、品牌或首页图片资源，必须遵守现有媒体公开访问和对象存储安全边界。

## 6. UI / UE 约束

- 本需求不新增可见分享按钮；优先使用微信小程序右上角原生分享入口和页面已有分享按钮。
- 已有商品详情页底部分享按钮若保留，MUST 调起微信原生分享能力或保持与页面分享配置一致。
- 分享相关改造不得遮挡自定义导航栏、微信原生胶囊、页面内容或底部操作栏。
- 分享直达页面的错误态、空状态和加载态文案应保持当前小程序深色品牌风格，不新增临时视觉主题。
- 商品列表页分享前后标题应一致，避免用户点击分享后进入“全部商品”但原本分享的是搜索或分类结果。

## 8. 技术与实现影响预判

| 领域 | 影响预判 |
|---|---|
| 小程序 | 预计影响 `src/miniapp/pages/index`、`tile-detail`、`product-list`、`brand-detail` 的页面级分享配置与埋点。 |
| API | 默认不新增 API；若后续要求后台配置分享图或文案，需另行评审并同步 OpenAPI。 |
| 数据库 | 默认不新增表或字段。 |
| Web 管理端 | 不涉及；后台分享配置不在本期范围。 |
| Web 展示端 | 不涉及。 |
| Orval | 默认不需要；若新增接口 contract，则实现阶段再同步。 |
| Docker Compose | 不涉及。 |
| 对象存储 / 媒体 | 仅使用现有公开可访问图片或本地兜底图；不得放宽 MinIO 权限。 |
| 测试 | 后续 `req-complete` / OpenSpec 阶段需补充页面分享矩阵、路径参数和分享直达验收。 |
