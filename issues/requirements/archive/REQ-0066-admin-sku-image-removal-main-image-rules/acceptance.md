# REQ-0066 验收标准

## 1. 功能 AC

### 1.1 图片移除入口

- [ ] **AC-001** SKU 编辑弹窗中每张商品图片都展示移除入口，入口可通过鼠标和键盘触达。
- [ ] **AC-002** 移除入口有可访问名称，能区分“移除图片”和“设为主图”。
- [ ] **AC-003** 只读账号不展示 SKU 编辑弹窗或图片移除入口。

### 1.2 移除非主图

- [ ] **AC-004** 移除非主图图片后，当前主图仍为主图。
- [ ] **AC-005** 移除非主图图片后，剩余图片顺序稳定，`sort_order` 重新连续计算。
- [ ] **AC-006** 保存后再次打开编辑弹窗，被移除的非主图图片不再出现。

### 1.3 设置主图并前置

- [ ] **AC-007** 点击非主图图片“设为主图”后，该图片立即带“主图”标签。
- [ ] **AC-008** 新主图自动移动到图片列表第一位。
- [ ] **AC-009** 原主图取消主图标记，并保留在剩余图片列表中。
- [ ] **AC-010** 保存 payload 中仅一张图片 `is_main=true`，且该图片 `sort_order=0`。

### 1.4 移除当前主图

- [ ] **AC-011** 移除当前主图且移除前后一张图片存在时，后一张图片自动成为新主图。
- [ ] **AC-012** 移除当前主图且当前主图为最后一张、但仍有其他图片时，移除后列表第一张图片自动成为新主图。
- [ ] **AC-013** 新主图在弹窗内即时显示为第一张，不等待保存后才变化。
- [ ] **AC-014** 保存后重新打开弹窗，主图兜底结果与保存前弹窗显示一致。

### 1.5 移除全部图片

- [ ] **AC-015** 移除最后一张图片后，商品图片列表为空。
- [ ] **AC-016** 图片列表为空时不显示“主图”标签。
- [ ] **AC-017** 图片列表为空时仍显示“继续添加图片”入口。
- [ ] **AC-018** 保存草稿允许缺图片/缺主图，并沿用既有素材完整度显示。

### 1.6 数据与回归

- [ ] **AC-019** SKU 更新请求不包含已移除图片。
- [ ] **AC-020** 后端保存后 `tile_images` 中同一 SKU 至多一张图片 `is_main=1`。
- [ ] **AC-021** 本需求不要求删除 MinIO 中的对象文件；验收只检查 SKU 图片关联变化。
- [ ] **AC-022** 前端组件测试覆盖：移除非主图、设主图前置、移除主图兜底、移除全部图片。
- [ ] **AC-023** 若 API contract 未改变，不需要重新生成 Orval；若实现改变 images 请求/响应字段，必须同步 OpenAPI 与 Orval。

## 2. 横切 AC（knowledge-base）

> 来源：`docs/knowledge-base/best-practices/admin-modal-width-css-cascade.md`、`docs/knowledge-base/best-practices/admin-media-upload-chain.md` — 预防 Sprint 002/003 复发类缺陷。

- [ ] **AC-XCUT-001** `TileSkuFormModal` 或后续拆分组件不得同时挂载通用 `modal-card` 与 `sku-modal-card` 等专属弹窗类。
- [ ] **AC-XCUT-002** 在 1440px 视口下验收 SKU 编辑弹窗 Computed width 与既有 SKU 弹窗规格一致，不得被全局 `.modal-card` 宽度覆盖。
- [ ] **AC-XCUT-003** 弹窗相关 Vitest 或等价检查覆盖可能冲突的 admin CSS 栈，确保 SKU 弹窗宽度类不被其他管理端 CSS 覆盖。
- [ ] **AC-XCUT-004** 矮视口下 SKU 编辑弹窗 body 区域可滚动，底部操作区可访问，图片网格不溢出遮挡。
- [ ] **AC-XCUT-005** 图片上传仍保持 `idle→uploading→done/failed` 状态机；本需求新增移除能力不得破坏上传状态显示。
- [ ] **AC-XCUT-006** 同一弹窗会话中新上传图片必须即时回显，且可继续执行“设为主图”和“移除”操作。
- [ ] **AC-XCUT-007** 图片上传失败必须在上传控件或图片区域内展示错误，不得只依赖全局 toast。
- [ ] **AC-XCUT-008** 如本 Change 触及上传链路或大小限制，必须经 `http://localhost:3000` 验证边界文件上传；若仅实现本地移除和 payload 重排，可标记 N/A — 未改变上传链路。
