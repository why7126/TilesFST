# REQ-0066 用户故事

## US-001 后台运营移除错误商品图片

**作为** 后台运营，  
**我希望** 在 SKU 编辑弹窗中可以移除任意已添加商品图片，  
**以便** 及时清理错误图片、过期图片或不适合展示的图片。

验收要点：

- SKU 编辑弹窗中的每张图片均有移除入口。
- 移除非主图图片后，当前主图保持不变。
- 保存后再次打开弹窗，被移除图片不再出现。
- 移除图片不影响 SKU 名称、品牌、类目、价格、视频等其他字段。

## US-002 后台运营设置主图并自动前置

**作为** 后台运营，  
**我希望** 设置某张图片为主图后它自动移动到第一位，  
**以便** 通过图片顺序直观看到前台和列表优先展示的首图。

验收要点：

- 点击“设为主图”后，目标图片立即成为唯一主图。
- 新主图自动移动到图片列表第一位。
- 原主图取消主图标记并保留在剩余图片中。
- 保存 payload 中主图 `is_main=true` 且 `sort_order=0`。

## US-003 后台运营移除当前主图时自动兜底

**作为** 后台运营，  
**我希望** 移除当前主图后系统能自动选择下一张图片作为新主图，  
**以便** SKU 在仍有图片时不意外变成缺主图状态。

验收要点：

- 移除当前主图时，优先选择移除前当前主图后一张图片为新主图。
- 若当前主图已是最后一张但仍有其他图片，移除后列表第一张成为新主图。
- 新主图变化在弹窗内即时显示。
- 保存后主图状态与弹窗内显示一致。

## US-004 后台管理员确认数据一致性

**作为** 后台管理员，  
**我希望** SKU 图片移除和主图调整在保存后稳定回填，  
**以便** 管理端列表、详情和后续前台展示都使用一致的图片事实源。

验收要点：

- 保存后重新查询 SKU 详情，图片顺序与主图标记一致。
- `tile_images.is_main` 仅有一张为真，除非图片列表为空。
- 图片列表为空时，素材完整度沿用缺图片/缺主图规则。
- 本需求不要求物理删除 MinIO 对象，避免误删复用媒体。

## 非目标用户说明

只读账号不进入本需求写操作范围；小程序和店主端仅消费保存后的图片顺序与主图结果，本期不调整端侧展示规则。
