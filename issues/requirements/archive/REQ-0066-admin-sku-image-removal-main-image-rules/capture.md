# 一句话

管理端瓷砖 SKU 编辑弹窗应支持移除已添加的商品图片，并在图片顺序和主图变化时自动维护主图位置与兜底主图。

# 原始描述

管理端，瓷砖 SKU 编辑弹窗中，商品图片不支持移除，只支持添加。主图默认移动到第一个位置；当当前主图被移除时，如果还有其他图片，默认设置当前主图后一张图片为主图。

# 背景与关联

- 父需求：`REQ-0006-tile-sku-management`
- 涉及端：Web 管理端
- 涉及对象：瓷砖 SKU 编辑弹窗、商品图片列表、主图标记
- 当前现状：编辑弹窗中商品图片仅支持添加，缺少移除入口；主图与图片顺序的联动规则需要明确。
- 业务价值：让运营在维护 SKU 商品图片时可以闭环完成新增、移除、调整主图，避免错误图片残留或主图缺失。

# 建议验收要点

- [ ] SKU 编辑弹窗中的每张商品图片都提供清晰的移除操作，且不会误删未确认的图片。
- [ ] 设置某张图片为主图后，该图片默认移动到图片列表第一个位置。
- [ ] 移除非主图图片时，当前主图保持不变，并仍位于第一个位置。
- [ ] 移除当前主图时，如果主图后一张图片存在，则默认将该图片设为新主图。
- [ ] 移除当前主图时，如果不存在后一张但仍有其他图片，则按确认后的兜底策略选择剩余图片作为主图。
- [ ] 移除最后一张图片后，商品图片列表为空，主图状态清空且保存行为有明确校验或提示。
- [ ] 保存后再次打开 SKU 编辑弹窗，图片列表顺序与主图状态和保存结果一致。

# 待澄清

- [ ] 移除已保存图片时是否需要二次确认，还是在弹窗保存前允许撤销。
- [ ] “当前主图后一张图片”以移除前的图片顺序为准，还是以主图已移动到第一位后的顺序为准。
- [ ] SKU 是否允许无商品图片保存；如不允许，缺图时需要给出怎样的错误提示。
- [ ] 图片移除是否仅解除 SKU 关联，还是需要清理对象存储中的媒体文件。
