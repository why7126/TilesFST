# 业务流程

## 1. 总流程

```text
用户打开小程序
  |
  v
点击“品牌”入口
  |
  v
进入品牌列表页
  |
  +-- 加载品牌轮播
  |       |
  |       +-- 点击有效轮播项
  |              |
  |              v
  |        品牌详情 / 品牌商品列表 / 商品详情 / 搜索 / 门店信息
  |
  +-- 加载品牌双列列表
          |
          +-- 点击品牌卡片
          |       |
          |       v
          |  品牌详情页/主页（REQ-0058）
          |       |
          |       +-- 未交付时：品牌商品列表降级
          |
          +-- 无品牌 / 网络失败
                  |
                  v
             空状态 / 错误态 / 重试
```

## 2. 品牌入口流程

| 步骤 | 说明 | 输出 |
|---|---|---|
| 1 | 用户在小程序首页、TabBar 或既有导航入口看到“品牌” | 入口文案与品牌频道一致 |
| 2 | 用户点击入口 | 进入品牌列表页 |
| 3 | 页面初始化 | 并行或串行加载品牌轮播与品牌列表 |
| 4 | 加载成功 | 展示品牌轮播与双列品牌卡片 |
| 5 | 加载失败 | 展示可恢复错误态和重试入口 |

## 3. 品牌轮播流程

| 步骤 | 说明 | 输出 |
|---|---|---|
| 1 | 页面获取品牌轮播数据 | 轮播图、标题、副标题、跳转类型、跳转目标 |
| 2 | 轮播数据有效 | 展示与首页轮播一致的自动播放和指示点 |
| 3 | 用户点击轮播 | 按配置跳转到品牌、商品、搜索或门店信息 |
| 4 | 跳转目标不可达 | 安全降级，展示轻量提示 |
| 5 | 无轮播或图片异常 | 隐藏异常项或展示品牌化兜底 |

## 4. 品牌列表流程

| 步骤 | 说明 | 输出 |
|---|---|---|
| 1 | 页面获取公开可展示品牌 | 仅包含启用且公开品牌 |
| 2 | 按排序值和稳定规则排序 | 一行 2 个品牌卡片 |
| 3 | 用户点击品牌卡片 | 进入品牌详情页/主页或品牌商品列表 |
| 4 | 品牌不可访问 | 卡片不可进入无效页，并给出轻提示 |
| 5 | 列表为空 | 展示“暂无可展示品牌”空状态 |

## 5. 与父 REQ 和关联需求差异

| 关联需求 | 差异 |
|---|---|
| `REQ-0005-brand-management` | 父需求关注品牌主数据维护；本需求只关注小程序前台品牌列表浏览，不新增管理端维护能力。 |
| `REQ-0041-miniapp-home` | 首页已提供 Banner 轮播基准；本需求复用其轮播体验，但页面定位为品牌频道。 |
| `REQ-0054-brand-card-common-component` | 该需求沉淀品牌卡片展示逻辑；本需求要求双列列表态，可能需要扩展样式模式。 |
| `REQ-0058-brand-detail-home-page` | 本需求定义列表入口和跳转关系；品牌详情页内容由 `REQ-0058` 承接。 |

## 6. 非目标流程

- 不进入管理端品牌新增、编辑、启停或排序流程。
- 不在本需求中实现品牌详情页完整内容管理。
- 不实现品牌收藏、预约、询价或下单流程。
- 不新增 Web 展示端品牌页。
- 不为品牌列表新增重复品牌数据源。
