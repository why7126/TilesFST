# REQ-0060 新增品牌列表页

## 1. 需求背景

菲尚特微信小程序已经具备首页、搜索、分类、商品列表和商品详情等基础浏览能力，首页中也已经存在 Banner 轮播与“品牌馆”入口提示。随着品牌主数据、品牌卡片组件和品牌详情页/主页需求逐步沉淀，用户需要一个独立的品牌列表页，用品牌维度浏览可公开展示的瓷砖品牌，并从品牌卡片进入品牌详情页或品牌相关商品。

当前用户反馈明确提出：新增品牌列表页，并将入口中的“找砖”调整为“品牌”。结合探索结论，本需求将页面范围收敛为微信小程序品牌频道：顶部提供与首页轮播一致的品牌轮播，主体提供一行 2 个品牌卡片的品牌列表。

本需求不直接建设管理端品牌维护能力；品牌数据、品牌 Logo、上下架状态和排序规则优先复用现有品牌管理能力。

## 2. 目标用户

| 用户 | 诉求 |
|---|---|
| 装修业主 | 通过品牌维度了解菲尚特代理或推荐品牌，并继续查看品牌主页或品牌商品。 |
| 室内设计师 | 快速浏览品牌矩阵，用品牌风格、定位和商品线索辅助选材沟通。 |
| 门店销售 | 在小程序中向客户展示品牌资源，并从品牌列表快速进入品牌详情或相关产品。 |

## 3. 需求目标

1. 新增微信小程序品牌列表页，承接首页或导航中的品牌入口。
2. 顶部品牌轮播与首页轮播保持一致的视觉、自动播放和指示器体验。
3. 品牌列表采用一行 2 个品牌卡片，适合移动端快速浏览。
4. 入口文案从“找砖”调整为“品牌”，让导航名称与实际品牌频道一致。
5. 点击品牌卡片后进入品牌详情页/主页或品牌商品列表，具体目标与 `REQ-0058-brand-detail-home-page` 对齐。

## 4. 范围

### 4.1 本期包含

| 范围 | 说明 |
|---|---|
| 小程序品牌列表页 | 新增品牌列表页面，页面标识建议为 `pages/brands/index` 或实现阶段按项目路由规范确认。 |
| 品牌轮播 | 页面顶部展示品牌轮播，视觉和交互对齐首页 Banner 轮播。 |

## 5. 功能要求

### FR-001 品牌列表页入口

- 小程序 MUST 提供可进入品牌列表页的入口。
- 原“找砖”入口文案 MUST 调整为“品牌”，并与页面标题、导航标题保持一致。
- 若现有入口是首页快捷入口、TabBar、侧边栏或其他导航形态，后续实现 MUST 选择当前产品实际存在的入口位置进行一致性调整。
- 入口点击 MUST 进入品牌列表页，不得继续降级到搜索页或找砖页。
- 若品牌列表页加载失败，入口跳转后 MUST 展示可恢复的错误态，不得停留在空白页。

### FR-002 品牌轮播

- 品牌列表页顶部 MUST 展示品牌轮播区域。
- 品牌轮播视觉与交互 MUST 对齐小程序首页轮播，包括自动播放、循环播放、指示点和品牌金激活态。
- 轮播项 SHOULD 支持图片、标题、副标题和跳转目标。
- 轮播图片 MUST 使用公开可访问的安全 URL，不暴露对象存储原始 key。
- 点击轮播项时，若配置了品牌、商品、搜索或门店等可达目标，MUST 执行跳转；若目标不可达，MUST 安全降级并提示，不得报错。
- 无轮播数据时 SHOULD 使用品牌化默认图或隐藏轮播区；不得展示破图。

### FR-003 双列品牌卡片列表

- 品牌列表 MUST 使用一行 2 个卡片的移动端网格布局。
- 品牌卡片 MUST 至少展示品牌 Logo 和品牌名称。
- 品牌卡片 SHOULD 展示品牌简介、品牌标语、商品数量或“查看品牌主页”等辅助文案中的一种。
- 品牌 Logo 加载失败时 MUST 使用品牌名称首字、品牌占位或统一占位图，不得出现破图。
- 卡片点击区域 MUST 覆盖整张卡片，并提供小程序原生按压反馈。
- 品牌列表 MUST 仅展示公开可见、启用状态的品牌。

### FR-004 品牌卡片跳转

- 点击品牌卡片 SHOULD 进入品牌详情页/主页。
- 若品牌详情页/主页尚未完成，后续实现 MAY 临时进入品牌商品列表，并携带品牌筛选参数。
- 跳转目标 MUST 与 `REQ-0058-brand-detail-home-page` 保持一致，避免列表页和详情页分别定义互相冲突的路由。
- 不可访问、已下架或缺少跳转目标的品牌 MUST 禁止进入无效页面，并给出轻量提示。

### FR-005 数据加载与分页

- 页面首次加载 MUST 获取品牌轮播与品牌列表所需数据。
- 品牌列表 SHOULD 支持分页或下拉加载更多；若品牌数量较少，可一次加载并保持实现简单。
- 列表排序 SHOULD 优先使用后台排序值；排序值相同按创建时间或品牌名称稳定排序。
- 加载中 MUST 展示骨架屏或轻量加载态。
- 网络异常 MUST 提供重试入口。
- 无品牌数据时 MUST 展示品牌化空状态，例如“暂无可展示品牌”。

### FR-006 数据与接口边界

- 品牌列表数据 MUST 复用已有品牌管理数据源，不新增重复品牌表。
- 品牌轮播 SHOULD 优先复用已有 Banner 管理能力；若需要品牌页专属轮播位，必须在后续 OpenSpec 中明确配置来源和兼容边界。
- 小程序端 MUST 通过后端接口获取数据，不得直连未授权对象存储。
- API 响应 MUST 不暴露管理端内部字段、未公开品牌、内部备注或未授权素材。
- 若实现阶段新增或调整 API，MUST 同步 OpenAPI、Orval、接口文档和测试。

### FR-007 埋点与行为记录

- 页面曝光 SHOULD 记录 `brand_list_page_view`。
- 点击品牌卡片 SHOULD 记录 `brand_card_click`，参数至少包含品牌 ID、位置索引和来源入口。
- 点击品牌轮播 SHOULD 记录 `brand_banner_click`，参数至少包含轮播 ID、跳转类型和位置索引。
- 埋点 MUST 避免记录用户敏感信息。

## 6. UI / UE 约束

- 页面视觉 MUST 与微信小程序首页保持一致的暗色旗舰风和品牌金强调。
- 品牌轮播 MUST 对齐首页轮播的高度比例、圆角、指示点和标题层级；允许根据品牌页内容密度做等比例收敛。
- 双列品牌卡片在 320 到 430 pt 宽度下 MUST 不出现横向滚动、文字重叠或卡片挤压变形。
- 卡片 Logo 区域 SHOULD 稳定尺寸，避免图片加载后造成布局跳动。
- 品牌名称最多展示 1 到 2 行，超长使用省略号或合理换行。
- 辅助文案 SHOULD 保持短句，不得压缩主要品牌名称可读性。
- 页面顶部、轮播、列表和底部安全区 MUST 避免与小程序自定义导航或 TabBar 重叠。
- UI 不得使用大面积促销红、纯黑、纯白或与瓷砖品牌调性不一致的装饰风格。

## 7. 数据字段建议

品牌卡片 SHOULD 映射以下字段；字段命名可在 API / 小程序实现阶段按项目约定转换：

| 字段 | 用途 | 首期要求 |
|---|---|---|
| `brandId` | 品牌跳转和埋点 | 必需 |
| `brandName` | 品牌名称 | 必需 |
| `logoUrl` | 品牌 Logo | 建议 |
| `description` | 品牌简介或标语 | 建议 |
| `productCount` | 品牌商品数量 | 可选 |
| `sortOrder` | 列表排序 | 建议 |
| `isEnabled` | 是否启用 | 必需 |
| `isPublic` | 是否公开展示 | 必需 |

品牌轮播 SHOULD 映射以下字段：

| 字段 | 用途 | 首期要求 |
|---|---|---|
| `id` | 轮播唯一标识 | 必需 |
| `title` | 轮播标题 | 建议 |
| `subtitle` | 轮播副标题 | 可选 |
| `imageUrl` | 轮播图片 | 必需 |
| `jumpType` | 跳转类型 | 建议 |
| `targetId` | 跳转目标 | 可选 |
| `sortOrder` | 展示排序 | 建议 |

## 9. 风险与待确认

| 风险 / 待确认 | 说明 |
|---|---|
| 入口形态 | 用户描述为 siderbar，但小程序可能实际是快捷入口或 TabBar；实现前需确认现有入口位置。 |
| 轮播配置来源 | 品牌页轮播是否复用 Banner 管理，还是需要品牌页专属轮播位，需要在 OpenSpec 阶段明确。 |
| 品牌详情依赖 | 若 `REQ-0058` 未交付，品牌卡片跳转目标需临时降级为品牌商品列表或轻提示。 |
| 双列卡片复用 | 现有品牌卡片更偏单行横向信息卡，双列样式可能需要扩展组件展示模式。 |
