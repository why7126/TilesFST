# 微信小程序商品卡片组件需求文档

## 1. 背景

菲尚特微信小程序已经具备首页、分类列表、搜索结果、商品列表和 SKU 详情等商品浏览链路。用户在不同列表页中识别商品时，需要依赖稳定一致的卡片信息层级、图片比例、价格表达和点击反馈。如果各页面分别实现商品卡片，容易出现字段缺失、跳转规则不一致、图片占位差异和后续维护成本上升。

本需求沉淀微信小程序端可复用的商品卡片组件，作为商品列表页通用组件的子能力，优先服务分类商品列表、搜索结果列表、首页推荐和后续收藏/品牌商品列表等场景。

## 2. 目标用户

| 用户 | 核心诉求 |
|---|---|
| 装修客户 | 在列表中快速判断商品外观、名称、规格和价格，进入详情继续了解 |
| 设计师 | 快速比较不同 SKU 的品牌、规格、色系和参考价格，辅助方案选型 |
| 门店导购 | 在小程序列表页中快速定位商品，并打开详情页向客户介绍 |
| 普通访客 | 在分类、搜索、推荐等入口获得一致的商品浏览体验 |

## 3. 范围

### 3.1 包含

- 微信小程序商品卡片组件的信息结构、展示规则、触控反馈和跳转行为。
- 商品主图、商品名称、SKU 编码、品牌、规格、参考价格等核心字段展示。
- 图片失败、字段缺失、商品不可查看等卡片级异常处理。
- 分类列表、搜索结果、首页推荐、品牌商品列表、收藏列表等商品列表场景复用约束。
- 卡片点击进入 SKU 详情页的参数传递、来源上下文和防重复点击规则。
- 商品卡片曝光与点击埋点建议。

### 3.2 不包含

- 商品列表页容器、筛选、排序、分页、下拉刷新和上拉加载更多状态机。
- SKU 详情页的信息架构、媒体轮播、收藏、分享和推荐模块。
- 管理端 Web 商品卡片、店主 Web 商品卡片或跨端组件库建设。
- 商品后台管理、商品录入、上下架、库存、价格编辑或商品数据模型新增。
- 购物车、在线下单、询价、客服咨询、对比和卡片内收藏快捷操作。
- 瀑布流、横滑大图、AR 预览、3D 展示或个性化推荐算法。

## 4. 功能要求

### FR-001 卡片基础结构

- 商品卡片 MUST 作为微信小程序可复用组件沉淀，支持被不同商品列表页引用。
- 卡片 MUST 接收单个商品 SKU 数据和列表来源上下文，不在组件内部直接拼装列表查询参数。
- 卡片 MUST 提供稳定的图片区、信息区和状态区，避免不同页面各自定义核心结构。
- 卡片 SHOULD 支持单列大卡和双列紧凑卡两种展示密度，具体启用方式在后续 `/req-complete` 阶段结合原型确认。

### FR-002 商品信息展示

- 商品卡片 MUST 展示商品主图、SKU 名称、品牌名称、规格和参考价格。
- 商品卡片 SHOULD 展示 SKU 编码、分类名称、适用空间或主色系等辅助信息，但不得挤压 SKU 名称和主图。
- SKU 名称 MUST 支持最多两行展示，超出后省略。
- 参考价格 MUST 使用“参考价格”语义表达，不得暗示在线成交价或库存承诺。
- 价格缺失、品牌缺失或规格缺失时，卡片 MUST 使用统一占位文案，不得展示空字符串或异常值。

### FR-003 商品图片

- 商品主图 MUST 使用固定比例容器，避免图片加载前后导致卡片高度跳动。
- 主图加载失败时 MUST 使用统一深色占位图或占位背景，不展示破图。
- 图片 SHOULD 使用适合列表场景的缩略图 URL，不直接加载详情页原图。
- 卡片图片不得抢占全部信息层级，文字信息在 320px 小屏宽度下仍需可读。

### FR-004 点击与跳转

- 整个商品卡片 MUST 可点击，并跳转到 SKU 详情页。
- 跳转路径 MUST 携带 `skuId`，并 SHOULD 携带 `sourcePage`、`sourceModule`、`categoryId`、`keyword` 等来源上下文。
- 点击期间 MUST 提供小程序触控反馈，并防止连续点击重复打开详情页。
- 商品已下架、不可查看或缺少 `skuId` 时，卡片 MUST 阻止跳转并给出可理解的提示或不可用状态。
- v1 不在卡片内部提供收藏、分享、加入询价、购物车或联系客服快捷按钮。

### FR-005 复用与列表适配

- 卡片 MUST 能在分类商品列表、搜索结果 SKU 列表、首页推荐商品列表中复用。
- 卡片 SHOULD 预留品牌商品列表、收藏列表和同品牌推荐列表的复用能力。
- 不同列表页 MAY 传入不同来源上下文、曝光位置和展示密度，但不得改变核心字段语义。
- 列表容器负责分页、筛选、排序、加载态和空状态；卡片组件只负责单个商品的展示与交互。

### FR-006 卡片级异常状态

- 商品主图缺失或加载失败时，卡片 MUST 保持整体可点击和信息可读。
- 商品名称缺失时，卡片 SHOULD 展示“未命名商品”或后端兜底名称，并上报数据异常。
- 商品不可查看时，卡片 SHOULD 展示“暂不可查看”状态，并禁用详情跳转。
- 单个卡片异常不得导致整个商品列表页不可用。

### FR-007 埋点

商品卡片 SHOULD 支持以下埋点事件：

- `product_card_exposure`
- `product_card_click`
- `product_card_image_failed`
- `product_card_unavailable_click`

公共参数 SHOULD 包含 `skuId`、`skuCode`、`sourcePage`、`sourceModule`、`listContext`、`index`、`categoryId`、`brandId`、`keyword` 和 `requestId`。

## 5. UI 约束

- 视觉风格 MUST 与菲尚特微信小程序深色企业轻奢风保持一致。
- 卡片圆角、阴影、分割线、价格强调色和触控反馈 SHOULD 与现有小程序首页和商品列表页保持统一。
- 商品图片区域 MUST 使用稳定比例；双列模式下卡片高度差异 SHOULD 控制在合理范围内。
- 文本层级 SHOULD 优先保证 SKU 名称、品牌/规格和参考价格可快速扫描。
- 可点击区域有效尺寸 SHOULD 不小于 44px，适配小程序触控场景。
- 卡片内不堆叠过多操作按钮，v1 优先保证浏览效率和进入详情路径清晰。
- 图片占位、价格缺失、不可查看等状态 MUST 使用统一视觉语义。

## 6. 关联需求

- `REQ-0047-product-list-common-component-application`：父需求，商品卡片组件属于小程序商品列表通用能力的子能力。

- `REQ-0045-category-list-page`：分类页二级分类进入商品列表后复用商品卡片。
- `REQ-0046-search-component-application`：搜索结果 SKU 列表复用商品卡片。
- `REQ-0041-miniapp-home`：首页推荐商品区域可复用商品卡片。
