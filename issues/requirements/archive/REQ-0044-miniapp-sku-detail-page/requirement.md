# REQ-0044 微信小程序瓷砖 SKU 详情页需求文档

- 版本：v1.0
- 状态：approved
- 产品：菲尚特瓷砖微信小程序
- 页面：瓷砖 SKU 详情页
- 更新日期：2026-07-18

## 1. 背景与目标

菲尚特微信小程序面向装修客户、设计师、门店导购与品牌访客，核心定位是数字瓷砖展厅。SKU 详情页承接首页、搜索、分类、品牌馆、收藏页及分享卡片的商品访问流量，帮助用户快速了解单个瓷砖 SKU、查看高清纹理与视频、收藏备选商品，并分享给客户或家人。

本期不提供购物车、在线下单、支付、库存及询价功能。

## 2. 用户角色

| 角色 | 核心诉求 |
|---|---|
| 装修客户 | 快速查看产品外观、规格、工艺、价格并收藏备选 |
| 设计师 | 查看纹理细节、规格参数，分享给业主确认 |
| 门店导购 | 通过微信分享商品，强化品牌与商品信息表达 |
| 普通访客 | 从品牌、分类或搜索入口继续浏览相关商品 |

## 3. 已确认产品决策

1. 页面同时服务“快速了解产品、辅助选砖、分享客户”。
2. 图片与多个视频统一混合轮播，支持全屏沉浸预览。
3. SKU 信息采用卡片分组展示。
4. 参考价格直接展示具体金额与计价单位。
5. 收藏粒度为 SKU。
6. 分享卡片展示商品主图、SKU 名称、参考价格和品牌名称。
7. 底部固定操作栏包含“收藏、品牌、分享”。
8. 多级类目使用完整面包屑展示。
9. 备注说明独立卡片、默认完整展开。
10. 顶部媒体区采用大图布局，约占首屏内容区 70%。
11. 品牌入口进入品牌主页。
12. 一个 SKU 对应一个详情页，不在页内切换规格。
13. 页面底部展示“同系列商品”和“同品牌推荐”。
14. 图片预览支持双击放大、双指缩放、惯性拖动与查看原图。
15. 品牌信息在 SKU 名称上方突出展示。
16. 商品参数默认全部展开。
17. 媒体区右下角分别展示图片数和视频数。
18. 同品牌推荐采用横向滑动。
19. 视觉风格沿用首页 v6 的深色企业轻奢风。

## 4. 页面入口与出口

### 4.1 入口

- 首页商品卡片
- 搜索联想中的 SKU 项
- 搜索结果页 SKU 卡片
- 分类列表 SKU 卡片
- 品牌主页 SKU 卡片
- 收藏页 SKU 卡片
- 微信好友/群分享卡片

### 4.2 出口

- 返回上一页
- 进入品牌主页
- 进入同系列 SKU 详情
- 进入同品牌 SKU 详情
- 收藏或取消收藏
- 调起微信原生分享
- 进入图片/视频全屏预览

## 6. 字段定义

| 字段 | 类型 | 必填 | 展示规则 |
|---|---|---:|---|
| skuId | string | 是 | 页面主键，不直接展示 |
| skuName | string | 是 | 最多两行，超出省略 |
| skuCode | string | 是 | 显示“SKU 编码：xxx” |
| brandId | string | 是 | 用于品牌跳转 |
| brandName | string | 是 | 品牌卡和参数卡均展示 |
| brandLogo | url | 否 | 无 Logo 时显示品牌首字占位 |
| categoryPath | array | 是 | 依层级顺序以“›”连接 |
| specification | string | 是 | 如 600×1200mm |
| surfaceProcess | string | 否 | 空值显示“—” |
| mainColor | string | 否 | 空值显示“—” |
| referencePrice | number | 是 | 两位小数可按配置去尾零 |
| priceUnit | enum | 是 | 片、㎡、箱等 |
| remark | richText | 否 | 支持换行，禁止执行脚本 |
| images | array | 是 | 至少 1 张；仅一张时不自动轮播 |
| primaryImageId | string | 是 | 主图在混合轮播中排第一 |
| videos | array | 否 | 支持多个，展示封面、时长、URL |
| isFavorite | boolean | 是 | 控制收藏初始状态 |
| sameSeries | array | 否 | 最多返回 10 条 |
| sameBrand | array | 否 | 最多返回 10 条 |

## 7. 业务规则

### 7.1 媒体排序

1. 主图固定为第 1 项。
2. 其余图片按后台排序值升序。
3. 视频接在图片之后，并按后台排序值升序。
4. 媒体总数为图片数与视频数之和。
5. 右下角固定显示“图片 N、视频 M”；视频为 0 时仅显示图片数。

### 7.2 价格

- 格式：`¥89/片`、`¥198/㎡`。
- 页面文案标注“参考价格”。
- 价格为 0 或空值属于数据异常，前端显示“暂无参考价”并上报埋点。
- 价格不作为交易承诺，备注区下方显示“实际价格以门店为准”。

### 7.3 收藏

- 收藏对象为当前 SKU。
- 未登录点击收藏：先静默调起微信授权流程；授权失败保持未收藏。
- 收藏成功后按钮变为实心金色心形，并展示 Toast“已加入收藏”。
- 再次点击取消收藏，展示 Toast“已取消收藏”。
- 防重复点击：请求期间按钮禁用，最多 3 秒超时恢复。

### 7.4 分享

- 点击底部“分享”调起微信分享能力。
- 分享标题：`{SKU名称}｜{品牌名称}`。
- 分享图：优先使用后台分享图，未配置时使用商品主图裁切。
- 分享路径携带 `skuId` 与来源参数 `source=share`。
- 分享卡片摘要包含参考价格，不展示备注全文。

### 7.5 推荐

- 同系列优先展示同系列其他 SKU，不包含当前 SKU。
- 同品牌推荐排除当前 SKU 与已出现的同系列 SKU。
- 推荐卡片点击进入新的 SKU 详情页，并重置页面滚动位置为顶部。
- 无推荐数据时整个模块不展示，不保留空白。

## 8. 页面状态

| 状态 | 页面表现 |
|---|---|
| 首次加载 | 顶部媒体、标题、参数和推荐使用骨架屏 |
| 正常 | 展示完整内容 |
| SKU 不存在/已下架 | 空状态：商品暂不可查看，提供返回首页 |
| 网络失败 | 错误状态：加载失败，提供重新加载 |
| 图片失败 | 使用深色纹理占位图，不影响其他媒体浏览 |
| 视频失败 | 封面保留，显示“视频加载失败，点击重试” |
| 收藏失败 | 状态回滚并 Toast 提示 |
| 推荐为空 | 隐藏对应推荐模块 |

## 9. 非功能要求

- 首屏可交互时间目标：4G 网络下不超过 2.5 秒。
- 主图使用 WebP/AVIF 优先，提供 JPEG 降级。
- 首屏只预加载主图和下一项媒体；其余懒加载。
- 图片原图预览应使用独立原图 URL，避免列表缩略图放大失真。
- 视频默认静音、不自动播放；用户点击后播放。
- 视频播放时暂停轮播；退出播放后不强制恢复自动轮播。
- 页面需适配 320–430px 逻辑宽度和底部安全区。
- 所有点击目标有效尺寸不小于 44×44px。

## 10. 接口建议

### 10.1 查询详情

`GET /api/miniapp/skus/{skuId}`

返回 SKU 主体、媒体、品牌、收藏状态与推荐数据。首屏数据建议一次返回，避免串行请求。

### 10.2 收藏

- `POST /api/miniapp/favorites` body: `{skuId}`
- `DELETE /api/miniapp/favorites/{skuId}`

接口需保证幂等。

## 11. 埋点

| 事件 | 触发 |
|---|---|
| sku_detail_view | 页面成功展示 |
| sku_media_swipe | 切换媒体 |
| sku_image_preview | 打开图片全屏预览 |
| sku_video_play | 视频开始播放 |
| sku_favorite | 收藏成功 |
| sku_unfavorite | 取消收藏成功 |
| sku_share_click | 点击分享 |
| sku_brand_click | 点击品牌入口 |
| sku_recommend_click | 点击推荐 SKU |
| sku_load_error | 详情加载失败 |

## 12. 验收标准

1. 页面视觉与首页 v6 使用相同 Design Token。
2. 主图位于第一项，图片和视频可连续左右切换。
3. 图片全屏预览支持缩放、拖动、左右切换及序号展示。
4. 视频播放期间轮播不自动切换。
5. 收藏状态与收藏页保持一致，失败时可正确回滚。
6. 品牌卡及底部品牌按钮均进入对应品牌主页。
7. 类目完整展示所有层级且顺序正确。
8. 所有已定义字段完整展示，空值按规则处理。
9. 同系列与同品牌推荐不包含当前 SKU，卡片可横向滑动。
10. 分享卡片包含主图、名称、品牌和参考价格。
11. 下架、网络失败、图片失败、视频失败均有明确状态。
12. HTML 原型在 Chromium、Chrome、Safari 移动端模式可直接预览。
