# 用户故事

## US-001 查看 SKU 核心信息

作为装修客户，我希望从首页、分类、搜索、品牌页、收藏页或分享卡片进入 SKU 详情页后，能快速看到品牌、SKU 名称、编码、参考价格、规格、工艺、色系和类目路径，以便判断该瓷砖是否适合我的装修需求。

验收要点：

- 详情页可根据 `skuId` 加载并展示完整 SKU 主信息。
- SKU 名称、编码、品牌、参考价格、规格、表面工艺、主色系和完整类目路径均按字段规则展示。
- 空字段显示为约定占位或隐藏对应模块，不出现未处理的 `null`、`undefined` 或接口字段名。

## US-002 浏览图片与视频

作为设计师，我希望在详情页顶部连续浏览瓷砖高清图片和视频，并能进入全屏预览查看纹理细节，以便和客户确认材质、纹理和质感。

验收要点：

- 主图固定为首项，图片和视频按规则混合排序。
- 媒体区显示图片数和视频数，视频项有播放入口。
- 图片全屏预览支持缩放、拖动、切换和原图加载；视频播放不自动抢占用户操作。

## US-003 收藏与取消收藏

作为选砖用户，我希望能收藏或取消收藏当前 SKU，并看到清晰反馈，以便稍后在收藏页继续比较。

验收要点：

- 收藏粒度为 SKU，初始状态由详情接口返回。
- 收藏和取消收藏成功后按钮状态、Toast 和收藏页数据保持一致。
- 请求失败时回滚前端状态，并给出可理解的失败提示。

## US-004 分享给客户或家人

作为门店导购或装修客户，我希望能通过微信分享 SKU 详情卡片，以便对方直接打开同一 SKU。

验收要点：

- 分享标题包含 SKU 名称和品牌名称。
- 分享卡片包含主图、SKU 名称、品牌和参考价格。
- 分享路径携带 `skuId` 和来源参数，分享进入后可正常展示详情。

## US-005 继续浏览品牌和推荐 SKU

作为普通访客，我希望能从详情页进入品牌主页、同系列商品或同品牌推荐，以便继续浏览相关瓷砖。

验收要点：

- 品牌卡和底部品牌按钮均进入对应品牌主页。
- 同系列与同品牌推荐不包含当前 SKU，并支持横向滑动。
- 推荐卡点击后进入新的 SKU 详情页，并重置页面滚动位置。

## US-006 处理异常与边界状态

作为小程序用户，我希望下架、网络失败、媒体失败等情况有明确提示和恢复方式，以便知道下一步能做什么。

验收要点：

- SKU 不存在或已下架时展示空状态和返回首页入口。
- 网络失败时提供重新加载。
- 单张图片或单个视频失败不阻断其他内容展示。
