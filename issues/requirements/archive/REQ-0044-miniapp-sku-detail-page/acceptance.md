# 验收标准

## 功能 AC

- [ ] AC-001 页面可从首页商品卡片、分类列表、搜索结果、品牌主页、收藏页和微信分享卡片通过 `skuId` 进入，并在缺少或非法 `skuId` 时展示可恢复错误状态。
- [ ] AC-002 首次进入页面时展示稳定骨架屏，媒体区高度不因加载状态变化产生明显布局跳动。
- [ ] AC-003 详情页完整展示 SKU 名称、SKU 编码、品牌、参考价格、规格、表面工艺、主色系、完整类目路径和备注说明。
- [ ] AC-004 空字段按规则显示“—”或隐藏对应模块，不出现 `null`、`undefined`、空白异常块或接口字段名。
- [ ] AC-005 顶部媒体区主图固定为第一项，其余图片和视频按后台排序值组成混合轮播。
- [ ] AC-006 媒体区右下角准确展示图片数和视频数；视频为 0 时仅展示图片数。
- [ ] AC-007 图片全屏预览支持左右切换、双击缩放、双指缩放、拖动和原图加载；失败时可重试。
- [ ] AC-008 视频默认不自动播放；播放期间轮播不自动切换，页面隐藏或跳转时暂停播放。
- [ ] AC-009 收藏粒度为当前 SKU；收藏/取消收藏成功后按钮状态、Toast 和收藏页数据保持一致。
- [ ] AC-010 收藏请求失败时前端状态回滚，并展示可理解的失败提示；请求期间防重复点击。
- [ ] AC-011 点击分享调起微信分享能力，分享标题包含 SKU 名称和品牌名称，分享卡片包含主图、名称、品牌和参考价格。
- [ ] AC-012 分享路径携带 `skuId` 和来源参数，从分享进入后页面能力与普通进入一致，并记录来源埋点。
- [ ] AC-013 品牌卡和底部品牌按钮均进入对应品牌主页，入口参数使用 `brandId`。
- [ ] AC-014 同系列商品和同品牌推荐不包含当前 SKU；同品牌推荐同时排除已出现在同系列中的 SKU。
- [ ] AC-015 推荐模块无数据时整体隐藏；有数据时横向滑动可用，点击推荐卡进入新 SKU 详情并重置滚动位置。
- [ ] AC-016 SKU 不存在、已下架、网络失败、图片失败、视频失败均有明确状态，不阻断可用内容。
- [ ] AC-017 页面不出现购物车、立即购买、在线下单、支付、库存、优惠券、促销倒计时或询价承诺。

## UI AC

- [ ] AC-018 页面视觉延续小程序首页 v6 深色企业轻奢风，品牌金仅用于价格、主按钮、激活态和重点强调。
- [ ] AC-019 顶部媒体区采用大图布局，约占首屏内容区 70%，不使用小卡片包裹主视觉。
- [ ] AC-020 品牌信息位于 SKU 名称上方，并提供清晰品牌主页入口。
- [ ] AC-021 商品参数默认全部展开，类目路径完整展示所有层级且顺序正确。
- [ ] AC-022 备注说明独立成卡片并默认完整展开；无备注时不展示空模块。
- [ ] AC-023 底部固定操作栏包含收藏、品牌、分享，其中分享为品牌金主按钮。
- [ ] AC-024 页面适配 320-430px 逻辑宽度和底部安全区，所有主要点击目标不小于 44x44px。
- [ ] AC-025 原型 `prototype/miniapp/sku-detail.html` 可在 Chromium、Chrome 和 Safari 移动端模式直接预览。

## API / 数据 AC

- [ ] AC-026 若新增 `GET /api/miniapp/skus/{skuId}` 或调整现有接口，必须同步 OpenAPI、Orval、接口文档和后端集成测试。
- [ ] AC-027 若新增收藏接口或调整收藏 contract，收藏与取消收藏接口必须具备幂等行为，并覆盖未登录、失败回滚和超时恢复测试。
- [ ] AC-028 若新增 SKU 字段、媒体关系或公开状态字段，必须同步 SQLite/MySQL schema、迁移、数据库文档和测试。
- [ ] AC-029 图片、视频和分享图必须通过后端授权或公开安全 URL 返回，不允许小程序端直连未授权对象存储。

## 非功能 AC

- [ ] AC-030 4G 网络下首屏可交互时间目标不超过 2.5 秒，首屏只预加载主图和下一项媒体。
- [ ] AC-031 图片优先使用 WebP/AVIF 等适合小程序的压缩资源，并提供兼容降级。
- [ ] AC-032 备注富文本只支持安全展示能力，不执行脚本、不暴露内部路径或未授权资源。
- [ ] AC-033 埋点覆盖详情展示、媒体切换、图片预览、视频播放、收藏、取消收藏、分享、品牌点击、推荐点击和加载失败。

## 横切 AC（knowledge-base）

无横切 AC。本 REQ 为微信小程序 SKU 详情页，不命中 `req-complete` 当前定义的 `admin-list`、`admin-form`、`admin-modal`、`media-upload` 管理端横切标签；后续若扩展管理端上传或表单能力，应另行引用对应 best-practices。
