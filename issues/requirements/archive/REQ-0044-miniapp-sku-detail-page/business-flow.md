# 业务流程

## 主流程

```text
用户从入口点击 SKU
  ├─ 首页商品卡片
  ├─ 分类列表 / 搜索结果 / 品牌主页
  ├─ 收藏页
  └─ 微信分享卡片
        ↓
小程序进入 SKU 详情页，读取 skuId/source
        ↓
展示骨架屏并请求详情数据
        ↓
详情接口返回 SKU、媒体、品牌、收藏状态、推荐数据
        ↓
页面渲染媒体区、品牌信息、SKU 摘要、参数、备注、推荐和底部操作栏
        ↓
用户执行后续动作
  ├─ 浏览图片/视频 → 全屏预览或播放
  ├─ 收藏/取消收藏 → 更新按钮、Toast、收藏页一致性
  ├─ 点击品牌 → 进入品牌主页
  ├─ 点击推荐 SKU → 进入新 SKU 详情页
  └─ 点击分享 → 调起微信分享
```

## 异常流程

```text
详情加载失败
  ├─ 网络失败 → 错误状态 + 重新加载 + 返回首页
  ├─ SKU 不存在/已下架 → 空状态 + 返回首页
  └─ 部分媒体失败 → 展示单项失败状态，不阻断其他内容

收藏请求失败
  ├─ 回滚收藏状态
  └─ Toast 提示操作失败

分享进入缺少 skuId
  └─ 展示参数错误状态，引导返回首页
```

## 与父需求差异

父需求 `REQ-0006-tile-sku-management` 关注 SKU 资料管理和 SKU 数据本身；本需求关注微信小程序端的单个 SKU 详情页展示、浏览、收藏、分享和推荐承接。

本需求不包含：

- 管理端 SKU 新增、编辑、上下架能力。
- 购物车、在线下单、支付、库存、优惠券、促销倒计时。
- 店主 Web 展示端 SKU 详情页。
- 新增对象存储上传流程；仅消费已有安全媒体 URL。

## 数据与接口边界

- 优先通过一个详情接口返回首屏所需数据，避免小程序端串行请求导致首屏过慢。
- 若需新增或调整接口 contract，后续 OpenSpec Change 必须同步 OpenAPI、Orval、docs 和测试。
- 若需要新增 SKU 字段、媒体关系或公开状态字段，后续 Change 必须同步 SQLite/MySQL schema、迁移、数据库文档和测试。

## UI 与原型策略

- 原型位于 `prototype/miniapp/`，包含 HTML、上下文说明、交互说明和多状态 PNG。
- 视觉延续小程序首页 v6 深色企业轻奢风，避免电商红、纯白大背景、购物交易能力和超范围营销组件。
- 小程序实现需遵守微信平台限制，不直接复用 Web 浏览器专属 API。
