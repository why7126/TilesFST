# REQ-0005 用户管理页面需求文档

## 1. 需求背景

TILESFST 管理后台需要提供统一的用户管理能力，用于维护前台业务用户与后台管理用户。用户体系需要兼容当前内部管理场景，并为后续开放注册、店主自助注册、前台账号运营管理预留扩展能力。

本需求页面继承管理后台首页的视觉与交互：左侧固定 Sidebar、深色工业石材风、品牌金强调色、极细分割线、卡片式工作台内容区。

## 2. 页面目标

- 支持高级权限用户在后台创建、维护、冻结、删除和授权用户。
- 支持对用户进行模糊搜索、条件筛选、分页浏览。
- 支持展示用户总数、当前筛选结果数量、分页信息。
- 支持用户头像、昵称、角色等基础信息维护。
- 支持生成随机密码用于重置密码。
- 限制用户管理菜单仅后台管理员可见和可操作。

## 3. 角色定义

| 角色 | 是否允许进入后台 | 说明 | 用户管理权限 |
|---|---:|---|---|
| 前台用户 | 否 | 面向业务前台、店主或外部注册用户，仅可访问前台业务系统 | 无 |
| 后台运营 | 是 | 可进入后台进行 SKU、品牌、类目、Banner 等日常运营维护 | 不可见用户管理菜单 |
| 后台管理员 | 是 | 可进入后台，并拥有系统级管理能力，例如用户管理、权限维护、用户冻结等 | 可见并可编辑用户管理 |

> 本版本不单独设置“超级管理员”，避免与后台管理员职责重复。后续如需要更细粒度 RBAC，可在权限模型中扩展能力点，而不是新增重复角色名称。

## 4. 权限规则

1. 仅“后台管理员”可以看到左侧 SYSTEM 下的“用户管理”菜单。
2. 仅“后台管理员”可以访问用户管理页面。
3. “后台运营”进入后台后不可见用户管理菜单。
4. “前台用户”不允许进入后台管理端。
5. 如用户直接访问用户管理 URL，但无权限，系统应返回无权限页面或重定向至后台首页。
6. 后台管理员可以维护所有普通用户；如未来存在更高内置系统账号，应通过后端策略保护，不在前端仅做限制。

## 5. 页面信息架构

```text
管理后台用户管理
├─ Sidebar
│  ├─ OPERATIONS
│  └─ SYSTEM
│     ├─ 用户管理（当前激活）
│     └─ 系统设置
└─ Main Content
   ├─ 页面标题区
   ├─ 用户搜索与筛选区
   ├─ 用户数量统计区
   ├─ 用户列表区
   ├─ 分页区
   └─ 添加/编辑用户弹窗（独立状态页面）
```

## 6. 列表页面功能

### 6.1 用户模糊搜索

支持输入关键词进行模糊搜索，匹配范围：

- 用户名
- 昵称
- 邮箱或手机号（如后续账号体系启用）

搜索交互：

- 输入关键词后点击“搜索”触发查询。
- 支持回车触发查询。
- 点击“重置”清空关键词与筛选条件。

### 6.2 条件筛选

筛选项包括：

| 筛选项 | 类型 | 选项 |
|---|---|---|
| 角色 | 下拉单选 | 全部角色、前台用户、后台运营、后台管理员 |
| 状态 | 下拉单选 | 全部状态、正常、已冻结、已删除 |
| 登录情况 | 下拉单选 | 全部、从未登录、最近 7 天登录、超过 30 天未登录 |

### 6.3 用户数量展示

页面需要显示：

- 用户总数
- 当前筛选结果数
- 正常用户数
- 已冻结用户数

列表标题处需要显示：`用户列表 · 共 N 个用户`。

### 6.4 用户列表字段

| 字段 | 说明 |
|---|---|
| 头像 | 显示用户头像；无头像时显示用户名首字母或默认占位 |
| 用户名 | 用户唯一登录标识，创建后不可修改 |
| 昵称 | 用户可展示名称，默认为空，允许修改 |
| 角色 | 前台用户、后台运营、后台管理员 |
| 状态 | 正常、已冻结、已删除 |
| 最后登录时间 | 未登录过显示“从未登录” |
| 创建时间 | 用户创建时间 |
| 操作 | 编辑、重置密码、冻结/解冻、删除 |

## 7. 添加/编辑用户弹窗

### 7.1 弹窗信息顺序

添加/编辑用户弹窗中，每行一个信息，从上到下顺序必须为：

1. 用户名
2. 头像
3. 昵称
4. 角色

### 7.2 添加用户规则

| 字段 | 是否必填 | 规则 |
|---|---:|---|
| 用户名 | 是 | 4–32 位，创建后不可修改，不允许重复 |
| 头像 | 否 | 支持上传图片；无头像时使用默认占位头像 |
| 昵称 | 否 | 默认为空，最多 32 个字符 |
| 角色 | 是 | 前台用户、后台运营、后台管理员 |


用户名格式规则（与上表「4–32 位」一致）：

- 长度：4～32 个字符
- 仅限小写字母、数字、下划线 `_`、连字符 `-`、点号 `.`
- 必须以字母开头，不能以数字或符号开头
- 不允许连续两个及以上特殊符号（如 `__`、`--`、`..`）
- 禁止使用系统保留关键字（如 admin、root、system、test、user、login、administrator、anonymous、null、true、false 等，不区分大小写）


新用户创建后：

- 状态默认正常。
- 不在添加弹窗中展示状态字段。
- 后端生成初始随机密码或由创建成功反馈给管理员。

### 7.3 编辑用户规则

- 用户名在编辑弹窗中只读，不允许修改。
- 头像允许更换。
- 昵称允许修改，可为空。
- 角色允许修改。
- 状态不在编辑弹窗中展示，状态通过列表操作“冻结/解冻”维护。

## 8. 重置密码

- 操作入口：用户列表操作列“重置密码”。
- 点击后弹出确认提示。
- 确认后由系统生成随机密码。
- 生成后在二次弹窗中展示一次性随机密码，并提供复制按钮。
- 随机密码展示后，刷新或关闭弹窗不可再次查看，只能再次重置。

随机密码建议规则：

- 长度至少 12 位。
- 包含大写字母、小写字母、数字、特殊字符。
- 排除易混淆字符，如 O/0、I/l。

## 9. 冻结用户

- 操作入口：用户列表操作列“冻结”。
- 冻结后用户状态变更为“已冻结”。
- 已冻结用户不允许登录前台或后台。
- 冻结用户的操作按钮变为“解冻”。
- 解冻后状态恢复为“正常”。

## 10. 删除用户

- 删除为软删除。
- 只有“从未登录”的用户允许删除。
- 已登录过的用户不允许删除，操作按钮置灰并显示原因。
- 删除后状态变更为“已删除”，默认列表可通过筛选查看。
- 已删除用户不允许登录。

## 11. 分页规则

- 支持分页展示。
- 默认每页 10 条。
- 支持每页 10 / 20 / 50 条切换。
- 分页区展示当前范围，例如：`1-10 / 126`。

## 12. 交互与状态

| 场景 | 反馈 |
|---|---|
| 创建用户成功 | Toast：用户已创建 |
| 编辑用户成功 | Toast：用户信息已更新 |
| 重置密码成功 | 弹窗展示随机密码并可复制 |
| 冻结成功 | Toast：用户已冻结 |
| 解冻成功 | Toast：用户已恢复正常 |
| 删除成功 | Toast：用户已删除 |
| 无权限访问 | 展示无权限提示或重定向 |

## 13. UI 一致性要求

- 必须继承管理后台首页 Sidebar、卡片、表格、下拉框、按钮、分页视觉。
- 页面背景使用暗色工业风，不允许改为浅色后台模板。
- 品牌金仅用于 Logo、激活导航、主按钮、关键数字和重点状态。
- 表格行高、边框、文字透明度需要与后台首页保持一致。
- 页面右侧内容独立滚动，Sidebar 固定 100vh。
- HTML Golden Reference 为开发最高优先级视觉源。
