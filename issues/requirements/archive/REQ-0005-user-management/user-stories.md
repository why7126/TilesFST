## US-001 后台管理员维护系统用户

**作为** 后台管理员，  
**我希望** 在管理后台统一创建、编辑、冻结、删除用户并重置密码，  
**以便** 控制谁能访问前台与后台、保障账号安全。

### 验收要点

- 仅 `admin` 角色可见 SYSTEM 下「用户管理」菜单，可访问 `/admin/users`。
- 支持关键词模糊搜索（用户名、昵称、邮箱/手机号）。
- 支持按角色、状态、登录情况筛选，并展示用户总数、筛选结果数、正常/已冻结统计。
- 列表展示头像、用户名、昵称、角色、状态、最后登录、创建时间及操作列。
- 可通过弹窗添加用户（用户名 4–32 位、头像、昵称、角色），编辑时用户名只读。
- 可重置密码并在二次弹窗一次性展示随机密码（可复制）。
- 可冻结/解冻用户；仅「从未登录」用户可软删除。
- 分页默认 10 条，支持 10/20/50 切换。

### 关联功能

- FR-001：用户列表与筛选
- FR-002：添加/编辑用户弹窗
- FR-003：重置密码
- FR-004：冻结/解冻
- FR-005：软删除
- FR-006：权限与菜单可见性

## US-002 后台运营不可访问用户管理

**作为** 后台运营人员，  
**我希望** 进入后台后只看到日常运营菜单、看不到用户管理，  
**以便** 职责边界清晰，避免误操作账号与权限。

### 验收要点

- `employee` 角色登录后 Sidebar SYSTEM 分组不展示「用户管理」。
- 直接访问 `/admin/users` 时返回无权限提示或重定向至 `/admin/dashboard`。
- 后端用户管理 API 对非 `admin` 返回 HTTP 403。

### 关联功能

- FR-006：权限与菜单可见性

## US-003 前台用户由管理员代管（本期）

**作为** 前台业务用户（店主等），  
**我希望** 账号由管理员在后台创建与维护，  
**以便** 在尚未开放自助注册前也能正常使用前台系统。

### 验收要点

- 管理员可将用户角色设为「前台用户」（对应 `store_owner`）。
- 前台用户不允许登录管理端；冻结或删除后不允许登录前台。
- 昵称可为空，前台展示可回退为用户名。

### 关联功能

- FR-002：添加/编辑用户弹窗
- FR-004：冻结/解冻
