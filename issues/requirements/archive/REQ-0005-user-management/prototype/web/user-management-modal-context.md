# REQ-0005 用户管理页面 v1 - 添加用户弹窗原型上下文工程

## 1. 原型目标

本文件用于指导 Cursor / AI 前端开发还原 TILESFST 管理后台用户管理页面的添加/编辑用户弹窗状态。

开发时应优先参考：

1. `user-management-modal.html`
2. `user-management-modal.png`
3. `issues/requirements/archive/REQ-0005-user-management/requirement.md`

## 2. 页面状态

该原型为“列表页 + 添加用户弹窗打开”的状态。

背景页面保持用户管理列表页，覆盖一层暗色遮罩，中央显示添加用户弹窗。

## 3. 弹窗结构

```text
.modal-backdrop
└─ .modal-card
   ├─ .modal-head
   │  ├─ 标题：添加用户
   │  └─ 关闭按钮
   ├─ .modal-body
   │  ├─ 用户名
   │  ├─ 头像
   │  ├─ 昵称
   │  └─ 角色
   └─ .modal-footer
      ├─ 取消
      └─ 创建用户
```

## 4. 字段顺序

弹窗中每行一个信息，顺序必须固定为：

1. 用户名
2. 头像
3. 昵称
4. 角色

## 5. 字段规则

### 用户名

- 添加状态：可输入，必填。
- 编辑状态：只读，不允许修改。
- 规则：4–32 位；仅限小写字母、数字、`_`、`-`、`.`；须以字母开头；禁止连续特殊符号与保留关键字（见 `requirement.md` §7.2）。
- 辅助说明：`用户名创建后不可修改。`

### 头像

- 一行展示头像预览与上传按钮。
- 支持更换头像。
- 无头像时显示默认首字母头像。

### 昵称

- 可为空。
- 默认为空。
- 辅助说明：`未填写时前台可展示用户名。`

### 角色

可选项：

- 前台用户
- 后台运营
- 后台管理员

不再展示超级管理员。

## 6. 状态字段

添加/编辑用户弹窗中不展示状态字段。

新用户创建后默认状态为正常，状态变更通过列表操作冻结/解冻完成。

## 7. 视觉规范

- 弹窗宽度：520px。
- 背景：`rgba(33,30,22,.98)`。
- 边框：`0.5px solid rgba(255,255,255,.1)`。
- 遮罩：`rgba(0,0,0,.62)`。
- 主按钮：品牌金实底。
- 输入框高度：42px。
- 每个表单项之间间距：16px。
- 不允许表单项左右双列布局。

## 8. 一致性检查清单

- [ ] 弹窗字段为单列。
- [ ] 字段顺序为用户名、头像、昵称、角色。
- [ ] 不展示状态字段。
- [ ] 角色选项只有前台用户、后台运营、后台管理员。
- [ ] 背景列表页被遮罩但仍可识别。
- [ ] 弹窗按钮样式与后台页面一致。
