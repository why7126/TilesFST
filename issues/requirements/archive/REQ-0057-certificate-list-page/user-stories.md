# 用户故事

## US-001 浏览公开证书列表

作为装修客户，我希望在小程序“证书”Tab 中看到企业公开证书，以便快速判断品牌资质和可信度。

验收要点：

- 证书 Tab 展示真实列表，不再停留在建设中占位页。
- 列表仅展示后台设置为前台展示且未删除的证书。
- 每张证书卡片展示证书名称、所属品牌、证书类型和有效期信息。
- 首屏加载、空状态和网络失败状态均有稳定反馈。

## US-002 搜索和筛选证书

作为设计师，我希望按关键词、品牌、类型和有效状态筛选证书，以便快速找到客户关心的资质材料。

验收要点：

- 关键词支持证书名称、证书编号、发证机构和品牌名称。
- 类型、品牌和有效状态筛选变化后列表回到第一页。
- 空结果区分“暂无公开证书”和“没有符合筛选条件的证书”。
- 筛选状态可清除，清除后恢复默认列表。

## US-003 预览证书文件

作为门店导购，我希望点击证书卡片后能预览图片或 PDF 证书，以便在现场向客户展示证据材料。

验收要点：

- 图片证书可通过小程序图片预览能力查看。
- PDF 证书使用受控 URL 打开、复制或等价方式处理。
- 文件 URL 缺失、类型未知或预览失败时展示稳定提示。
- 预览失败不影响用户返回证书列表继续浏览。

## US-004 在移动端稳定浏览

作为普通访客，我希望证书页在不同手机宽度下布局稳定，以便不被导航栏、TabBar 或长文本遮挡。

验收要点：

- 页面使用全局自定义导航栏，标题为“证书”。
- 320/375/430 pt 视口下首屏内容、筛选区、卡片和底部 TabBar 不互相遮挡。
- 长证书名称、长品牌名、PDF 占位和图片失败占位不撑破卡片。
- 下拉刷新和触底加载更多不会重复请求或造成明显布局跳动。

## US-005 记录证书浏览行为

作为产品运营人员，我希望记录证书页曝光、筛选、点击和预览行为，以便评估证书内容对用户信任的帮助。

验收要点：

- 页面曝光记录来源入口和初始筛选条件。
- 搜索/筛选记录筛选类型、筛选值和结果数量。
- 证书点击记录证书 ID、品牌 ID、证书类型和列表位置。
- 预览点击记录文件类型、预览方式和成功/失败结果。
