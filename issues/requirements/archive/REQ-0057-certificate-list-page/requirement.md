# REQ-0057 新增证书列表页

## 1. 需求背景

菲尚特瓷砖微信小程序已将底部 TabBar 规划为“首页、分类、找砖、收藏、证书”，其中“证书”页当前仍为建设中占位页。后台已通过 `REQ-0038-brand-certificate-management` 建立品牌证书主数据、展示控制、文件上传和管理端证书列表，具备向公开端输出品牌资质、检测报告、绿色建材与荣誉证书的基础。

本需求将小程序“证书”Tab 从占位页升级为公开证书聚合列表页，让装修客户、设计师和门店访客可以集中浏览企业可公开展示的证书资料，增强品牌信任感，并为后续证书详情、品牌详情联动预留空间。

## 2. 目标用户

1. 装修客户：希望快速查看品牌资质、检测报告和环保认证，辅助选砖决策。
2. 设计师：需要向客户展示企业资质与产品可信度。
3. 门店导购：需要在小程序中快速打开证书页，辅助现场介绍。

## 3. 产品目标

1. 将小程序 TabBar“证书”占位页升级为可浏览的公开证书列表页。
2. 仅展示后台设置为前台展示、未删除且可公开访问的证书。
3. 支持按证书关键词、类型、品牌和有效状态进行轻量筛选或搜索。
4. 为图片证书和 PDF 证书提供稳定的缩略图、占位和预览入口。
5. 保持小程序暗色高端视觉体系，与首页、分类页和商品列表页体验一致。

## 4. 范围

### 4.1 包含

- 小程序 `pages/certificates/index` 证书列表页。
- 证书 TabBar 入口从占位页切换为真实列表页。
- 证书列表、筛选、搜索、分页或加载更多。
- 证书卡片展示：证书名称、所属品牌、证书类型、证书编号或发证机构、有效期、有效状态、文件缩略图或 PDF 占位。
- 图片证书预览与 PDF 受控文件 URL 打开能力。
- 加载态、空状态、错误态、图片失败兜底和重复点击防护。
- 公开证书列表 API 或等价后端能力，必须只返回允许公开展示的数据。
- 小程序埋点：页面曝光、筛选切换、证书点击、预览点击和加载失败。

### 4.2 不包含

- 管理端品牌证书新增、编辑、上传、显示/隐藏、删除能力。
- 后台证书主数据模型重建。
- 证书审批、真伪校验、OCR 识别、电子签章。
- 证书与 SKU 单独绑定。
- 完整证书详情页，除非后续需求明确。
- 展示后台草稿、隐藏、软删除或内部备注字段。
- 前端直连未授权对象存储或暴露原始对象 Key。

## 5. 功能要求

### FR-001 证书 Tab 真实可用

- 用户点击底部 TabBar“证书”后，进入证书列表页。
- 页面不得继续展示“功能建设中”作为主体验。
- 首屏加载期间展示与最终布局一致的骨架屏。
- 网络异常时展示重试入口；若存在可用缓存，可先展示缓存并静默刷新。

### FR-002 公开证书数据范围

- 列表仅展示后台未删除且 `is_visible=true` 的证书。
- 若证书所属品牌已被停用或不可公开展示，默认不得出现在公开列表中。
- 证书文件 URL 必须为后端控制的可读 URL 或等价安全引用，不得暴露对象存储原始 Key。
- 后端必须按排序值、品牌或更新时间提供稳定排序，避免翻页重复或遗漏。

### FR-003 证书卡片展示

- 每张证书卡片必须展示证书名称、所属品牌和证书类型。
- 证书编号存在时优先展示证书编号；不存在时可展示发证机构或文件名。
- 有效期展示需覆盖长期有效、起止日期、仅到期日期、未设置和已过期。
- 图片证书展示稳定比例缩略图；PDF 证书展示统一 `PDF` 文件占位。
- 图片加载失败、文件 URL 缺失或文件类型未知时展示统一占位，不显示破图。

### FR-004 搜索与筛选

- 页面支持按关键词搜索证书名称、证书编号、发证机构和品牌名称。
- 页面支持按证书类型筛选：质量体系、产品检测、绿色环保、荣誉资质、其他证书等后台枚举。
- 页面支持按品牌筛选，品牌筛选项只包含存在公开证书的品牌或后端返回的可筛选品牌。
- 页面支持按有效状态筛选：全部、长期有效、有效、即将到期、已过期、未设置。
- 筛选条件变化后应回到第一页，并保持列表加载状态明确。

### FR-005 分页与加载更多

- V1 优先采用小程序常见的下拉刷新和触底加载更多。
- 首次加载、加载更多和刷新应有独立状态，避免重复请求。
- 无更多数据时展示轻量提示，不遮挡列表内容。
- 空结果需区分“暂无公开证书”和“没有符合筛选条件的证书”。

### FR-006 预览与跳转

- 用户点击证书缩略图、卡片主区域或“查看”入口时，可预览图片证书或打开 PDF。
- 图片证书优先使用小程序图片预览能力。
- PDF 证书可通过受控 URL 打开或复制提示，具体方式以后续技术设计确认。
- 预览失败时展示稳定错误提示，并允许用户返回列表继续浏览。
- V1 不强制实现证书详情页；若后续需要详情页，应拆分或扩展需求。

### FR-007 埋点

- 页面曝光：记录来源入口和筛选初始状态。
- 搜索/筛选：记录筛选类型、筛选值和结果数量。
- 证书点击：记录证书 ID、品牌 ID、证书类型、列表位置。
- 预览点击：记录文件类型、预览方式和是否成功。
- 加载失败：记录错误码、是否有缓存和当前筛选条件。

## 6. UI 约束

- 视觉风格延续小程序“工业石材 · 暗色旗舰风”，与首页、分类页、商品列表页保持统一。
- 页面使用全局自定义导航栏，标题为“证书”。
- 列表卡片应适合移动端单列浏览，不使用管理端表格布局。
- 证书缩略图、PDF 占位、Badge 和按钮需有稳定尺寸，避免加载后布局跳动。
- 筛选控件应轻量，不遮挡主要浏览路径；筛选状态应可清除。
- 长文本必须截断或换行，不得撑破卡片。
- 空状态、错误态和加载态不得出现技术解释文案。

## 7. 接口建议

```http
GET /api/v1/miniapp/certificates
```

建议查询参数：

| 参数 | 说明 |
|---|---|
| `keyword` | 证书名称、编号、发证机构、品牌名称 |
| `brand_id` | 所属品牌 |
| `type` | 证书类型 |
| `validity_status` | 有效状态 |
| `page` | 页码 |
| `page_size` | 每页条数 |

响应要求：

- 返回 `items`、`pagination` 和筛选辅助数据或等价结构。
- 列表项应包含证书 ID、名称、类型、编号、发证机构、品牌 ID、品牌名称、文件 URL、文件名、MIME、有效期字段、有效状态和排序字段。
- API 必须复用统一响应 envelope。
- API 变更后必须同步 OpenAPI、Orval、小程序服务层和接口测试。

## 8. 数据与权限约束

- 数据事实源为后台品牌证书主数据。
- 小程序端只读公开证书，不提供任何维护操作。
- 后端必须过滤隐藏、软删除和不可公开品牌证书。
- 文件读取必须遵守 MinIO 单桶和后端受控 URL 策略。
- 不得把后台备注、审计字段、内部用户字段返回给公开端。
