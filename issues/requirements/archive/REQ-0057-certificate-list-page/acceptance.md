# 验收标准

## 功能 AC

- [ ] AC-001 小程序 TabBar「证书」进入 `pages/certificates/index` 后展示真实证书列表页，不再以“功能建设中”作为主体验。
- [ ] AC-002 公开证书列表仅返回并展示未删除、`is_visible=true` 且所属品牌允许公开的证书。
- [ ] AC-003 API 响应不包含后台备注、审计字段、内部用户字段、对象存储原始 Key 或未授权直连地址。
- [ ] AC-004 证书卡片展示证书名称、所属品牌、证书类型、证书编号或发证机构、有效期/有效状态。
- [ ] AC-005 图片证书使用固定比例缩略图；PDF 证书展示统一 PDF 占位；文件缺失、类型未知或图片加载失败时不出现破图。
- [ ] AC-006 关键词搜索支持证书名称、证书编号、发证机构和品牌名称。
- [ ] AC-007 类型、品牌和有效状态筛选生效；筛选变化后回到第一页并展示明确加载状态。
- [ ] AC-008 下拉刷新刷新第一页；触底加载更多追加后续数据；重复触发不会产生并发重复请求。
- [ ] AC-009 空状态区分“暂无公开证书”和“没有符合筛选条件的证书”，筛选空结果提供清除筛选入口。
- [ ] AC-010 网络失败保留可用已加载数据或缓存，并提供重试入口；失败状态不导致页面白屏。
- [ ] AC-011 图片证书可通过小程序图片预览能力查看。
- [ ] AC-012 PDF 证书通过受控 URL 打开、复制提示或等价方式处理；预览失败展示稳定提示并可返回列表。
- [ ] AC-013 证书点击、预览点击和筛选操作具备防重复触发能力。
- [ ] AC-014 页面曝光、搜索/筛选、证书点击、预览点击、加载失败均记录埋点，参数包含证书、品牌、类型、列表位置和结果状态。
- [ ] AC-015 `GET /api/v1/miniapp/certificates` 或等价公开接口使用统一响应 envelope；API 变更同步 OpenAPI、Orval、小程序服务层和接口测试。

## UI AC

- [ ] AC-016 页面使用全局自定义导航栏，标题为“证书”，首屏内容不被状态栏、胶囊 reserve、自定义导航或 TabBar 遮挡。
- [ ] AC-017 320/375/430 pt DevTools 视口下，筛选区、证书卡片、加载态、空状态和错误态无横向滚动、无文本溢出、无按钮不可达。
- [ ] AC-018 卡片采用移动端单列浏览布局，不复用管理端表格结构。
- [ ] AC-019 长证书名称、长品牌名、长发证机构和未知有效状态均有截断、换行或降级展示策略，不撑破卡片。
- [ ] AC-020 视觉风格延续小程序深色高端体系，品牌金仅用于重点 Badge、激活态或主操作。
- [ ] AC-021 自定义导航、页面 offset 和截图 evidence 参考 `docs/knowledge-base/best-practices/miniapp-custom-navigation.md` 执行；真机不可用时必须标记 blocked 或 follow_up，不得写作真机通过。

## 数据与安全 AC

- [ ] AC-022 后端过滤隐藏、软删除和不可公开品牌证书，前端不得依赖本地过滤作为唯一安全边界。
- [ ] AC-023 文件读取遵守 MinIO 单桶和后端受控 URL 策略，不暴露真实 AccessKey、SecretKey、对象 Key 或本机路径。
- [ ] AC-024 小程序端只读公开证书，不提供新增、编辑、上传、显示/隐藏、删除或内部备注展示。
- [ ] AC-025 证书列表排序稳定，翻页或加载更多不会出现重复、漏项或顺序明显跳变。
