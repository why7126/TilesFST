## US-001 从侧栏菜单修改本人密码

**作为** 已登录的后台管理员或内部员工，  
**我希望** 在侧栏底部用户菜单点击「密码修改」后打开修改密码弹窗，  
**以便** 在不离开当前业务页的情况下安全更新登录密码。

### 验收要点

- 用户菜单含「个人资料、密码修改、退出登录」；点击「密码修改」打开 520px 居中弹窗，非跳转页面。
- 弹窗含原密码、新密码、确认新密码三字段；默认聚焦原密码；打开时禁止主体滚动。
- 三字段均支持显隐切换；新密码下方展示三条规则提示。
- 取消、×、Esc 可关闭；有输入时关闭须二次确认。
- 校验通过后保存；成功 Toast 并跳转登录页。

### 关联功能

- FR-001 ~ FR-005、FR-010（见 `requirement.md`）

## US-002 明确知道密码规则与错误原因

**作为** 正在修改密码的后台用户，  
**我希望** 在输入过程中看到规则是否满足，并在出错时看到字段级提示，  
**以便** 快速修正输入并成功保存。

### 验收要点

- 前端即时校验：长度 8–32、字母+数字、新旧不同、两次一致。
- 不满足规则时字段边框危险色 + inline 文案（与 FR-005 表一致）。
- 原密码错误、弱密码、过于频繁等后端错误展示接口 message。
- 保存中按钮禁用，文案「保存中…」，不可重复提交。

### 关联功能

- FR-002、FR-005、FR-008

## US-003 改密后全端登录态失效

**作为** 系统安全策略，  
**我希望** 用户成功修改密码后所有已签发 token 失效，  
**以便** 防止旧密码对应的 JWT 在其他 tab 或设备继续访问后台。

### 验收要点

- 改密成功后端 `token_version += 1`；旧 JWT 调用任意受保护接口返回 401。
- 当前浏览器清 token、清 remember 凭证并跳转 `/admin/login`。
- 使用新密码可重新登录；新 JWT 含更新后的 `tv`。

### 关联功能

- FR-006、FR-007、FR-009
