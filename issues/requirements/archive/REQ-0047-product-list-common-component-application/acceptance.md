# 验收标准

## 功能 AC

- [ ] AC-001 分类页二级分类点击后进入 `pages/product-list/index?categoryId={id}`。
- [ ] AC-002 商品列表页可接收 `categoryId`、`keyword`、`brandId`、`spec`、`priceRange`、`sort`、`page` 和 `pageSize` 等基础参数。
- [ ] AC-003 商品列表容器支持分类、搜索、品牌、推荐等上下文复用。
- [ ] AC-004 商品列表容器统一承载首屏加载、刷新、加载更多、无更多数据、空状态和错误状态。
- [ ] AC-005 商品卡片展示主图、商品名称、SKU 编码、品牌、规格和参考价格。
- [ ] AC-006 商品卡片可展示分类名称或适用空间等辅助信息，但不得挤压核心标题和图片。
- [ ] AC-007 商品图片加载失败时展示统一占位，不展示破图。
- [ ] AC-008 商品卡片整卡点击进入 SKU 详情页。
- [ ] AC-009 v1 不在商品卡片上提供收藏、加入询价、购物车或快捷联系商家按钮。
- [ ] AC-010 列表页提供品牌、分类、规格等基础筛选入口。
- [ ] AC-011 完整筛选使用底部抽屉呈现，并支持重置和确认。
- [ ] AC-012 筛选确认后重置分页并重新拉取第一页。
- [ ] AC-013 已选筛选项在列表顶部以紧凑标签展示，并支持快速移除。
- [ ] AC-014 筛选项数量基于当前上下文动态聚合；接口暂不支持时可隐藏数量。
- [ ] AC-015 列表页支持默认排序、最新、价格升序和价格降序。
- [ ] AC-016 切换排序后重置分页、回到列表顶部并重新请求第一页。
- [ ] AC-017 v1 不支持个性化推荐排序、人工置顶配置或复杂多字段排序编辑。
- [ ] AC-018 首屏请求第一页数据。
- [ ] AC-019 下拉刷新清空旧分页游标并重新请求第一页。
- [ ] AC-020 上拉加载更多防止重复请求。
- [ ] AC-021 接口返回无更多数据时展示统一无更多状态。
- [ ] AC-022 请求失败时保留已加载数据，并提供重试入口。
- [ ] AC-023 参数无效、分类或品牌已下架时展示可恢复的空状态或返回提示。
- [ ] AC-024 搜索上下文下的商品列表保留关键词展示，筛选和排序请求不得丢失关键词。
- [ ] AC-025 无商品时按上下文展示分类无商品、搜索无结果或筛选无匹配文案。
- [ ] AC-026 搜索无结果时提示用户缩短关键词、替换品牌/规格或清空筛选。
- [ ] AC-027 商品列表记录列表页浏览、商品曝光、商品点击、筛选打开、筛选应用、排序切换、刷新、加载更多和加载失败事件。
- [ ] AC-028 埋点公共参数包含 sourcePage、categoryId、brandId、keyword、filterSnapshot、sort、page、pageSize、resultCount、skuId 和 requestId。
- [ ] AC-029 日志不得记录手机号等与商品浏览无关的个人敏感信息。

## UI AC

- [ ] AC-UI-001 视觉风格与菲尚特小程序深色企业轻奢风保持一致。
- [ ] AC-UI-002 商品列表采用统一卡片密度，优先保证商品图、名称、SKU、品牌、规格和参考价格可扫读。
- [ ] AC-UI-003 商品图片区域使用稳定比例，加载前后无明显布局跳动。
- [ ] AC-UI-004 筛选、排序、搜索上下文标签保持紧凑，不遮挡商品内容。
- [ ] AC-UI-005 底部筛选抽屉适配小程序触控场景，重置和确认按钮清晰可见。
- [ ] AC-UI-006 加载态、空状态、错误态和无更多状态使用统一视觉语义。
- [ ] AC-UI-007 可点击区域不小于 44px。
- [ ] AC-UI-008 实现验收优先以 `prototype/miniapp/prototype.html` 与后续导出的 PNG 为视觉基准，`prototype/miniapp/context.md` 为交互补充说明。

## 横切 AC（knowledge-base）

无横切 AC。本需求为微信小程序商品列表体验，不命中 `admin-list`、`admin-form`、`admin-modal`、`media-upload` 标签；管理端列表、表单、弹窗、上传相关 best-practices 不适用。
