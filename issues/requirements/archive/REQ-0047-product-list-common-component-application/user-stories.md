# 用户故事

## US-001 用户从分类进入商品列表

作为装修客户或普通访客，我希望点击分类页二级分类后进入商品列表，以便浏览该分类下的瓷砖 SKU。

验收要点：

- 二级分类点击后进入 `pages/product-list/index?categoryId={id}`。
- 商品列表展示当前分类名称或上下文提示。
- 默认展示该分类下可用 SKU，按后台推荐或默认排序。
- 分类无商品时展示分类无商品空状态，不自动跳转其他分类。

## US-002 用户浏览商品卡片并进入详情

作为装修客户、设计师或门店导购，我希望在列表中快速识别商品关键信息，以便选择目标 SKU 进入详情页。

验收要点：

- 商品卡片展示主图、商品名称、SKU 编码、品牌、规格和参考价格。
- 商品图片区域比例稳定，加载前后不造成列表明显跳动。
- 图片加载失败时展示统一占位，不出现破图。
- 整个商品卡片可点击并进入 SKU 详情页。

## US-003 用户使用筛选缩小列表范围

作为设计师或门店导购，我希望按品牌、分类、规格和价格区间筛选商品，以便快速缩小候选范围。

验收要点：

- 列表页提供筛选入口，完整筛选使用底部抽屉。
- 筛选支持重置和确认。
- 确认筛选后重置分页并重新请求第一页。
- 已选筛选项在列表顶部以紧凑标签展示，并支持快速移除。

## US-004 用户调整排序

作为正在比较商品的用户，我希望可以切换默认、最新、价格升序或价格降序，以便按当前决策方式浏览商品。

验收要点：

- 列表页支持默认排序。
- v1 支持最新、价格升序、价格降序等基础排序。
- 切换排序后回到列表顶部，并重新加载第一页。
- v1 不支持个性化推荐排序、人工置顶配置或复杂排序编辑。

## US-005 用户刷新或继续加载列表

作为用户，我希望列表支持下拉刷新和上拉加载更多，以便持续浏览商品且不会重复加载。

验收要点：

- 首屏请求第一页数据。
- 下拉刷新清空旧分页游标并重新请求第一页。
- 上拉加载更多防止重复请求。
- 无更多数据时展示统一无更多状态。
- 加载更多失败时保留已加载数据，并提供重试入口。

## US-006 用户从搜索结果复用商品列表

作为通过搜索找砖的用户，我希望搜索结果中的 SKU 列表与分类商品列表体验一致，以便用同一种方式继续筛选、排序和进入详情。

验收要点：

- 搜索结果 SKU 列表可复用商品列表容器和商品卡片。
- 页面保留搜索关键词展示。
- 搜索上下文下的筛选和排序不会丢失关键词。
- 搜索无结果时展示关键词、调整建议和清空筛选入口。

## US-007 产品团队追踪商品列表行为

作为产品运营人员，我希望记录商品列表浏览、点击、筛选和异常行为，以便后续分析商品发现效率。

验收要点：

- 记录列表页浏览、商品曝光、商品点击、筛选打开、筛选应用、排序切换、刷新、加载更多和加载失败事件。
- 埋点公共参数包含 sourcePage、categoryId、brandId、keyword、filterSnapshot、sort、page、pageSize、resultCount、skuId 和 requestId。
- 日志不得记录手机号等与商品浏览无关的个人敏感信息。
