# 业务流程

## 1. 总流程

```text
用户进入小程序入口页面
  |
  +-- 分类页二级分类 ------+
  |                       |
  +-- 搜索结果 SKU Tab ----+--> 组装商品列表上下文
  |                       |
  +-- 首页推荐/品牌页 -----+
                          |
                          v
                    商品列表页通用容器
                          |
          +---------------+---------------+
          |                               |
          v                               v
     加载第一页商品                  展示筛选/排序入口
          |                               |
          +---------------+---------------+
                          |
           +--------------+--------------+
           |                             |
           v                             v
        有商品                        无商品/异常
           |                             |
           v                             v
      展示商品卡片                 空状态/错误/重试
           |
           +-- 点击商品 --> SKU 详情页
           |
           +-- 下拉刷新 / 上拉加载更多
           |
           +-- 筛选 / 排序 --> 重置分页并重新加载
```

## 2. 分类商品列表流程

| 步骤 | 说明 | 输出 |
|---|---|---|
| 1 | 用户在分类页点击二级分类 | 进入 `pages/product-list/index?categoryId={id}` |
| 2 | 商品列表页解析 `categoryId` | 展示分类上下文标题 |
| 3 | 请求第一页商品 | 展示骨架屏或加载态 |
| 4 | 接口返回商品 | 渲染商品卡片列表 |
| 5 | 用户点击商品卡片 | 进入 SKU 详情页 |
| 6 | 当前分类无商品 | 展示分类无商品空状态 |

## 3. 搜索结果 SKU 列表流程

| 步骤 | 说明 | 规则 |
|---|---|---|
| 1 | 搜索页提交关键词或切换 SKU Tab | 携带 `keyword` 进入商品列表上下文 |
| 2 | 商品列表容器加载 SKU 结果 | 保留关键词展示 |
| 3 | 用户使用筛选或排序 | 请求参数继续携带关键词 |
| 4 | 搜索无结果或筛选无匹配 | 展示关键词、调整建议和清空筛选入口 |

## 4. 筛选与排序流程

```text
打开列表页
  |
  v
点击筛选或排序入口
  |
  +-- 筛选 --> 底部抽屉选择品牌/分类/规格/价格
  |              |
  |              +-- 重置 --> 清空筛选草稿
  |              |
  |              +-- 确认 --> 更新 filterSnapshot
  |
  +-- 排序 --> 选择默认/最新/价格
                 |
                 v
           重置分页到第一页
                 |
                 v
           回到列表顶部并重新加载
```

## 5. 分页与异常流程

| 场景 | 处理 |
|---|---|
| 首屏加载 | 展示与最终布局一致的骨架屏 |
| 下拉刷新 | 清空分页游标，重新请求第一页 |
| 上拉加载更多 | 请求下一页，防止重复请求 |
| 无更多数据 | 展示统一无更多状态 |
| 加载更多失败 | 保留已加载数据，提供重试 |
| 首屏失败 | 展示错误状态和重试入口 |
| 图片失败 | 商品卡片内展示统一占位，不影响列表 |

## 6. 与关联需求差异

| 关联需求 | 差异 |
|---|---|
| REQ-0041-miniapp-home | 首页仅提供推荐或搜索入口，本需求承接商品列表展示与复用 |
| REQ-0044-miniapp-sku-detail-page | SKU 详情页是列表出口，本需求不改变详情页字段与布局 |
| REQ-0045-category-list-page | 分类页负责类目浏览，本需求负责二级分类后的商品列表页 |
| REQ-0046-search-component-application | 搜索组件负责搜索输入、联想和结果入口，本需求只复用 SKU 列表展示能力 |

## 7. 非目标流程

- 不进入 Web 管理后台商品管理、上下架、批量操作或价格维护流程。
- 不进入店主 Web 商品列表流程。
- 不提供购物车、询价、在线下单、收藏或客服找砖流程。
- 不处理商品数据模型新增、后台录入和复杂推荐算法。
