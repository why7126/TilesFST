# REQ-0041 微信小程序首页用户故事

## 1. 用户故事总览

| ID | 用户 | 优先级 | 故事 | 验收要点 |
|---|---|---|---|---|
| US-001 | 装修业主 | P1 | 作为正在选砖的业主，我希望进入小程序后立即看到品牌、搜索、Banner 和推荐商品，以便快速判断门店是否有适合我的瓷砖。 | 首页首屏展示品牌导航、搜索、Banner、快捷入口和至少一个推荐模块。 |
| US-002 | 室内设计师 | P1 | 作为设计师，我希望按空间、规格、风格、颜色快速找砖，以便根据客户方案筛选候选商品。 | 五个快捷入口可点击并进入带预置条件的搜索或筛选结果页。 |
| US-003 | 潜在到店客户 | P1 | 作为到店前的客户，我希望搜索商品名称、编号或规格，以便快速找到目标型号。 | 搜索页支持关键词查询，结果可进入商品详情页。 |
| US-004 | 装修业主 | P1 | 作为用户，我希望点击新品、热销或 Banner 商品后查看详情，以便确认图片、规格、品牌、适用空间和价格信息。 | 首页推荐、搜索结果和有效 Banner 跳转均可到达商品详情。 |
| US-005 | 潜在到店客户 | P1 | 作为用户，我希望从首页查看门店基础信息，以便了解门店名称和必要联系方式来源。 | 点击门店名称或门店入口进入门店信息页；联系方式字段来源待产品确认。 |
| US-006 | 门店销售 | P2 | 作为门店销售，我希望首页视觉接近高保真原型，以便在向客户展示时保持专业可信的品牌形象。 | 375x812 与 390x844 视口下，布局、色彩、圆角、间距和品牌金使用与原型一致。 |
| US-007 | 产品负责人 | P1 | 作为产品负责人，我希望首期实现分享、咨询和热销行为统计，同时仍排除收藏和预约，以便兼顾转化闭环和范围控制。 | 分享、咨询和行为统计有可验收闭环；收藏、预约、后台入口配置仍不进入本期。 |
| US-008 | 开发 / 测试人员 | P1 | 作为开发和测试人员，我希望首页数据复用现有后台/店主端能力，以便避免重复数据源和多套展示口径。 | 首页聚合接口只组合已有公开数据；不新增重复业务表；API 变化同步 OpenAPI/Orval/docs/tests。 |
| US-009 | 装修业主 | P1 | 作为用户，我希望能把首页或商品分享给家人、设计师，以便共同讨论选砖方案。 | 首页或商品详情支持微信原生分享，并记录分享行为。 |
| US-010 | 潜在到店客户 | P1 | 作为用户，我希望能从首页或商品详情发起咨询，以便快速联系门店确认商品与服务。 | 至少一种咨询方式可用；缺失配置时安全隐藏或降级，并记录咨询行为。 |
| US-011 | 运营 / 产品负责人 | P1 | 作为运营人员，我希望热销推荐能参考详情访问、分享、咨询等行为，以便推荐更贴近用户兴趣的商品。 | 记录详情访问、分享、咨询等行为；热销排序人工配置优先，行为统计作为补充。 |

## 2. 关键验收线索

- 首期闭环是：首页 → 搜索页 / 筛选结果 → 商品详情页，以及首页 → 门店信息页。
- Banner 跳转必须安全降级：配置目标不可达时不得出现空白页、小程序报错或死链。
- 原型中的收藏心形、预约等能力不得被误实现为半成品入口。
- 图片展示必须通过后端授权 URL 或现有安全适配层，不得让小程序直连未授权对象存储。
- 后台快捷入口配置、服务入口配置、收藏、预约和复杂用户画像仍应拆分后续需求。
