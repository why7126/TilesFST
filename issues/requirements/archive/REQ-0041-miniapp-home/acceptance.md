# REQ-0041 微信小程序首页验收标准

## 1. 功能 AC

- [ ] AC-001 首页使用原生微信小程序实现，页面代码归属 `src/miniapp/`，不写入 `src/web/`、项目根目录或 `scripts/`。
- [ ] AC-002 首页首屏在 375x812 与 390x844 视口下展示品牌导航、搜索入口、Banner、快捷入口和至少一个推荐模块。
- [ ] AC-003 顶部品牌导航展示菲尚特 Logo 与门店名称，点击门店名称或门店区域进入门店信息页。
- [ ] AC-004 搜索入口提示语为“搜索瓷砖名称、编号或规格”，点击后进入搜索页。
- [ ] AC-005 搜索页支持按商品名称、商品编号、品牌名称、规格、系列名称进行查询，搜索结果可进入商品详情页。
- [ ] AC-006 Banner 复用已有后台 Banner 配置能力，展示图片、标题、副标题和有效跳转；无有效跳转目标时安全降级，不出现空白页或小程序报错。
- [ ] AC-007 快捷入口固定展示“按空间、按规格、按风格、按颜色、全部分类”，点击后进入带预置筛选条件的搜索页或筛选结果页。
- [ ] AC-008 新品推荐展示 3 到 6 个商品卡片；无新品数据时隐藏模块，不展示空白卡片。
- [ ] AC-009 热销推荐人工配置优先；详情访问、分享、咨询统计作为辅助依据；统计不足时按时间或排序字段降级。
- [ ] AC-010 点击首页推荐商品、搜索结果或有效 Banner 商品目标后，可进入商品详情页。
- [ ] AC-011 商品详情页展示公开商品核心字段，至少包含商品名称、商品编号、主图/纹理图、规格、品牌、类目、适用空间、风格标签和价格展示信息。
- [ ] AC-012 首页服务区如展示“联系门店”等文案，必须至少支持一种可用咨询方式；缺少配置的咨询方式必须隐藏或安全降级。
- [ ] AC-013 底部 TabBar 首页为选中态；收藏功能暂不实现时，不展示收藏 Tab 或按产品确认方案隐藏。
- [ ] AC-014 首页不出现新增、编辑、上下架、库存、订单、客户管理等后台管理入口。
- [ ] AC-015 首页或商品详情支持微信原生分享，并记录分享行为。
- [ ] AC-016 首页或商品详情支持咨询入口，并记录咨询行为。
- [ ] AC-017 商品详情访问行为被记录，可用于热销推荐统计。

## 2. 范围控制 AC

- [ ] AC-SCOPE-001 本期不实现收藏功能，包括首页收藏按钮、商品收藏状态、收藏列表和收藏统计。
- [ ] AC-SCOPE-002 本期实现分享和咨询，但不实现复杂用户画像、分享裂变活动或多渠道咨询工单管理。
- [ ] AC-SCOPE-003 本期不实现预约表单，包括到店预约入口、表单提交和预约管理。
- [ ] AC-SCOPE-004 本期不定义到店询价规则；价格展示规则和 `priceDisplayMode` 需产品确认后再实现。
- [ ] AC-SCOPE-005 快捷入口后台配置、服务入口后台配置必须拆分后续 REQ，不得混入本 Change。
- [ ] AC-SCOPE-006 热销行为统计不包含收藏量，因为收藏功能不在本期。

## 3. UI / UE AC

- [ ] AC-UI-001 首页视觉与 `prototype/miniapp/prototype.png`、`prototype/miniapp/prototype.html` 保持一致，采用暖白背景、墨黑主视觉和品牌金点缀。
- [ ] AC-UI-002 页面在 320 到 430 pt 宽度下无页面级横向滚动、明显内容截断、控件重叠或底部 TabBar 遮挡。
- [ ] AC-UI-003 所有可点击区域不小于 44x44 pt，适合单手触控。
- [ ] AC-UI-004 Banner、搜索框、商品卡片、快捷入口图标底板的圆角和间距与原型保持同等级视觉比例。
- [ ] AC-UI-005 瓷砖图片优先展示纹理清晰的商品图或空间效果图，不使用与瓷砖无关的装饰图。
- [ ] AC-UI-006 原型中的收藏心形按钮在本期实现中必须移除、隐藏或置为非交互展示，不能产生半成品收藏入口。
- [ ] AC-UI-007 页面不采用电商促销红色风格，不引入复杂表格、密集表单、过度渐变、玻璃拟态或强阴影。

## 4. 状态与降级 AC

- [ ] AC-STATE-001 首屏加载时展示骨架屏或轻量加载态，不出现大面积空白。
- [ ] AC-STATE-002 Banner、推荐商品或图片模块加载失败时模块级降级，不阻塞首页其他模块展示。
- [ ] AC-STATE-003 推荐商品为空时隐藏对应推荐模块；所有商品为空时展示品牌化空状态。
- [ ] AC-STATE-004 网络异常时展示重试入口，点击后重新请求首页数据。
- [ ] AC-STATE-005 图片加载失败时展示统一瓷砖占位图或品牌占位图。
- [ ] AC-STATE-006 Banner 配置指向未实现能力时，必须安全降级，不能导致小程序路由错误。

## 5. API / 数据 / 安全 AC

- [ ] AC-API-001 首页数据复用现有店主端/后台接口和数据模型聚合，不新增重复业务数据表。
- [ ] AC-API-002 如新增或调整 `GET /api/miniapp/home` 或等价聚合接口，必须说明请求、响应、错误码，并同步 OpenAPI、Orval、docs 和测试。
- [ ] AC-API-003 聚合接口只返回允许公开展示的数据，不暴露后台内部字段、未授权素材、库存管理、内部备注或敏感配置。
- [ ] AC-API-004 小程序端只能访问后端授权接口和后端返回的安全图片 URL，不得直连未授权对象存储。
- [ ] AC-API-005 商品卡片至少可映射 `productId`、`productName`、`skuCode`、`coverImage`、`specification`；其他字段按 PRD 表格逐项确认。
- [ ] AC-API-006 行为统计至少支持商品详情访问事件；分享和咨询事件应按实际入口记录。
- [ ] AC-API-007 若现有行为日志不足以支持热销统计，可新增或扩展统计事件/聚合字段，但必须同步数据库文档、API 文档、OpenAPI、Orval 和测试。
- [ ] AC-API-008 行为统计不得记录用户敏感信息；必要时只记录匿名事件、商品 ID、事件类型和时间。

## 6. 测试与验证 AC

- [ ] AC-TEST-001 后续实现阶段必须补充小程序首页核心流程测试或等价 smoke 验收：首页加载、搜索入口、快捷入口、商品详情跳转、门店信息跳转、分享、咨询。
- [ ] AC-TEST-002 后续实现阶段必须验证 375x812、390x844，以及 320 到 430 pt 宽度范围内的布局表现。
- [ ] AC-TEST-003 若新增后端 API contract，必须补充后端集成测试、失败场景测试，并运行 OpenAPI / Orval 同步校验。
- [ ] AC-TEST-004 Docker Compose 默认 N/A：除非新增或修改代理、环境变量、Nginx、对象存储或容器网络边界，否则不要求 Docker Compose 验证。
- [ ] AC-TEST-005 验收报告必须记录不做项检查结果，确认收藏、预约、后台配置、到店询价和复杂用户画像未被误纳入。
- [ ] AC-TEST-006 验收报告必须记录热销行为统计证据，至少包含详情访问事件，建议包含分享和咨询事件。

## 7. 知识库横切 AC

本 REQ 不命中 `req-complete` 规定的 `admin-list`、`admin-form`、`admin-modal`、`media-upload` 标签，因此无 `AC-XCUT-*` 管理端横切验收项。参考的复盘经验为：`docs/knowledge-base/retrospectives/sprint-007-retrospective.md` 中关于 XL 能力分层验收、成功路径摘要输出和范围控制的经验，已转化到 API / 数据 / 测试 / 范围控制 AC。
