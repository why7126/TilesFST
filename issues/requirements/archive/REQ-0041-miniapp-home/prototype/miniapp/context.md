# 微信小程序首页原型上下文

## 1. 原型文件

| 文件 | 说明 |
|---|---|
| `prototype.html` | 外部附件复制的高保真 HTML 原型，用于布局、层级和交互参考。 |
| `prototype.png` | 外部附件复制的静态截图，用于视觉 golden reference。 |
| `assets/logo.png` | 原型依赖的菲尚特 Logo。 |

## 2. 实现优先级

后续实现与验收优先级：

```text
prototype.html > prototype.png > requirement.md > acceptance.md > rules/ui-design.md
```

若原型与 PRD 范围冲突，以 PRD 和 acceptance 的范围控制为准。例如原型中出现收藏心形，但本期仍不实现收藏能力，因此实现阶段应隐藏、移除或置为非交互展示；联系门店和分享能力已纳入本期，应具备可用闭环。

## 3. 视觉关键词

- 暖白页面背景。
- 墨黑主视觉 Banner。
- 品牌金用于激活态、重点按钮、价格或强调信息。
- 现代、克制、轻奢、专业建材展厅感。
- 商品图片和空间效果图应优先表达瓷砖纹理，不使用无关装饰图。

## 4. 首期闭环

```text
首页
├─ 门店名称 -> 门店信息页
├─ 搜索框 -> 搜索页 -> 商品详情页
├─ 快捷入口 -> 搜索/筛选结果页 -> 商品详情页
├─ 新品/热销商品 -> 商品详情页
│                    ├─ 记录详情访问
│                    ├─ 分享商品
│                    └─ 咨询门店
└─ Banner -> 商品详情页 / 搜索结果页 / 门店信息页 / 安全降级
```

## 5. 明确不做

- 收藏。
- 到店预约。
- 到店询价规则。
- 快捷入口后台配置。
- 服务入口后台配置。
- 收藏量驱动的热销算法。
- 复杂用户画像或多渠道咨询工单管理。

## 6. 验收视口

实现阶段至少检查：

- `375x812`
- `390x844`
- `320 到 430 pt` 宽度范围

验收重点是首屏可见性、触控区域、无横向滚动、TabBar 安全区和文字不重叠。
