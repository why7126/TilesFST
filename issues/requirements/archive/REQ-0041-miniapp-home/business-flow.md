# REQ-0041 微信小程序首页业务流程

## 1. 首期业务闭环

```text
用户打开微信小程序
  |
  v
首页加载聚合数据
  |
  +--> 读取门店基础信息
  +--> 读取可展示 Banner
  +--> 读取新品推荐商品
  +--> 读取热销推荐商品（人工配置优先/行为统计辅助/排序字段降级）
  +--> 读取默认快捷入口（固定配置）
  |
  v
首页首屏展示
  |
  +--> 点击门店名称 ----------> 门店信息页
  |
  +--> 点击搜索框 ------------> 搜索页
  |                              |
  |                              v
  |                         搜索结果页
  |                              |
  |                              v
  |                         商品详情页
  |
  +--> 点击快捷入口 ----------> 搜索页/筛选结果页
  |                              |
  |                              v
  |                         商品详情页
  |
  +--> 点击新品/热销商品 ------> 商品详情页
  |                              |
  |                              +--> 记录详情访问行为
  |                              +--> 用户分享商品 -> 记录分享行为
  |                              +--> 用户咨询门店 -> 记录咨询行为
  |
  +--> 点击有效 Banner --------> 商品详情页 / 搜索结果页 / 门店信息页
                                 |
                                 v
                           不可达目标安全降级
```

## 2. 页面与数据关系

| 页面 / 模块 | 数据来源 | 本期处理 |
|---|---|---|
| 首页品牌导航 | 门店基础信息、品牌 Logo | 复用现有配置或后端返回；点击进入门店信息页。 |
| Banner | 已有后台 Banner 配置 | 复用已有配置能力；不可达跳转安全降级。 |
| 快捷入口 | 固定默认入口 | 不依赖后台配置；入口配置另拆需求。 |
| 新品推荐 | 现有 SKU / 商品数据 | 以新品标识、时间或已有排序字段展示。 |
| 热销推荐 | 现有 SKU / 商品数据 + 行为统计 | 人工配置优先；详情访问、分享、咨询统计作为辅助；统计不足时按时间/排序字段降级。 |
| 搜索页 | 现有商品、品牌、规格、系列数据 | 支持关键词检索并进入详情。 |
| 商品详情页 | 现有公开 SKU / 商品数据 | 仅展示公开字段。 |
| 门店信息页 | 门店基础信息 | 联系方式字段来源待确认。 |

## 3. 异常与降级流程

```text
首页请求失败
  |
  +--> 网络异常：展示重试入口
  +--> Banner 失败：隐藏 Banner 或展示占位，不阻塞推荐商品
  +--> 推荐为空：隐藏对应模块
  +--> 全部商品为空：展示品牌化空状态
  +--> 图片失败：展示统一瓷砖/品牌占位图
  +--> Banner 跳转目标不可达：停留当前页或跳转安全 fallback，不报错
```

## 4. 与现有能力差异

| 能力 | 现状 / 依赖 | REQ-0041 处理 |
|---|---|---|
| SKU 管理 | 后台维护瓷砖 SKU 和素材 | 只消费公开展示字段，不新增后台编辑能力。 |
| Banner 管理 | 后台已有完整配置能力 | 复用已有 Banner 配置。 |
| 小程序预览 | REQ-0021 是 SKU 上线前预览能力 | 本需求是终端客户实际小程序首页，不作为 REQ-0021 refinement。 |
| 分享 | 微信小程序原生能力 | 本期纳入首页/商品分享，并记录分享行为。 |
| 咨询 | 门店联系方式 / 微信客服配置待确认 | 本期至少支持一种咨询方式；缺失配置时隐藏或降级，并记录咨询行为。 |
| 收藏 / 预约 | 当前无对应首期闭环 | 收藏和预约仍明确排除，后续独立需求。 |
| 快捷入口 / 服务入口配置 | 后台暂无配置能力 | 本期固定展示，后台配置另拆。 |
| 热销算法 | 缺历史行为基础 | 本期采集详情访问、分享、咨询行为；人工配置优先，统计不足时按时间/排序字段降级。 |

## 5. 实施顺序建议

1. 确认原生小程序目录、构建方式和页面路由。
2. 定义首页聚合响应或复用接口组合方式，确保只消费公开数据。
3. 搭建首页基础布局与 TabBar。
4. 接入 Banner、新品、热销、快捷入口和门店信息。
5. 补齐搜索页、商品详情页、门店信息页、分享与咨询的首期闭环。
6. 接入详情访问、分享、咨询行为统计，并用于热销推荐辅助排序。
7. 完成加载、空、异常、图片失败与不可达跳转降级。
8. 使用原型在 375x812、390x844、320 到 430 pt 宽度范围内做视觉与触控验收。

## 6. 父子需求关系

本需求暂无父需求。它与 `REQ-0021-sku-web-miniapp-preview` 都涉及小程序展示视图，但业务目标不同：

- `REQ-0021`：管理端在 SKU 上线前预览 Web / 小程序展示效果。
- `REQ-0041`：面向终端客户的真实小程序首页首期闭环。

因此本需求保持独立 REQ，后续若需要管理端配置快捷入口、服务入口、收藏、预约或复杂咨询管理，应拆分新的 peer REQ。
