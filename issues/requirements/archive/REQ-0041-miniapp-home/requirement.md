# REQ-0041 微信小程序首页

## 1. 需求背景

菲尚特已具备瓷砖信息后台管理能力，后台可维护品牌、类目、规格、SKU、Banner、商品素材等基础数据。当前项目需要补齐面向终端客户的微信小程序首页，让用户在微信内快速完成“进入门店展厅 - 查看主推内容 - 按条件找砖 - 搜索商品 - 查看详情 - 了解门店”的首期闭环。

本需求不把小程序做成后台管理入口，也不在首页承载新增、编辑、库存、订单、客户管理等内部操作。小程序首页应作为品牌数字展厅的第一屏，优先服务装修业主、设计师、潜在客户和门店销售的展示转发场景。

本需求基于 `capture.md` 与已沉淀原型：

- `prototype/miniapp/prototype.html`
- `prototype/miniapp/prototype.png`
- `prototype/miniapp/assets/logo.png`

附件中的外部版本号不作为本项目需求版本事实；本 PRD 版本以 frontmatter `version: v1` 为准。

## 2. 目标用户

| 用户 | 诉求 |
|---|---|
| 装修业主 | 在微信内快速了解菲尚特门店、浏览新品/热销瓷砖，并进入商品详情查看纹理、规格和适用空间。 |
| 室内设计师 | 按空间、规格、风格或颜色快速筛选产品，用于客户沟通和方案选材。 |
| 潜在到店客户 | 到店前远程查看门店主推商品、搜索目标型号，并了解门店基础信息。 |
| 门店销售 | 使用小程序作为轻量数字展厅，辅助给客户展示首页、搜索结果和商品详情。 |

## 3. 需求目标

- 建立原生微信小程序首页，承接品牌展示与选砖入口。
- 首屏展示品牌、搜索、Banner、快捷入口与推荐商品，降低用户理解成本。
- 复用现有店主端/后台数据能力聚合首页内容，不新增重复数据源。
- 首期闭环覆盖首页、门店信息页、搜索页、商品详情页。
- 对暂不具备数据基础或业务闭环的收藏、预约功能明确排除；分享、咨询和热销行为统计纳入本期，但必须按首期闭环收敛实现。
- 保持与原型一致的暖白背景、墨黑主视觉和品牌金点缀，形成现代、克制、可信的瓷砖展厅质感。

## 5. 功能要求

### FR-001 原生微信小程序首页

- 小程序首页 MUST 使用原生微信小程序技术形态实现。
- 首页 MUST 适配微信小程序安全区、胶囊区域、底部 TabBar 和主流手机宽度。
- 页面 MUST 不依赖 Web 浏览器专属 API。
- 代码实现阶段 MUST 遵守 `src/miniapp/` 目录边界，不得把小程序业务代码放入根目录、`src/web/` 或 `scripts/`。

### FR-002 顶部品牌导航与门店信息入口

- 首页顶部 MUST 展示菲尚特品牌 Logo 与门店名称，例如“菲尚特瓷砖馆”。
- 单门店场景 SHOULD 隐藏或弱化门店切换箭头；多门店切换能力不作为本期必须能力。
- 点击门店名称或门店区域 MUST 进入门店信息页。
- 门店信息页 MUST 展示门店基础信息；价格展示规则、客服/电话/微信号/导航配置来源仍需产品确认。

### FR-003 搜索入口与搜索页闭环

- 首页 MUST 展示搜索入口，默认提示语为“搜索瓷砖名称、编号或规格”。
- 点击搜索入口 MUST 进入搜索页。
- 搜索页 SHOULD 支持按商品名称、商品编号、品牌名称、规格、系列名称进行查询。
- 搜索结果 MUST 可点击进入商品详情页。
- 搜索页是否展示历史搜索、热门搜索，可在后续 `/req-complete` 或 OpenSpec 阶段根据现有数据能力细化；不得为了历史/热门搜索新增超出本期的数据采集能力。

### FR-004 Banner 展示与跳转

- 首页 MUST 展示品牌 Banner 区域，视觉参考原型主视觉“质感空间，由砖而生”。
- Banner 数据 MUST 复用已有后台 Banner 配置能力，不新增重复 Banner 数据源。
- Banner SHOULD 支持图片、主标题、副标题、跳转类型、跳转目标、排序、上下架状态、生效时间等已有字段。
- Banner SHOULD 支持轮播指示器、自动轮播和手势切换。
- 点击 Banner 时，若跳转目标在首期闭环内，MUST 跳转至商品详情、搜索结果或门店信息等可达页面；若配置指向暂未实现能力，MUST 安全降级，不得出现空白页或小程序报错。

### FR-005 快捷找砖入口

- 首页 MUST 展示五个默认快捷入口：
  - 按空间
  - 按规格
  - 按风格
  - 按颜色
  - 全部分类
- 点击快捷入口 MUST 进入搜索页或筛选结果页，并带入对应预置筛选条件。
- 快捷入口首期为固定配置；后台可配置顺序、名称、图标属于后续拆分需求。
- 快捷入口图标与文案 SHOULD 与原型保持一致，点击区域不小于 44x44 pt。

### FR-006 新品推荐

- 首页 MUST 展示新品推荐模块；无新品数据时 SHOULD 隐藏该模块，不展示空白卡片。
- 新品推荐 SHOULD 横向展示 3 到 6 个商品卡片。
- 商品卡片 MUST 至少展示商品主图、商品编号/名称、规格和价格展示信息。
- 若商品具备新品标识，卡片 SHOULD 展示“新品”角标。
- 点击商品卡片 MUST 进入商品详情页。
- 点击“更多” SHOULD 进入搜索页或商品列表结果页，并带入新品筛选条件。

### FR-007 热销推荐

- 首页 SHOULD 展示热销推荐模块；无热销数据时 SHOULD 隐藏该模块。
- 热销推荐 SHOULD 使用大卡片或双列卡片，突出瓷砖纹理或空间效果图。
- 排序策略 MUST 以人工配置优先；行为统计数据积累不足时按时间或排序字段展示。
- 本期 SHOULD 记录商品详情访问、分享、咨询等行为，用于热销推荐排序。
- 本期 MUST NOT 依赖收藏量，因为收藏功能不在本期范围。
- 原型中的收藏心形按钮不属于本期功能，实现阶段 SHOULD 移除或隐藏。

### FR-008 商品详情页闭环

- 首页推荐商品、搜索结果和有效 Banner 跳转 MUST 能进入商品详情页。
- 商品详情页 MUST 展示商品核心字段，至少包括商品名称、商品编号、主图/纹理图、规格、品牌、类目、适用空间、风格标签和价格展示信息。
- 商品详情页 MUST 仅展示允许公开的瓷砖信息，不暴露后台管理字段、库存管理、内部备注或未授权素材。
- 商品详情页 MUST 支持微信原生分享或等价分享入口，并记录商品分享行为。
- 商品详情页 MUST 支持门店咨询入口，并记录商品咨询行为。
- 商品详情页的收藏、预约入口不作为本期必须能力。

### FR-009 品牌服务区

- 首页 SHOULD 展示品牌服务区，用于呈现“正品保障”“免费选砖”“门店咨询”等服务承诺。
- 本期服务区中的咨询入口 MUST 可用，支持微信客服、拨打电话或复制微信号中的至少一种，具体方式以门店配置字段可用性为准。
- 若某种咨询方式缺少配置，MUST 隐藏该方式或安全降级，不得出现可点击失败入口。
- 咨询入口触发后 MUST 记录咨询行为，用于热销行为统计。
- 服务入口后台配置能力另拆需求。

### FR-010 底部 TabBar

- 小程序 MUST 提供底部 TabBar，首期至少包含：首页、分类、找砖、我的。
- 若收藏功能暂不实现，TabBar MUST 不展示“收藏”入口，或以产品确认后的方式隐藏/延后。
- 首页 Tab MUST 为选中态，选中态使用品牌金色，未选中态使用中性灰。
- TabBar MUST 预留底部安全区，不遮挡页面内容。

### FR-011 页面状态与降级

- 首屏加载时 MUST 使用骨架屏或轻量加载态，不出现大面积空白。
- Banner、推荐商品、图片等模块加载失败时 MUST 模块级降级，不阻塞整个首页渲染。
- 推荐商品为空时 SHOULD 隐藏对应模块。
- 所有商品为空时 SHOULD 展示品牌化空状态和“暂无商品”类提示。
- 网络异常时 MUST 提供重新加载入口。
- 图片加载失败 MUST 使用统一瓷砖占位图或可接受的品牌占位图。

### FR-012 分享功能

- 首页 SHOULD 支持微信小程序原生分享能力，分享标题、路径和封面图应与菲尚特品牌和当前页面匹配。
- 商品详情页 MUST 支持商品分享，分享路径 SHOULD 能回到对应商品详情页。
- 分享触发 SHOULD 记录行为事件，至少区分首页分享和商品分享。
- 分享失败或取消时不得阻断用户继续浏览。

### FR-013 热销行为统计

- 本期 MUST 记录商品详情访问行为，用于热销推荐排序的数据基础。
- 本期 SHOULD 记录分享行为和咨询行为，用于辅助热销推荐排序。
- 热销排序 MUST 支持人工配置优先；行为统计作为自动排序的补充依据。
- 行为统计 MUST 不依赖收藏量，因为收藏功能仍不在本期范围。
- 行为统计数据采集 MUST 避免记录用户敏感信息，必要时只记录匿名事件、商品 ID、事件类型和时间。

### FR-014 数据复用与接口边界

- 首页数据 MUST 复用现有店主端/后台接口和数据模型聚合，不新增重复数据源。
- 若实现阶段需要 `GET /api/miniapp/home` 或等价小程序首页聚合接口，该接口 MUST 仅聚合现有公开数据，不新增独立业务表。
- 小程序端 MUST 只访问后端授权接口，不得直连未授权对象存储。
- API 响应 MUST 不暴露管理端专用字段、内部状态或敏感配置。
- 若 API contract 发生变化，后续 OpenSpec 与实现阶段 MUST 同步 OpenAPI、Orval、docs 与测试。
- 若行为统计需要新增或扩展事件接口、统计表或聚合字段，MUST 同步数据库文档、API 文档、OpenAPI、Orval 和测试。

## 6. UI / UE 约束

- UI MUST 参考 `prototype/miniapp/prototype.png` 与 `prototype/miniapp/prototype.html` 的布局、层级和视觉气质。
- 页面 SHOULD 采用暖白背景、墨黑主视觉、品牌金点缀，保持现代轻奢、专业建材展厅感。
- 首页首屏 SHOULD 在 375x812 与 390x844 视口下展示品牌、搜索、Banner、快捷入口和至少一个推荐模块。
- 点击区域 MUST 不小于 44x44 pt。
- 文本 MUST 在 320 到 430 pt 宽度下不出现不可接受的截断、重叠或横向滚动。
- 瓷砖图片 SHOULD 优先展示纹理可辨识的真实商品图或空间效果图，不使用与瓷砖无关的装饰图。
- Banner 圆角、搜索框圆角、商品卡片圆角、模块间距 SHOULD 与原型保持一致；实现阶段可根据原生小程序单位做等比例适配。
- 页面 MUST 避免电商促销红色风格、复杂表格、密集表单、过度渐变、玻璃拟态或强阴影。

## 7. 数据与字段要求

首页商品卡片 SHOULD 映射以下字段；字段命名可在 API / 小程序实现阶段按项目约定转换：

| 字段 | 用途 | 首期要求 |
|---|---|---|
| `productId` | 商品详情跳转 | 必需 |
| `productName` | 商品名称 | 必需 |
| `skuCode` | 商品编号 | 必需 |
| `coverImage` | 商品主图 | 必需 |
| `specification` | 规格 | 必需 |
| `categoryName` | 类目 | 建议 |
| `brandName` | 品牌 | 建议 |
| `styleTags` | 风格标签 | 建议 |
| `applicableSpaces` | 适用空间 | 建议 |
| `retailPrice` | 价格 | 待产品确认展示规则 |
| `priceDisplayMode` | 价格展示方式 | 待产品确认 |
| `isNew` | 新品角标 | 建议 |
| `isHot` | 热销标识 | 建议 |
| `isFavorite` | 收藏状态 | 本期不使用 |

行为统计 SHOULD 至少支持以下事件：

| 事件 | 用途 | 首期要求 |
|---|---|---|
| `product_detail_view` | 商品详情访问量 | 必需 |
| `product_share` | 商品分享量 | 建议 |
| `home_share` | 首页分享量 | 建议 |
| `product_contact_click` | 商品咨询量 | 建议 |
| `home_contact_click` | 首页咨询量 | 建议 |

## 8. 依赖与拆分建议

| 依赖 / 后续项 | 关系 |
|---|---|
| Banner 管理 | 本期复用已有后台 Banner 配置能力。 |
| SKU / 品牌 / 类目 / 规格管理 | 本期复用已有数据作为首页、搜索、详情的数据来源。 |
| 门店信息配置 | 本期需要门店基础信息与咨询方式来源；电话、微信号、微信客服、导航等字段来源待确认。 |
| 快捷入口后台配置 | 后续独立 REQ；本期固定默认入口。 |
| 服务入口后台配置 | 后续独立 REQ；本期仅静态展示或安全跳转。 |
| 收藏能力 | 后续独立 REQ；本期不实现。 |
| 预约能力 | 后续独立 REQ；本期不实现。 |
| 分享能力 | 本期纳入，优先使用微信原生分享。 |
| 咨询能力 | 本期纳入，至少支持一种可用门店咨询方式。 |
| 行为统计与热销算法 | 本期纳入详情访问、分享、咨询等行为统计；不包含收藏量。 |

建议后续拆分需求：

- 后台快捷入口配置管理。
- 后台服务入口配置管理。
- 小程序收藏能力。
- 小程序预约能力。
- 更复杂的用户画像、收藏驱动热销算法或跨渠道咨询管理。

**建议 OpenSpec change 命名**：`add-miniapp-home`。
