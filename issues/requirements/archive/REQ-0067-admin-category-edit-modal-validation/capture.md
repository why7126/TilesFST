# 一句话

管理端类目编辑弹窗应隐藏系统生成的类目编码，并对上级类目、类目名称和排序权重提供必填标识及类目名称合法性、同层级去重校验。

# 原始描述

管理端-类目编辑弹窗，类目编码由系统自动生成，不需要显示给用户，也不需要用户填写，生成规则以CAT-开头

类目名称，最多不能超过10个字符，只能使用中文、英文、数字；同一个类目同一层级类目名称要校验，不允许重复

上级类目、类目名称、排序权重均为必填项，需要有必填项的标识

# 背景与关联

- 父需求：`REQ-0005-tile-category-management`
- 涉及端：Web 管理端
- 涉及对象：瓷砖类目编辑弹窗、类目创建/编辑表单、类目名称校验、类目编码生成规则
- 当前诉求：类目编码应作为系统内部字段自动生成，不暴露给运营填写；类目编辑弹窗需要明确必填项和输入校验，避免重复或非法类目名称进入主数据。
- 业务价值：降低运营填写成本，统一类目主数据规则，避免同层级重复类目和不可控编码影响后续商品挂载与前台分类展示。

# 建议验收要点

- [ ] 类目编辑弹窗不展示类目编码输入项，也不要求用户填写类目编码。
- [ ] 新建类目时系统自动生成唯一类目编码，编码格式以 `CAT-` 开头。
- [ ] 编辑既有类目时不允许用户修改类目编码；如页面需要内部追踪，可仅在非表单区域或调试信息中弱展示，默认不面向用户展示。
- [ ] 上级类目、类目名称、排序权重均为必填项，并在表单标签或等价位置展示清晰必填标识。
- [ ] 类目名称最多 10 个字符。
- [ ] 类目名称仅允许中文、英文和数字，不允许空格、符号、emoji 或其他特殊字符。
- [ ] 同一上级类目下的类目名称必须唯一；创建或编辑为同层级已存在名称时应阻止保存并展示明确错误提示。
- [ ] 编辑类目时，若名称未变化或仅保存其他字段，不应误判自身为重复。
- [ ] 前端表单校验与后端 API 校验保持一致；后端必须兜底阻止非法名称、缺失必填项和同层级重复名称。

# 待澄清

- [ ] 顶级类目的“上级类目”是否通过固定选项“无上级/顶级类目”表达，还是必须选择一个虚拟根节点。
- [ ] 类目名称 10 个字符按 Unicode 字符数计算，还是按中英文混排展示宽度计算。
- [ ] 类目编码 `CAT-` 后缀采用纯数字递增、随机串、日期序列还是现有编码生成器规则。
- [ ] 同层级重复名称校验是否区分英文大小写。
- [ ] 排序权重的允许范围、默认值和是否只允许正整数需要在后续 PRD/Change 中确认。
