# 业务流程

## 1. 总流程

```text
用户进入小程序分类页
  |
  v
浏览一级分类导航与二级分类宫格
  |
  +-- 切换一级分类 --------------------+
  |                                   |
  +-- 点击一级分类商品入口             |
  |       |                           |
  |       v                           |
  |  product-list?categoryLevel=primary
  |       |
  |       v
  |  聚合展示该一级分类下所有启用二级分类商品
  |
  +-- 点击二级分类卡片
          |
          v
     product-list?categoryLevel=secondary
          |
          v
     展示该二级分类下商品
          |
          v
     筛选 / 排序 / 分页 / 商品详情
```

## 2. 一级分类聚合商品列表流程

| 步骤 | 说明 | 输出 |
|---|---|---|
| 1 | 用户在分类页选中某个一级分类 | 右侧展示该一级分类名称与二级分类宫格 |
| 2 | 用户点击一级分类商品列表入口 | 进入商品列表页 |
| 3 | 商品列表页解析分类上下文 | `categoryId={primaryId}`、`categoryLevel=primary`、`sourcePage=category` |
| 4 | 查询商品 | 展开该一级分类下所有启用二级分类，并聚合其商品 |
| 5 | 展示商品列表 | 用户继续筛选、排序、分页或进入 SKU 详情 |
| 6 | 无商品或参数异常 | 展示可恢复空状态或错误重试 |

## 3. 二级分类精确商品列表流程

| 步骤 | 说明 | 输出 |
|---|---|---|
| 1 | 用户点击二级分类卡片 | 进入商品列表页 |
| 2 | 商品列表页解析分类上下文 | `categoryId={secondaryId}`、`categoryLevel=secondary`、`sourcePage=category` |
| 3 | 查询商品 | 仅查询该二级分类下商品 |
| 4 | 展示商品列表 | 用户继续筛选、排序、分页或进入 SKU 详情 |
| 5 | 无商品或参数异常 | 展示可恢复空状态或错误重试 |

## 4. 分类页交互边界

```text
左侧一级分类项
  |
  +-- tap: 切换当前一级分类

右侧一级分类标题区
  |
  +-- tap/CTA: 查看该一级分类全部商品

右侧二级分类卡片
  |
  +-- tap: 查看该二级分类商品
```

该边界用于避免左侧导航点击语义过载：左侧继续专注“切换分类”，右侧提供“进入商品列表”。

## 5. 与父 REQ 差异

| 关联需求 | 差异 |
|---|---|
| REQ-0045-category-list-page | 父需求定义分类页两级展示和二级分类进入商品列表；本需求补齐一级分类聚合商品列表入口和分类层级参数 |
| REQ-0047-product-list-common-component-application | 相关需求定义商品列表通用能力；本需求只定义分类页如何把一级/二级分类上下文传入商品列表 |

## 6. 非目标流程

- 不在分类页直接加载商品列表或商品卡片。
- 不进入管理端分类或商品维护流程。
- 不改变商品详情页字段与布局。
- 不新增购物车、收藏、询价、下单或客服能力。
