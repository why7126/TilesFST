# 微信小程序分类列表页支持一二级分类商品列表入口需求文档

## 1. 背景

菲尚特微信小程序分类页已经提供一级分类导航和二级分类宫格，用户可以点击二级分类进入对应商品列表。实际浏览中，用户也会希望在尚未明确二级分类时，直接从一级分类查看该大类下的全部商品，再通过商品列表页继续筛选和浏览。

本需求补齐分类列表页中一级分类到商品列表页的入口规则，同时保持二级分类进入商品列表页的既有能力，让不同分类层级都能作为稳定的商品浏览入口。

## 2. 目标用户

| 用户 | 核心诉求 |
|---|---|
| 装修客户 | 从大类快速浏览全部候选瓷砖，再逐步缩小选择范围 |
| 设计师 | 按一级品类聚合浏览素材，避免频繁切换二级分类 |
| 门店导购 | 在客户需求尚模糊时，快速打开某个大类下的商品集合进行介绍 |
| 普通访客 | 点击一级或二级分类时都能获得可预期的商品列表结果 |

## 3. 范围

### 3.1 包含

- 微信小程序分类列表页一级分类商品列表入口。
- 微信小程序分类列表页二级分类商品列表入口的规则确认与延续。
- 一级分类商品列表展示该一级分类下所有启用二级分类商品的聚合结果。
- 二级分类商品列表展示该二级分类下的商品结果。
- 分类入口到商品列表页的上下文参数，包括分类 ID、分类名称、分类层级和来源页面。
- 商品列表页基于分类层级展示标题、空状态、错误提示和埋点上下文。

### 3.2 不包含

- Web 管理端、店主 Web 展示端分类页入口调整。
- 后台分类管理、商品管理、上下架、库存、价格或批量操作能力。
- 新增分类层级模型或支持三级及以上分类。
- 在分类列表页直接展示商品卡片、价格、筛选栏或排序栏。
- 购物车、询价、在线下单、收藏或客服找砖能力。
- 复杂推荐排序、人工置顶或个性化算法。

## 4. 功能要求

### FR-001 一级分类商品列表入口

- 分类列表页 MUST 允许用户从一级分类进入商品列表页。
- 一级分类商品列表 MUST 展示该一级分类下所有启用二级分类商品的聚合结果。
- 一级分类入口 MUST 保留当前一级分类切换能力，不得让普通切换和进入商品列表产生误触冲突。
- 若一级分类无启用二级分类或聚合后无商品，商品列表页 MUST 展示可恢复的空状态。

### FR-002 二级分类商品列表入口

- 分类列表页 MUST 允许用户从二级分类进入商品列表页。
- 二级分类商品列表 MUST 展示该二级分类下的商品结果。
- 二级分类入口 SHOULD 复用既有商品列表页通用组件、分页、筛选、排序、加载态和空状态。
- 连续点击一级或二级分类商品列表入口时 MUST 做防重复触发，避免重复打开页面。

### FR-003 分类层级与跳转参数

- 分类页进入商品列表页时 MUST 传递分类 ID、分类名称、分类层级和来源页面。
- 推荐参数语义：
  - `categoryId`：当前点击的分类 ID。
  - `categoryName`：当前点击的分类名称。
  - `categoryLevel`：`primary` 或 `secondary`。
  - `sourcePage`：`category`。
- 商品列表查询 MUST 能区分一级分类聚合查询和二级分类精确查询。
- 若后端接口暂不支持分类层级参数，后续 OpenSpec Change MUST 明确兼容策略，不得让一级分类入口错误地只查询直接挂载在一级分类下的商品。

### FR-004 商品列表展示语义

- 一级分类列表页标题 SHOULD 使用一级分类名称，例如“地面砖”。
- 二级分类列表页标题 SHOULD 使用二级分类名称，例如“通体大理石”。
- 一级分类上下文中的筛选项 SHOULD 限制在该一级分类聚合结果相关范围内。
- 二级分类上下文中的筛选项 SHOULD 限制在该二级分类结果相关范围内。
- 商品列表页返回分类页后 SHOULD 保持分类页原有一级分类选中态、左侧滚动位置和右侧滚动位置。

### FR-005 状态、异常与埋点

- 一级分类聚合结果为空时 SHOULD 展示“该分类暂未上架商品”或等价文案。
- 二级分类结果为空时 SHOULD 展示“该分类暂未上架商品”或等价文案。
- 分类已下架、参数无效或接口返回无权限访问时 MUST 展示可恢复提示，不得白屏。
- 一级分类入口点击 SHOULD 记录 `primary_category_product_list_click` 或在既有 `primary_category_click` 中增加进入商品列表语义参数。
- 二级分类入口点击 SHOULD 继续记录 `secondary_category_click`，并补充商品列表跳转上下文。
- 商品列表页浏览埋点 SHOULD 包含 `categoryId`、`categoryName`、`categoryLevel`、`sourcePage`、`resultCount` 和 `requestId`。

## 5. UI 约束

- 分类列表页 MUST 继续沿用“工业石材 · 暗色旗舰风”，与现有小程序首页、分类页和商品列表页保持一致。
- 一级分类的“切换当前分类”和“进入商品列表”交互 SHOULD 有清晰区分，例如在右侧当前一级分类标题区域提供进入入口，避免左侧导航点击语义过载。
- 可点击区域 SHOULD 不小于 44px，符合小程序触控场景。
- 新增入口不得破坏分类页现有左右双栏布局、底部 TabBar 和二级分类三列宫格。
- 商品列表页标题、搜索提示、空状态和错误状态 SHOULD 能体现一级/二级分类上下文，但不得堆叠过多解释性文字。
