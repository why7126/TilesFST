# 业务流程

## 1. 总流程

```text
用户进入密码设置入口
  |
  v
输入新密码
  |
  v
前端即时校验：长度 5-32 + 英文 + 数字
  |
  +-- 不通过 --> 字段下方展示具体原因，阻止提交
  |
  v
提交 API
  |
  v
后端统一密码校验函数再次校验
  |
  +-- 不通过 --> 返回统一错误 message / error code
  |
  v
继续执行入口原有业务逻辑
  |
  +-- 修改本人密码 --> 校验原密码、更新 hash、保持 token 失效策略
  |
  +-- 创建用户 --> 创建账号并保存新密码 hash
  |
  +-- 重置密码 --> 生成或保存新密码 hash，并反馈结果
  |
  v
固定 Toast 或字段反馈展示结果
```

## 2. 与父需求 REQ-0015 的差异

| 维度 | REQ-0015 原规则 | REQ-0063 调整后 |
|---|---|---|
| 长度下限 | 8 位 | 5 位 |
| 长度上限 | 32 位 | 32 位，保持不变 |
| 字符组成 | 至少字母 + 数字 | 包含英文字符 + 数字，语义更明确 |
| 弱密码/限流/token 失效 | 已纳入修改密码能力 | 不自动取消，继续按既有安全能力执行 |
| 影响范围 | 主要修改本人密码弹窗 | 扩展到所有设置新密码的入口 |

## 3. 入口影响

| 入口 | 前端行为 | 后端行为 | 备注 |
|---|---|---|---|
| 修改本人密码 | 弹窗内新密码提示和即时校验更新 | `POST /api/v1/admin/profile/password` 使用新规则 | 原密码、确认密码、新旧不同等逻辑保持 |
| 创建用户 | 初始密码字段提示和提交校验更新 | 创建用户接口使用新规则 | 若无手输密码字段，则生成密码需满足新规则 |
| 重置密码 | 重置表单或结果提示更新 | 重置接口使用新规则 | 复制密码等交互不在本需求新增 |
| 初始化管理员 | 若存在初始化校验，提示和后端规则更新 | 初始化流程使用新规则 | 具体入口待实现阶段确认 |

## 4. 异常流程

```text
密码长度 < 5
  -> 前端提示“密码至少需要 5 位”
  -> 后端兜底返回策略错误

密码长度 > 32
  -> 前端提示“密码至多 32 位”
  -> 后端兜底返回策略错误

密码无英文字符
  -> 前端提示“密码需包含英文字符”
  -> 后端兜底返回策略错误

密码无数字
  -> 前端提示“密码需包含数字”
  -> 后端兜底返回策略错误
```

## 5. 待后续 Change 确认

- 是否允许空格、中文、全角数字、非 ASCII 字母。
- 是否需要调整弱密码表，使其与 5 位下限匹配。
- 创建用户和重置密码是否均为用户输入密码，还是部分入口为系统自动生成。
