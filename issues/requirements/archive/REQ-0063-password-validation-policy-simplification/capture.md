# 一句话

将系统密码校验规则简化为 5 至 32 位，且必须同时包含英文字符和数字。

# 原始描述

密码校验简化成以下：

1. 至少 5 位长度
2. 至多 32 位长度
3. 包含英文字符
4. 包含数字

# 背景与关联

- 父需求：`REQ-0015-password-change`
- 涉及端：后端认证/用户密码校验、管理端用户创建/重置/修改密码表单
- 业务价值：降低密码规则复杂度，使用户更容易理解并通过校验，同时保留基础长度与字符组合约束
- 预期后续：需要在 OpenSpec Change 中确认适用范围，并同步后端校验、前端表单提示、API 错误提示与测试用例

# 建议验收要点

- [ ] 密码长度少于 5 位时校验失败，并提示至少 5 位。
- [ ] 密码长度超过 32 位时校验失败，并提示至多 32 位。
- [ ] 密码不包含英文字符时校验失败。
- [ ] 密码不包含数字时校验失败。
- [ ] 同时满足 5 至 32 位、包含英文字符、包含数字的密码可以通过校验。
- [ ] 管理端相关密码输入位置的校验提示与后端规则一致。

# 待澄清

- [ ] 新规则是否适用于登录密码的所有入口：创建用户、重置密码、修改密码、初始化管理员密码。
- [ ] 英文字符是否限定为 ASCII 字母 `A-Z` / `a-z`。
- [ ] 是否允许空格、中文、符号等其他字符与英文/数字共同出现。
- [ ] 是否需要保留既有历史密码，还是仅影响新设置/重置的密码。
