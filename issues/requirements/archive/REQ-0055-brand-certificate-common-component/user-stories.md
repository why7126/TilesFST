# 用户故事

## US-001 运营人员识别证书文件

作为管理端运营人员，我希望在证书列表和相关弹窗中看到一致的证书缩略图、PDF 标识和文件占位，以便快速判断证书文件类型和可预览性。

验收要点：

- 图片证书展示稳定尺寸缩略图。
- PDF 证书展示统一 `PDF` 文件占位。
- 文件 URL 缺失、类型未知或图片加载失败时展示统一文件占位。
- 缩略图加载前后不造成列表行高明显跳动。

## US-002 运营人员查看证书状态

作为管理端运营人员，我希望证书有效期、有效状态和前台展示状态使用统一 Badge 与文案，以便在不同证书场景中快速识别风险。

验收要点：

- 有效期展示覆盖长期有效、起止日期、仅到期日期和未设置。
- 有效状态覆盖长期有效、有效、即将到期、已过期和未设置。
- 前台展示状态覆盖展示中和已隐藏。
- 未知状态降级展示原始文本，不导致页面渲染失败。

## US-003 运营人员预览证书

作为管理端运营人员，我希望图片和 PDF 证书使用一致的预览入口，以便从列表或组件上下文快速打开原文件。

验收要点：

- 缩略图或预览按钮可触发统一预览行为。
- 图片和 PDF v1 均通过新窗口打开文件 URL。
- 文件 URL 缺失时阻止预览，并返回可展示的失败原因。
- 预览失败文案复用 `文件暂时无法预览，请稍后重试或下载查看`。

## US-004 运营人员在弹窗中确认上传文件

作为管理端运营人员，我希望新增/编辑证书弹窗内的文件卡片清晰展示未上传、上传中、已就绪和失败状态，以便知道何时可以保存证书。

验收要点：

- `idle` 状态展示未上传提示。
- `uploading` 状态展示文件名、进度和保存阻塞提示。
- `done` 状态展示缩略图或文件类型、文件名、文件大小、重新上传和删除入口。
- `failed` 状态展示失败原因和重新上传入口。

## US-005 管理端开发者复用证书展示组件

作为管理端开发者，我希望证书缩略图、信息单元、Badge、预览入口和文件卡片有清晰的 props 契约，以便在品牌证书列表、品牌详情快捷区或后续后台页面中复用。

验收要点：

- 组件 props 面向展示模型，不暴露页面筛选、分页或保存状态。
- 组件复用现有证书 label、日期格式和 PDF 判断 helper。
- 状态切换、删除确认、接口调用仍由页面或业务容器负责。
- 组件导出路径清晰，后续页面无需从品牌证书管理页内部复制代码。

## US-006 产品/验收人员检查组件状态矩阵

作为产品或验收人员，我希望通过原型和验收清单确认所有关键组件状态，以便在后续开发中避免只覆盖成功态。

验收要点：

- 原型展示图片、PDF、加载失败、长期有效、即将到期、已过期、隐藏、上传中和上传失败等状态。
- 验收清单覆盖列表列、弹窗文件卡片、预览入口和横切 knowledge-base gate。
- PNG 可后续导出，但 HTML 与 context 必须足以指导首轮实现。
