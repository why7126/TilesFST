REQ-0027 / WEB ADMIN MOBILE ADAPTATION

Web 管理端移动端基础适配验收矩阵

本原型用于表达验收范围与检查顺序：只覆盖当前已实现的 Web 管理端，不覆盖店主 Web 或微信小程序。 目标是基础可用：无页面级横向溢出、控件不重叠、表格可读可滚动、弹窗可关闭且底部操作可达。

web-admin only admin-list admin-form admin-modal media-upload regression

1. 必测视口

375×812手机小屏。优先发现弹窗、分页、筛选重叠问题。
390×844主流手机。验收主要管理操作可用。
768×1024小屏平板。验收 Shell、Sidebar 与表格容器滚动。
1440×1024桌面回归。确认移动端修复不破坏桌面管理端。

2. 必测页面

路由 页面 组件类型 移动端通过标准
/admin/login登录页auth form单栏居中，输入与登录按钮可操作。
/admin/dashboard管理首页shell / metrics / table指标卡、快捷操作、最近更新表格不重叠。
/admin/tile-skusSKU 管理list / wide modal / media宽表格容器滚动，SKU 弹窗可滚动，媒体字段可操作。
/admin/brands品牌管理list / modal / uploadLogo 列可读，品牌弹窗与启停确认可操作。
/admin/users用户管理list / modal / confirm分页基准不回归，重置密码确认可操作。
/admin/logs日志审计filters / table / drawer多条件筛选可单列，日志详情抽屉可关闭可滚动。
/admin/settings/basic系统设置form / tabs / confirm设置导航、保存、恢复默认、dirty 切换均可操作。

3. 横切检查点

列表页
  • 筛选区在手机宽度下单列可操作。
  • 表格仅在容器内横向滚动。
  • 分页 DOM 与用户管理基准一致。
  • 状态操作使用 DS confirm，无 window.confirm。
表单页
  • 系统设置与个人资料小屏单列可读。
  • 全页仅一处主要保存 CTA。
  • 保存反馈使用 fixed toast，不推挤布局。
  • dirty 切换与恢复默认使用 DS modal。
弹窗 / 抽屉
  • 375px 宽度下关闭按钮可见。
  • 矮视口下 body 可滚动。
  • footer 主操作可达。
  • 宽弹窗验收 computed width / max-width。
媒体上传回归
  • 上传状态机 idle / uploading / done / failed 不被隐藏。
  • 同会话上传成功后即时回显。
  • 失败提示不遮挡底部操作按钮。
  • 本 REQ 不新增上传 API 或 Docker 配置。

4. 通过判定

  • 必测视口与必测页面均完成 smoke，结果写入 trace。
  • 无页面级不可控横向溢出；宽表格在容器内滚动。
  • 弹窗、抽屉和确认框可关闭、可滚动、可完成主操作。
  • 横切 AC 全部通过，或有明确 N/A 理由。
  • 桌面 1440×1024 回归不破坏现有管理端布局。
