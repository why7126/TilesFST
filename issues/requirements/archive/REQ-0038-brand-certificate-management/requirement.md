# REQ-0038 管理后台 - 瓷砖品牌证书管理

## 1. 需求背景

瓷砖品牌通常需要维护质量体系、产品检测、绿色建材、环保认证、专利荣誉等证明材料，并在店主端品牌详情中按需展示。当前品牌主数据仅包含品牌名称、Logo 与介绍，无法支撑证书的结构化管理、有效期治理和前台展示控制。

本需求新增独立一级“品牌证书”管理页面及“新增/编辑证书”弹窗，支持跨品牌查询，并通过必填品牌字段建立一对多关系：一个品牌可关联多个证书，一个证书只能关联一个品牌。页面继承 TILESFST 管理后台工业石材暗色旗舰风，保持固定 Sidebar、右侧独立滚动、品牌金激活态、极细分割线和低圆角工业风一致。

## 2. 产品目标

1. 将品牌证书从非结构化附件升级为可检索、可排序、可展示、可治理的主数据。
2. 让运营人员快速识别即将到期、已过期和未启用证书。
3. 为店主端品牌详情页提供稳定、结构化的证书展示数据。
4. 保证证书原图/文件可追溯，且编辑、启停、删除操作可审计。

## 3. 页面入口与导航

- 左侧 OPERATIONS 导航在“瓷砖规格”下方新增独立菜单“品牌证书”。
- 页面路由：`/admin/brand-certificates`。
- 当前左侧导航高亮“品牌证书”，不再高亮“瓷砖品牌”。
- 品牌列表页操作列可保留“证书”快捷入口，点击后进入 `/admin/brand-certificates?brand_id={brand_id}`，自动应用品牌筛选。
- 证书主体管理、跨品牌到期治理和新增证书均在一级“品牌证书”页面完成。
- 不使用 `/admin/brands/{brand_id}/certificates` 作为主要页面，仅可作为兼容重定向。

### 3.1 信息架构结论

采用“一级证书管理 + 品牌列表快捷筛选入口”的组合：

1. 一级页适合跨品牌查看即将到期、已过期和隐藏证书。
2. 品牌列表快捷入口保留品牌上下文，减少运营查找成本。
3. 证书数据必须保存 `brand_id`，数据库关系为 `brand 1 : N certificate`。
4. 一个证书记录只能存在一个 `brand_id`，不支持多品牌共享同一证书记录。

## 4. 页面范围

### 4.1 包含

- 证书指标概览。
- 关键词、所属品牌、证书类型、有效状态、展示状态即时筛选。
- 证书列表、排序、分页、每页显示数。
- 新增、编辑、启用、停用、删除证书。
- 证书封面/文件上传与预览。
- 长期有效与有效期管理。
- 前台展示开关。
- 到期状态自动计算。

### 4.2 不包含

- 批量操作、导出、OCR 自动识别。
- 证书审批流、电子签章、证书真伪校验。
- 多语言证书名称。
- 证书与 SKU 单独绑定。
- 店主端展示页面实现。

## 7. 数据概览

| 指标 | 统计口径 |
|---|---|
| 证书总数 | 当前品牌所有未删除证书 |
| 有效证书 | 长期有效，或当前日期处于有效期内 |
| 即将到期 | 距到期日 0–30 天，且未过期 |
| 已过期 | 当前日期晚于到期日 |

指标数量点击后可快速应用相应筛选；“证书总数”清空有效状态筛选。

## 8. 搜索与筛选

筛选区不显示独立标题。

| 字段 | 控件 | 说明 |
|---|---|---|
| 关键词 | Input | 支持证书名称、证书编号、发证机构模糊搜索 |
| 所属品牌 | Select/SearchSelect | 全部品牌；支持按品牌名称搜索；可由品牌列表快捷入口预设 |
| 证书类型 | Select | 全部、质量体系、产品检测、绿色环保、荣誉资质、专利证书、其他 |
| 有效状态 | Select | 全部、有效、即将到期、已过期、未设置有效期 |
| 展示状态 | Select | 全部、展示中、已隐藏 |
| 重置 | Ghost Button | 清空全部筛选条件并回到第 1 页 |

筛选不提供“查询”按钮。关键词输入采用 300ms 防抖后自动生效；任一下拉项变化后立即生效。筛选条件同步到 URL Query，刷新页面后保持。每次筛选条件变化后当前页重置为第 1 页。

## 9. 证书列表

| 字段 | 说明 |
|---|---|
| 证书 | 证书封面缩略图、证书名称、证书编号 |
| 所属品牌 | 品牌名称；点击可打开品牌编辑或应用品牌筛选 |
| 证书类型 | 类型 Badge |
| 发证机构 | 最多两行，超出省略 |
| 有效期 | 长期有效或开始日期至结束日期 |
| 有效状态 | 有效、即将到期、已过期、未设置 |
| 前台展示 | 展示中/已隐藏 |
| 排序 | 正整数，越小越靠前 |
| 更新时间 | 最近维护时间 |
| 操作 | 预览、编辑、显示/隐藏、删除 |

### 10.1 缩略图规则

- 图片证书显示图片缩略图。
- PDF 显示统一 PDF 文档封面并标注 `PDF`。
- 加载失败显示文件类型占位，不显示破图。
- 点击缩略图或“预览”打开大图/PDF 新窗口预览。

### 10.2 有效状态计算

```text
长期有效 = is_permanent = true
有效 = end_date >= today + 31 days
即将到期 = today <= end_date <= today + 30 days
已过期 = end_date < today
未设置有效期 = 非长期有效 且 end_date 为空
```

状态由服务端返回，前端不作为唯一计算源。长期有效使用品牌金 Badge；即将到期使用警示色；已过期使用风险色；未设置使用弱色。

### 10.3 删除规则

- 任意证书均允许删除，但必须二次确认。
- 删除为软删除，不立即物理清理对象存储文件。
- 删除确认标题：`删除证书`。
- 确认文案：`删除后店主端将不再展示“{证书名称}”，该操作可由管理员在审计记录中追溯。`
- 按钮：`取消`、`删除证书`。

## 10. 分页

底部分为左右两组：左侧显示 `共 x 个证书`；右侧依次显示 `上一页 / 页码 / 下一页 / 每页显示 20条、50条、100条`。不提供跳页输入框。

- 上一页和下一页使用完整文字按钮，不使用单独箭头。
- 当前页使用品牌金实底。
- 切换每页显示数后回到第 1 页，并保留全部筛选条件。
- 保留全部筛选条件。
- 无数据时展示空状态，不隐藏筛选区和新增按钮。

## 11. 新增/编辑证书弹窗

### 12.1 尺寸与滚动

- 宽度：760px。
- 最大高度：`calc(100vh - 80px)`。
- 弹窗头部和底部固定。
- 主体区域 `overflow:auto`。
- 遮罩为深色半透明，不使用大圆角。

### 12.2 字段布局

```text
第一行：所属品牌（全宽）
第二行：证书名称（2/3）、证书排序（1/3）
第三行：证书类型（1/2）、证书编号（1/2）
第四行：发证机构（全宽）
第五行：证书文件（全宽）
第六行：长期有效（Switch）
第七行：生效日期（1/2）、到期日期（1/2）
第八行：前台展示（Switch）、备注（全宽）
```

长期有效开启时：生效日期和到期日期禁用并清空；关闭后恢复可编辑。

### 12.3 字段定义

| 字段 | 必填 | 控件 | 规则 |
|---|---:|---|---|
| 所属品牌 | 是 | SearchSelect | 选择一个启用或停用品牌；一个证书仅允许关联一个品牌 |
| 证书名称 | 是 | Input | 最大 80 字，同品牌内不可重复 |
| 证书排序 | 是 | Input Number | 正整数，默认 10 |
| 证书类型 | 是 | Select | 枚举值 |
| 证书编号 | 否 | Input | 最大 80 字 |
| 发证机构 | 否 | Input | 最大 100 字 |
| 证书文件 | 是 | Upload | JPG、PNG、WebP、PDF；单文件；最大 20MB |
| 长期有效 | 否 | Switch | 默认关闭 |
| 生效日期 | 否 | Date | 不得晚于到期日期 |
| 到期日期 | 条件必填 | Date | 非长期有效时必填 |
| 前台展示 | 否 | Switch | 默认开启 |
| 备注 | 否 | Textarea | 最大 300 字，仅后台可见 |

### 12.4 上传交互

- 初始态：虚线框，支持点击和拖拽。
- 上传中：显示文件名、进度条、取消上传。
- 成功态：显示缩略图/文件类型、文件名、文件大小、重新上传、删除文件。
- 失败态：显示失败原因与“重新上传”。
- 上传采用后端授权直传对象存储；保存业务数据前必须拿到 `file_url` 和 `file_key`。
- 编辑时未更换文件，不重复上传。

### 12.5 校验文案

| 场景 | 文案 |
|---|---|
| 证书名称为空 | 请输入证书名称 |
| 名称重复 | 该品牌已存在同名证书，请更换 |
| 排序非法 | 请输入正整数 |
| 类型为空 | 请选择证书类型 |
| 文件为空 | 请上传证书文件 |
| 文件格式不支持 | 仅支持 JPG、PNG、WebP 或 PDF |
| 文件过大 | 文件大小不能超过 20MB |
| 到期日为空 | 请选择到期日期或设为长期有效 |
| 日期顺序错误 | 到期日期不得早于生效日期 |

## 12. 页面状态

### 13.1 加载中

- 指标卡片使用骨架。
- 表格保留表头，展示 5 行骨架。
- 关键词输入使用防抖，避免连续触发接口请求；下拉筛选变化时取消尚未完成的旧请求。

### 13.2 空状态

- 首次无证书：`暂未维护品牌证书`，说明 `上传质量体系、检测报告或荣誉资质，完善品牌可信资料。`，提供 `新增证书` 主按钮。
- 筛选无结果：`未找到符合条件的证书`，提供 `清空筛选`。

### 13.3 异常状态

- 列表加载失败：卡片内展示错误提示与“重新加载”。
- 保存失败：弹窗保持打开，保留用户输入，并在底部展示服务端错误。
- 文件预览失败：提示 `文件暂时无法预览，请稍后重试或下载查看`。

## 13. 数据关系与约束

```text
brand.id 1 ─── N brand_certificate.brand_id
```

- `brand_certificate.brand_id` 必填且不可为数组。
- 同品牌内证书名称建议建立联合唯一约束：`UNIQUE(brand_id, certificate_name, deleted_at)`。
- 删除品牌前，如存在未删除证书，应阻止删除或要求先迁移/删除证书。
- 品牌停用不自动删除证书；店主端不展示停用品牌及其证书。
- 编辑证书允许切换所属品牌，但需重新校验目标品牌下名称唯一性。

## 14. 接口建议

### 14.1 列表

`GET /admin/brand-certificates`

Query：`keyword`、`brand_id`、`type`、`validity_status`、`display_status`、`page`、`page_size`。

### 14.2 新增

`POST /admin/brand-certificates`

```json
{
  "brand_id": 1001,
  "name": "ISO 9001 质量管理体系认证",
  "sort_order": 10,
  "certificate_type": "QUALITY_SYSTEM",
  "certificate_no": "CN-QMS-2026-0831",
  "issuer": "中国质量认证中心",
  "file_url": "https://...",
  "file_key": "brands/1001/certificates/...pdf",
  "file_name": "iso9001.pdf",
  "file_mime_type": "application/pdf",
  "file_size": 2483200,
  "is_permanent": false,
  "effective_date": "2026-01-01",
  "expiry_date": "2028-12-31",
  "is_visible": true,
  "remark": "年度复审通过"
}
```

### 14.3 其他接口

- `PUT /admin/brand-certificates/{certificate_id}`：编辑，可变更 `brand_id`。
- `POST /admin/brand-certificates/{certificate_id}/show`：前台显示。
- `POST /admin/brand-certificates/{certificate_id}/hide`：前台隐藏。
- `DELETE /admin/brand-certificates/{certificate_id}`：软删除。
- `POST /admin/uploads/authorize`：获取上传授权。

## 15. 错误码建议

|---|---|
| BRAND_NOT_FOUND | 品牌不存在或已删除 |
| CERTIFICATE_NAME_DUPLICATED | 该品牌已存在同名证书，请更换 |
| CERTIFICATE_FILE_REQUIRED | 请上传证书文件 |
| CERTIFICATE_FILE_TYPE_INVALID | 仅支持 JPG、PNG、WebP 或 PDF |
| CERTIFICATE_FILE_TOO_LARGE | 文件大小不能超过 20MB |
| CERTIFICATE_DATE_INVALID | 到期日期不得早于生效日期 |
| CERTIFICATE_NOT_FOUND | 证书不存在或已删除 |

## 17. 审计要求

记录新增、编辑、显示、隐藏、删除操作，至少包含：操作人、品牌 ID、证书 ID、操作类型、变更前后摘要、时间、IP。文件替换需记录旧 `file_key` 与新 `file_key`。

## 18. 前端实现约束

- React + Tailwind + shadcn/ui。
- 页面优先复用 `AdminListPage`、Sidebar、Pagination、Badge、Upload 等现有组件。
- TSX/CSS 使用语义 Token；不得新增裸 Hex 或硬编码 rgba。
- `className` 合并使用 `cn()`。
- 弹窗使用统一 admin-modal 结构，避免页面级 CSS 覆盖基础组件。
- 当前 HTML 原型中的颜色值仅作为视觉交付的 token 映射参考，正式代码需替换为 Design System Token。

## 19. 验收标准

1. 左侧导航在“瓷砖规格”下方展示“品牌证书”，并由该菜单独立高亮。
2. 页面不展示品牌摘要栏或返回品牌列表栏，直接由页面头部进入 4 个指标卡。
3. 页面具备所属品牌筛选、列表所属品牌列和新增/编辑弹窗所属品牌必填字段。
4. 页面无导出、无批量操作。
5. 有效、即将到期、已过期、未设置、长期有效状态显示正确。
6. 列表支持预览、编辑、显示/隐藏、删除。
7. 新增/编辑弹窗字段、顺序、必填及联动符合本需求。
8. 长期有效开启后日期字段清空并禁用。
9. 文件支持 JPG、PNG、WebP、PDF，单文件最大 20MB，并具备上传各状态。
10. 弹窗头尾固定、主体可滚动；小屏幕宽度自适应。
11. 筛选、页码、每页显示数刷新后可恢复；页面中不存在“查询”按钮。
12. 权限、错误态、空状态、加载态均可实现，不依赖占位文案。
13. UI 与品牌管理 V7 在 Sidebar、卡片、表格、按钮、Badge、输入框、分页和弹窗上保持一致。

5. 删除跳至页码输入框，降低分页区域操作复杂度。
