# 微信小程序新增品牌主页/详情页需求文档

## 1. 背景

品牌是瓷砖 SKU、证书资质和品牌故事展示的重要主数据。当前小程序已有首页、分类、搜索、商品列表、商品详情和证书相关能力，但缺少围绕“品牌”组织内容的前台展示页。用户若想按品牌浏览商品或查看品牌资质，需要在多个页面之间跳转，品牌形象、商品集合和证书背书没有形成统一入口。

本需求新增微信小程序品牌入口页与单品牌主页/详情页。品牌入口页提供与首页轮播一致的品牌轮播，以及一行 2 个品牌卡片的品牌列表；品牌主页/详情页上半部分展示品牌信息与品牌图片，下半部分通过 Tab 展示该品牌下的商品列表和证书列表。

## 2. 目标用户

| 用户 | 核心诉求 |
|---|---|
| 装修客户 | 快速了解品牌形象、品牌介绍和该品牌下的商品，降低选择成本 |
| 设计师 | 按品牌查看商品集合和品牌资质，便于向客户推荐 |
| 门店导购 | 通过品牌主页集中讲解品牌、商品和证书背书 |
| 普通访客 | 从品牌列表进入单品牌页面，不需要理解分类和筛选规则也能浏览 |

## 3. 范围

### 3.1 包含

- 微信小程序品牌入口页，包含品牌轮播和品牌列表。
- 品牌轮播的视觉、交互、指示器、图片比例和降级策略与首页轮播保持一致。
- 品牌列表采用一行 2 个品牌卡片的双列布局。
- 品牌卡片展示品牌图片或 Logo、品牌名称和必要的辅助信息，并可点击进入品牌主页/详情页。
- 品牌主页/详情页上半部分展示品牌信息，包括品牌图片、品牌名称、品牌介绍等基础内容。
- 品牌主页/详情页下半部分采用 Tab 展示内容，首期包含商品列表和证书列表。
- 商品 Tab 展示当前品牌下的商品列表，优先复用既有商品卡片和商品列表展示策略。
- 证书 Tab 展示当前品牌关联证书，优先复用既有品牌证书展示规则。
- 页面需覆盖首屏加载、刷新、加载更多、空状态、错误状态和点击跳转。

### 3.2 不包含

- Web 管理端品牌主数据管理能力重做。
- 店主 Web 展示端品牌主页。
- 品牌主页可视化装修、复杂楼层配置或自定义 Tab。
- 品牌视频、门店信息、联系方式、地图、客服、询价、购物车或在线下单。
- 品牌收藏、分享海报、SEO、小程序码生成。
- 新增品牌、商品或证书的数据模型，除非后续 OpenSpec Change 发现现有字段无法支撑。
- 管理端单独维护品牌轮播位置，除非后续明确 Banner 管理无法复用。

## 4. 功能要求

### FR-001 品牌入口页信息架构

- 小程序 MUST 提供品牌入口页，用于集中展示可公开访问的品牌。
- 品牌入口页 MUST 包含顶部品牌轮播和下方品牌列表。
- 品牌入口页 SHOULD 与小程序既有页面导航、状态栏、底部 TabBar 和暗色旗舰风保持一致。
- 品牌入口页首屏 SHOULD 同时露出轮播和至少一部分品牌卡片内容。
- 品牌入口页 MUST 支持加载中、空状态、错误状态和重试。

### FR-002 品牌轮播

- 品牌轮播 MUST 与首页轮播的 `swiper` 视觉和交互保持一致。
- 品牌轮播 MUST 支持多张图片自动轮播、手动滑动和指示器。
- 品牌轮播图片 MUST 使用稳定比例，加载前后不得造成明显布局跳动。
- 轮播项点击 SHOULD 支持跳转到对应品牌主页/详情页。
- 无可用轮播时 MUST 展示与首页轮播一致的降级视觉或隐藏轮播区，不得白屏。
- 品牌轮播数据来源 SHOULD 优先复用现有 Banner 管理能力；若现有 Banner 展示端/位置无法表达品牌页轮播，后续 OpenSpec Change 中再定义扩展。

### FR-003 品牌列表双列卡片

- 品牌列表 MUST 以一行 2 个品牌卡片展示。
- 品牌卡片 MUST 展示品牌图片或 Logo、品牌名称。
- 品牌卡片 MAY 展示品牌简称、英文名、商品数量或简短介绍，但不得挤压主图和名称。
- 品牌图片或 Logo 缺失时 MUST 展示统一占位图或品牌名称占位。
- 品牌名称过长时 MUST 限制行数或截断，不得撑破卡片。
- 整个品牌卡片 MUST 可点击并进入对应品牌主页/详情页。

### FR-004 品牌主页/详情页上半部分

- 品牌主页/详情页 MUST 根据品牌 ID 加载品牌基础信息。
- 页面上半部分 MUST 展示品牌图片或 Logo、品牌名称和品牌介绍。
- 品牌图片 SHOULD 优先使用品牌主图；若无主图，可回退到品牌 Logo。
- 品牌介绍为空时 MUST 使用隐藏策略或简短兜底文案，不得展示异常空字段。
- 品牌不存在、下架、禁用或不可公开访问时 MUST 展示可恢复错误态，不得白屏。

### FR-005 品牌主页/详情页 Tab 内容区

- 品牌主页/详情页下半部分 MUST 使用 Tab 分区展示内容。
- 首期 Tab MUST 包含“商品”和“证书”。
- 默认 Tab SHOULD 为“商品”。
- 切换 Tab 时 SHOULD 保留页面顶部品牌信息不丢失。
- Tab 内容加载中、为空、失败和重试状态 MUST 独立展示，不影响其他 Tab。
- Tab 标题、数量徽标或空态文案 SHOULD 清晰表达当前品牌上下文。

### FR-006 商品 Tab

- 商品 Tab MUST 展示当前品牌下的商品列表。
- 商品列表 SHOULD 复用既有商品列表双列卡片展示策略，与 `REQ-0056-product-list-card-only-layout` 保持一致。
- 商品卡片 MUST 支持点击进入 SKU 详情页。
- 商品 Tab MUST 支持分页、下拉刷新、上拉加载更多、无更多和加载失败重试。
- 当前品牌无商品时 MUST 展示品牌上下文相关空态。

### FR-007 证书 Tab

- 证书 Tab MUST 展示当前品牌关联的证书列表。
- 证书列表 SHOULD 复用既有品牌证书展示规则，包括证书图片、证书名称、证书类型和有效状态等公开可展示字段。
- 证书项点击 MAY 支持预览证书图片或进入证书详情，具体交互在 `/req-complete` 阶段确认。
- 当前品牌无证书时 MUST 展示品牌上下文相关空态。
- 不可公开、已下架或失效且不应展示的证书 MUST 不出现在前台列表中。

### FR-008 路由、入口与埋点

- 品牌入口页 MAY 由首页导航、分类/品牌入口或其他运营入口进入。
- 品牌主页/详情页 MUST 支持通过 `brandId` 路由参数访问。
- 品牌卡片点击、轮播点击、品牌主页浏览、Tab 切换、商品点击和证书点击 SHOULD 保留埋点。
- 埋点公共参数 SHOULD 包含 `sourcePage`、`brandId`、`brandName`、`tab`、`page`、`pageSize`、`resultCount` 和 `requestId`。
- 埋点失败不得阻断页面加载、Tab 切换或详情跳转。

## 5. UI 约束

- 页面视觉 MUST 延续小程序“工业石材 · 暗色旗舰风”。
- 品牌轮播 MUST 与首页轮播保持一致，不单独发明另一套轮播样式。
- 品牌列表一行 2 个卡片的间距、图片比例和文字层级 MUST 在常见小程序手机宽度下稳定。
- 品牌主页/详情页上半部分 SHOULD 以品牌图片和品牌名称形成首屏识别点。
- Tab 区域 SHOULD 固定在品牌信息下方，切换时不造成页面明显跳动。
- 商品卡片和证书卡片 SHOULD 优先复用现有组件或视觉规则。
- 点击区域 SHOULD 不小于 44px，适配小程序触控场景。
- 长品牌名、长商品名、长证书名 MUST 截断或限制行数，不得遮挡其他内容。
