# 验收标准

## 功能 AC

- [ ] AC-001 搜索入口组件支持关键词输入、清空、提交、取消/返回和禁用态。
- [ ] AC-002 搜索入口组件可接收初始关键词、占位文案、搜索范围和来源页面，并可被首页、分类页、SKU 列表页、品牌相关页面和独立搜索页复用。
- [ ] AC-003 搜索框、返回按钮和清空按钮符合小程序触控规范，可点击区域不小于 44px。
- [ ] AC-004 搜索首页展示最近搜索、热门搜索和最近浏览。
- [ ] AC-005 最近搜索本机保存最多 20 条，重复关键词去重后置顶，并支持单条删除和全部清空。
- [ ] AC-006 最近浏览最多展示 10 条，并按最近访问倒序排列。
- [ ] AC-007 中文输入 1 字、英文或数字输入 2 字后触发实时联想。
- [ ] AC-008 联想请求使用 300ms 防抖，旧请求响应不会覆盖新关键词结果。
- [ ] AC-009 联想结果展示 6 到 10 条，可包含最近搜索、SKU、品牌、类目、规格和普通关键词建议。
- [ ] AC-010 证书不参与实时联想，但可在完整搜索结果中展示。
- [ ] AC-011 可直达实体建议点击后进入对应详情或列表页，不可直达建议点击后进入搜索结果页。
- [ ] AC-012 搜索结果页提供综合、SKU、品牌、类目、证书类型 Tab，综合默认激活。
- [ ] AC-013 综合页展示最多 1 条最佳匹配，并按分区展示结果。
- [ ] AC-014 SKU 卡片展示主图、产品名称、SKU 编码、品牌、规格和参考价格。
- [ ] AC-015 SKU 卡片整卡点击进入 SKU 详情页，不放置收藏、分享等额外快捷操作。
- [ ] AC-016 搜索结果页支持品牌、类目、规格快捷筛选。
- [ ] AC-017 完整筛选使用底部抽屉展示品牌、类目、规格、价格区间，支持重置和确认。
- [ ] AC-018 筛选选项和数量基于当前搜索结果动态聚合，确认按钮展示应用筛选后的结果数量。
- [ ] AC-019 搜索排序优先级满足完整 SKU 精确匹配、SKU 前缀匹配、产品名称精确匹配、品牌精确匹配、品牌拼音/首字母匹配、标签组合匹配、普通模糊匹配。
- [ ] AC-020 同等级结果按有图优先、热度优先、最近更新优先排序。
- [ ] AC-021 无结果页展示当前关键词、调整建议和推荐搜索词。
- [ ] AC-022 v1 不展示联系商家、提交找砖需求、购物车、询价或在线下单入口。
- [ ] AC-023 搜索行为记录搜索页浏览、输入、联想曝光、联想点击、搜索提交、结果曝光、结果点击、筛选应用、无结果和历史操作事件。
- [ ] AC-024 埋点公共参数包含 keyword、normalizedKeyword、scope、entityType、resultCount、sourcePage、filterSnapshot 和 requestId。
- [ ] AC-025 日志不得记录手机号等与搜索无关的个人敏感信息。

## UI AC

- [ ] AC-UI-001 视觉风格与菲尚特小程序首页深色企业轻奢风保持一致。
- [ ] AC-UI-002 搜索框固定在顶部安全区下方，输入区域高度参考 44px。
- [ ] AC-UI-003 结果页类型 Tab 可横向滚动，综合 Tab 默认激活。
- [ ] AC-UI-004 筛选选中态使用品牌金边框与浅金背景。
- [ ] AC-UI-005 底部筛选抽屉覆盖页面约 72%，背景遮罩约 62%。
- [ ] AC-UI-006 搜索空状态、加载态、联想态和结果态切换时无明显布局跳动。
- [ ] AC-UI-007 实现验收优先以 `prototype/*.html` 与 `prototype/*.png` 为视觉基准，`prototype/context.md` 为交互补充说明。

## 横切 AC（knowledge-base）

无横切 AC。本需求为微信小程序搜索体验，不命中 `admin-list`、`admin-form`、`admin-modal`、`media-upload` 标签；管理端列表、表单、弹窗、上传相关 best-practices 不适用。
