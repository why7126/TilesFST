# 微信小程序搜索通用组件并应用需求文档

## 1. 背景

菲尚特微信小程序已经具备首页、分类、SKU 详情等商品浏览路径，用户需要通过关键词快速找到目标瓷砖 SKU、品牌、类目、规格或证书信息。为避免不同页面重复实现搜索输入、联想、结果展示、筛选与空状态，本需求沉淀微信小程序端可复用的搜索通用组件与搜索页面能力。

本需求仅覆盖微信小程序端搜索体验与组件应用，不包含 Web 管理端、店主 Web、后台搜索配置中心或管理端列表搜索改造。

## 2. 目标用户

| 用户 | 核心诉求 |
|---|---|
| 装修客户 | 输入 SKU、品牌、规格或风格关键词，快速找到可参考的瓷砖商品 |
| 设计师 | 根据空间、色系、规格等关键词筛选候选 SKU，辅助方案沟通 |
| 门店导购 | 在小程序内快速检索商品，并进入 SKU 详情页分享给客户 |
| 普通访客 | 从首页或浏览路径进入搜索，继续探索品牌、类目和证书信息 |

## 3. 范围

### 3.1 包含

- 微信小程序搜索入口组件：搜索框、清空、提交、返回和关键词状态。
- 微信小程序搜索首页：最近搜索、热门搜索、最近浏览。
- 微信小程序实时联想：关键词防抖、建议列表、实体建议跳转。
- 微信小程序搜索结果页：综合结果、类型切换、SKU/品牌/类目/证书结果展示。
- 微信小程序搜索筛选：品牌、类目、规格、价格区间等筛选能力。
- 微信小程序无结果状态：关键词反馈、搜索建议和推荐词。
- 搜索行为埋点：搜索输入、提交、联想点击、结果点击、筛选和无结果。
- 原型范围内的页面与交互约束：`prototype/context.md` 及搜索页面原型。

### 3.2 不包含

- Web 管理后台搜索组件或管理端列表页搜索应用。
- 店主 Web 展示端搜索组件。
- 后台“搜索配置中心”、热门词维护、同义词维护、自然语言词典维护和搜索统计管理页。
- 管理端搜索配置接口，例如 `/api/admin/search/*`。
- 基于大模型的自然语言搜索理解。
- 购物车、询价、在线下单、客服找砖或联系商家入口。

## 4. 页面与入口

| 入口 | 默认搜索范围 | 行为 |
|---|---|---|
| 小程序首页顶部搜索框 | 综合 | 点击进入搜索首页，输入后展示联想或结果 |
| 小程序分类页搜索框 | SKU | 保留当前类目上下文，搜索结果优先展示当前类目相关 SKU |
| 小程序 SKU 列表搜索框 | SKU | 关键词直接用于 SKU 搜索与筛选 |
| 小程序品牌相关页面搜索框 | 品牌/SKU | 品牌词命中可进入品牌页，SKU 命中进入 SKU 详情 |
| 小程序独立搜索页 | 综合 | 承载历史、热门、联想、结果、筛选和无结果状态 |

## 5. 功能要求

### FR-001 搜索通用组件

- 组件 MUST 支持搜索关键词输入、清空、提交、取消/返回和禁用态。
- 组件 MUST 支持外部传入初始关键词、占位文案、搜索范围和来源页面。
- 组件 MUST 在小程序可点击区域规范内提供返回按钮与清空按钮，点击区域不小于 44px。
- 组件 MUST 支持回车/键盘搜索和搜索按钮触发。
- 组件 SHOULD 支持被首页、分类页、SKU 列表页、品牌相关页面和独立搜索页复用。

### FR-002 搜索首页

- 搜索首页 MUST 展示最近搜索、热门搜索和最近浏览。
- 最近搜索 MUST 本机保存，最多 20 条，重复关键词去重后置顶。
- 最近搜索 MUST 支持单条删除和全部清空。
- 最近浏览 SHOULD 最多展示 10 条，按最近访问倒序。
- 热门搜索词 MUST 仅用于小程序端展示，具体来源和管理方式留待后续 OpenSpec 细化。

### FR-003 实时联想

- 中文输入达到 1 个字符、英文或数字输入达到 2 个字符后 MUST 触发联想。
- 联想请求 MUST 使用 300ms 防抖。
- 组件 MUST 避免旧请求结果覆盖新关键词结果。
- 联想结果 SHOULD 展示 6 到 10 条。
- 联想结果 MAY 包含最近搜索、SKU、品牌、类目、规格和普通关键词建议。
- 证书 MUST 不参与实时联想。
- 可直达实体点击后 MUST 进入对应详情或列表页；不可直达建议点击后进入搜索结果页。

### FR-004 搜索结果

- 搜索结果页 MUST 提供类型 Tab：综合、SKU、品牌、类目、证书。
- 综合页 SHOULD 采用最佳匹配和分区结果结构。
- 最佳匹配最多展示 1 条。
- SKU 卡片 MUST 展示主图、产品名称、SKU 编码、品牌、规格和参考价格。
- SKU 卡片 MUST 整卡点击进入 SKU 详情页，不放置额外快捷操作。
- 证书仅在完整搜索结果中展示，不参与联想。

### FR-005 筛选

- 搜索结果页 MUST 支持快捷筛选：品牌、类目、规格。
- 完整筛选 SHOULD 使用底部抽屉呈现，并支持品牌、类目、规格、价格区间。
- 筛选项和数量 SHOULD 基于当前搜索结果动态聚合。
- 筛选 MUST 支持重置和确认。
- 确认按钮 SHOULD 展示应用筛选后的结果数量。

### FR-006 排序

搜索结果 SHOULD 按以下优先级排序：

1. 完整 SKU 精确匹配。
2. SKU 前缀匹配。
3. 产品名称精确匹配。
4. 品牌精确匹配。
5. 品牌拼音或首字母匹配。
6. 标签组合匹配。
7. 普通模糊匹配。

同等级结果 SHOULD 按有图优先、热度优先、最近更新优先排序。v1 不支持人工置顶和个性化浏览权重。

### FR-007 无结果

- 无结果页 MUST 展示当前关键词。
- 无结果页 SHOULD 提示用户检查编码、缩短关键词、替换品牌/规格/名称。
- 无结果页 SHOULD 展示推荐搜索词。
- v1 不提供联系商家或提交找砖需求入口。

### FR-008 埋点

小程序搜索 SHOULD 记录以下事件：

- `search_page_view`
- `search_input`
- `search_suggestion_exposure`
- `search_suggestion_click`
- `search_submit`
- `search_result_exposure`
- `search_result_click`
- `search_filter_apply`
- `search_no_result`
- `search_history_click`
- `search_history_delete`
- `search_history_clear`

公共参数 SHOULD 包含 keyword、normalizedKeyword、scope、entityType、resultCount、sourcePage、filterSnapshot 和 requestId。

## 6. UI 约束

- 视觉风格 MUST 与菲尚特小程序首页深色企业轻奢风保持一致。
- 搜索框 SHOULD 固定在顶部安全区下方，输入区域高度参考 44px。
- 结果页类型 Tab SHOULD 支持横向滚动，综合默认激活。
- 筛选选中态 SHOULD 使用品牌金边框与浅金背景。
- 底部筛选抽屉 SHOULD 覆盖页面约 72%，背景遮罩约 62%。
- 搜索空状态、加载态、联想态和结果态之间切换 MUST 避免布局跳动。

## 7. 关联需求

- `REQ-0041-miniapp-home`：小程序首页可作为搜索入口。
- `REQ-0044-miniapp-sku-detail-page`：搜索联想和搜索结果中的 SKU 可进入 SKU 详情页。
- `REQ-0045-category-list-page`：分类页可复用小程序搜索组件并携带类目上下文。
