# 微信小程序品牌卡片组件用户故事

## US-001 装修客户识别商品品牌

作为装修客户，我希望在 SKU 详情页看到稳定清晰的品牌卡片，以便快速理解当前商品所属品牌，并进入品牌相关内容继续浏览。

验收要点：

- SKU 详情页展示品牌 Logo、品牌名称和入口提示。
- 品牌名称较长时仍能在移动端窄屏保持可读，不撑破布局。
- Logo 缺失或加载失败时展示统一占位，不出现破图。
- 点击品牌卡片后进入品牌主页、品牌商品列表或品牌搜索结果入口。

## US-002 设计师沿品牌继续选型

作为设计师，我希望从 SKU 详情页进入同品牌内容，以便比较同品牌不同系列或类似风格产品。

验收要点：

- 卡片点击携带品牌 ID、品牌名称和来源页面上下文。
- 有 `brand_entry_path` 时优先使用配置入口。
- 无配置入口时 fallback 到品牌关键词搜索页。
- 入口不可用时提供可理解提示，不发生空白页或无响应。

## US-003 门店导购讲解品牌

作为门店导购，我希望品牌卡片展示风格稳定、点击反馈明确，以便在向客户介绍商品时顺畅切换到品牌相关内容。

验收要点：

- 整个卡片可点击，触控区域不小于 44px。
- 点击期间有小程序触控反馈。
- 连续点击不得重复触发多次跳转。
- 卡片异常不得影响 SKU 详情页其他模块展示。

## US-004 小程序开发者复用组件

作为小程序开发者，我希望品牌卡片沉淀为独立组件，以便后续品牌商品列表、同品牌推荐、首页品牌推荐等场景复用同一信息结构和异常策略。

验收要点：

- 组件接收单个品牌展示数据，不在组件内部请求列表或详情接口。
- 页面容器负责页面布局、加载态和列表状态机，组件只负责单品牌展示与交互。
- 组件参数预留 `sourcePage`、`sourceModule`、`skuId`、`listContext` 等上下文。
- 组件新增后替换 SKU 详情页现有内联品牌卡片。
