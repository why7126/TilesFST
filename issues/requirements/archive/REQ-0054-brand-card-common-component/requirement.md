# 微信小程序品牌卡片组件需求文档

## 1. 背景

菲尚特微信小程序已经在 SKU 详情页展示品牌信息，并通过品牌入口引导用户查看同品牌内容。当前品牌信息以页面内联结构实现，后续若品牌主页、品牌商品列表、搜索结果推荐和详情页推荐模块都需要展示品牌，容易出现 Logo fallback、文案层级、点击跳转和埋点规则不一致。

本需求沉淀微信小程序端可复用的品牌卡片组件，首版优先替换 SKU 详情页中的内联品牌卡片，并为后续品牌主页入口、同品牌推荐、品牌商品列表等场景预留复用能力。

## 2. 目标用户

| 用户 | 核心诉求 |
|---|---|
| 装修客户 | 在商品详情中快速识别所属品牌，并进入品牌相关内容继续浏览 |
| 设计师 | 判断 SKU 所属品牌、风格定位和后续同品牌选型入口 |
| 门店导购 | 在介绍商品时快速切换到品牌相关商品或品牌故事，辅助推荐 |
| 普通访客 | 通过一致的品牌入口理解品牌关系，减少跳转路径困惑 |

## 3. 范围

### 3.1 包含

- 微信小程序品牌卡片组件的信息结构、展示规则、触控反馈和跳转行为。
- 品牌 Logo、品牌名称、品牌简介或入口提示等核心字段展示。
- Logo 缺失、Logo 加载失败、品牌名称缺失、品牌入口不可用等卡片级异常处理。
- SKU 详情页现有内联品牌卡片的组件化替换约束。
- 预留品牌主页入口、品牌商品列表、同品牌推荐等后续复用场景。
- 品牌卡片点击和图片异常等埋点建议。

### 3.2 不包含

- 品牌主页完整信息架构、品牌故事页面、品牌商品列表容器和分页状态机。
- 管理端 Web 品牌管理页、新增/编辑/启停/删除等主数据维护操作。
- 店主 Web 展示端品牌卡片或跨端 React 组件库建设。
- 品牌证书列表、品牌资质详情、证书上传或证书展示组件。
- 品牌 API、数据库字段、品牌 Logo 上传、MinIO 对象存储策略新增或变更。
- 收藏、分享、在线咨询、询价、品牌对比等卡片内复杂操作。

## 4. 功能要求

### FR-001 组件基础结构

- 品牌卡片 MUST 作为微信小程序可复用组件沉淀，支持被 SKU 详情页和后续品牌相关页面引用。
- 组件 MUST 接收单个品牌展示数据，不在组件内部直接请求品牌列表、SKU 列表或品牌详情接口。
- 组件 MUST 提供稳定的 Logo 区、品牌信息区和入口提示区，避免不同页面各自定义核心结构。
- 组件 SHOULD 支持传入来源上下文，用于跳转参数、埋点和后续复用场景区分。

### FR-002 品牌信息展示

- 品牌卡片 MUST 展示品牌 Logo 和品牌名称。
- 品牌卡片 SHOULD 支持展示品牌简介、品牌定位、入口提示或等价副文案，但不得挤压品牌名称可读性。
- 品牌名称 MUST 支持长文本截断，移动端窄屏下不得撑破卡片布局。
- 品牌简介或副文案缺失时，组件 MUST 使用统一占位或隐藏策略，不展示空字符串、`null`、`undefined` 等异常值。
- 品牌 Logo 的替代文本或无障碍语义 SHOULD 基于品牌名称生成，便于后续测试和可访问性检查。

### FR-003 Logo 与图片异常

- 品牌 Logo MUST 使用稳定尺寸容器，避免图片加载前后导致卡片高度跳动。
- Logo 缺失或加载失败时 MUST 使用统一深色占位、品牌首字或默认图片，不展示破图。
- Logo SHOULD 使用适合小程序展示的缩略图 URL，不直接依赖管理端原始上传对象路径。
- 图片异常不得影响品牌名称、入口提示和卡片点击能力。

### FR-004 点击与跳转

- 整个品牌卡片 MUST 可点击，并跳转到品牌主页、品牌商品列表或品牌搜索结果中的一个明确入口。
- 当后端或页面数据提供 `brand_entry_path` 时，组件 SHOULD 优先使用该入口。
- 当 `brand_entry_path` 缺失时，组件 MAY fallback 到品牌关键词搜索页，但 MUST 对品牌名称做 URL 编码。
- 品牌 ID、品牌名称和来源页面 SHOULD 纳入跳转上下文或埋点参数。
- 品牌数据缺失、入口不可用或品牌名称不可用时，组件 MUST 阻止跳转并给出可理解的提示或不可用状态。

### FR-005 复用与页面适配

- 首版 MUST 支持替换 SKU 详情页现有内联品牌卡片。
- 组件 SHOULD 预留在品牌商品列表、同品牌推荐、首页品牌推荐等场景复用的参数结构。
- 不同页面 MAY 传入不同入口提示、来源上下文和展示密度，但不得改变 Logo、品牌名称和点击入口的核心语义。
- 页面容器负责页面布局、加载态、错误页和品牌商品列表状态机；品牌卡片只负责单个品牌的展示与交互。

### FR-006 卡片级异常状态

- 品牌 Logo 缺失或加载失败时，卡片 MUST 保持整体信息可读。
- 品牌名称缺失时，卡片 SHOULD 展示“品牌信息待完善”或等价兜底文案，并上报数据异常。
- 品牌入口不可用时，卡片 SHOULD 展示禁用态或点击后提示“品牌内容暂不可查看”。
- 单个品牌卡片异常不得导致 SKU 详情页或所在列表页不可用。

### FR-007 埋点

品牌卡片 SHOULD 支持以下埋点事件：

- `brand_card_click`
- `brand_card_image_failed`
- `brand_card_unavailable_click`

公共参数 SHOULD 包含 `brandId`、`brandName`、`sourcePage`、`sourceModule`、`skuId`、`listContext`、`index` 和 `requestId`。

## 5. UI 约束

- 视觉风格 MUST 与菲尚特微信小程序深色企业轻奢风保持一致。
- 卡片圆角、阴影、分割线、品牌金强调色和触控反馈 SHOULD 与现有小程序 SKU 详情页和商品卡片组件保持统一。
- Logo 区域 MUST 使用固定尺寸，品牌名称和副文案在 320px 小屏宽度下仍需可读。
- 可点击区域有效尺寸 SHOULD 不小于 44px，适配小程序触控场景。
- 卡片内不堆叠编辑、启停、删除等管理操作，首版优先保证品牌识别和进入品牌内容路径清晰。
- Logo 占位、品牌名称缺失和入口不可用状态 MUST 使用统一视觉语义。

## 6. 关联需求

- `REQ-0005-brand-management`：父需求，品牌主数据和 Logo 来源由品牌管理能力维护。
- `REQ-0044-miniapp-sku-detail-page`：首版优先替换 SKU 详情页内联品牌卡片。
- `REQ-0049-miniapp-product-card-component`：小程序商品卡片组件，可参考其组件边界、图片 fallback、点击跳转和埋点策略。
- `REQ-0047-product-list-common-component-application`：后续品牌商品列表或推荐模块可能与商品列表通用能力联动。
- `REQ-0038-brand-certificate-management`：品牌证书属于后续品牌详情/资质展示范围，首版品牌卡片不直接展示证书列表。
