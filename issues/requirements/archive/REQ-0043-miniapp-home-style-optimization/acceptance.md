# 验收标准

## 功能 AC

- [ ] AC-001 首页使用深色视觉体系，页面背景为 `#18160F`，卡片/搜索框/模块底色为 `#211E16`，品牌金为 `#C8A055`，不得恢复暖白或浅灰主背景。
- [ ] AC-002 首页不模拟微信系统状态栏、分享按钮、关闭按钮或胶囊控件；页面 Header 与微信真实导航环境不重叠、不重复。
- [ ] AC-003 首页搜索入口文案为“搜索瓷砖名称 / 型号 / 规格”或等价中文，点击后进入搜索页。
- [ ] AC-004 Banner 使用管理端 Banner 管理中小程序首页轮播位的上线数据；正常状态不得显示前端 Mock Banner。
- [ ] AC-005 Banner 图片使用后端受控媒体 URL，图片加载失败时展示品牌化占位或降级 Hero，页面不白屏。
- [ ] AC-006 快捷入口只展示“选瓷砖、品牌馆、新品榜、热销榜”四项。
- [ ] AC-007 四个快捷入口采用统一的图标在上、文案在下结构，单个入口点击区域不小于 72x72px 或小程序等效尺寸。
- [ ] AC-008 “新品榜”入口可显示 `NEW` 徽章，但新品推荐商品卡片图片区域不得显示“新品”角标或标签。
- [ ] AC-009 新品推荐横向展示商品卡片，卡片展示 SKU 主图、SKU 编号或名称、规格和 `price_display`。
- [ ] AC-010 热销推荐使用双列大卡片，展示 SKU 主图、系列/商品名、空间/工艺标签和 `price_display`。
- [ ] AC-011 商品卡片点击进入商品详情页；不可达目标必须安全降级，不得出现白屏或路由错误。
- [ ] AC-012 热销推荐下方展示“全部产品”模块。
- [ ] AC-013 全部产品使用双列瀑布流布局，图片高度允许错落，不得用固定等高双列网格冒充瀑布流。
- [ ] AC-014 首页首次加载 10 到 20 个全部产品；推荐分页大小为 12。
- [ ] AC-015 页面接近底部时可自动加载下一页，加载中不得重复发起并发请求。
- [ ] AC-016 下一页加载成功后追加商品，不覆盖已有列表，并按 `product_id` 去重。
- [ ] AC-017 无更多数据时展示“已经到底了”或等价状态，并停止继续请求。
- [ ] AC-018 加载下一页失败时展示可重试状态，且保留已加载商品。
- [ ] AC-019 首页没有商品时仍保留搜索、Banner、快捷入口等非商品模块。
- [ ] AC-020 底部 TabBar 目标文案为：首页、分类、找砖、收藏、证书；如收藏/证书页未完整实现，点击必须安全降级。
- [ ] AC-021 “我的”Tab 不作为首页目标导航出现；若保留用户中心能力，必须另行定义入口与范围。
- [ ] AC-022 320 到 430 pt 宽度范围内无页面级横向滚动、明显重叠、严重截断或 TabBar 遮挡。
- [ ] AC-023 所有主要可点击区域不小于 44x44 pt。
- [ ] AC-024 首页埋点至少预留或记录搜索点击、快捷入口点击、商品点击、瀑布流加载、加载失败、收藏视觉点击和证书 Tab 点击。
- [ ] AC-025 埋点不得记录聊天内容、Authorization、Cookie、手机号等敏感信息。

## 非范围 AC

- [ ] AC-NON-001 不新增收藏持久化 API、收藏列表、收藏统计或用户维度收藏状态。
- [ ] AC-NON-002 不新增完整证书聚合页、证书详情页或公开证书 API。
- [ ] AC-NON-003 不新增品牌馆独立页、品牌详情页、新品榜独立页或热销榜独立页。
- [ ] AC-NON-004 不新增快捷入口后台配置能力。
- [ ] AC-NON-005 小程序首页不出现订单、库存、新增、编辑、上下架、客户管理等后台能力。

## 横切 AC（knowledge-base）

本 REQ 为微信小程序首页 UI 与信息架构优化，不命中 `req-complete` 规定的管理端横切标签（admin-list / admin-form / admin-modal / media-upload），因此无 AC-XCUT 条目。
