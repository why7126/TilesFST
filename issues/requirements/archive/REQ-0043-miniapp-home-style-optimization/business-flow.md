# 业务流程

## 1. 首页浏览主流程

```text
用户打开小程序首页
  |
  v
加载首页聚合数据 + 首批全部产品
  |
  +-- 成功 --> 展示品牌 Header / 搜索 / Banner / 四入口 / 新品 / 热销 / 全部产品
  |
  +-- 部分失败 --> 模块级降级，其他模块继续可见
  |
  +-- 网络失败 --> 展示重试状态
```

## 2. 搜索与快捷入口流程

```text
首页搜索框
  └─ 点击 --> 搜索页 --> 商品结果 --> 商品详情

四入口
  ├─ 选瓷砖 --> 分类 Tab 或筛选页
  ├─ 品牌馆 --> 首期安全降级到搜索/占位，完整品牌馆另拆
  ├─ 新品榜 --> 搜索页 section=new
  └─ 热销榜 --> 搜索页 section=hot
```

## 3. 推荐商品与瀑布流流程

```text
新品推荐 / 热销推荐 / 全部产品瀑布流
  |
  +-- 点击卡片 --> 商品详情页
  |
  +-- 图片失败 --> 深色占位图
  |
  +-- 价格缺失 --> 价格待维护
```

## 4. 全部产品分页状态机

```text
idle
  |
  | 触底 200-300px 且 hasMore=true
  v
loading
  |
  +-- success(items, hasMore=true) --> append + dedupe --> idle
  |
  +-- success(items, hasMore=false) --> append + dedupe --> finished
  |
  +-- failed --> retry
```

约束：

- `loading=true` 期间不得并发触发下一页请求。
- 请求成功后追加，不覆盖已有商品。
- 按 `product_id` 去重。
- 页面离开后应忽略过期回调，避免回写已卸载页面状态。

## 5. 与父需求 REQ-0041 差异

| 维度 | REQ-0041 | REQ-0043 |
|---|---|---|
| 首页目标 | 小程序首页首期可用闭环 | 基于附件原型做首页体验升级 |
| 视觉体系 | 首期首页结构与基本品牌展示 | 深色高端展厅风、明确色彩/字号/卡片约束 |
| 快捷入口 | 按空间、规格、风格、颜色、全部分类 | 选瓷砖、品牌馆、新品榜、热销榜 |
| 商品承接 | 新品、热销推荐 | 新品、热销 + 全部产品瀑布流 |
| 底部 Tab | 首页、分类、找砖、我的 | 首页、分类、找砖、收藏、证书目标；未完成页需安全降级 |
| 收藏/证书 | 明确不纳入首期 | 仅允许视觉/占位/安全降级；完整能力仍后置 |
