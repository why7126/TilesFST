# REQ-0043 微信小程序首页样式与信息架构优化

## 1. 需求背景

`REQ-0041-miniapp-home` 已建立微信小程序首页首期能力，包含首页聚合、Banner、新品/热销推荐、搜索、详情、门店信息和底部 TabBar。随着新的附件产品原型明确，首页需要进一步从“功能可用”升级为“高端深色数字瓷砖展厅”：更突出品牌识别、搜索效率、四个核心入口、持续浏览能力和产品库承接。

本需求基于以下附件与探索结论生成：

- `附件 requirement.md`
- `附件 context.md`
- `附件 prototype.html`
- `附件 prototype.png`
- `/req-explore REQ-0043` 结论：首期优先落地首页视觉结构与全部产品瀑布流，收藏持久化、证书聚合页、品牌馆独立页等能力后置。

## 2. 目标用户

| 用户 | 诉求 |
|---|---|
| 装修业主 | 在微信内快速浏览菲尚特瓷砖主推、新品、热销和更多产品，形成咨询意向。 |
| 设计师 | 通过搜索、分类、榜单和连续浏览快速找到合适 SKU，用于方案沟通。 |
| 门店访客 | 到店前查看产品库与品牌信息，减少线下沟通成本。 |
| 门店销售 | 用首页作为移动端展示工具，快速向客户展示产品纹理、规格和价格。 |

## 3. 需求目标

- 将首页视觉升级为深色品牌体系，强化“专业选砖、品质生活”的高端展厅感。
- 将快捷入口从旧的筛选维度入口升级为四个业务入口：选瓷砖、品牌馆、新品榜、热销榜。
- 在热销推荐下方新增“全部产品”双列瀑布流，让用户无需跳转即可持续浏览公开 SKU。
- 复用现有小程序公开商品接口与 SKU/Banner/媒体数据，避免新增重复业务数据源。
- 明确收藏、证书、品牌馆等大范围能力的首期边界，避免首页优化演变为多个独立业务模块的一次性交付。

## 4. 范围

### 4.1 本期包含

| 范围 | 说明 |
|---|---|
| 首页深色视觉 | 页面背景、卡片、文字、品牌金、搜索框、Banner、商品卡片按附件原型调整。 |
| 品牌 Header | 首页顶部展示 Logo、店名和品牌副文案；必须适配微信小程序真实导航环境，不模拟系统胶囊按钮。 |
| 搜索入口 | 搜索框提示“搜索瓷砖名称 / 型号 / 规格”，点击进入已有搜索页。 |
| Banner 区 | 继续使用管理端 Banner 管理中“展示端=小程序首页、展示位置=小程序首页轮播”的上线 Banner。 |
| 四入口快捷导航 | 固定展示选瓷砖、品牌馆、新品榜、热销榜，采用图标在上、文案在下的四列结构。 |
| 新品推荐 | 横向商品卡片展示 SKU 主图、SKU 编号/名称、规格、价格；商品图片区域不显示“新品”角标。 |
| 热销推荐 | 双列大卡片展示主图、系列/商品名、空间/工艺标签、价格；可显示非持久化收藏视觉图标。 |
| 全部产品瀑布流 | 热销推荐下方展示双列瀑布流，支持分页加载、触底追加、失败重试、无更多提示、去重。 |
| 底部 TabBar 文案调整 | 页面视觉目标按附件原型表达：首页、分类、找砖、收藏、证书；首期如页面未实现，允许占位或安全降级。 |
| 埋点 | 补充首页搜索、快捷入口、瀑布流曝光/点击/加载、收藏视觉点击、证书 Tab 点击等事件定义或预留。 |

### 4.2 本期不包含

| 不包含 | 说明 |
|---|---|
| 收藏持久化 | 不新增收藏 API、收藏列表、收藏统计或用户维度持久化；热销排序不得依赖收藏量。 |
| 证书聚合页完整实现 | 证书 Tab 可作为占位/安全降级；公开证书列表、证书详情和品牌证书聚合另拆需求。 |
| 品牌馆独立页 | 品牌馆入口首期可跳转已有搜索/筛选或占位页；品牌列表页、品牌详情页另拆需求。 |
| 新品榜/热销榜独立页 | 首期“查看更多”或入口可复用已有搜索页 section 参数；独立榜单页面另拆需求。 |
| 后台配置快捷入口 | 四入口默认固定；入口名称、图标、顺序和目标后台配置另拆需求。 |
| 自定义全局导航栏 | 是否使用 `navigationStyle: custom` 另行评审；本需求不强制全局自定义导航。 |
| 订单/库存/后台编辑 | 小程序首页不得出现订单、库存、新增、编辑、上下架、客户管理等后台能力。 |

## 5. 功能要求

### FR-001 首页深色视觉体系

- 首页 MUST 使用深色体系：页面背景 `#18160F`，卡片/搜索框/模块底色 `#211E16`，品牌金 `#C8A055`。
- 主要文字 SHOULD 使用 `#EDE8DF`，辅助文字 SHOULD 使用 `rgba(237,232,223,0.5)`。
- 页面 MUST NOT 使用纯黑、暖白或浅灰作为主背景。
- 字体层级 SHOULD 贴近附件原型：Banner 标题约 38px，模块标题约 22px，卡片标题约 16px，价格/正文约 13px，规格/辅助信息约 11px。

### FR-002 品牌 Header 与真实小程序导航环境

- 首页 SHOULD 展示品牌 Logo、门店名称“菲尚特瓷砖馆”和品牌副文案“专业选砖 · 品质生活”。
- Header MUST 避免模拟微信系统状态栏、分享按钮、关闭按钮或胶囊控件。
- 如果使用微信原生导航栏，页面内容 SHOULD 从品牌 Header 或搜索框开始，并避免与原生标题重复造成冗余。
- 如果后续采用 `navigationStyle: custom`，MUST 额外处理状态栏高度、胶囊位置、安全区和不同机型适配。

### FR-003 搜索入口

- 首页 MUST 展示搜索入口，默认文案为“搜索瓷砖名称 / 型号 / 规格”。
- 搜索入口 MUST 具备清晰点击反馈，点击后进入已有搜索页。
- 搜索入口 SHOULD 支持根据关键字搜索商品名称、SKU 编号、规格、品牌或系列。

### FR-004 Banner 轮播

- Banner MUST 使用管理端 Banner 管理数据，不使用前端 Mock 数据作为正常展示来源。
- Banner 数据范围 MUST 限定为小程序首页轮播位，即展示端为小程序首页、展示位置为小程序首页轮播、状态上线且有效期内。
- Banner 图片 MUST 使用后端受控媒体 URL，图片加载失败时展示品牌化占位或降级 Hero。
- Banner SHOULD 支持自动轮播、手势切换、分页指示器和点击跳转。

### FR-005 四入口快捷导航

- 首页 MUST 仅展示四个快捷入口：选瓷砖、品牌馆、新品榜、热销榜。
- 四个入口 MUST 使用统一的“图标在上、文案在下”结构。
- 每个入口点击区域 MUST 不小于 72x72px 或小程序等效尺寸。
- 新品榜 MAY 使用品牌金实心圆形背景和 `NEW` 徽章。
- 点击策略：
  - 选瓷砖：进入分类/筛选页或已有分类 Tab。
  - 品牌馆：首期可安全降级到搜索/筛选或占位提示，完整品牌馆另拆。
  - 新品榜：进入已有搜索页并带入 `section=new`。
  - 热销榜：进入已有搜索页并带入 `section=hot`。

### FR-006 新品推荐

- 新品推荐 MUST 使用横向滚动商品卡片。
- 商品卡片 MUST 展示 SKU 主图、SKU 编号或商品名称、规格和价格展示。
- 商品图片区域 MUST NOT 展示“新品”角标或标签，新品属性由模块标题表达。
- 点击商品卡片 MUST 进入商品详情页。
- “查看更多” SHOULD 进入新品筛选结果或安全降级到搜索页。

### FR-007 热销推荐

- 热销推荐 SHOULD 使用双列大卡片展示。
- 卡片 MUST 展示 SKU 主图、系列/商品名、空间/工艺标签和价格展示。
- 卡片整卡点击 MUST 进入商品详情页。
- 收藏心形图标如出现，首期 MUST 仅作为视觉反馈或占位，不得承诺持久化收藏。
- 热销推荐排序首期 SHOULD 复用现有热销排序或公开商品排序，后续可接入更完整行为统计。

### FR-008 全部产品瀑布流

- 热销推荐下方 MUST 展示“全部产品”模块。
- 全部产品 MUST 使用双列瀑布流布局，图片高度允许错落，不得用固定等高网格冒充瀑布流。
- 首次进入首页 SHOULD 加载 10 到 20 个商品，推荐默认分页大小为 12。
- 页面滚动接近底部 200 到 300px 时 SHOULD 自动加载下一页。
- 请求期间 MUST 防止重复触发；请求成功后 MUST 追加数据，不覆盖已有产品。
- 返回重复商品时 MUST 按 `product_id` 或等价商品 ID 去重。
- `hasMore=false` 时 MUST 展示“已经到底了”或等价状态，并停止继续请求。
- 加载失败时 MUST 展示“加载失败，点击重试”或等价状态，并保留已有商品。

### FR-009 全部产品数据接口复用

- 首期 SHOULD 复用现有 `/api/v1/miniapp/products` 公开商品列表接口实现瀑布流分页。
- 若接口缺少 `hasMore`，实现阶段 SHOULD 基于 `total/page/page_size` 推导，或在 OpenSpec 阶段扩展响应字段。
- 商品卡片 SHOULD 使用已有字段映射：`product_id`、`product_name`、`sku_code`、`cover_image`、`specification`、`category_name`、`price_display`。
- 小程序端 MUST 使用后端受控 `/media/` 地址规范化后的可访问图片 URL，不得直连未授权对象存储。

### FR-010 底部 TabBar 调整

- 首页目标 TabBar 为：首页、分类、找砖、收藏、证书。
- 首期若收藏页或证书页未实现，Tab 点击 MUST 安全降级，例如展示“功能建设中”提示或跳转占位页，不得出现白屏或路由错误。
- TabBar 图标 SHOULD 使用统一线性风格，选中态使用品牌金，未选中态使用辅助文字色。
- “我的”Tab SHOULD 从首页目标导航中移除；如仍需用户中心能力，必须另行定义入口和范围。

### FR-011 页面状态与降级

- 首次加载 MUST 展示骨架屏或轻量 loading。
- Banner、商品图或商品模块加载失败时 MUST 模块级降级，不影响其他模块展示。
- 首页没有商品时 MUST 保留搜索、Banner、快捷入口等非商品模块。
- 瀑布流加载失败、无更多数据、空商品均 MUST 有明确状态。
- 图片加载失败 MUST 使用统一深色占位图。

### FR-012 埋点与行为记录

- 首页 SHOULD 记录搜索点击、快捷入口点击、新品商品点击、热销商品点击、全部产品曝光/点击、瀑布流加载、加载失败等行为。
- 收藏视觉点击如存在，SHOULD 记录前端交互事件，但不得作为持久化收藏事实。
- 证书 Tab 点击 SHOULD 记录事件，用于判断是否需要优先建设证书聚合页。
- 埋点 MUST 不记录聊天内容、Authorization、Cookie、手机号等敏感信息。

## 6. UI / UE 约束

- UI MUST 以附件 `prototype.png` 和 `prototype.html` 为视觉基准。
- 首页 MUST 保持高端、克制、真实可开发的瓷砖展厅气质。
- 快捷入口图标 MUST 风格统一，不混用彩色、面性和不同粗细图标。
- 商品主图 SHOULD 展示真实瓷砖纹理或空间图，不使用与瓷砖无关的装饰图。
- 320 到 430 pt 宽度范围内不得出现页面级横向滚动、明显重叠、严重截断或 TabBar 遮挡。
- 主要可点击区域 MUST 不小于 44x44 pt。
- 禁止出现 3D 实景体验、免费设计服务、全国门店等本需求未定义能力。

## 7. 数据与字段要求

### 7.1 首页商品卡片字段

| 字段 | 用途 | 首期要求 |
|---|---|---|
| `product_id` | 商品详情跳转 | 必需 |
| `product_name` | 商品名称 | 必需 |
| `sku_code` | SKU 编号 | 必需 |
| `cover_image` | SKU 主图 | 必需 |
| `specification` | 规格 | 必需 |
| `category_name` | 空间/类目展示 | 建议 |
| `price_display` | 价格或待维护文案 | 必需 |
| `is_new` | 新品模块筛选 | 建议 |
| `is_hot` | 热销模块筛选 | 建议 |

### 7.2 瀑布流分页字段

| 字段 | 用途 | 首期要求 |
|---|---|---|
| `items` | 当前页商品 | 必需 |
| `page` | 当前页码 | 必需 |
| `page_size` | 每页数量 | 必需 |
| `total` | 商品总数 | 必需 |
| `has_more` | 是否还有下一页 | 建议；若缺失可由前端推导 |
