# REQ-0062 业务流程

## 1. 总体流程

```text
运营进入 Banner 管理
  |
  +-- 列表加载
  |     |
  |     +-- 后端仅返回小程序范围 Banner
  |     +-- summary / 分页 / 筛选仅统计小程序范围
  |
  +-- 新增或编辑 Banner
  |     |
  |     +-- 展示端固定为“小程序”
  |     +-- 展示位置选择：首页轮播 / 品牌列表页轮播
  |     +-- 配置图片、跳转类型、排序、有效期
  |     +-- 若跳转类型为品牌详情：搜索并选择品牌，可使用品牌 Logo 或自定义上传图片
  |     +-- 后端校验合法组合
  |     +-- 保存草稿或更新
  |
  +-- 上线 / 下线 / 删除
        |
        +-- 状态变更走二次确认
        +-- 删除仍遵守已上线需先下线规则
```

## 2. 旧数据删除流程

```text
实现阶段迁移或清理脚本启动
  |
  +-- 查找 banners 中不在新范围的数据
  |     |
  |     +-- display_client != 小程序
  |     +-- 或 position 不属于首页轮播 / 品牌列表页轮播
  |
  +-- 删除匹配 Banner 业务记录
  |
  +-- 记录删除条件和删除数量
  |
  +-- 验证列表 API / summary / 小程序查询均不再返回旧数据
```

边界：

- 删除的是 Banner 业务记录，不直接删除 MinIO 对象。
- 可明确映射为小程序首页轮播的历史小程序首页 Banner 可保留。
- Web 首页、专题页、首页中部运营位和未知枚举均删除。

## 3. 小程序查询流程

```text
小程序首页
  |
  +-- 请求首页聚合数据
  +-- 后端查询：首页轮播
  +-- 返回在线且有效期内 Banner

品牌列表页
  |
  +-- 请求品牌列表页数据或轮播数据
  +-- 后端查询：品牌列表页轮播
  +-- 返回在线且有效期内 Banner
```

降级策略：

- 首页无轮播数据：沿用首页既有降级策略，不白屏。
- 品牌列表页无轮播数据：隐藏轮播或展示品牌化兜底，不使用首页轮播兜底。

## 4. 品牌详情跳转流程

```text
运营新增或编辑 Banner
  |
  +-- 跳转类型选择：品牌详情
  |
  +-- 搜索并选择已启用品牌
  |
  +-- 可点击“使用品牌 Logo”
  |     |
  |     +-- 品牌有 Logo：写入品牌 logo_object_key 作为 Banner 图片
  |     +-- 品牌无 Logo：提示自定义上传
  |
  +-- 保存 Banner
  |
  +-- 小程序轮播点击
        |
        +-- 进入 pages/brand-detail/index?brandId=...
```

边界：

- 品牌详情跳转只使用 `brand_id`，不得同时携带 SKU、专题或外链目标。
- 品牌必须存在且已启用。
- 使用品牌 Logo 不改变品牌本身资料；Banner 也可使用自定义上传运营图。

## 5. 与父需求差异

| 项目 | REQ-0016 原范围 | REQ-0062 调整后 |
|---|---|---|
| 展示端 | Web 首页、小程序首页、专题页 | 小程序 |
| 展示位置 | 多个端下多个位置 | 首页轮播、品牌列表页轮播 |
| 旧数据 | 可继续存在 | 不在新范围内的旧 Banner 删除 |
| 小程序品牌页 | 可复用首页轮播 | 独立品牌列表页轮播 |
| 品牌跳转 | 未覆盖品牌详情跳转 | Banner 可跳转品牌详情 |
| 实现重点 | 新增 Banner 管理能力 | 收敛枚举、清理数据、拆分查询 |

## 6. 依赖与风险

| 风险 | 处理方式 |
|---|---|
| 枚举命名不统一 | OpenSpec 阶段确定最终枚举，并同步 API、DB、Orval。 |
| 旧数据删除误伤 | 删除条件只匹配 Banner 业务记录，不物理删除媒体对象。 |
| 品牌列表页仍复用首页轮播 | 实现任务必须新增或调整品牌列表页轮播查询。 |
| 管理端 UI 与后端校验不一致 | 验收同时覆盖前端选项和后端非法枚举拒绝。 |
| 品牌 Logo 缺失 | 品牌详情 Banner 必须允许自定义上传兜底。 |
