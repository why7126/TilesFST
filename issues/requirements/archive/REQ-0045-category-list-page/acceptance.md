# 验收清单

## 功能 AC

- [ ] AC-001 分类页作为微信小程序底部 TabBar「分类」一级频道存在，页面路径为 `pages/category/index`。
- [ ] AC-002 页面展示一级分类 + 二级分类两级结构，不在本页展示商品卡片、价格、收藏按钮或筛选排序栏。
- [ ] AC-003 一级分类在左侧纵向导航中按后台排序值升序展示，排序相同时按创建时间升序展示。
- [ ] AC-004 一级分类超过可视区域时左栏可独立滚动，切换分类后左栏滚动位置不回到顶部。
- [ ] AC-005 当前一级分类同时使用品牌金指示条、深色选中背景和高亮文字表达，不仅依赖颜色区分。
- [ ] AC-006 右侧顶部展示当前一级分类名称，切换一级分类后右侧内容滚动到顶部。
- [ ] AC-007 二级分类按三列宫格展示，卡片仅展示分类名称，名称最多一行并超长省略。
- [ ] AC-008 二级分类不展示图片、商品数量、简介、价格或运营 Banner。
- [ ] AC-009 分类列表页不得消费二级分类封面图，也不得自动取第一款商品主图替代分类展示图。
- [ ] AC-010 仅展示 `status=enabled` 的一级和二级分类；分类下架后刷新页面不再展示。
- [ ] AC-011 点击二级分类整张卡片跳转 `pages/product-list/index?categoryId={secondaryCategoryId}&categoryName={encodedName}`。
- [ ] AC-012 二级分类点击在 300ms 内防重复触发；跳转失败时提示“页面打开失败，请重试”。
- [ ] AC-013 从分类商品列表或商品详情返回分类页后，恢复当前一级分类、左侧滚动位置和右侧滚动位置。
- [ ] AC-014 分类列表页不展示店铺 Logo/Header 模块、搜索框或搜索跳转入口。
- [ ] AC-015 页面首次加载展示与最终布局一致的骨架屏，左栏和右栏分别有深色骨架块，避免整体闪动或亮白闪烁。
- [ ] AC-016 当前一级分类没有二级分类时，右侧保留标题并展示“该分类暂未配置二级分类”，不得自动跳转其他一级分类。
- [ ] AC-017 二级分类不渲染图片资源，因此不存在图片加载失败占位或破图状态；分类名称和点击能力保持可用。
- [ ] AC-018 无缓存且网络异常时展示错误说明和“重新加载”按钮；有缓存时继续显示缓存并静默刷新。
- [ ] AC-019 类目接口一次返回两级可用类目，支持数据版本号；切换一级分类不得重新请求接口。
- [ ] AC-020 类目数据本地缓存建议 24 小时，进入页面时通过版本号静默刷新，更新失败不得清空已展示缓存。
- [ ] AC-021 分类列表页不得加载二级分类封面缩略图或原始大图。
- [ ] AC-022 正常 4G 环境下页面可交互时间不超过 1.5s；类目接口首屏 P95 响应时间不超过 800ms。
- [ ] AC-023 埋点覆盖 `category_page_view`、`primary_category_click`、`secondary_category_click`、`category_load_failed`。

## 视觉与交互 AC

- [ ] AC-024 页面沿用“工业石材 · 暗色旗舰风”，与小程序首页保持统一的深色高端视觉。
- [ ] AC-025 页面背景、内容面板、主文字、次文字、品牌金、分割线符合 requirement.md §11 的视觉验收约束，不使用纯黑或纯白作为大面积背景。
- [ ] AC-026 页面标题、左右双栏分类主体、底部 TabBar 顺序和选中态与首页深色视觉一致。
- [ ] AC-027 页面标题固定在分类主体上方；左右两栏分别滚动；底部 TabBar 始终固定。
- [ ] AC-028 一级分类按钮高度在 56-60px；二级分类卡片可访问名称为 `{分类名称}`，不得依赖图片替代文本。
