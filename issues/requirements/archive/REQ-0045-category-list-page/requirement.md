# 菲尚特瓷砖微信小程序「分类列表页」需求文档

- 文档版本：v1.1
- 页面标识：`pages/category/index`
- 页面入口：底部 TabBar「分类」
- 关联页面：首页、分类商品列表页、商品详情页

## 1. 需求背景

菲尚特瓷砖微信小程序定位为面向装修客户、设计师和门店访客的数字瓷砖展厅。用户需要通过清晰、低学习成本的类目结构快速定位目标瓷砖品类，再进入对应分类商品列表进行浏览、筛选和查看详情。

本页面是 TabBar 一级频道，不承担商品编辑、库存管理、订单交易等后台能力。

## 2. 页面目标

1. 清晰展示一级、二级类目关系。
2. 在一级类目数量超过 10 个时仍可快速切换。
3. 通过简洁的二级分类名称宫格降低浏览成本。
4. 用户点击二级分类后直接进入对应分类商品列表页。
5. 与现有首页保持统一的深色高端视觉体系。

## 3. 已确认产品决策

| 决策项 | V1 方案 |
|---|---|
| 页面定位 | TabBar「分类」= 分类列表页 |
| 类目层级 | 一级分类 + 二级分类 |
| 一级分类布局 | 左侧固定纵向导航 |
| 一级分类数量 | 10 个以上，支持纵向滚动 |
| 一级分类样式 | 纯文字，不展示图标或数量 |
| 二级分类样式 | 纯文字名称卡片 |
| 二级分类布局 | 三列宫格 |
| 二级分类图片 | 分类列表页不展示图片 |
| 二级分类排序 | 严格按照后台排序值 |
| 二级分类数量 | 不显示商品数量 |
| 二级分类简介 | 不展示 |
| 右侧标题 | 展示当前一级分类名称 |
| 点击二级分类 | 进入分类商品列表页 |
| 卡片状态 | 无持续选中态，仅提供系统点击反馈 |
| 左侧滚动位置 | 切换分类时保持当前位置 |
| 一级分类搜索 | V1 不支持 |
| 热门分类 | V1 不上线，预留后续扩展能力 |

## 4. 用户故事

### US-01 浏览一级分类

作为访客，我希望通过左侧导航查看全部一级分类，以便快速找到目标品类。

**验收标准：**
- 一级分类按后台排序展示。
- 超出可视区域时可独立纵向滚动。
- 当前分类使用品牌金色指示条、深色选中背景和高亮文字。
- 切换一级分类后，左侧滚动位置不回到顶部。

### US-02 浏览二级分类

作为访客，我希望看到当前一级分类下的二级分类名称，以便快速理解品类。

**验收标准：**
- 右侧顶部展示当前一级分类名称。
- 二级分类采用三列宫格。
- 名称最多展示一行，超长使用省略号。
- 不展示图片、商品数量和分类简介。

### US-03 进入分类商品列表

作为访客，我希望点击二级分类后进入对应商品列表，以便继续筛选和浏览商品。

**验收标准：**
- 整个二级分类卡片均可点击。
- 点击后跳转 `pages/product-list/index?categoryId={id}`。
- 点击期间提供原生按压反馈，不保留选中状态。
- 连续点击需防抖，避免重复打开页面。

## 6. 数据模型

```json
{
  "primaryCategories": [
    {
      "id": "floor",
      "name": "地面砖",
      "sort": 10,
      "status": "enabled",
      "children": [
        {
          "id": "polished-marble",
          "name": "通体大理石",
          "coverUrl": "https://.../cover.jpg",
          "sort": 10,
          "status": "enabled"
        }
      ]
    }
  ]
}
```

约束：
- 仅展示 `status=enabled` 的分类。
- 一级、二级分类分别按 `sort ASC` 排序；排序相同时按创建时间升序。
- `coverUrl` 为接口兼容字段；分类列表页不消费、不渲染二级分类图片。
- 前端不得自动取第一款商品主图替代分类展示图。

## 7. 接口建议

```http
GET /api/miniapp/categories/tree?depth=2
```

返回：

```json
{
  "code": 0,
  "data": {
    "version": "20260718-01",
    "items": []
  }
}
```

接口要求：
- 一次返回两级可用类目，避免每次切换一级分类重新请求。
- 返回类目数据版本号，支持本地缓存更新判断。
- 若接口返回 `coverUrl`，应使用安全公开 URL，不暴露原始 object key；分类列表页不加载该资源。

## 8. 状态与异常

### 8.1 首次加载
- 展示与最终布局一致的骨架屏。
- 一级和二级区域分别使用骨架块，避免整体闪动。

### 8.2 一级分类无二级分类
- 右侧保留一级分类标题。
- 展示空状态：“该分类暂未配置二级分类”。
- 不自动跳转其他一级分类。

### 8.3 二级分类图片
- 分类列表页不渲染二级分类图片。
- 不存在图片加载失败占位或破图状态。
- 分类名称和点击能力保持可用。

### 8.4 网络异常
- 首屏加载失败展示重试状态。
- 有缓存时优先展示缓存，并静默请求最新数据。
- 更新失败不清空已展示缓存。

### 8.5 分类被下架
- 当前页刷新后移除该分类。
- 若当前选中的一级分类被移除，默认选中新的第一个可用一级分类。

## 9. 性能要求

- 类目接口首屏 P95 响应时间不超过 800ms。
- 页面可交互时间不超过 1.5s（正常 4G 环境）。
- 类目数据本地缓存建议 24 小时，进入页面时通过版本号静默刷新。
