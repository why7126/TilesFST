# 菲尚特瓷砖微信小程序「分类列表页」交互说明

## 1. 页面进入

1. 用户点击 TabBar「分类」进入页面。
2. 页面优先读取本地缓存类目数据并渲染。
3. 默认选中后台排序第一的可用一级分类。
4. 后台请求完成后，若数据版本变化，则无闪动更新页面。
5. 页面不记忆跨会话的一级分类选中项；同一次页面驻留期间保持当前选中项。

## 2. 一级分类切换

- 左侧一级分类区域独立纵向滚动。
- 点击任一一级分类：
  1. 立即更新左侧选中态；
  2. 更新右侧标题；
  3. 将右侧内容滚动到顶部；
  4. 渲染对应二级分类；
  5. 左侧自身滚动位置保持不变。
- 不进行页面跳转，不重新请求接口。
- 选中态：左侧 3px 品牌金指示条 + `#211E16` 背景 + 主文字色。

## 3. 二级分类浏览

- 采用三列宫格，纵向滚动。
- 每个卡片包含 1:1 图片和单行名称。
- 卡片整块为点击热区，热区不得小于视觉卡片范围。
- 按压反馈：透明度降低或缩放至 0.96，时长约 120～180ms。
- 不显示持续选中态、勾选态或描边高亮。

## 4. 二级分类跳转

点击卡片后：

```text
pages/product-list/index
  ?categoryId={secondaryCategoryId}
  &categoryName={encodedName}
```

规则：
- 300ms 内禁止重复触发。
- 跳转失败时 Toast 提示“页面打开失败，请重试”。
- 返回分类页后恢复：
  - 当前一级分类；
  - 左侧滚动位置；
  - 右侧滚动位置。

## 5. 搜索入口

- 搜索框为跳转入口，不在当前页直接唤起键盘。
- 点击进入全局搜索页。
- 搜索范围包括瓷砖名称、型号、规格和品牌。
- 搜索框样式、圆角、按钮和占位文案与首页保持一致。

## 6. 滚动规则

- 页面头部、搜索框和页面标题固定在分类主体上方。
- 左右两栏分别滚动，互不抢占滚动位置。
- 当手指从左栏开始滑动，仅滚动左栏；从右栏开始滑动，仅滚动右栏。
- 禁止左右栏整体随页面滚出视口，底部 TabBar 始终固定。
- iOS 回弹效果使用系统默认，但不得造成底部露白。

## 7. 页面恢复

从分类商品列表或商品详情返回时：
- 不重新初始化当前一级分类。
- 保持左右栏滚动位置。
- 缓存数据仍有效时不显示骨架屏。
- 若后台类目版本变化，可在恢复后静默更新；当前分类失效时回退至第一个可用分类。

## 8. 加载状态

### 骨架屏
- 左栏展示 8～10 个文字骨架。
- 右栏展示标题骨架和 3×3 图片宫格骨架。
- 骨架背景仅使用深色层级，不使用亮白闪烁。

### 空状态
- 保留当前一级分类标题。
- 中部展示弱化图形和文案。
- 不提供“联系商家”或“提交需求”等额外入口。

### 错误状态
- 无缓存：展示错误图形、说明和“重新加载”按钮。
- 有缓存：继续显示缓存，不阻断用户操作；可轻提示“网络异常，已展示缓存内容”。

## 9. 图片规则

- 使用 `aspectFill`，固定 1:1。
- 图片懒加载。
- 加载前使用 `#29261D` 占位背景。
- 失败后使用统一的深灰纹理占位图。
- 禁止拉伸图片或根据原图比例改变卡片高度。

## 10. TabBar

- 当前「分类」Tab 使用品牌金色高亮。
- 其他 Tab 使用次要文字色。
- TabBar 顺序固定：首页、分类、找砖、收藏、证书。
- 点击当前 Tab：右侧列表回到顶部；若已在顶部则不执行额外动作。

## 11. 无障碍与触控

- 一级分类按钮高度 56～60px。
- 二级分类名称与图片共同作为可访问名称。
- 图片需配置替代文本：`{分类名称}分类封面`。
- 文字与背景对比度应满足移动端可读性要求。
- 不仅依赖颜色表达一级分类选中状态，必须同时使用左侧指示条。
