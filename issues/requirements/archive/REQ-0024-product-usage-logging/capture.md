# 一句话

需要记录用户在产品使用过程中的行为埋点数据，并存储接口请求日志及日志详情，便于后续追踪使用情况、排查问题和审计关键操作。

# 原始描述

需要记录用户在产品使用上的行为埋点数据，存储接口请求日志以及日志的详情。

# 背景与关联

- 涉及端：Web 管理端、店主 Web 展示端、微信小程序、后端 API
- 涉及能力：用户行为埋点、接口请求日志、日志详情存储与后续查询
- 业务价值：帮助团队了解功能使用情况，定位接口异常与用户操作路径，并为关键操作审计提供基础数据

# 待澄清

- [ ] 行为埋点覆盖哪些事件：页面访问、按钮点击、搜索筛选、表单提交、媒体上传、登录退出或其他关键路径
- [ ] 接口请求日志需要记录哪些字段：请求路径、方法、状态码、耗时、操作者、客户端、IP、User-Agent、请求参数摘要、响应摘要或错误详情
- [ ] 日志详情中是否允许保存请求体/响应体，以及敏感字段脱敏、截断、保留周期和访问权限边界
- [ ] 日志查看入口放在管理端哪个模块，是否需要筛选、分页、导出和详情页
- [ ] 小程序与 Web 展示端的匿名用户行为如何标识，是否需要会话 ID 或设备标识
- [ ] 日志数据量增长后的归档、清理、索引与生产 MySQL 兼容策略
