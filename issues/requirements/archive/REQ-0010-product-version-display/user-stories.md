## US-001 管理员在后台快速识别产品版本

**作为** 后台管理员或内部员工，  
**我希望** 在管理端侧边栏顶部产品名称旁看到当前产品版本号，  
**以便** 报障或沟通时能准确说明正在使用的系统版本。

### 验收要点

- 登录后任意 `/admin/*` 页面，侧边栏顶部可见 `TILESFST` + 版本 pill（如 `v0.0.1`）。
- 版本 pill 位于产品名称右侧、同一视觉组，参照 SoulKing 参考图布局。
- 不展示 API / 后端版本号。
- 导航、用户菜单布局无回归。

### 关联功能

- FR-001、FR-002、FR-004（见 `requirement.md`）

---

## US-002 店主在目录页识别产品版本

**作为** 瓷砖零售店店主，  
**我希望** 在店主端侧边栏顶部看到与管理端一致的产品版本号，  
**以便** 与平台方沟通时能快速对齐系统版本。

### 验收要点

- `LandingPage` / `ListPage` 等带筛选侧栏的页面，侧栏最上方可见品牌名 + 版本 pill。
- 版本值与管理端一致（共用单一常量）。
- 版本展示在侧栏内，而非仅顶栏 `SiteNav`。
- 筛选区、商品列表布局无回归。

### 关联功能

- FR-001、FR-003、FR-004（见 `requirement.md`）

---

## US-003 发布时人工维护产品版本

**作为** 发布负责人，  
**我希望** 产品版本号由单一常量人工维护，  
**以便** 发版时有明确、可控的版本标识，且不依赖构建流水线自动递增。

### 验收要点

- 版本定义位于 `src/shared/` 单一模块，发版时手动更新。
- 不读取 `package.json`、FastAPI `version` 或 Git 信息。
- 更新常量后，管理端与店主端展示同步变化。
- release checklist 含「更新产品版本常量」检查项。

### 关联功能

- FR-001、FR-005（见 `requirement.md`）
