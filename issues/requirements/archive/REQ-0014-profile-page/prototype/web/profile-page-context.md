# REQ-0014 个人资料页面原型上下文工程 v1

## 0. 关联与 API（req-complete 定稿）

- **REQ-ID**：REQ-0014-profile-page
- **路由**：`/admin/profile`
- **API**：`GET/PATCH /api/v1/profile/me`；`GET /api/v1/profile/me/activities`（limit **5**，v1.1）
- **修改密码**：打开 REQ-0015 弹窗（非独立页面）
- **保存成功**：inline `save-tip`「资料已更新」（非 toast）

## 1. 原型目标

本文件用于指导 Cursor / AI 前端开发还原「个人资料」页面。开发时应优先参考 `prototype/web/profile-page.html`；`prototype/images/profile-page.png` 为 Golden Reference。

## 2. 设计继承关系

- 继承 `ui-design.md` 的工业石材暗色旗舰风。
- 继承后台管理页面左侧导航结构与底部用户菜单。
- 视觉密度、按钮、输入框、卡片、表格细线风格与 SKU 管理页保持一致。

## 3. 页面布局

```text
左侧固定导航 264px
右侧主内容区：
  顶部 Page Head
  两列内容区：
    左侧 1fr：个人资料主卡片
    右侧 360px：账号安全卡片 + 操作记录卡片
```

## 4. 左侧用户菜单

底部用户菜单需要展开，结构如下：

```text
Admin User / admin@tilesfst.com
┌─────────────┐
│ 个人资料     │ 高亮
│ 密码修改     │
├─────────────┤
│ 退出登录     │ danger/弱强调
└─────────────┘
```

## 5. 个人资料主卡片

字段顺序：

```text
头像区域
用户名（只读）
昵称（可编辑）
联系邮箱（可编辑）
手机号码（可编辑）
备注说明（可编辑，max 200）

> 所属角色、账号状态 **不在** 主卡片表单重复展示，仅在 §6 账号安全卡片展示（BUG-0022 UX 定稿）。
```

按钮：

- 重置：幽灵按钮
- 保存修改：品牌金实底主按钮

## 6. 右侧信息卡片

账号安全卡片：

- 登录账号
- 账号状态
- 所属角色
- 最后登录
- 修改密码按钮 → REQ-0015 modal

操作记录卡片：

- 2026-06-16 10:24 资料更新
- 2026-06-15 21:08 登录后台
- 2026-06-12 09:35 头像更新
