# 用户故事

## US-001 管理员跨列表页保持一致操作心智

作为企业内部管理员，  
我希望 SKU、品牌、类目、规格、Banner、用户、日志和接口文档等管理端列表页使用一致的页面结构、筛选行为、表格操作列和分页，  
以便我在不同页面之间切换时不需要重新学习操作方式。

验收要点：

- 页面模块顺序稳定为「标题模块 → 指标卡模块 → 筛选/搜索模块 → 列表模块」。
- 筛选变化、重置、分页切换的交互一致。
- 表格最后一列操作在横向滚动时保持可见。

## US-002 前端开发复用列表页模板

作为前端开发人员，  
我希望新增管理端列表页时可以复用 `AdminListPage` 模板或等价组合，  
以便减少重复实现筛选区、分页、表格卡片和操作列导致的 DOM 漂移。

验收要点：

- 模板输入可以描述标题、主操作、指标卡、筛选项、表格列、行操作和分页状态。
- 业务页面仍可保留领域特有列、筛选项和行操作。
- 不需要在每个页面内重复手写分页 DOM、sticky action column 和 fixed toast 约束。

## US-003 QA 使用设计验收页检查回归

作为测试 / QA，  
我希望 `/design-system` 中有 AdminListPage 验收样例和边界态，  
以便在新增列表页或重构列表页时快速检查模块顺序、分页、操作列和状态反馈是否回归。

验收要点：

- 验收页展示标准列表、空态、加载 / 错误态和分页边界。
- 验收页可以作为后续视觉截图或并排对照的基准。
- 验收页说明受 BUG-0055 影响的 8 个页面矩阵。

## US-004 产品负责人沉淀 BUG-0055 经验

作为产品负责人，  
我希望 BUG-0055 的修复经验沉淀为模板契约和验收门禁，  
以便未来不再通过逐页修复来维持管理端列表一致性。

验收要点：

- 需求文档、验收清单和 trace 显式关联 BUG-0055。
- 横切 AC 引用知识库 `admin-list-page-consistency.md`。
- 后续 `/req-opsx` 的 design 能够复用本需求的 knowledge_base_refs。

## US-005 运营人员避免误操作与布局跳动

作为企业内部运营人员，  
我希望列表页的启停、上架、删除、重置密码等危险操作保持确认弹窗，且操作反馈不推挤页面布局，  
以便我在高频维护数据时减少误操作和视觉干扰。

验收要点：

- 状态变更、删除、重置密码等危险操作必须使用 Design System confirm modal。
- 不允许使用 `window.confirm` 或 `window.alert`。
- 操作反馈使用 fixed toast，不导致 hero、筛选区、表格或分页纵向位移。
