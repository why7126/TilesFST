# 用户故事

## US-001 后台管理员识别系统品牌

作为后台管理员，我希望进入管理端后在左侧品牌区看到菲尚特 Logo、`菲尚特FST`、当前版本号与 `家居建材资料库`，以便确认自己正在操作正确的企业资料库。

验收要点：

- Sidebar 顶部品牌区在展开态完整展示 Logo、主标题、版本号和副标题。
- 文案必须分别为 `菲尚特FST` 与 `家居建材资料库`。
- 品牌区不影响当前导航菜单识别与点击。

## US-002 后台员工在浏览器标签中识别系统

作为后台员工，我希望浏览器标签页显示菲尚特 Logo 图标，以便在多个后台页面或多个系统标签之间快速找到本系统。

验收要点：

- favicon 不再是框架默认图标。
- 标签图标使用菲尚特 Logo 或由其派生的清晰小尺寸图标。

## US-003 维护人员保留已有版本号与侧栏交互

作为项目维护人员，我希望品牌区改造只替换品牌露出，不破坏现有产品版本号来源和侧栏展开/收起逻辑，以便后续升级版本时不需要重复改品牌布局。

验收要点：

- 版本号继续从现有常量或组件来源读取。
- 展开/收起按钮仍保持可点击、可访问、无重叠。
- 本需求不引入 API、数据库和 Orval 变更。

## US-004 视觉验收人员对照原型确认还原度

作为视觉验收人员，我希望有明确的原型参考与差异说明，以便确认品牌区布局与上传图片、原型图和最新用户文案保持一致。

验收要点：

- 原型承载页仅用于验证 Sidebar 品牌区。
- 若原型旧文案与本需求冲突，以本需求中的 `家居建材资料库` 为准。
