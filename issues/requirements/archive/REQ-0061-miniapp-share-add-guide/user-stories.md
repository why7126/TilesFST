# REQ-0061 小程序添加到我的小程序引导语用户故事

## US-001 进入小程序时看到添加引导

作为装修业主，我希望进入小程序时看到一条简短提示，知道可以通过右上角入口把小程序添加到“我的小程序”，以便下次更快找回。

验收要点：

- 用户进入小程序后可看到“添加到我的小程序”含义明确的引导语。
- 引导语靠近右上角微信原生菜单 / 分享入口，但不遮挡系统胶囊。
- 引导语不阻塞首页搜索、分类、商品推荐等主要浏览内容。
- 引导语展示失败时不影响页面主功能。

## US-002 用户可以手工关闭引导

作为小程序访客，我希望不需要这条提示时可以手工关闭，以便继续专注浏览瓷砖内容。

验收要点：

- 引导语提供可识别的关闭入口。
- 点击关闭后，引导语立即隐藏。
- 关闭入口不与微信右上角系统胶囊重叠。
- 关闭状态至少在当前会话内有效；更长记忆周期待评审确认。

## US-003 不同入口进入时体验稳定

作为门店销售，我希望客户从扫码、分享卡片或普通入口进入小程序时，引导语都不会遮挡关键内容，以便演示过程稳定。

验收要点：

- 普通启动、扫码进入、分享进入时页面不因引导语错位而出现遮挡。
- 加载态、空状态、错误态下引导语不遮挡提示和重试入口。
- 获取胶囊位置信息失败时使用安全降级策略或不展示引导。

## US-004 避让自定义导航与微信原生胶囊

作为开发 / 测试人员，我希望引导语复用小程序导航安全区和胶囊避让规则，以便后续实现不会破坏现有自定义导航。

验收要点：

- 引导语位置基于状态栏、菜单按钮和胶囊 reserve 信息计算。
- 首页品牌导航、非首页标题、返回按钮、搜索入口均不被引导语遮挡。
- 禁止自绘模拟微信分享按钮、更多按钮、关闭按钮或系统胶囊。
- 320、375、430 pt 视口下无横向滚动和明显重叠。

## US-005 后续可评估引导效果

作为产品负责人，我希望后续可以知道用户是否看见并关闭这条引导，以便判断提示是否有效或是否需要调整频率。

验收要点：

- 本期不强制新增埋点接口。
- 若后续接入埋点，应至少区分展示、关闭和相关入口点击意向事件。
- 埋点不得记录用户敏感信息。
- 若引入 API 或 DB 变化，必须另行通过 OpenSpec、API、DB 文档和测试同步。
