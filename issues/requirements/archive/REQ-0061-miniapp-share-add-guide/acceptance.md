# REQ-0061 小程序添加到我的小程序引导语验收标准

## 1. 功能 AC

- [ ] AC-001 用户进入小程序后，可看到表达“添加到我的小程序，方便下次找回”含义的引导语。
- [ ] AC-002 引导语靠近右上角微信原生菜单 / 分享入口，但不遮挡系统胶囊。
- [ ] AC-003 引导语以轻提示、气泡或等价形式呈现，不阻塞首页搜索、分类、Banner、商品卡片等主内容。
- [ ] AC-004 引导语文案短句可读，在 320 到 430 pt 宽度下不出现明显截断、横向滚动或重叠。
- [ ] AC-005 页面加载失败或无法获取胶囊位置时，引导语安全降级隐藏或使用安全兜底位置，不影响页面主功能。

## 2. 手工关闭 AC

- [ ] AC-CLOSE-001 引导语提供清晰可识别的关闭入口。
- [ ] AC-CLOSE-002 用户点击关闭入口后，引导语立即隐藏。
- [ ] AC-CLOSE-003 关闭入口热区不小于 32x32 pt，且不与微信原生胶囊重叠。
- [ ] AC-CLOSE-004 关闭后至少当前会话内不再展示；若评审确认当天或长期记忆，实现和验收必须同步记录。
- [ ] AC-CLOSE-005 本地关闭状态写入失败时，页面主功能不受影响。

## 3. 展示频率与入口 AC

- [ ] AC-FREQ-001 未关闭状态下，普通启动进入小程序时展示引导语。
- [ ] AC-FREQ-002 未关闭状态下，扫码或分享卡片进入时引导语不遮挡当前页面核心内容。
- [ ] AC-FREQ-003 已关闭且关闭记忆仍有效时，不重复展示引导语。
- [ ] AC-FREQ-004 引导语不得在授权弹窗、系统弹窗、关键加载或错误重试区域上方抢占焦点。
- [ ] AC-FREQ-005 展示频率最终策略需在评审记录中确认，避免“每次进入”与“允许关闭”互相冲突。

## 4. 微信原生胶囊与自定义导航 AC

- [ ] AC-WX-001 引导语位置基于微信小程序状态栏、菜单按钮或项目统一 fallback 计算。
- [ ] AC-WX-002 引导语容器、箭头、关闭按钮不得进入微信原生分享 / 更多 / 关闭胶囊区域。
- [ ] AC-WX-003 页面不得在 WXML / WXSS 中自绘模拟分享按钮、更多按钮、关闭按钮或系统胶囊。
- [ ] AC-WX-004 首页品牌 Logo、门店名称、搜索入口、Banner 不被引导语遮挡。
- [ ] AC-WX-005 非首页返回按钮、页面标题、搜索入口和内容起点不被引导语遮挡。
- [ ] AC-WX-006 320、375、430 pt DevTools 视口下，胶囊 reserve、引导语和页面首屏内容无重叠。

## 5. API / 数据 / 安全 AC

- [ ] AC-API-001 本需求默认不新增 API；关闭状态优先使用小程序本地存储。
- [ ] AC-API-002 本需求默认不新增或修改数据库表、字段和迁移。
- [ ] AC-API-003 本需求默认不需要 Orval；若后续新增服务端配置或埋点接口，必须同步 OpenAPI、Orval、docs 和测试。
- [ ] AC-API-004 若后续接入埋点，不得记录手机号、地址、微信身份标识、Authorization header、Cookie、`.env` 或真实客户数据。
- [ ] AC-API-005 引导语展示失败、关闭状态写入失败或本地存储不可用时，不得阻断商品浏览、搜索、分享和返回。

## 6. 测试与验证 AC

- [ ] AC-TEST-001 后续实现阶段必须补充小程序静态测试或等价检查，确认没有自绘系统胶囊和伪分享按钮。
- [ ] AC-TEST-002 后续实现阶段必须记录 DevTools 320、375、430 pt 视口 evidence，覆盖引导位置、关闭按钮和首屏遮挡。
- [ ] AC-TEST-003 后续实现阶段必须按条件记录真机 evidence；若真机不可用，必须标记 blocked 或 follow_up，不得写作真机通过。
- [ ] AC-TEST-004 后续实现阶段必须验证普通启动、扫码进入、分享进入至少一种直达场景下引导不遮挡关键内容。
- [ ] AC-TEST-005 若 API / DB / Orval 未变化，验收报告需明确标记 N/A；若发生变化，必须补充对应同步记录。
- [ ] AC-TEST-006 必须验证关闭后当前会话内不重复展示。

## 7. 范围控制 AC

- [ ] AC-SCOPE-001 本需求不修改微信原生菜单、分享能力或系统胶囊行为。
- [ ] AC-SCOPE-002 本需求不新增后台配置页面。
- [ ] AC-SCOPE-003 本需求不新增 Web 展示端或企业管理端能力。
- [ ] AC-SCOPE-004 本需求不包含营销活动弹窗、优惠券、用户画像定向展示。
- [ ] AC-SCOPE-005 本需求不创建 OpenSpec Change；后续开发前必须先通过 `/req-review` 和 `/req-opsx`。

## 横切 AC（knowledge-base）

本 REQ 为微信小程序留存引导 UI，不命中 `req-complete` 规定的 `admin-list`、`admin-form`、`admin-modal`、`media-upload` 标签，因此无管理端 `AC-XCUT-*` 横切验收项。

小程序导航相关参考：`docs/knowledge-base/best-practices/miniapp-custom-navigation.md`。后续实现必须按该文档完成状态栏、胶囊 reserve、页面 offset、DevTools / 真机 evidence 和敏感信息边界验收。
