# REQ-0061 小程序添加到我的小程序引导语业务流程

## 1. 总流程

```text
用户进入微信小程序
  |
  v
页面初始化并获取状态栏 / 胶囊位置信息
  |
  v
读取引导关闭状态
  |
  +-- 已关闭且仍在记忆有效期内
  |     |
  |     +-- 不展示引导语
  |
  +-- 未关闭或记忆已失效
        |
        v
      在右上角入口附近展示添加引导语
        |
        +-- 用户关闭：立即隐藏，并记录关闭状态
        |
        +-- 用户继续浏览：引导语不阻塞页面主功能
```

## 2. 页面与入口流程

```text
普通启动 / 扫码进入 / 分享卡片进入
  |
  v
识别当前页面形态
  |
  +-- 首页：靠近品牌导航右上角展示，避让搜索和 Banner
  |
  +-- 非首页：避让返回按钮、页面标题和原生胶囊
  |
  +-- 异常状态：加载失败 / 空状态 / 网络错误
        |
        +-- 优先保证错误提示和重试入口可见
        +-- 引导语可降级隐藏
```

## 3. 关闭状态流程

```text
用户点击关闭入口
  |
  v
引导语立即隐藏
  |
  v
写入本地关闭状态
  |
  +-- 写入成功：按确认后的会话 / 日期 / 长期策略控制再次展示
  |
  +-- 写入失败：保持当前页面隐藏；下次进入可按默认规则再展示
```

## 4. 与关联需求关系

| 关联项 | 复用点 | 差异 |
|---|---|---|
| `REQ-0041-miniapp-home` | 首页作为首期主要进入场景，需避让品牌导航和首屏内容。 | 本需求只新增添加引导，不改变首页模块结构。 |
| `REQ-0048-miniapp-global-custom-navigation-bar` | 复用状态栏、胶囊 reserve、内容 offset 和禁止伪系统控件规则。 | 本需求不是导航栏改造，只在导航附近增加轻提示。 |
| `REQ-0053-miniapp-custom-navigation-best-practice` | 复用截图矩阵、真机/DevTools evidence、敏感信息边界。 | 本需求新增引导语关闭和展示频率验收。 |

## 5. 风险控制

| 风险 | 控制方式 |
|---|---|
| 引导语遮挡微信原生胶囊 | 使用菜单按钮位置信息和右侧 reserve；验收时检查胶囊可见可用。 |
| 每次进入都展示造成打扰 | 必须支持手工关闭；关闭后至少当前会话不再展示，长期策略评审确认。 |
| 多页面位置不一致 | 优先做共享引导组件或统一定位逻辑；页面接入需遵守同一 offset 规则。 |
| 获取胶囊信息失败导致错位 | 使用项目统一 fallback 或隐藏引导，不影响页面浏览。 |
| 后续埋点扩大范围 | 默认不新增 API / DB；如需服务端统计另走 OpenSpec。 |
