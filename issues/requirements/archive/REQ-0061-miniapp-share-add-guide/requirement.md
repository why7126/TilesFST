# REQ-0061 小程序添加到我的小程序引导语

## 1. 需求背景

用户每次进入微信小程序时，右上角微信原生胶囊区域包含分享、更多等系统入口，但普通用户未必知道可通过该入口将小程序添加到“我的小程序”。为降低用户下次找回小程序的成本，需要在小程序入口场景展示一条靠近右上角入口的引导语，提示用户点击对应系统入口并添加到我的小程序。

本需求只定义小程序端留存引导体验，不改造微信原生菜单能力，不手绘模拟分享按钮或系统胶囊，也不新增 Web、管理端、后端业务管理能力。

## 2. 目标用户

| 用户 | 诉求 |
|---|---|
| 装修业主 | 首次或再次进入小程序时能知道如何收藏小程序，方便下次继续看砖、找门店和查看商品。 |
| 设计师 | 给客户转发或演示后，能引导客户把小程序保留在微信常用入口中。 |
| 门店销售 | 向客户推荐小程序时，减少口头解释“怎么下次再找到”的成本。 |
| 开发 / 测试人员 | 有明确的展示位置、关闭行为、胶囊避让和频率规则，便于后续实现与验收。 |

## 3. 需求目标

- 在微信小程序进入场景提供“添加到我的小程序”引导语。
- 引导语应与右上角微信原生菜单 / 分享入口形成视觉关联，但不得遮挡或模拟系统胶囊。
- 用户必须可以手工关闭引导语。
- 引导展示与关闭策略必须可验收，避免每次进入都造成强干扰。
- 引导语在不同机型、安全区和自定义导航栏页面上不得与现有顶部导航、搜索、标题或内容重叠。

## 4. 范围

### 4.1 本期包含

| 范围 | 说明 |
|---|---|
| 小程序全局引导 | 在用户进入小程序后展示“添加到我的小程序”相关引导语。 |
| 右上角入口关联 | 引导语位置应靠近微信原生右上角胶囊 / 菜单区域，形成操作提示。 |
| 手工关闭 | 引导语必须提供关闭入口，用户点击后立即隐藏。 |
| 关闭状态记忆 | 关闭后是否当前会话、当天或长期不再展示需在后续补全阶段确认；实现应预留可调整策略。 |
| 安全区适配 | 引导语必须避让状态栏、胶囊按钮、自定义导航栏和页面内容。 |
| 页面覆盖 | 首期建议以首页进入场景为主；若为全局引导，需保证非首页和分享进入页面不冲突。 |

### 4.2 本期不包含

| 不包含 | 说明 |
|---|---|
| 修改微信原生菜单 | 不改变微信右上角菜单、分享、收藏或添加入口的系统行为。 |
| 手绘系统按钮 | 不在页面内模拟微信分享按钮、更多按钮、关闭按钮或胶囊控件。 |
| 后台配置 | 不提供后台配置引导文案、频率、样式或开关的管理界面。 |
| 新增 API / DB | 默认不新增后端接口、数据库表或字段；关闭记忆可优先使用小程序本地存储。 |
| Web / 管理端改造 | 不涉及 Web 展示端和企业管理端。 |
| 复杂运营弹窗 | 不做活动弹窗、营销卡片、优惠券或用户画像定向展示。 |

## 5. 功能要求

### FR-001 进入小程序展示引导语

- 小程序 SHOULD 在用户进入小程序后展示一条添加引导语。
- 引导语文案 SHOULD 表达“点击右上角按钮，添加到我的小程序，方便下次找回”的含义。
- 引导语 SHOULD 以轻提示、气泡或等价形式呈现，不应阻塞用户浏览首页内容。
- 引导语首次出现时 SHOULD 靠近右上角微信原生胶囊区域，帮助用户理解操作入口。
- 引导语不得依赖 Web 浏览器 API，必须使用微信小程序可用能力实现。

### FR-002 右上角胶囊与分享入口避让

- 引导语 MUST 根据微信小程序菜单按钮、状态栏和安全区信息计算位置。
- 引导语、箭头、关闭按钮或容器 MUST NOT 覆盖微信原生分享 / 更多 / 关闭胶囊区域。
- 若当前页面已有自定义导航栏，引导语 MUST 与导航栏标题、品牌 Logo、返回按钮和搜索入口错开。
- 在 320 到 430 pt 宽度范围内，引导语不得产生横向滚动或遮挡核心操作。
- 不得手绘模拟微信系统分享按钮、更多按钮或胶囊。

### FR-003 手工关闭

- 引导语 MUST 提供清晰的关闭入口。
- 用户点击关闭后，引导语 MUST 立即隐藏。
- 关闭入口点击热区 SHOULD 不小于 32x32 pt，并不得与微信系统胶囊重叠。
- 关闭后再次展示的策略需在后续阶段明确，建议至少支持当前会话内不再展示。
- 若使用本地存储记录关闭状态，MUST 保证存储失败时安全降级，不影响页面主功能。

### FR-004 展示频率与触发策略

- 产品原始描述为“每次进入小程序时”展示；后续需确认是否严格每次进入都展示。
- 若用户未关闭，引导语 MAY 在每次冷启动或进入首页时展示。
- 若用户已关闭，引导语 SHOULD 按确认后的记忆策略控制再次展示，避免频繁打扰。
- 从分享卡片进入、扫码进入、搜索进入和普通启动进入时，引导逻辑 SHOULD 表现一致，除非后续验收明确区分。
- 引导语不应在页面加载失败、授权弹窗、系统弹窗或关键交互过程中抢占焦点。

### FR-005 页面状态与降级

- 首页加载态、空状态、错误态下，引导语不得遮挡主要反馈和重试入口。
- 若无法获取菜单按钮位置信息，页面 SHOULD 使用安全兜底位置或不展示引导，避免错位。
- 引导语展示失败不得影响首页、分类、商品列表、详情、搜索等已有页面访问。
- 低端机或慢网络下，引导语不应导致页面首屏明显抖动。

### FR-006 可观测性预留

- 后续实现 MAY 记录引导展示、关闭和相关入口点击意向事件，用于评估留存引导效果。
- 若引入埋点，事件不得记录用户敏感信息，优先只记录匿名事件类型、页面、时间和需求来源。
- 本 PRD 不强制新增后端埋点接口；若后续需要新增接口或表结构，必须通过 OpenSpec 与 API / DB 文档同步。

## 6. UI / UE 约束

- 引导语视觉 SHOULD 轻量、克制，与小程序现有深色品牌导航和瓷砖展厅风格协调。
- 文案应短句表达，避免长段说明遮挡首屏内容。
- 气泡或提示容器可带有指向右上角菜单区域的视觉线索，但不得指到胶囊内部或遮挡胶囊。
- 关闭按钮应易识别，建议使用小型关闭图标或等价控件。
- 引导语层级应高于页面内容，但低于微信系统控件；不得影响原生菜单点击。
- 首页、商品详情、分类列表、搜索页等已有页面的核心 CTA、返回、搜索、商品卡片不得被引导语遮挡。
- 深浅模式或主题变化若存在，应保持文字对比度可读。

| 领域 | 影响预判 |
|---|---|
| 小程序 | 预计影响 `src/miniapp` 的全局入口、首页或共享引导组件，以及必要的本地关闭状态存储。 |
| API | 默认不新增 API；若后续需要服务端控制频率或埋点，再同步 OpenAPI / docs / tests。 |
| 数据库 | 默认不新增表或字段；关闭状态优先本地存储。 |
| Web 管理端 | 不涉及。 |
| Web 展示端 | 不涉及。 |
| Orval | 默认不需要。 |
| Docker Compose | 不涉及。 |
| 测试 | 后续需补充小程序静态测试、设备截图或 DevTools/真机 evidence，覆盖位置避让、关闭行为和展示频率。 |
