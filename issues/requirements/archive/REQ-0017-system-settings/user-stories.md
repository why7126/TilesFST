## US-001 配置平台基础信息

**作为** 后台管理员，  
**我希望** 在系统设置「基础信息」分组中维护平台名称、语言、时区、公告与首页展示开关，  
**以便** 全站展示一致且无需改代码或 `.env`。

### 验收要点

- 侧栏 SYSTEM「系统设置」进入 `/admin/settings/basic`（或 `/admin/settings` 重定向）。
- 可编辑平台名称（2–64 字）、默认语言、默认时区、客服邮箱、维护窗口、系统公告等。
- 保存后立即持久化；dashboard 等平台文案可读取 effective 平台名称（实现范围以 OpenSpec 为准）。
- dirty 时展示「有未保存修改」；保存成功 inline 提示。

### 关联功能

- FR-001、FR-002、FR-004；requirement §8.1

## US-002 调整媒体上传限制

**作为** 后台管理员，  
**我希望** 在「媒体与存储」分组调整图片/视频大小与 MIME 允许列表，并查看只读的桶与 Key 规则，  
**以便** 运营期灵活控制上传策略且与对象存储架构对齐。

### 验收要点

- 「媒体与存储」Tab 可 PATCH 大小与 MIME 列表；保存后**下一次上传**即按新限制校验（无需重启）。
- 存储桶名、Key 生成规则只读展示，文案与 REQ-0012 spec 一致；不可在线修改。
- 「恢复默认」将该分组恢复 env/seed 默认值。

### 关联功能

- FR-004；requirement §8.3；REQ-0012

## US-003 统一安全策略

**作为** 后台管理员，  
**我希望** 在「安全策略」分组配置密码规则与会话超时，  
**以便** 建用户、重置密码、自助改密均遵循同一策略。

### 验收要点

- 可配置密码最小长度、复杂度四项开关、密码有效期、会话超时、首次登录强制改密。
- 修改后：admin 重置密码、REQ-0015 改密、随机密码生成均 enforcement 新规则。
- JWT 超时仅影响**新签发** token。
- 登录失败锁定（P1b）为可选；未交付时不展示或 disabled 并 documented。

### 关联功能

- FR-005；requirement §8.2；REQ-0015

## US-004 管理审计策略并查看变更

**作为** 后台管理员，  
**我希望** 在「审计配置」分组设置日志保留与脱敏策略，并查看最近系统设置变更，  
**以便** 满足内控与问题追溯。

### 验收要点

- 可配置保留天数、导出权限开关、敏感操作强制记录、脱敏展示。
- 每次 settings PATCH/reset 写入 `audit_logs`（domain=`system_settings`）。
- 页面展示最近变更列表（修改人、时间、摘要）。
- 物理 purge job、导出文件下载本期可不实现。

### 关联功能

- FR-006；requirement §8.5；REQ-0014（audit 表统一）

## US-005 配置通知开关（不含发信）

**作为** 后台管理员，  
**我希望** 在「通知设置」分组开启/关闭各类预警并设置容量阈值，  
**以便** 为后续通知通道预留配置；本期不要求真实发送。

### 验收要点

- 持久化：账号冻结通知、SKU 待处理、存储容量预警及阈值（50–95%）。
- 通知模板只读列表 + 「查看」入口（modal 占位即可）。
- 切换开关保存成功；**不验收**邮件/短信/站内信是否发出。

### 关联功能

- FR-007；requirement §8.4

## US-006 后台运营不得访问系统设置（边界）

**作为** 后台运营人员，  
**我希望** 看不到系统设置入口且无法通过 URL 修改平台配置，  
**以便** 平台级参数仅由管理员维护。

### 验收要点

- `employee` 侧栏 SYSTEM 仅「用户管理」，无「系统设置」。
- 直链 `/admin/settings/*` → `/admin/forbidden` 或等价 403 页。
- 调用 system-settings API 返回 403。

### 关联功能

- FR-001、FR-003
