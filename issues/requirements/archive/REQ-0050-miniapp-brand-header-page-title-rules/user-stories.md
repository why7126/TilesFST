# REQ-0050 小程序 brand-header 页面标题规则用户故事

## US-001 首页看到稳定品牌双行文案

作为装修业主，我希望进入首页时顶部 `brand-header` 显示“菲尚特瓷砖馆”和“质感空间，由砖而生”，以便快速识别品牌和小程序调性。

验收要点：

- 首页第一行固定显示 `菲尚特瓷砖馆`。
- 首页第二行固定显示 `质感空间，由砖而生`。
- 首页不显示返回按钮。
- 首页右侧避让微信原生分享 / 关闭胶囊。
- 首页搜索框、Banner 和首屏入口不被导航栏遮挡。

## US-002 非首页只看到当前页面标题

作为普通访客，我希望在搜索、分类、详情、收藏、证书等页面只看到一行页面标题，以便快速确认自己所在页面，不被副文案干扰。

验收要点：

- search 页面标题为 `搜索`。
- category 页面标题为 `全部分类` 或等价分类页标题。
- tile-detail 页面标题本期固定为 `商品详情`。
- favorites 页面标题为 `收藏`。
- certificates 页面标题为 `证书`。
- 非首页不显示第二行 subtitle、SKU 编号副标题或品牌副文案。

## US-003 非首页可稳定返回

作为小程序访客，我希望非首页顶部有返回按钮，以便从搜索、列表、详情、收藏、证书和门店信息等页面回到上一层。

验收要点：

- 非首页左侧显示返回按钮，首页不显示。
- 有上一页页面栈时点击返回按钮回到上一页。
- 无上一页页面栈时点击返回按钮兜底回到首页。
- 加载态、空态、错误态下返回按钮仍可用。
- 返回按钮热区不小于 44x44 pt。

## US-004 右侧原生胶囊不被遮挡

作为设计师或门店销售，我希望在页面顶部继续使用微信原生分享 / 关闭能力，以便分享商品或退出小程序时不被自定义导航影响。

验收要点：

- 自定义标题和返回按钮不进入微信右侧原生胶囊区域。
- 页面不自绘分享按钮、关闭按钮或胶囊控件。
- 支持分享的页面继续保留页面级分享配置。
- tile-detail 分享路径保留当前 SKU 参数。

## US-005 开发测试可按统一规则验收

作为开发 / 测试人员，我希望首页和非首页有明确的文案与布局规则，以便实现阶段能快速发现 subtitle 残留、标题换行、胶囊遮挡和内容被 fixed header 遮挡的问题。

验收要点：

- 覆盖页面使用统一 `custom-navigation` 或等价导航契约。
- 首页和非首页的差异由页面类型或组件属性决定，不由各页零散复制样式实现。
- 标题单行截断，不横向滚动。
- 320、375、430 pt 宽度下返回按钮、标题和胶囊避让区无重叠。
- 新增接入自定义导航的非首页默认遵守单行标题规则。
