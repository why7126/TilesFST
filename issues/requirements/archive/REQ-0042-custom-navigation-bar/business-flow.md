# REQ-0042 微信小程序首页品牌自定义导航栏业务流程

## 1. 首页顶部结构

```text
用户打开小程序首页
  |
  v
展示搜索框上方模块
  |
  +--> 自定义导航栏（本 REQ 范围）
  |      |
  |      +--> store-logo / 品牌 Logo
  |      +--> store-name / 菲尚特瓷砖馆
  |      +--> store-subtitle / 专业选砖 · 品质生活
  |      +--> 右侧原生分享按钮 / 原生关闭按钮（本 REQ 范围，使用小程序原生能力）
  |
  +--> 门店信息入口（明确排除）
          |
          +--> store-link / 门店信息
          +--> openStoreInfo 跳转
```

## 2. 首页浏览关系

```text
自定义导航栏
  |
  v
搜索框（导航栏下方）
  |
  +--> 点击进入搜索页
  |
  v
Banner
  |
  v
四入口：选瓷砖 / 品牌馆 / 新品榜 / 热销榜
  |
  v
新品推荐 / 热销推荐 / 全部产品瀑布流
```

约束：

- 自定义导航栏只承担品牌识别，不承担门店信息跳转。
- 导航栏右侧保留小程序原生分享和关闭按钮；页面不得手绘模拟分享、关闭或胶囊控件。
- 搜索框保持在导航栏下方，不并入导航栏。
- 导航栏调整不得影响 `REQ-0043` 的四入口、商品推荐、瀑布流和底部 TabBar。

## 3. 状态与降级流程

```text
首页数据加载
  |
  +--> 成功：使用 store.logo_url / store.name / store.description 展示品牌导航
  |
  +--> 失败或字段缺失：使用本地 Logo / “菲尚特瓷砖馆” / “专业选砖 · 品质生活”兜底
  |
  v
搜索框、Banner 和后续模块按 REQ-0043 继续展示或降级
```

## 4. 与 REQ-0043 的一致性

| 维度 | REQ-0043 基准 | REQ-0042 约束 |
|---|---|---|
| 视觉 | 深色高端展厅，背景 `#18160F` | 导航栏必须沿用深色体系 |
| Header | 品牌 Logo、店名、副文案 | 这三类品牌展示元素组成自定义导航栏 |
| 原生右侧操作 | 微信小程序原生分享 / 关闭 | 右侧保留原生能力，品牌内容做避让 |
| 门店信息 | 可作为页面辅助入口 | 不纳入自定义导航栏 |
| 搜索 | “搜索瓷砖名称 / 型号 / 规格” | 搜索框保持在导航栏下方 |
| 快捷入口 | 选瓷砖、品牌馆、新品榜、热销榜 | 不恢复旧五入口 |
| 商品承接 | 新品、热销、全部产品瀑布流 | 不影响瀑布流结构和触底加载 |
| TabBar | 首页、分类、找砖、收藏、证书目标 | 不因导航栏调整恢复“我的”Tab |

## 5. 页面与配置关系

| 对象 | 本期处理 |
|---|---|
| 首页 WXML | 将 `brand-header` 中品牌展示元素定义为自定义导航栏；排除 `store-link`。 |
| 首页 WXSS | 导航栏样式服从 REQ-0043 深色视觉；保持与搜索框间距稳定。 |
| 首页 TS | 不把自定义导航栏整体默认绑定 `openStoreInfo`；按小程序原生分享能力承接分享；如需门店入口另行定义。 |
| 搜索页 | 仅作为搜索框跳转目标，不改搜索能力本身。 |
| 门店信息页 | 不属于本导航栏范围。 |
| 后端 API | 默认不改。 |
| 数据库 | 默认不改。 |

## 6. 父子需求关系

本需求仍是 `REQ-0041-miniapp-home` 的 refinement，并以 `REQ-0043-miniapp-home-style-optimization` 为当前首页基准：

- 不新增收藏持久化、证书聚合页、品牌馆独立页。
- 不新增快捷入口后台配置。
- 不新增多门店切换。
- 不默认新增 API / DB / Orval。
- 不手绘模拟微信系统控件；右侧分享和关闭使用小程序原生能力。
