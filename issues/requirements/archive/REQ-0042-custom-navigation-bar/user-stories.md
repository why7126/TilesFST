# REQ-0042 微信小程序首页品牌自定义导航栏用户故事

## 1. 用户故事总览

| ID | 用户 | 优先级 | 故事 | 验收要点 |
|---|---|---|---|---|
| US-001 | 装修业主 | P1 | 作为正在选砖的业主，我希望首页搜索框上方先展示简洁的菲尚特品牌导航，以便快速确认当前品牌。 | 搜索框上方展示 Logo、店名和副文案组成的自定义导航栏。 |
| US-002 | 设计师 | P1 | 作为设计师，我希望顶部品牌导航不混入门店信息入口，以便页面首屏更干净、聚焦。 | “门店信息”文案和跳转不属于自定义导航栏。 |
| US-003 | 门店客户 | P1 | 作为用户，我希望搜索入口仍在品牌导航下方清晰展示，以便继续快速搜索瓷砖名称、型号或规格。 | 搜索框保持在导航栏下方，点击进入搜索页。 |
| US-004 | 门店销售 | P1 | 作为门店销售，我希望顶部导航沿用 `REQ-0043` 的深色展厅视觉，以便演示时整体质感一致。 | 导航栏不回退暖白背景，不恢复旧五入口，不破坏瀑布流和 TabBar 目标。 |
| US-005 | 开发 / 测试人员 | P1 | 作为开发和测试人员，我希望明确当前 `brand-header` 中哪些元素进入导航栏，以便实现时不误保留门店信息入口。 | 包含 `store-logo`、`store-name`、`store-subtitle`；排除 `store-link` 和默认门店跳转。 |
| US-006 | 产品负责人 | P1 | 作为产品负责人，我希望该调整不扩散为全小程序壳层、后台配置或新接口，以便控制需求范围。 | 不做全局导航壳、后台配置、多门店切换、独立 API / DB。 |
| US-007 | 小程序用户 | P1 | 作为用户，我希望导航栏右侧仍使用微信小程序原生分享和关闭按钮，以便符合小程序习惯操作。 | 右侧保留原生分享 / 关闭能力，不使用页面自绘按钮模拟。 |

## 2. 关键验收线索

- 自定义导航栏来源是当前首页搜索框上方的 `brand-header` 品牌展示部分。
- “门店信息”入口明确排除，不属于自定义导航栏内容。
- 搜索框保持在自定义导航栏下方，不并入导航栏。
- 右侧分享和关闭按钮使用微信小程序原生能力，品牌内容需要避让原生按钮区域。
- `REQ-0043` 仍是首页样式与信息架构基准。
- 页面不得手绘模拟微信状态栏、分享按钮、关闭按钮或胶囊控件。

## 3. 非目标用户故事

| ID | 说明 | 处理 |
|---|---|---|
| NUS-001 | 用户从自定义导航栏点击进入门店信息页 | 本期不实现；门店信息入口不属于导航栏。 |
| NUS-002 | 用户在所有小程序页面获得统一 custom navigation 壳层 | 本期不实现，后续按页面体验另拆或在 OpenSpec 中明确。 |
| NUS-003 | 页面中绘制微信胶囊、分享按钮、关闭按钮作为 UI 元素 | 禁止；分享和关闭必须直接使用小程序原生能力。 |
| NUS-004 | 运营人员后台配置导航栏 Logo、颜色、搜索提示语和入口排序 | 本期不实现，后续独立 REQ。 |
| NUS-005 | 系统为导航栏新增独立数据接口或数据库表 | 默认不实现；仅后续 OpenSpec 明确字段缺口后评估。 |
