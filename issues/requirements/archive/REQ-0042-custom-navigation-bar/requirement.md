# REQ-0042 微信小程序首页品牌自定义导航栏

## 1. 需求背景

`REQ-0043-miniapp-home-style-optimization` 已明确微信小程序首页以深色高端展厅视觉为当前基准，并在首页搜索框上方形成了品牌 Header 模块。当前小程序首页实现中，搜索框上方模块为 `brand-header`，包含品牌 Logo、门店名称、品牌副文案和右侧“门店信息”入口。

本需求进一步明确：将当前小程序首页搜索框上方模块中的品牌展示部分作为首页自定义导航栏；右侧“门店信息”入口不纳入自定义导航栏。导航栏右侧需要保留小程序原生分享和关闭两个系统按钮，直接使用微信小程序原生能力，不在页面内手绘模拟。搜索框仍保持在自定义导航栏下方，不并入导航栏。

一句话：自定义导航栏 = 当前首页 `brand-header` 中除“门店信息”入口外的品牌展示区域 + 右侧微信小程序原生分享 / 关闭按钮避让区。

## 2. 目标用户

| 用户 | 诉求 |
|---|---|
| 装修业主 | 打开首页后先看到稳定的菲尚特品牌导航，并可使用小程序右侧原生分享 / 关闭能力，再进入搜索和商品浏览。 |
| 设计师 | 展示首页时顶部品牌区简洁高级，不被“门店信息”等辅助入口干扰。 |
| 门店销售 | 使用首页演示时，顶部导航只承担品牌识别，不混入门店详情跳转。 |
| 开发 / 测试人员 | 有明确的 DOM / 信息边界，知道 `brand-header` 哪些内容进入自定义导航栏、哪些必须排除。 |

## 3. 需求目标

- 将当前首页搜索框上方的品牌展示模块定义为自定义导航栏。
- 自定义导航栏仅包含品牌 Logo、门店名称和品牌副文案等品牌识别元素。
- 自定义导航栏右侧必须保留小程序原生分享和关闭两个按钮，不使用自定义绘制按钮替代。
- 自定义导航栏必须排除“门店信息”入口、门店详情跳转和多门店切换暗示。
- 搜索框保持在自定义导航栏下方，仍作为首页主搜索入口。
- 自定义导航栏必须服从 `REQ-0043` 的深色视觉、Header、四入口、瀑布流和 TabBar 目标。
- 页面不得模拟微信系统状态栏、分享按钮、关闭按钮或胶囊控件。

## 4. 范围

### 4.1 本期包含

| 范围 | 说明 |
|---|---|
| 品牌自定义导航栏 | 以当前首页 `brand-header` 中的品牌展示区域作为自定义导航栏。 |
| 品牌元素 | 导航栏包含 Logo、店名“菲尚特瓷砖馆”和副文案“专业选砖 · 品质生活”或等价品牌文案。 |
| 原生右侧按钮 | 导航栏右侧保留小程序原生分享和关闭两个按钮，并做好安全区和胶囊位置避让。 |
| 门店信息排除 | 右侧“门店信息”文案、入口和点击跳转不属于自定义导航栏。 |
| 搜索框下置 | 搜索框位于自定义导航栏下方，不并入导航栏，不产生重复搜索入口。 |
| 深色视觉一致 | 导航栏背景、文字、边界和间距服从 `REQ-0043` 深色首页。 |
| 顶部稳定性 | 导航栏在加载、错误、返回和不同屏幕宽度下保持稳定展示。 |

### 4.2 本期不包含

| 不包含 | 说明 |
|---|---|
| 门店信息入口 | “门店信息”按钮、门店详情跳转、多门店切换入口均不纳入自定义导航栏。 |
| 手绘系统按钮 | 不在 WXML / WXSS 中模拟分享按钮、关闭按钮或胶囊控件；必须直接使用小程序原生能力。 |
| 搜索栏并入导航 | 搜索框仍是导航栏下方的首页主搜索入口，不作为自定义导航栏内部元素。 |
| 全小程序统一导航壳 | 分类、找砖、收藏、证书、搜索页、详情页等页面是否统一改造，后续另行确认。 |
| 后台配置导航栏 | Logo、颜色、文案、搜索提示语、入口排序等后台配置不在本期。 |
| 新增独立 API / DB | 默认复用 `REQ-0043` 首页已有 store / brand 数据，不新增导航栏独立数据源。 |
| 收藏 / 证书完整能力 | 不因导航栏新增收藏持久化、证书聚合页或品牌馆独立页。 |

## 5. 功能要求

### FR-001 自定义导航栏内容来源

- 自定义导航栏 MUST 取自当前首页搜索框上方模块的品牌展示部分。
- 自定义导航栏 SHOULD 对应当前实现中的 `store-logo`、`store-name`、`store-subtitle` 或等价元素。
- 自定义导航栏 MUST NOT 包含当前实现中的 `store-link` “门店信息”入口。
- 自定义导航栏 MUST NOT 让整个品牌区域默认绑定进入门店信息页，除非后续需求另行明确。
- 自定义导航栏右侧 MUST 为微信小程序原生分享和关闭按钮预留位置，品牌内容不得压到原生按钮区域。

### FR-002 门店信息排除

- “门店信息”文案、按钮、箭头、跳转和多门店切换暗示 MUST 从自定义导航栏范围中排除。
- 若首页仍需要门店信息入口，MUST 由其他页面区域或后续需求定义，不得混入本导航栏。
- 单门店场景下，导航栏不得出现可切换门店的视觉暗示。

### FR-003 搜索框位置

- 搜索框 MUST 保持在自定义导航栏下方。
- 搜索框文案 SHOULD 与 `REQ-0043` 保持一致：“搜索瓷砖名称 / 型号 / 规格”或等价中文。
- 导航栏内部 MUST NOT 再新增第二个主搜索入口。
- 搜索框点击后进入现有搜索页。

### FR-004 与 REQ-0043 一致

- 自定义导航栏 MUST 以 `REQ-0043-miniapp-home-style-optimization` 为视觉和信息架构基准。
- 页面背景、Header、搜索、Banner、四入口、新品、热销、全部产品瀑布流和底部 TabBar MUST 不因导航栏调整而偏离 `REQ-0043`。
- 本需求 MUST NOT 要求恢复 `REQ-0041` 的暖白首页主背景或旧五入口结构。

### FR-005 真实小程序导航环境

- 页面 MUST 直接使用微信小程序原生分享和关闭按钮，不得在页面内模拟系统状态栏、分享按钮、关闭按钮或胶囊控件。
- 若后续实现启用 `navigationStyle: custom`，MUST 处理状态栏高度、原生右侧按钮位置、安全区和不同机型适配。
- 原生分享能力 SHOULD 通过小程序标准分享能力承接，例如 `onShareAppMessage` 或平台菜单能力；关闭按钮 MUST 使用小程序原生关闭能力。
- 若未启用 `navigationStyle: custom`，验收时将 custom navigation 安全区门禁标为 N/A，但仍需确认品牌导航栏与微信真实导航环境不重叠。

### FR-006 状态稳定

- 加载态下，自定义导航栏高度和品牌元素位置 MUST 稳定。
- 网络异常态下，自定义导航栏 SHOULD 展示品牌兜底文案，例如“菲尚特瓷砖馆”。
- 从搜索页、商品详情页、分类/找砖、收藏/证书占位页或分享返回首页后，自定义导航栏展示不得错位、闪烁或高度异常。

## 6. UI / UE 约束

- UI MUST 以 `REQ-0043` 附件 `prototype.png` / `prototype.html` 为视觉基准。
- 首页 MUST 使用 `REQ-0043` 的深色体系：页面背景 `#18160F`、卡片/搜索框/模块底色 `#211E16`、品牌金 `#C8A055`。
- 自定义导航栏应简洁承载品牌识别，不额外叠加与原型无关的标题条、胶囊控件或门店信息按钮。
- 自定义导航栏右侧原生分享 / 关闭按钮区域必须与品牌 Logo、店名、副文案保持安全间距。
- 自定义导航栏与下方搜索框之间的间距应稳定，不能导致搜索框、Banner 或四入口被挤压。
- 320 到 430 pt 宽度范围内，Logo、店名、副文案不重叠、不严重截断、不造成页面级横向滚动。
- 所有主要可点击区域 MUST 不小于 44x44 pt；若导航栏无点击行为，则不得表现为强可点击控件。
