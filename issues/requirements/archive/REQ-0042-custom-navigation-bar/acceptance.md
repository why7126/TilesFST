# REQ-0042 微信小程序首页品牌自定义导航栏验收标准

## 1. 功能 AC

- [ ] AC-001 当前小程序首页搜索框上方模块中的品牌展示区域被定义为自定义导航栏。
- [ ] AC-002 自定义导航栏包含品牌 Logo、店名“菲尚特瓷砖馆”和副文案“专业选砖 · 品质生活”或等价品牌文案。
- [ ] AC-003 自定义导航栏不得包含右侧“门店信息”文案、按钮、箭头或点击入口。
- [ ] AC-004 当前 `brand-header` 如仍保留“门店信息”入口，该入口必须从自定义导航栏语义和验收范围中拆出。
- [ ] AC-005 自定义导航栏整体不得默认绑定进入门店信息页；门店信息跳转需由导航栏外的入口或后续需求定义。
- [ ] AC-006 搜索框保持在自定义导航栏下方，且不并入导航栏内部。
- [ ] AC-007 搜索框点击后进入现有搜索页。
- [ ] AC-008 首页不得出现两个同等主级搜索入口。
- [ ] AC-009 自定义导航栏右侧保留小程序原生分享和关闭两个按钮。
- [ ] AC-010 分享和关闭按钮必须使用微信小程序原生能力，不得使用 WXML / WXSS 自绘按钮模拟。

## 2. 与 REQ-0043 一致性 AC

- [ ] AC-BASE-001 首页顶部方案以 `REQ-0043-miniapp-home-style-optimization` 为视觉与信息架构基准。
- [ ] AC-BASE-002 首页使用 `REQ-0043` 的深色视觉体系，不得恢复暖白或浅灰主背景。
- [ ] AC-BASE-003 快捷入口保持 `REQ-0043` 的四项：选瓷砖、品牌馆、新品榜、热销榜，不恢复旧五入口。
- [ ] AC-BASE-004 热销推荐下方的“全部产品”瀑布流不得因导航栏调整被移除、遮挡或改为固定等高网格。
- [ ] AC-BASE-005 底部 TabBar 目标文案保持 `REQ-0043`：首页、分类、找砖、收藏、证书；不因导航栏调整恢复“我的”Tab。
- [ ] AC-BASE-006 页面不得模拟微信系统状态栏、分享按钮、关闭按钮或胶囊控件；右侧分享和关闭必须来自小程序原生能力。

## 3. UI / UE AC

- [ ] AC-UI-001 UI 以 `REQ-0043` 附件 `prototype.png` / `prototype.html` 为视觉基准。
- [ ] AC-UI-002 自定义导航栏只承载品牌识别，不额外叠加与原型无关的标题条、胶囊控件或门店信息按钮。
- [ ] AC-UI-003 导航栏与下方搜索框之间的间距稳定，不导致搜索框、Banner 或四入口被挤压。
- [ ] AC-UI-004 320 到 430 pt 宽度范围内，Logo、店名、副文案不重叠、不严重截断、不造成页面级横向滚动。
- [ ] AC-UI-005 所有主要可点击区域不小于 44x44 pt；自绘区域若无点击行为，则不得表现为强可点击控件；原生分享 / 关闭按钮遵循微信小程序原生交互规范。
- [ ] AC-UI-006 页面不使用强促销色、复杂渐变、重阴影或与 `REQ-0043` 瓷砖展厅气质不一致的装饰。
- [ ] AC-UI-007 品牌展示区域必须避让右侧原生分享 / 关闭按钮，不能遮挡、重叠或压缩到不可读。

## 4. 状态与降级 AC

- [ ] AC-STATE-001 首页加载态下，自定义导航栏高度和品牌元素位置稳定。
- [ ] AC-STATE-002 首页数据缺少 `store.logo_url` 时，使用本地安全 Logo 或等价品牌兜底。
- [ ] AC-STATE-003 首页数据缺少 `store.name` 或 `store.description` 时，分别使用“菲尚特瓷砖馆”和“专业选砖 · 品质生活”或等价文案兜底。
- [ ] AC-STATE-004 网络异常态下，自定义导航栏仍展示品牌兜底文案，并与错误重试区不重叠。
- [ ] AC-STATE-005 从搜索页、商品详情页、分类/找砖、收藏/证书占位页或分享返回首页后，自定义导航栏展示不出现错位、闪烁或高度异常。

## 5. API / 数据 / 安全 AC

- [ ] AC-API-001 自定义导航栏默认复用 `REQ-0043` 首页已有 store / brand 信息，不新增独立导航栏接口。
- [ ] AC-API-002 若导航栏展示门店名称或 Logo，数据必须来自后端允许公开展示的字段。
- [ ] AC-API-003 小程序端不得为了导航栏直连未授权对象存储；Logo 或图片必须使用后端返回的安全 URL 或本地安全资源。
- [ ] AC-API-004 若后续新增或调整 API contract，必须同步 OpenAPI、Orval、docs 和测试。
- [ ] AC-API-005 本需求默认不修改数据库；若后续新增字段，必须同步 SQLite/MySQL 文档、schema / migration 和测试。

## 6. 范围控制 AC

- [ ] AC-SCOPE-001 本期不实现全小程序统一 custom navigation 壳层；其他页面是否采用自定义导航栏需后续单独确认。
- [ ] AC-SCOPE-002 本期不新增后台配置导航栏能力，包括 Logo、颜色、文案、搜索提示语、入口排序等配置。
- [ ] AC-SCOPE-003 本期不新增多门店切换能力；单门店场景不得出现误导性的切换箭头。
- [ ] AC-SCOPE-004 本期不新增收藏持久化、证书聚合页、品牌馆独立页、新品榜/热销榜独立页。
- [ ] AC-SCOPE-005 本期默认不新增独立 API、数据库表或字段；若后续实现发现字段缺口，必须经 OpenSpec 明确并同步 API / DB / Orval / docs / tests。

## 7. 测试与验证 AC

- [ ] AC-TEST-001 后续实现阶段必须对照当前首页 `brand-header` 验证自定义导航栏仅包含品牌 Logo、店名和副文案。
- [ ] AC-TEST-002 后续实现阶段必须验证“门店信息”入口不属于自定义导航栏，且不会因导航栏点击触发门店信息跳转。
- [ ] AC-TEST-003 后续实现阶段必须对照 `REQ-0043` 验证首页 Header、搜索、Banner、四入口、新品、热销、全部产品瀑布流和 TabBar 目标未被导航栏调整破坏。
- [ ] AC-TEST-004 若未启用 `navigationStyle: custom`，验收报告需明确说明 custom navigation 安全区门禁为 N/A。
- [ ] AC-TEST-005 若未改 API / DB / Orval，验收报告需明确说明 N/A；若发生变化，必须补充对应测试与同步记录。
- [ ] AC-TEST-006 后续实现阶段必须在真机或开发者工具中验证右侧原生分享和关闭按钮可见、可用，且不是页面自绘控件。

## 8. 横切 AC（knowledge-base）

本 REQ 不命中 `req-complete` 规定的 `admin-list`、`admin-form`、`admin-modal`、`media-upload` 标签，因此无 `AC-XCUT-*` 管理端横切验收项。
