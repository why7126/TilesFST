# REQ-0048 小程序全局自定义导航栏用户故事

## US-001 首页继续展示品牌导航

作为装修业主，我希望进入首页时仍看到当前品牌 `brand-header`，以便快速识别菲尚特瓷砖馆并进入搜索和商品浏览。

验收要点：

- 首页不展示左侧返回按钮。
- 首页品牌 Logo、门店名称、品牌副文案保持当前表达。
- 首页右侧继续避让微信原生分享 / 关闭胶囊。
- 首页搜索框、Banner 和入口区不被导航栏遮挡。

## US-002 非首页可稳定返回上一页

作为小程序访客，我希望在搜索、商品详情、分类、收藏、证书和门店信息页看到左侧返回按钮，以便不依赖系统手势也能回到上一层。

验收要点：

- search、tile-detail、category、product-list、favorites、certificates、store-info 页面均展示返回按钮。
- 有页面栈时点击返回按钮回到上一页。
- 无页面栈时返回首页或进入明确兜底路径。
- 返回按钮热区不小于 44x44 pt。

## US-003 详情页保留分享能力

作为设计师或门店销售，我希望在 SKU 详情页使用微信原生分享能力，以便把当前商品分享给客户或家人。

验收要点：

- tile-detail 自定义导航不遮挡微信原生胶囊。
- 分享路径保留当前 SKU 参数。
- 分享标题、封面和路径沿用页面级分享配置。
- 页面不出现自绘伪分享按钮。

## US-004 内容不被 fixed header 遮挡

作为普通访客，我希望进入任何非首页页面后，顶部内容从导航栏下方开始展示，以便首屏内容完整可读。

验收要点：

- 搜索框、媒体轮播、分类列表、商品列表、收藏列表、证书列表和门店信息首屏内容均不被遮挡。
- 加载态、空状态、错误态和下拉刷新区域同样避让导航栏。
- 320 到 430 pt 宽度范围内无横向滚动和元素重叠。

## US-005 开发测试可复用统一导航模块

作为开发 / 测试人员，我希望不同页面使用统一导航模块与统一高度计算，以便降低多页面重复 header 造成的样式漂移和回归风险。

验收要点：

- 首页形态与非首页形态由同一模块或同一布局契约承接。
- 状态栏高度、胶囊位置和内容顶部占位由统一计算结果控制。
- 新增非首页默认接入该导航模块，除非需求明确豁免。
