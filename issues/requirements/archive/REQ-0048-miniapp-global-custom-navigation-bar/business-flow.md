# REQ-0048 小程序全局自定义导航栏业务流程

## 1. 总流程

```text
用户进入小程序页面
  |
  v
识别当前页面是否首页
  |
  +-- 是：首页导航形态
  |     |
  |     +-- 展示当前 brand-header
  |     +-- 不展示左侧返回按钮
  |     +-- 右侧避让微信原生胶囊
  |     +-- 内容区从首页导航下方开始
  |
  +-- 否：非首页导航形态
        |
        +-- 展示左侧返回按钮
        +-- 展示品牌/页面标题信息
        +-- 右侧避让微信原生胶囊
        +-- 内容区从 fixed header 下方开始
        |
        v
      用户点击返回？
        |
        +-- 有上一页：navigateBack
        +-- 无上一页：兜底返回首页
```

## 2. 页面覆盖流程

```text
首页
  -> search
      -> tile-detail
  -> category
      -> product-list
          -> tile-detail
  -> favorites
      -> tile-detail
  -> certificates
  -> store-info

所有非首页节点：
  - 顶部显示返回按钮
  - 右侧避让微信原生胶囊
  - 主体内容避让导航栏高度
```

## 3. 与父 REQ 差异

| 对比项 | REQ-0042 首页品牌自定义导航栏 | REQ-0048 全局自定义导航栏 |
|---|---|---|
| 页面范围 | 仅首页 | 首页 + search、tile-detail、category、product-list、favorites、certificates、store-info 等非首页 |
| 左侧返回 | 首页不涉及 | 非首页必须新增返回按钮，首页不显示 |
| 右侧避让 | 首页避让原生分享 / 关闭胶囊 | 所有覆盖页面都必须避让原生分享 / 关闭胶囊 |
| 内容避让 | 首页搜索框在导航下方 | 所有页面主体、加载态、空态、错误态均需避让 fixed header |
| 复用目标 | 明确首页 `brand-header` 边界 | 建立统一导航模块与页面接入契约 |

## 4. 异常与兜底流程

```text
用户从分享卡片直达非首页
  |
  v
页面栈为空或无上一页
  |
  v
点击返回按钮
  |
  +-- 回到首页兜底路径
  +-- 保留当前页面分享、刷新、错误重试能力
```

## 5. 风险控制

| 风险 | 控制方式 |
|---|---|
| 多页面各自硬编码顶部 padding | 后续实现阶段使用统一导航高度变量或工具函数。 |
| 非首页标题挤压右侧胶囊 | 标题单行截断并为胶囊区域预留固定安全宽度。 |
| 从分享进入时返回无效 | 无页面栈时兜底返回首页。 |
| fixed header 遮挡骨架屏 | 加载态、空态、错误态均接入同一内容避让规则。 |
| 页面自绘系统胶囊 | 明确禁止自绘分享 / 关闭按钮，保留微信原生能力。 |
