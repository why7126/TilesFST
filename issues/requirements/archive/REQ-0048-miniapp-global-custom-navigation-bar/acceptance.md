# REQ-0048 小程序全局自定义导航栏验收标准

## 1. 功能 AC

- [ ] AC-001 首页继续保留当前品牌 `brand-header`，不新增左侧返回按钮。
- [ ] AC-002 首页品牌 Logo、门店名称、品牌副文案或等价品牌信息展示稳定。
- [ ] AC-003 非首页使用同一导航模块的非首页形态，左侧展示返回按钮。
- [ ] AC-004 非首页导航模块不得引入与当前深色品牌风格冲突的新视觉主题。
- [ ] AC-005 导航模块支持按页面类型区分首页与非首页形态，首页不会误显示返回按钮。

## 2. 页面覆盖 AC

- [ ] AC-PAGE-001 search 页面接入非首页导航栏，返回按钮可见可点。
- [ ] AC-PAGE-002 tile-detail 页面接入非首页导航栏，返回按钮可见可点。
- [ ] AC-PAGE-003 category 页面接入非首页导航栏，返回按钮可见可点。
- [ ] AC-PAGE-004 product-list 页面接入非首页导航栏，返回按钮可见可点。
- [ ] AC-PAGE-005 favorites 页面接入非首页导航栏，返回按钮可见可点。
- [ ] AC-PAGE-006 certificates 页面接入非首页导航栏，返回按钮可见可点。
- [ ] AC-PAGE-007 store-info 页面接入非首页导航栏，返回按钮可见可点。
- [ ] AC-PAGE-008 实现阶段发现的其他同类非首页页面如接入该模块，必须满足同一返回、避让和内容不遮挡验收。

## 3. 返回行为 AC

- [ ] AC-BACK-001 有上一页页面栈时，点击返回按钮回到上一页。
- [ ] AC-BACK-002 无上一页页面栈时，点击返回按钮进入明确兜底路径，建议返回首页。
- [ ] AC-BACK-003 从分享卡片直达 tile-detail 时，返回按钮不失效、不报错。
- [ ] AC-BACK-004 加载态、空状态、错误态下返回按钮仍可使用。
- [ ] AC-BACK-005 返回按钮有效点击区域不小于 44x44 pt。

## 4. 微信原生胶囊与分享 AC

- [ ] AC-WX-001 所有覆盖页面的导航栏右侧均避让微信原生分享 / 关闭胶囊。
- [ ] AC-WX-002 品牌信息、页面标题、返回按钮和搜索入口不得进入微信原生胶囊区域。
- [ ] AC-WX-003 页面不得在 WXML / WXSS 中自绘模拟分享按钮、关闭按钮或系统胶囊。
- [ ] AC-WX-004 支持分享的页面继续保留微信小程序原生分享能力。
- [ ] AC-WX-005 tile-detail 分享路径保留当前 SKU 参数。
- [ ] AC-WX-006 非分享页面不得出现不可点击的伪分享按钮。

## 5. 状态栏与内容避让 AC

- [ ] AC-LAYOUT-001 导航栏高度计算包含状态栏高度、胶囊高度和导航内容高度。
- [ ] AC-LAYOUT-002 页面主体内容从导航栏下方开始，不被 fixed header 遮挡。
- [ ] AC-LAYOUT-003 搜索框、媒体轮播、分类列表、商品列表、收藏列表、证书列表和门店信息首屏内容均不被遮挡。
- [ ] AC-LAYOUT-004 加载态、空状态、错误态、骨架屏和下拉刷新区域同样避让导航栏。
- [ ] AC-LAYOUT-005 320 到 430 pt 宽度范围内，返回按钮、标题/品牌信息和右侧胶囊避让区不重叠。
- [ ] AC-LAYOUT-006 页面不产生横向滚动。

## 6. API / 数据 / 安全 AC

- [ ] AC-API-001 本需求默认不新增 API，导航栏复用已有品牌或页面数据。
- [ ] AC-API-002 本需求默认不修改数据库表或字段。
- [ ] AC-API-003 导航栏 Logo 或图片不得直连未授权对象存储，必须使用后端返回的安全 URL 或本地安全资源。
- [ ] AC-API-004 若后续实现阶段新增或调整接口 contract，必须同步 OpenAPI、Orval、docs 和测试。
- [ ] AC-API-005 若后续实现阶段新增字段，必须同步 SQLite/MySQL schema、数据库文档和测试。

## 7. 范围控制 AC

- [ ] AC-SCOPE-001 本期不新增业务页面，只改造已有或已规划页面的导航接入。
- [ ] AC-SCOPE-002 本期不新增后台导航配置能力。
- [ ] AC-SCOPE-003 本期不调整底部 TabBar 文案、图标和路由。
- [ ] AC-SCOPE-004 本期不做小程序整体视觉重设计。
- [ ] AC-SCOPE-005 本期不创建 OpenSpec Change；后续开发前必须先通过 `/req-review` 和 `/req-opsx`。

## 8. 测试与验证 AC

- [ ] AC-TEST-001 后续实现阶段必须在微信开发者工具或真机验证返回行为。
- [ ] AC-TEST-002 后续实现阶段必须验证微信原生胶囊可见、可用，且没有被自定义导航遮挡。
- [ ] AC-TEST-003 后续实现阶段必须验证支持分享页面的分享标题、路径和参数。
- [ ] AC-TEST-004 后续实现阶段必须截图或记录 320、375、430 pt 宽度下导航栏和首屏内容无重叠。
- [ ] AC-TEST-005 若未改 API / DB / Orval，验收报告需明确标记 N/A；若发生变化，必须补充对应同步记录。
- [ ] AC-TEST-006 后续实现阶段必须检查页面内容顶部占位使用统一策略，不允许各页面散落互相冲突的硬编码高度。
