# 用户故事

## US-001 管理员切换舒适主题

作为企业内部管理员，我希望在管理端切换为舒适暗色或浅色主题，以便长时间处理瓷砖 SKU、品牌、类目、用户和日志时减少眼睛疲劳。

验收要点：

- 管理端提供主题切换入口。
- 可选择系统默认、暗色旗舰、舒适暗色、浅色四类主题模式。
- 切换后当前页面即时生效，不丢失筛选条件、分页状态、表单输入和弹窗状态。

## US-002 用户主题偏好跨会话恢复

作为高频使用管理端的运营人员，我希望系统记住我的主题偏好，以便下次登录或换设备后继续使用熟悉的主题。

验收要点：

- 本地持久化用于首屏快速恢复。
- 账号级偏好用于跨设备和重新登录后恢复。
- 切换账号或退出登录后不会错误沿用其他用户的账号级主题。

## US-003 登录页保持品牌感与舒适度

作为管理端用户，我希望登录页在保持品牌专业感的同时不过暗或刺眼，以便在不同环境光下都能清晰登录。

验收要点：

- 登录页覆盖暗色旗舰、舒适暗色、浅色和系统默认模式。
- 表单、错误提示、语言切换和主 CTA 在各主题下清晰可读。
- 登录前选择主题时，登录后主题偏好可以延续。

## US-004 瓷砖 SKU 列表页主题验收

作为运营人员，我希望瓷砖 SKU 列表页在不同主题下仍保持清晰的指标、筛选、表格、操作列和分页，以便高频维护 SKU 时不因主题变化降低效率。

验收要点：

- 瓷砖 SKU 列表页作为首批列表页验收样例。
- 指标卡、筛选区、表格、sticky 操作列、分页在四类主题下不重叠、不裁切、不丢失层级。
- 成功 / 失败 toast 不引发布局位移。

## US-005 瓷砖 SKU 表单与弹窗主题验收

作为运营人员，我希望新建或编辑瓷砖 SKU 时，弹窗、表单、上传状态和确认操作在不同主题下都稳定清楚。

验收要点：

- 瓷砖 SKU 新建/编辑弹窗作为首批弹窗验收样例。
- 输入框、选择器、帮助文案、错误提示、保存 CTA、媒体上传状态在四类主题下可读。
- 危险操作和状态变更使用 Design System confirm modal，不使用原生 confirm。

## US-006 店主 Web 舒适主题

作为店主 Web 访问者，我希望除品牌展示页外的商品列表、筛选、详情阅读和询价路径也支持舒适主题，以便长时间浏览产品时更舒服。

验收要点：

- 品牌展示页、Hero 和品牌氛围区域允许保留暗色旗舰风。
- 商品列表、筛选、详情页和询价路径支持舒适主题。
- 店主端主题策略不破坏品牌识别和图片浏览体验。

## US-007 QA 使用 Design System 验收主题

作为 QA，我希望 `/design-system` 能集中展示主题 token、组件状态和代表页面样例，以便在实现前后快速判断主题适配是否完整。

验收要点：

- `/design-system` 展示四类主题模式。
- 展示 Button、Input、Badge、Card、Table、Pagination、Modal、Toast 等状态。
- 展示登录页、瓷砖 SKU 列表、瓷砖 SKU 弹窗/表单和店主 Web 舒适主题样例或链接。
