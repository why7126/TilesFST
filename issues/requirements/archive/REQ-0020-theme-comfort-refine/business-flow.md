# 业务流程

## 1. 总体流程

```text
用户进入 Web / 管理端
        |
        v
读取本地主题偏好
        |
        +-- 有本地偏好 -> 先应用本地主题，避免首屏无主题闪烁
        |
        +-- 无本地偏好 -> 应用系统默认 / 产品默认主题
        |
        v
用户认证完成后读取账号级主题偏好
        |
        +-- 账号偏好存在且与本地不同 -> 合并策略生效并同步当前主题
        |
        +-- 账号偏好不存在 -> 保持当前主题，必要时回写账号级偏好
        |
        v
用户通过主题入口切换主题
        |
        +-- 系统默认
        +-- 暗色旗舰
        +-- 舒适暗色
        +-- 浅色
        |
        v
即时更新 CSS variables / semantic token 映射
        |
        v
保存本地偏好 + 保存账号级偏好
        |
        v
登录页 / 管理端 / 店主 Web / design-system 使用同一主题事实源
```

## 2. 管理端主题切换流程

```text
Admin Shell / 用户菜单 / 设置页
        |
        v
打开主题选择控件
        |
        +-- 系统默认：跟随系统或产品默认策略
        +-- 暗色旗舰：保留当前工业石材品牌暗色
        +-- 舒适暗色：降低视觉压迫，适合长时间操作
        +-- 浅色：适合强光环境和偏好浅色用户
        |
        v
用户选择主题
        |
        v
当前页面即时应用
        |
        +-- 列表页：筛选、分页、表格状态不丢失
        +-- 表单页：dirty 状态与输入不丢失
        +-- 弹窗：当前弹窗不关闭，上传/错误状态不丢失
        |
        v
本地与账号级偏好同步
```

## 3. 首批验收页面矩阵

| 场景 | 代表页面 / 组件 | 验收重点 |
|---|---|---|
| 登录页 | Web 管理端登录页 | 品牌感、表单可读性、错误提示、登录前主题选择延续 |
| 列表页 | 瓷砖 SKU 管理列表 | 指标卡、筛选、表格、sticky 操作列、分页、toast |
| 表单页 | 瓷砖 SKU 编辑表单区域 | 输入、选择器、帮助文案、错误、保存 CTA |
| 弹窗 | 瓷砖 SKU 新建 / 编辑弹窗 | 宽度、遮罩、滚动、表单、媒体上传状态、确认操作 |
| Design System | `/design-system` 主题页 | token、基础组件、管理端组件、状态样例 |
| 店主 Web | 商品列表 / 详情 / 询价路径 | 舒适主题支持，品牌展示页保持暗色旗舰风 |

## 4. 与父需求差异

| 维度 | REQ-0000-build-design-system | REQ-0020-theme-comfort-refine |
|---|---|---|
| 目标 | 建立基础 Design System、token、组件和预览页 | 在既有 DS 上补齐主题舒适度和主题切换能力 |
| 默认视觉 | 工业石材 · 暗色旗舰风 | 保留暗色旗舰，同时新增系统默认、舒适暗色、浅色 |
| 浅色能力 | 预留，不做完整验收 | 本期纳入正式主题验收 |
| 用户偏好 | 未覆盖 | 本地 + 账号级持久化二者结合 |
| 验收页面 | 基础组件与 DS 预览 | 登录页、瓷砖 SKU 列表、SKU 表单/弹窗、店主 Web、/design-system |

## 5. API / DB / Orval 影响判断

本需求确认主题偏好采用本地持久化与账号级持久化二者结合，因此后续 `/req-opsx` 设计阶段需要评估并大概率新增账号级主题偏好能力：

| 项 | 判断 |
|---|---|
| API | 需要。预计提供读取 / 更新当前用户主题偏好的认证接口。 |
| 数据库 | 需要。预计在用户偏好表或用户配置字段中保存主题模式。 |
| Orval | 需要。若新增 API，必须同步 OpenAPI 与 Orval 生成。 |
| 权限 | 需要。主题偏好只能读写当前认证用户。 |
| Docker | 不需要改变部署拓扑，但实现后仍需常规 Web 构建验证。 |

## 6. 知识库横切模式

```text
admin-list  -> 瓷砖 SKU 列表页主题验收必须继承 AdminListPage / 分页 / toast / confirm gate
admin-form  -> 表单保存 CTA、dirty、恢复默认、fixed toast 不可因主题切换回归
admin-modal -> SKU 弹窗宽度、CSS 层叠、矮视口滚动不可因主题类名回归
media-upload -> SKU 媒体上传状态 UI 在主题切换后仍需 idle/uploading/done/failed 可见
```
