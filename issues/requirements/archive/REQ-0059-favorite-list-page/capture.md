# 一句话

新增收藏列表页，集中展示用户已收藏的内容，并支持从列表回到对应详情。

# 原始描述

新增收藏列表页

# 背景与关联

- 涉及端：待明确，倾向店主展示端或微信小程序用户侧页面
- 业务价值：让用户能够集中查看已收藏的瓷砖、SKU、品牌或其他对象，提升回访、对比和再次查看效率
- 预期后续：需要明确收藏对象范围、列表入口、排序规则、空状态、取消收藏交互，以及是否复用现有商品卡片或列表组件

# 待澄清

- [ ] 收藏列表页面向 Web 展示端、微信小程序，还是两端都需要
- [ ] 收藏对象是瓷砖 SKU、商品、品牌、分类，还是多个对象混合展示
- [ ] 页面入口位于底部导航、个人中心、首页快捷入口，还是其他位置
- [ ] 列表是否需要支持搜索、分类筛选、批量管理或仅展示基础卡片
- [ ] 空收藏状态、下架/删除对象、未登录状态应如何展示
- [ ] 是否需要登录态、用户身份与收藏数据模型支持
