# 新增收藏列表页需求文档

## 1. 背景

菲尚特瓷砖展示端和微信小程序已经具备首页、分类、搜索、商品列表和商品详情等浏览链路。用户在选砖过程中通常会反复比较多个 SKU、品牌或内容对象，如果缺少统一的收藏列表，用户只能依赖搜索、历史路径或外部截图记录，回访和对比效率较低。

本需求新增收藏列表页，用于集中展示用户已收藏的内容，并支持从列表回到对应详情页。由于 capture 阶段尚未明确具体端和收藏对象范围，本 PRD 先定义通用产品边界，后续在 `/req-complete` 阶段补齐端侧交互、数据模型和验收细节。

## 2. 目标用户

| 用户 | 核心诉求 |
|---|---|
| 装修客户 | 集中查看感兴趣的瓷砖 SKU，方便反复比较和再次进入详情 |
| 设计师 | 保存候选产品，后续按项目或方案继续筛选、沟通和复盘 |
| 门店导购 | 快速找回客户关注过的产品，辅助线下介绍和二次推荐 |
| 普通访客 | 在浏览过程中低成本保存内容，减少重复搜索 |

## 3. 范围

### 3.1 包含

- 新增用户侧收藏列表页，用于展示当前用户已收藏的内容。
- 收藏列表支持展示基础对象信息、收藏时间或默认排序、加载态、空状态和异常状态。
- 列表项支持点击进入对应详情页。
- 列表页支持取消收藏或从列表移除收藏项。
- 页面入口、收藏对象、登录态策略和跨端范围在后续阶段继续细化。
- 页面视觉与现有用户侧深色高端展示体系保持一致。

### 3.2 不包含

- 管理端收藏数据运营、批量导出或人工维护能力。
- 商品、品牌、分类等基础数据管理能力。
- 购物车、询价单、在线下单、客服会话或报价流程。
- 复杂收藏分组、项目清单、多人协作、分享收藏夹。
- 推荐算法、浏览历史、足迹列表或自动收藏能力。

## 4. 页面与入口

| 项 | v1 要求 |
|---|---|
| 页面定位 | 用户侧收藏列表页 |
| 端范围 | Web 展示端与微信小程序待确认，倾向至少覆盖微信小程序 |
| 推荐入口 | 个人中心、底部导航或首页快捷入口待确认 |
| 收藏对象 | SKU / 商品 / 品牌 / 分类范围待确认，v1 建议优先 SKU 或商品 |
| 跳转目标 | 列表项点击进入对应对象详情页 |
| 登录态 | 待确认；如收藏数据绑定用户，MUST 有明确登录或访客标识策略 |

## 5. 功能要求

### FR-001 收藏列表展示

- 系统 MUST 提供收藏列表页，集中展示当前用户已收藏的对象。
- 收藏列表 MUST 展示对象主图或占位、名称、关键辅助信息和收藏状态。
- 收藏列表 SHOULD 展示收藏时间或采用默认排序，便于用户找回最近收藏内容。
- 列表项 MUST 支持点击进入对应详情页。
- 当收藏对象已下架、删除或不可访问时，列表 MUST 给出稳定降级展示，不得导致整页不可用。

### FR-002 收藏对象范围

- v1 SHOULD 优先支持 SKU 或商品收藏，避免首版混合过多对象类型。
- 如果支持品牌、分类等多对象收藏，列表 MUST 清晰区分对象类型，并提供对应详情跳转。
- 收藏对象 ID、类型、展示摘要和跳转路径 MUST 保持一致，避免列表项无法定位详情。
- 后续 `/req-complete` MUST 明确最终对象范围和对象字段。

### FR-003 列表操作

- 用户 MUST 能从收藏列表中取消收藏或移除收藏项。
- 取消收藏 SHOULD 有明确反馈，避免用户误以为操作失败。
- 取消收藏后列表 SHOULD 即时更新；网络失败时 MUST 保留原状态并提示重试。
- v1 不要求批量管理、全选删除或收藏分组。

### FR-004 空状态与未登录状态

- 无收藏内容时 MUST 展示空状态，并提供返回首页、分类或搜索的引导入口。
- 如果收藏能力需要登录，未登录用户进入收藏列表时 MUST 展示登录引导或受控访客提示。
- 空状态文案 SHOULD 强调“先收藏感兴趣的产品”，而不是错误提示。
- 未登录、无数据、网络异常三种状态 MUST 使用不同视觉和文案。

### FR-005 加载、分页与刷新

- 收藏列表 MUST 支持首屏加载态。
- 收藏数量较多时 SHOULD 支持分页、上拉加载更多或等价的增量加载。
- 页面 SHOULD 支持下拉刷新或手动刷新，确保收藏状态与服务端一致。
- 加载更多时 MUST 防止重复请求。
- 网络异常时 MUST 提供重试入口，并尽量保留已加载内容。

### FR-006 数据一致性

- 收藏列表 SHOULD 与详情页收藏状态保持一致。
- 在详情页新增或取消收藏后，再进入收藏列表时 MUST 反映最新状态。
- 在收藏列表取消收藏后，返回详情页时 SHOULD 同步最新收藏状态。
- 如果存在本地缓存，缓存 MUST 有刷新策略，避免长期展示已失效收藏项。

### FR-007 埋点

收藏列表 SHOULD 记录以下事件：

- `favorite_list_page_view`
- `favorite_list_item_click`
- `favorite_list_remove`
- `favorite_list_empty_action_click`
- `favorite_list_load_failed`

公共参数 SHOULD 包含 terminal、objectType、objectId、index、sourcePage、hasLogin、resultCount 和 requestId。

## 6. UI 约束

- 页面 MUST 复用现有用户侧页面视觉语言，保持深色高端、品牌金色强调和清晰信息层级。
- 列表项 SHOULD 优先复用现有商品卡片或对象卡片能力，减少与商品列表页的样式分歧。
- 主图区域 MUST 使用稳定比例，避免图片加载导致布局跳动。
- 取消收藏操作 MUST 清晰但不过度抢占商品浏览视线。
- 空状态 MUST 提供明确下一步入口，例如去首页、分类或搜索。
- 触控端可点击区域 SHOULD 不小于 44px。
- 文案不得出现内部实现术语或后台管理语气。

## 7. 关联需求

- `REQ-0047-product-list-common-component-application`：如收藏对象为 SKU/商品，列表可复用商品卡片或列表能力。
- `REQ-0044-miniapp-sku-detail-page`：收藏列表项可跳转到 SKU 详情页。
- `REQ-0045-category-list-page`：空状态或探索入口可引导用户回到分类浏览。
- `REQ-0046-search-component-application`：空状态或探索入口可引导用户通过搜索查找商品。

## 8. 待确认问题

- [ ] 收藏列表首期覆盖 Web 展示端、微信小程序，还是仅微信小程序。
- [ ] 收藏对象首期是 SKU、商品、品牌、分类，还是多对象混合。
- [ ] 收藏能力是否必须登录；访客收藏是否允许本地保存。
- [ ] 页面入口位于个人中心、底部导航、首页快捷入口，还是多个入口。
- [ ] 是否需要收藏数量、筛选、搜索、排序或批量管理。
- [ ] 下架/删除对象是否保留灰态记录，还是自动从列表隐藏。
