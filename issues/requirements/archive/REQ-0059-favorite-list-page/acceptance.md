# 验收标准

## 功能 AC

- [ ] AC-001 收藏列表页存在明确入口，入口位置与首期端范围一致。
- [ ] AC-002 用户进入收藏列表后能看到页面标题和收藏列表上下文。
- [ ] AC-003 首屏加载期间展示与最终列表布局一致的加载态或骨架屏。
- [ ] AC-004 列表项展示主图或占位图、名称、对象类型和关键辅助信息。
- [ ] AC-005 首期若仅支持 SKU / 商品收藏，列表项必须按 SKU / 商品语义展示并进入对应详情页。
- [ ] AC-006 首期若支持多对象收藏，列表项必须展示对象类型标签，并按类型进入对应详情或列表页。
- [ ] AC-007 收藏对象下架、删除或不可访问时，列表使用灰态、隐藏或提示策略稳定降级，不导致整页不可用。
- [ ] AC-008 用户点击收藏项后进入对应详情页，跳转参数完整且可追溯。
- [ ] AC-009 收藏列表提供取消收藏或移除入口。
- [ ] AC-010 取消收藏成功后列表即时更新，并给出轻量成功反馈。
- [ ] AC-011 取消收藏失败时保留原收藏项，并提示用户重试。
- [ ] AC-012 无收藏内容时展示空状态，并提供去首页、分类或搜索的引导入口。
- [ ] AC-013 未登录状态不得误展示为空收藏，必须展示登录引导或访客态说明。
- [ ] AC-014 网络异常时展示错误状态和重试入口。
- [ ] AC-015 收藏数量较多时支持分页、上拉加载更多或等价增量加载。
- [ ] AC-016 加载更多失败时保留已加载内容，不清空列表。
- [ ] AC-017 详情页收藏状态变化后，再进入收藏列表时能反映最新收藏状态。
- [ ] AC-018 收藏列表取消收藏后，返回详情页时应同步最新收藏状态或触发刷新。
- [ ] AC-019 v1 不支持批量取消、收藏分组、分享收藏夹、购物车、询价单和在线下单。
- [ ] AC-020 埋点记录收藏列表浏览、收藏项点击、取消收藏、空状态行动点击和加载失败事件。
- [ ] AC-021 埋点公共参数包含 terminal、objectType、objectId、index、sourcePage、hasLogin、resultCount 和 requestId。
- [ ] AC-022 日志不得记录手机号、地址、客户姓名等与收藏浏览无关的个人敏感信息。

## UI AC

- [ ] AC-UI-001 页面视觉与现有用户侧深色高端展示体系保持一致。
- [ ] AC-UI-002 列表项优先复用现有商品卡片或对象卡片能力，减少与商品列表页视觉分歧。
- [ ] AC-UI-003 主图区域使用稳定比例，图片加载前后无明显布局跳动。
- [ ] AC-UI-004 取消收藏操作清晰可见，但不得抢占商品浏览主信息。
- [ ] AC-UI-005 空状态提供明确行动入口，文案不使用后台管理语气。
- [ ] AC-UI-006 未登录、无收藏、网络异常和对象失效状态使用不同文案和视觉层级。
- [ ] AC-UI-007 触控端可点击区域不小于 44px。
- [ ] AC-UI-008 若首期覆盖微信小程序，收藏列表顶部布局需按 `docs/knowledge-base/best-practices/miniapp-custom-navigation.md` 验收状态栏、胶囊 reserve、返回兜底和页面 offset。
- [ ] AC-UI-009 实现验收优先以 `prototype/web/prototype.html` 与 `prototype/web/context.md` 为视觉和交互参考；PNG 待后续设计导出。

## 横切 AC（knowledge-base）

无横切 AC。本需求为用户侧收藏列表页，不命中 `admin-list`、`admin-form`、`admin-modal`、`media-upload` 标签；管理端列表、表单、弹窗、上传相关 best-practices 不适用。
