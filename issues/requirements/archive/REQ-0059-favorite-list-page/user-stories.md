# 用户故事

## US-001 用户进入收藏列表

作为装修客户、设计师或门店导购，我希望从明确入口进入收藏列表，以便集中查看之前保存的候选内容。

验收要点：

- 收藏列表入口在产品确认的位置展示，例如个人中心、底部导航或首页快捷入口。
- 用户进入页面后能看到页面标题、收藏数量或等价上下文提示。
- 页面首屏加载期间展示与最终布局一致的加载态。
- 未登录或访客态策略明确，不把未登录态误展示为空收藏。

## US-002 用户浏览收藏内容

作为正在选砖的用户，我希望收藏列表展示足够的对象信息，以便快速判断是否继续查看详情。

验收要点：

- 收藏列表项展示主图或占位图、名称、对象类型和关键辅助信息。
- 如果首期仅支持 SKU 或商品收藏，列表文案和跳转均按 SKU / 商品语义展示。
- 如果支持多对象收藏，列表项必须展示类型标签，并跳转到对应详情页。
- 下架、删除或不可访问对象使用灰态或受控提示，不导致整页异常。

## US-003 用户从收藏进入详情

作为用户，我希望点击收藏项后进入对应详情，以便继续查看图片、规格、品牌和其他展示信息。

验收要点：

- 整个收藏列表项或明确主区域可点击。
- SKU / 商品收藏项进入 SKU 或商品详情页。
- 品牌、分类等非商品对象若被纳入首期，必须进入对应详情或列表页。
- 跳转参数完整，不能出现收藏项可见但无法打开目标页的状态。

## US-004 用户取消收藏

作为用户，我希望可以在收藏列表中取消收藏，以便整理不再关注的候选内容。

验收要点：

- 收藏列表项提供取消收藏或移除入口。
- 取消收藏成功后列表即时更新，并给出轻量反馈。
- 取消收藏失败时保留原收藏项，并提示用户重试。
- v1 不要求批量取消、全选或收藏分组。

## US-005 用户处理空状态和异常状态

作为用户，我希望在没有收藏或网络异常时看到清晰提示，以便知道下一步怎么做。

验收要点：

- 无收藏时展示空状态，并提供去首页、分类或搜索的行动入口。
- 网络异常时展示错误状态和重试入口。
- 加载更多失败时保留已加载内容，不清空页面。
- 未登录、无收藏、网络错误和对象失效状态使用不同文案。

## US-006 产品团队追踪收藏列表行为

作为产品运营人员，我希望追踪收藏列表浏览和操作行为，以便评估收藏能力是否帮助用户回访与选砖。

验收要点：

- 记录收藏列表浏览、收藏项点击、取消收藏、空状态行动点击和加载失败事件。
- 埋点参数包含 terminal、objectType、objectId、index、sourcePage、hasLogin、resultCount 和 requestId。
- 不记录手机号、地址、客户姓名等与收藏浏览无关的个人敏感信息。
