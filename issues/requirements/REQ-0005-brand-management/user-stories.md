## US-001 后台运营维护瓷砖品牌主数据

**作为** 后台运营人员，  
**我希望** 在管理后台统一维护瓷砖品牌名称、Logo、排序与启停状态，  
**以便** SKU 录入、前台展示与搜索筛选使用一致的品牌主数据。

### 验收要点

- 可访问 `/admin/brands`（或等价路由），Sidebar OPERATIONS 下「瓷砖品牌」高亮。
- 支持关键词模糊搜索（品牌名称、简称、英文名称）与状态筛选。
- 展示品牌总数、启用、停用、未关联 SKU 四个指标卡。
- 列表展示 Logo、名称、简称、英文名、排序、SKU 数量、状态、更新时间。
- 可新增/编辑品牌；弹窗字段顺序与校验符合 PRD；创建后默认启用（弹窗不展示状态）。
- 可启用/停用品牌；删除仅当 SKU 数量 = 0 且状态 = 停用时可用。
- 分页支持跳页与每页 20/50/100 切换；无导出、无批量操作。

### 关联功能

- FR-001：品牌列表与筛选
- FR-002：数据概览指标
- FR-003：新增/编辑品牌弹窗
- FR-004：启用/停用
- FR-005：条件删除
- FR-006：分页与每页显示数

## US-002 后台管理员监管品牌主数据

**作为** 后台管理员，  
**我希望** 与运营人员一样维护品牌，并在权限体系下受控访问品牌 API，  
**以便** 主数据质量可控、误删风险可防范。

### 验收要点

- 管理员与运营均可进入品牌管理（具体 RBAC 能力点见 requirement.md §14）。
- 删除、名称唯一、排序校验前后端双重校验。
- 非法删除返回 `BRAND_DELETE_FORBIDDEN`；名称重复返回 `BRAND_NAME_DUPLICATED`。

### 关联功能

- FR-005：条件删除
- FR-007：权限与 API 鉴权

---

## US-003 前台展示依赖准确品牌数据（受益）

**作为** 店主或前台访客，  
**我希望** 前台商品与筛选仅展示启用状态的品牌，  
**以便** 浏览体验准确、不出现已停用品牌。

### 验收要点

- 停用品牌不在前台品牌筛选/展示中出现（本期可实现为数据层预留，前台页面可后续迭代）。
- 品牌排序影响前台展示顺序（与 SKU/目录能力衔接）。

### 关联功能

- FR-004：启用/停用（下游消费）
