# REQ-0005 管理后台 - 瓷砖品牌管理

## 1. 需求背景
瓷砖品牌是瓷砖 SKU、前台商品展示、搜索筛选、品牌故事展示的重要主数据。本页面基于 admin-home V5 后台框架延展，保持 TILESFST 管理后台暗色旗舰风、固定 Sidebar、右侧独立滚动、品牌金激活态、极细分割线、低圆角工业风等视觉与交互一致性。

## 2. 本版本优化点
| 序号 | 优化项 | 说明 |
|---|---|---|
| 1 | 删除批量操作 | 页面不提供批量启用、批量停用、批量删除 |
| 2 | 删除导出功能 | 页面不出现导出按钮、导出入口、导出状态 |
| 3 | 删除标题行 | 删除“品牌列表”和“品牌检索”标题行 |
| 4 | 删除条件收敛 | 仅当 SKU数量=0 且 状态=停用 时允许删除 |
| 5 | 分页增强 | 页码跳转右侧新增“每页显示数” |
| 6 | 弹窗字段重排 | 按指定字段顺序布局 |
| 7 | 弹窗去冗余 | 不显示创建默认状态信息、字段规则说明、状态字段、国家/地区 |

## 3. 页面范围
包含：品牌列表、关键词搜索、状态筛选、新增品牌、编辑品牌、启用、停用、删除、分页、页码跳转、每页显示数、Logo上传区域、弹窗滚动容器。

不包含：导出、批量操作、国家/地区、弹窗状态字段、品牌合并、多语言、SEO、前台预览。

## 4. 页面结构
```text
admin-shell
├── sidebar：固定 264px，瓷砖品牌高亮
└── main-content
    ├── page-header：标题、说明、新增品牌
    ├── metric-grid：品牌总数/启用品牌/停用品牌/未关联SKU
    ├── filter-card：关键词、状态、查询、重置
    ├── brand-table-card：品牌列表
    └── pagination：页码、跳页、每页显示数
```

## 5. 页面头部
展示页面标题“瓷砖品牌”、页面说明、主按钮“新增品牌”。禁止出现导出按钮和批量操作入口。

## 6. 数据概览
| 指标 | 说明 |
|---|---|
| 品牌总数 | 当前系统全部品牌数量 |
| 启用品牌 | 状态为启用的品牌数量 |
| 停用品牌 | 状态为停用的品牌数量 |
| 未关联 SKU | SKU 数量为 0 的品牌数量 |

## 7. 搜索与筛选
筛选区不展示“品牌检索”标题。
| 字段 | 控件 | 说明 |
|---|---|---|
| 关键词 | 输入框 | 支持品牌名称、品牌简称、英文名称模糊搜索 |
| 状态 | Select | 全部、启用、停用 |
| 查询 | Button | 执行筛选 |
| 重置 | Ghost Button | 清空筛选条件 |

## 8. 品牌列表
品牌列表不展示“品牌列表”标题。
| 字段 | 说明 |
|---|---|
| Logo | 品牌 Logo 缩略图；无 Logo 时展示品牌首字母占位 |
| 品牌名称 | 中文名称，必须唯一 |
| 品牌简称 | 品牌简称 |
| 英文名称 | 英文名称 |
| 品牌排序 | 正整数，数字越小越靠前 |
| SKU 数量 | 已关联 SKU 数量 |
| 状态 | 启用 / 停用 |
| 更新时间 | 最近一次维护时间 |
| 操作 | 编辑、启用/停用、删除 |

## 9. 删除规则
删除按钮必须按以下规则控制：
```text
允许删除 = SKU数量 = 0 AND 状态 = 停用
```
| SKU 数量 | 状态 | 删除按钮 |
|---:|---|---|
| 0 | 停用 | 可点击 |
| 0 | 启用 | 置灰 |
| >0 | 停用 | 置灰 |
| >0 | 启用 | 置灰 |

置灰 hover 提示：`仅允许删除未关联SKU且已停用的品牌`。
服务端必须二次校验 `brand.sku_count == 0 AND brand.status == DISABLED`，否则返回：
```json
{"code":"BRAND_DELETE_FORBIDDEN","message":"仅允许删除未关联SKU且已停用的品牌"}
```
删除确认弹窗标题为“删除品牌”，按钮为“取消”和“删除品牌”。

## 10. 分页
分页底部结构：`共 128 条 / 上一页 / 页码 / 下一页 / 跳至页码 / 每页显示 20、50、100`。切换每页显示数后，当前页重置为第 1 页，并保留搜索筛选条件。

## 11. 新增/编辑品牌弹窗
弹窗宽度 720px，最大高度 `calc(100vh - 96px)`。头部和底部固定，主体超出时滚动。

字段排序：
```text
第一行：品牌名称、品牌排序
第二行：品牌简称、英文名称
第三行：品牌Logo
第四行：品牌介绍，宽度同品牌Logo
```

字段：
| 字段 | 必填 | 控件 | 规则 |
|---|---|---|---|
| 品牌名称 | 是 | Input | 必须唯一，最大 50 字 |
| 品牌排序 | 是 | Input Number | 正整数 |
| 品牌简称 | 否 | Input | 最大 30 字 |
| 英文名称 | 否 | Input | 最大 80 字 |
| 品牌 Logo | 否 | Upload | JPG/PNG/WebP，建议 1:1 |
| 品牌介绍 | 否 | Textarea | 最大 500 字 |

弹窗中不得出现：状态字段、创建默认状态信息、字段规则说明区块、国家/地区字段。业务规则：品牌创建后默认状态为启用，但不在弹窗 UI 展示提示。

## 12. 校验规则
品牌名称必须唯一。触发时机：blur、点击保存、服务端保存。错误文案：`品牌名称已存在，请更换`。服务端错误：`BRAND_NAME_DUPLICATED`。
品牌排序必填且必须为正整数，不允许 0、负数、小数、非数字。错误文案：`请输入正整数`。

## 13. 接口建议
- `GET /admin/brands`：keyword、status、page、page_size。
- `POST /admin/brands`：name、sort_order、short_name、english_name、logo_url、description；创建默认 ENABLED。
- `PUT /admin/brands/{brand_id}`：编辑品牌。
- `POST /admin/brands/{brand_id}/enable`：启用。
- `POST /admin/brands/{brand_id}/disable`：停用。
- `DELETE /admin/brands/{brand_id}`：仅允许 sku_count=0 且 status=DISABLED。

## 14. 权限点
| 权限点 | 说明 |
|---|---|
| brand:list | 查看品牌列表 |
| brand:create | 新增品牌 |
| brand:update | 编辑品牌 |
| brand:enable | 启用品牌 |
| brand:disable | 停用品牌 |
| brand:delete | 删除品牌 |

## 15. 验收标准
- 无导出、无批量操作、无“品牌列表/品牌检索”标题行。
- 左侧 Sidebar 与后台首页一致，瓷砖品牌高亮。
- 删除仅在 SKU数量=0 且状态=停用时可点击，其余置灰并展示提示。
- 分页右侧有每页显示数 20/50/100。
- 弹窗字段顺序完全符合要求。
- 品牌名称、品牌排序必填；品牌排序只能填写正整数。
- 品牌介绍宽度与品牌Logo上传区域一致。
- 弹窗无状态字段、无创建默认状态提示、无字段规则说明、无国家/地区。
- 弹窗主体超高可滚动，头部和底部固定。
