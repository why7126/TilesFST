# User Stories

## US-001 后端开发统一处理框架校验错误

作为后端开发，我希望 FastAPI / Pydantic 在请求进入业务逻辑前抛出的校验错误能被统一转换为项目 envelope，而不是在每个路由内重复处理。

**价值：**

- 减少管理端表单 API 的重复补丁。
- 保持 `AppError` 与框架校验错误边界清晰。
- 让后端测试可以使用统一断言验证参数错误。

**验收要点：**

- `RequestValidationError` 或等价校验异常返回 `{ code, message, data }`。
- 响应体不得只包含 FastAPI 默认 `detail`。
- 既有业务错误码和 HTTP 状态不被通用 handler 覆盖。

## US-002 前端开发稳定解析表单错误

作为 Web 前端开发，我希望管理端表单接口校验失败时总能读取 envelope 的 `message`，并在存在字段级错误时读取 `data.errors[]`，从而减少页面级兜底分支。

**价值：**

- 表单、弹窗、上传控件可以共享错误解析策略。
- Orval 类型不再把默认 `HTTPValidationError.detail` 作为唯一契约。
- 历史业务错误提示继续兼容。

**验收要点：**

- 前端错误解析优先读取 envelope `message`。
- `data.errors[]` 可映射字段错误，无法映射时安全降级为全局错误。
- 前端不得依赖裸 `detail[0].msg` 作为唯一错误来源。

## US-003 管理员获得明确中文错误提示

作为企业内部管理员或运营人员，我希望创建、编辑、提交、上传失败时看到清晰中文提示，而不是技术结构或“操作失败”。

**价值：**

- 降低表单提交失败后的理解成本。
- 减少重复提交与人工排障沟通。
- 上传缺文件、类型错误等问题能被快速定位。

**验收要点：**

- 参数错误展示可理解中文文案。
- 上传、弹窗、普通表单的失败反馈不造成布局明显跳动。
- 敏感字段不出现在用户可见错误文案中。

## US-004 QA 使用一致断言覆盖管理端表单 API

作为 QA，我希望用户、品牌、类目、SKU、规格、Banner、系统设置、个人资料、密码和上传接口的校验失败可以用统一结构断言。

**价值：**

- 降低回归测试维护成本。
- 提升 API 治理类改动的覆盖面。
- 能明确区分框架校验错误与业务错误。

**验收要点：**

- 后端测试覆盖 JSON body、路径 / 查询参数、multipart 参数校验失败。
- 前端测试覆盖 envelope message、字段级错误和兜底展示。
- `AppError` 保持原错误码、状态码和文案。

## US-005 运维和审计按统一信号定位参数错误

作为运维或排障人员，我希望参数校验失败继续作为 4xx 请求进入请求日志，并可按状态码、错误码和 request_id 定位。

**价值：**

- 保留 422 参数错误的排障语义。
- 避免运行时与文档、日志筛选规则漂移。
- 方便关联前端反馈与后端请求日志。

**验收要点：**

- 推荐保留 HTTP 422，响应体使用统一 envelope。
- 如改为 HTTP 400，必须同步日志筛选、文档和测试。
- request_id 等现有请求日志字段不受影响。

## US-006 安全负责人确认错误输出不泄露敏感信息

作为安全负责人，我希望字段级错误结构不包含密码、token、Authorization、MinIO 密钥、数据库连接串、真实文件路径或完整对象 key。

**价值：**

- 避免把原始输入或内部路径透出到 API 响应。
- 降低上传、密码、系统设置等敏感接口的误泄露风险。
- 保持错误治理与安全规范一致。

**验收要点：**

- `data.errors[]` 不返回原始敏感输入。
- 仅返回字段路径、归一化文案、错误类型和必要 location。
- 安全相关负例进入后端测试或检查清单。

## US-007 上传表单用户获得一致失败反馈

作为使用管理端上传控件的管理员，我希望缺文件、文件字段非法等框架校验失败和业务校验失败都能以一致方式展示。

**价值：**

- 上传链路不再因为 multipart 校验路径不同而显示通用失败。
- 不破坏已有“上传中 / 成功 / 失败”状态切换。
- 为品牌 Logo、Banner、SKU 图片和视频上传提供一致错误契约。

**验收要点：**

- 缺少 `file` 或文件参数非法时返回统一 envelope。
- 文件类型不允许等业务错误继续保留领域错误码。
- 失败展示不破坏同会话成功预览和上传控件状态机。
