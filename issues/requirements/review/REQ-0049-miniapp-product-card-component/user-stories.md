# 用户故事

## US-001 浏览商品核心信息

作为装修客户，我希望在商品列表中快速看到商品图片、名称、品牌、规格和参考价格，以便判断是否继续查看详情。

验收要点：

- 商品卡片展示主图、SKU 名称、品牌、规格和参考价格。
- SKU 名称最多两行，超出后省略。
- 价格使用“参考价格”语义，不表达在线成交承诺。
- 字段缺失时展示统一占位，不出现空白、`null`、`undefined` 或破坏布局的异常值。

## US-002 从卡片进入 SKU 详情

作为设计师，我希望点击任意商品卡片即可进入对应 SKU 详情页，以便查看完整图片、参数和推荐信息。

验收要点：

- 整张商品卡片都是可点击区域。
- 跳转携带 `skuId`，并保留来源上下文。
- 连续点击不会重复打开多个详情页。
- 缺少 `skuId` 或商品不可查看时不跳转，并展示可理解反馈。

## US-003 在不同列表页复用商品卡片

作为产品维护者，我希望分类列表、搜索结果、首页推荐等列表页复用同一商品卡片组件，以便保持展示一致并降低维护成本。

验收要点：

- 分类商品列表、搜索结果 SKU 列表和首页推荐可使用同一组件。
- 不同列表页可传入来源、曝光位置和展示密度。
- 列表容器负责分页、筛选和加载状态，卡片组件不重复维护列表状态机。
- 核心字段语义在不同列表页保持一致。

## US-004 处理图片和字段异常

作为普通访客，我希望单个商品图片加载失败或字段缺失时，仍能继续浏览列表并进入可用商品详情。

验收要点：

- 图片加载失败时展示统一占位。
- 价格、品牌、规格缺失时使用统一兜底文案。
- 单个卡片异常不影响其他商品卡片渲染。
- 商品不可查看时卡片呈现不可用状态，避免误导点击。

## US-005 记录卡片曝光与点击

作为产品运营人员，我希望商品卡片可以记录曝光、点击和异常数据，以便后续分析商品列表效果。

验收要点：

- 卡片支持曝光事件。
- 卡片支持点击事件。
- 图片失败和不可用点击可被记录。
- 埋点参数包含 SKU、来源页面、来源模块、列表上下文和位置序号。
