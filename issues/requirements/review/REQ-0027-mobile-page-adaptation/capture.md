# 一句话

新增移动端页面适配能力，让 Web 页面在手机、小屏平板等移动视口下具备可浏览、可操作、布局不溢出的基础体验。

# 原始描述

新增移动端页面的适配

# 背景与关联

- 涉及端：店主 Web 展示端、Web 管理端待确认；微信小程序是否纳入同一需求待确认。
- 业务价值：提升移动设备访问时的可用性，降低页面横向溢出、表格/筛选控件不可操作、图片和详情内容阅读困难等体验风险。
- 预期后续：需要明确适配页面清单、移动端断点、核心任务路径、视觉验收基准和是否需要单独的小程序适配策略。

# 待澄清

- [ ] 本需求覆盖哪些页面：店主端首页、商品列表、商品详情、管理端登录页、管理端列表/表单，还是全站页面。
- [ ] 移动端断点是否沿用现有 Design System 与 Tailwind 断点，或需要补充新的页面级响应式规则。
- [ ] 管理端在手机端是否只要求基础可用，还是需要完整移动工作流。
- [ ] 小程序页面适配是否纳入本 REQ，还是单独创建小程序专项需求。
- [ ] 是否需要为重点页面提供 prototype/PNG 作为移动端 Golden Reference。
