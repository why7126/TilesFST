# 用户故事

## US-001 用户从一级分类进入聚合商品列表

作为装修客户或普通访客，我希望在分类页点击当前一级分类的商品入口后，看到该一级分类下所有二级分类商品的聚合结果，以便先从大类浏览候选商品。

验收要点：

- 一级分类商品入口清晰可见，且不影响左侧一级分类切换。
- 进入商品列表页时携带 `categoryId`、`categoryName`、`categoryLevel=primary` 和 `sourcePage=category`。
- 商品列表展示该一级分类下所有启用二级分类商品的聚合结果。
- 聚合结果为空时展示分类无商品空状态，不白屏、不自动跳转其他分类。

## US-002 用户从二级分类进入精确商品列表

作为已明确品类方向的用户，我希望点击二级分类后只浏览该二级分类下的商品，以便快速缩小选择范围。

验收要点：

- 二级分类卡片整卡可点击。
- 进入商品列表页时携带 `categoryId`、`categoryName`、`categoryLevel=secondary` 和 `sourcePage=category`。
- 商品列表仅展示当前二级分类下的商品。
- 二级分类无商品时展示分类无商品空状态。

## US-003 用户在商品列表中理解分类上下文

作为正在浏览商品的用户，我希望商品列表页标题、搜索提示和筛选范围能反映一级或二级分类上下文，以便知道当前列表来源。

验收要点：

- 一级分类列表页标题使用一级分类名称。
- 二级分类列表页标题使用二级分类名称。
- 筛选项基于当前分类层级的结果范围提供，不展示明显无关的分类筛选。
- 空状态和错误提示使用分类上下文文案。

## US-004 用户返回分类页继续浏览

作为连续浏览分类的用户，我希望从商品列表或详情返回分类页后，分类页保持原有位置，以便继续查看其它分类。

验收要点：

- 返回分类页后保持当前一级分类选中态。
- 返回分类页后保持左侧一级分类滚动位置。
- 返回分类页后保持右侧二级分类滚动位置。
- 返回流程不清空分类页缓存数据。

## US-005 产品团队追踪分类入口效果

作为产品运营人员，我希望区分一级分类聚合入口和二级分类精确入口的点击与列表浏览行为，以便分析用户找砖路径。

验收要点：

- 一级分类商品入口点击有独立埋点或在既有一级分类点击事件中携带进入商品列表语义。
- 二级分类点击埋点继续保留，并补充商品列表跳转上下文。
- 商品列表页浏览埋点包含 `categoryId`、`categoryName`、`categoryLevel`、`sourcePage`、`resultCount` 和 `requestId`。
- 埋点不得记录手机号等与商品浏览无关的个人敏感信息。
