# 瓷砖类目管理 - 新增类目弹窗原型上下文

## 文件

- HTML：`prototype/web/tile-category-management-add.html`
- 原型图：`prototype/images/tile-category-management-add.png`

## 目标

指导 AI Coding 精准还原「新增类目」弹窗状态。该文件与列表页分离，便于开发按页面状态验收。

## 弹窗触发

在列表页点击顶部主按钮「＋ 新增类目」打开弹窗。弹窗为居中 Modal，背景列表页保留并添加暗色遮罩。

## 弹窗结构

- 宽度：560px。
- 头部：标题「新增类目」、关闭按钮。
- 表单：一行一个信息，从上到下依次为上级类目、类目名称、类目编码、排序权重、类目描述、状态。
- 底部：取消、保存类目。

## 字段规则

- 上级类目：为空时创建一级类目；选择上级后自动确定层级。
- 类目名称：必填。
- 类目编码：必填、唯一，推荐 `CAT-XXXX`。
- 排序权重：必填正整数，数字越小越靠前。
- 类目描述：选填。
- 状态：默认启用。

## 交互规则

- 点击取消或关闭按钮返回列表页，不保存。
- 点击保存类目后校验必填项，成功后关闭弹窗并刷新类目树、列表和数据概览。
- 当前最多三级类目，三级类目不能作为上级继续新增子级。

## 视觉一致性

Modal 使用卡片底色、0.5px 细边框、3px 圆角、暗色半透明遮罩和 blur。按钮、输入框、下拉框必须与列表页保持一致。
