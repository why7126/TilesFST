## US-001 后台运营维护瓷砖类目树

**作为** 后台运营人员，  
**我希望** 在管理后台维护最多三级的瓷砖类目、排序与启停状态，并通过类目树快速定位子类目，  
**以便** SKU 录入、前台目录与筛选导航使用一致的类目主数据。

### 验收要点

- 可访问 `/admin/tile-categories`（或等价路由），Sidebar OPERATIONS 下「瓷砖类目」高亮。
- 展示类目总数、启用类目、绑定 SKU、最大层级四个指标卡。
- 支持按名称/编码、状态、层级筛选；重置恢复默认条件。
- 左侧类目树展示层级与 SKU 数量；点击节点联动右侧列表。
- 可新增类目（弹窗一行一字段）；可编辑、启用/停用。
- 删除仅当 SKU 数量 = 0 且状态 = 停用时可用。
- 列表工具栏仅有「调整排序」；无导出、无批量操作。
- 分页支持每页 10/20/50。

### 关联功能

- FR-001：数据概览指标
- FR-002：类目检索与重置
- FR-003：类目树与列表联动
- FR-004：新增/编辑类目弹窗
- FR-005：启用/停用
- FR-006：条件删除
- FR-007：分页与每页显示数
- FR-008：调整排序入口

## US-002 后台管理员监管类目主数据

**作为** 后台管理员，  
**我希望** 类目编码唯一、层级深度受控，删除与写操作有服务端二次校验，  
**以便** 避免误删有 SKU 绑定的类目或破坏目录结构。

### 验收要点

- 具备 category:* 权限点（或 admin/employee 角色默认策略）。
- 三级类目不可再挂子级；非法创建返回明确错误码。
- 非法删除返回 `CATEGORY_DELETE_FORBIDDEN`；编码重复返回 `CATEGORY_CODE_DUPLICATED`。

### 关联功能

- FR-006：条件删除
- FR-009：层级深度校验
- FR-010：权限与 API 鉴权

---

## US-003 前台目录依赖准确类目数据（受益）

**作为** 店主或前台访客，  
**我希望** 前台仅展示启用状态的类目，并按排序权重呈现目录结构，  
**以便** 浏览路径清晰、不出现已停用类目。

### 验收要点

- 停用类目不在前台目录/筛选中出现（可与 SKU/目录能力后续衔接）。
- 类目排序影响同级展示顺序。

### 关联功能

- FR-005：启用/停用（下游消费）
