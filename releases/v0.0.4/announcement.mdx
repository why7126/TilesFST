# 产品版本 v0.0.4

发布时间：2026-07-20 23:39:49

关联 Sprint：sprint-009

## 新增功能

- 微信小程序商品浏览体验升级：统一商品卡片、商品列表双列卡片展示、分类入口到商品列表的跳转语义，并保留分页与入口上下文。
- 微信小程序品牌体验升级：新增品牌卡片、品牌列表页、品牌详情页入口能力，并沉淀自定义导航 best-practice。
- 微信小程序证书与收藏体验升级：新增公开证书列表页、证书预览路径和收藏列表页，方便用户集中查看证书与已收藏对象。
- 微信小程序添加引导：新增添加到我的小程序轻量引导，支持关闭状态和右上角入口附近展示。
- 小程序验收治理：新增 DevTools/真机验收 evidence 模板，明确自动化、静态视口、DevTools 和真机证据边界。
- 管理后台 Banner 治理：收敛 Banner 投放范围为小程序首页轮播和品牌列表页轮播，降低历史运营位混用风险。

## 修复 BUG

- 修复搜索组件与原型在搜索首页、联想、结果、筛选和无结果状态上的体验偏差。
- 修复首页新品榜、热销榜和推荐模块查看更多入口误跳搜索页的问题。
- 补齐小程序首页设备验收残留，避免把自动化侧证误写为真机或 DevTools 通过。
- 修复 SKU 商品详情页轮播视频 URL 生成字段错误，确保视频项可展示和播放。

## 发布注意事项

- 本版本发布范围来自 `sprint-009`，18 个 OpenSpec Change 已归档。
- 用户可见产品版本已更新为 `v0.0.4`。
- 小程序 DevTools 与真机验收必须按 evidence 模板区分证据类型，不得用静态测试替代真机结论。
- Banner 投放范围收敛后，仅应保留小程序首页轮播与品牌列表页轮播两个正式展示位置。

## 已知问题

- 暂无已知问题。

## 升级步骤

1. 完成升级前备份，确认小程序静态资源、媒体访问和 Banner 数据在目标环境可用。
2. 部署后端、管理端 Web 与微信小程序相关资源。
3. 执行后端、前端、OpenAPI/Orval、Docker Compose 和公告静态校验。
4. 完成小程序核心路径 smoke：首页、搜索、分类进入商品列表、商品列表、SKU 详情视频、品牌列表、品牌详情、证书列表、收藏列表和添加引导。

## 回滚说明

1. 回退到 v0.0.3 对应部署包或上一 Git tag。
2. 恢复上一版本小程序资源、Web 静态资源和后端服务。
3. 如 Banner 投放范围或数据库结构变更已应用，按迁移回滚说明恢复业务数据与 schema。
4. 复跑 smoke 与回归验证，确认首页、SKU 详情、品牌与证书入口恢复。
