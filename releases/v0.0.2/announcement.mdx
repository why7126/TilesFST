# TILESFST v0.0.2

发布时间：2026-07-04 08:19:15

关联 Sprint：sprint-004

## 发布摘要

v0.0.2 汇总 Sprint 004 的正式交付范围。本版本面向生产部署、管理端治理与运维可观测性：补齐 MySQL 生产部署路径、管理员保护策略、接口文档与 Swagger 深链、产品版本发布治理、日志审计能力，并统一管理端多列表页体验。

## 新增功能

- 新增生产环境部署与 MySQL 数据库支持，支持 `APP_ENV=production` 下连接客户已有 MySQL 8.0+，并保留本地 SQLite 默认体验。
- 新增管理端超级管理员账号保护，避免保底管理员被编辑、重置密码、冻结、删除或默认策略下本人改密。
- 新增管理端接口文档菜单，展示系统接口目录、OpenAPI 纳入状态、Swagger 入口和 Orval 方法映射。
- 新增接口文档列表行级「查看」入口，可打开 Swagger UI 并定位到具体 operation。
- 新增品牌区 Logo、`菲尚特FST` 文案、产品版本号与浏览器图标统一展示。
- 新增产品版本发布与公告管理，建立 `releases/` 发布对象、Mintlify 公告源文件和发布前门禁。
- 新增产品使用行为埋点与接口请求日志详情，支持日志审计列表、筛选、详情抽屉、request_id 复制和敏感字段脱敏。

## 修复 BUG

- 修复创建用户时用户名长度不足等校验失败只展示兜底错误的问题。
- 修复接口文档页 Swagger UI 入口跳转到 Web 首页的问题。
- 修复接口文档摘要指标卡与瓷砖 SKU 页视觉层级不一致的问题。
- 修复接口文档列表冗余标题与分页交互未对齐 SKU 页的问题。
- 修复管理端全局右侧内容区域内边距过大、内容宽度受限的问题。
- 修复瓷砖 SKU、品牌、类目、规格、Banner、用户、日志审计和接口文档等多列表页布局、筛选区、操作列与分页不统一的问题。

## 发布注意事项

- 本文件已完成 `/release-prepare v0.0.2`，发布确认前需保留门禁证据与公告源文件。
- `src/shared/product-version.ts` 的 `PRODUCT_VERSION` 已更新为 `v0.0.2`。
- 本版本包含 API 变更，OpenAPI 与 Orval 客户端已重新生成。
- 本版本包含 MySQL baseline、生产 Compose、Swagger Web 代理和日志存储能力，已完成本地门禁校验；目标生产环境仍需按真实网络和权限执行 smoke。
- 日志详情与接口文档不得泄露密码、Token、Cookie、数据库连接串、对象存储凭据或真实客户数据。
- 生产 Swagger 仍应保持只读或禁用 Try It Out，不得因 Web 代理与行级入口放宽调试策略。

## 已知问题

- 生产 MySQL、外部 MinIO 与真实域名环境仍需按目标部署执行 smoke。

## 升级步骤

1. 备份数据库、对象存储数据和当前部署配置。
2. 确认产品可见版本号为 `v0.0.2`。
3. 同步生产环境变量，确认 MySQL、MinIO、管理员初始化、日志保留和安全脱敏配置。
4. 执行数据库初始化或迁移，确认 SQLite 与 MySQL 路径均可用。
5. 重新生成 OpenAPI/Orval 客户端并构建 Web、Backend 镜像。
6. 启动 Docker Compose 或生产等价部署，验证登录、用户管理、接口文档、Swagger 代理、日志审计、媒体读取和管理端列表关键路径。
7. 执行 Mintlify 公告 build/preview 或等价校验。

## 回滚说明

1. 停止当前版本服务，回退到上一 Git tag 或上一部署包。
2. 恢复上一版本环境变量、Docker Compose 配置和 Web/Backend 镜像。
3. 恢复升级前数据库备份；如已执行 MySQL schema 变更，按迁移回滚说明处理。
4. 恢复对象存储备份或回退 MinIO/S3 配置。
5. 重新执行登录、用户管理、接口文档、媒体读取和管理端首页 smoke 验证。
