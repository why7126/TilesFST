# TILESFST v0.0.1

发布时间：2026-07-02 16:19:39

关联 Sprint：sprint-000、sprint-001、sprint-002、sprint-003

## 发布摘要

v0.0.1 是 TILESFST 的首个产品版本，合并 4 个已归档 Sprint 的正式交付范围。版本覆盖研发治理、管理端登录、管理端工作台、用户与瓷砖主数据、媒体对象存储、Banner 管理、系统设置、个人资料与产品版本号展示。

本公告面向公开页面展示。当前已于 2026-07-02 22:07:45 完成 `/release-publish v0.0.1` 发布确认。

## 新增功能

- 建立 Design System、API Governance、Testing Governance、Docker Compose 本地开发与演示部署基础。
- 新增 Web 管理端登录，支持账号密码登录、路由守卫、退出登录、记住登录状态与密码显隐。
- 新增管理端工作台首页与可折叠 Sidebar，展示版本号 `v0.0.1`。
- 新增用户管理、品牌管理、瓷砖类目管理、瓷砖 SKU 管理。
- 新增个人资料、修改密码、最近操作记录与侧栏用户菜单头像展示。
- 新增瓷砖规格管理，SKU 表单支持选择规格。
- 新增 Banner 管理，支持列表、筛选、上下线、弹窗编辑、图片上传与 SKU 主图引用。
- 新增系统设置，覆盖基础信息、媒体上传限制、安全策略、通知开关与审计配置。
- 新增 MinIO 对象存储上传读取链路，并统一对象 key 布局。

## 修复 BUG

- 修复登录服务重启后默认管理员账号密码不可用的问题。
- 修复品牌 Logo 上传、进度反馈、媒体读取与页面布局波动问题。
- 修复对象存储未写入 MinIO、legacy uploads 残留与品牌 Logo 代理展示问题。
- 修复类目、SKU、用户、规格、Banner、系统设置等管理页的列表、分页、弹窗、确认框与保存反馈一致性问题。
- 修复 SKU 视频上传回显、上传体积限制、规格提示样式与发布操作缺失问题。
- 修复个人资料重复字段、重复保存按钮、修改密码错误字段归属与显隐按钮布局问题。
- 修复 Banner 弹窗宽度、日期选择、跳转选择、图片引用与 CSS 层叠覆盖问题。

## 发布注意事项

- 必须在正式发布前确认所有纳入范围的 OpenSpec Change 已 archive。
- 必须执行后端、前端、发布校验相关测试，并记录证据。
- 本版本包含 API 变更，必须确认 OpenAPI 与 Orval 客户端已同步。
- 本版本包含数据库迁移，发布前必须备份 SQLite 数据库。
- 本版本包含 MinIO 对象 key 布局调整，发布前必须备份 MinIO 数据并先执行 dry-run。
- 本版本包含 Docker、Nginx 上传限制与环境变量调整，必须执行 Docker Compose 验证。
- 本版本公告使用 Mintlify 静态文档生成，正式发布前必须完成 build/preview 或等价校验。

## 升级步骤

1. 备份 SQLite 数据库与 MinIO 数据。
2. 同步实际环境变量与 `.env.example`，重点确认管理员初始化、上传大小、媒体 MIME、MinIO 单桶与对象 key 前缀。
3. 执行数据库迁移。
4. 如已有媒体对象，先执行对象 key 迁移 dry-run，确认无异常后再 apply。
5. 重新生成 OpenAPI/Orval 客户端。
6. 构建并启动 Web、Backend 与 MinIO。
7. 验证登录、用户、品牌、类目、SKU、规格、Banner、系统设置、上传与媒体读取链路。
8. 发布 Mintlify 静态公告页面。

## 回滚说明

1. 停止当前版本服务，回退到上一 Git tag 或上一部署包。
2. 恢复上一版本环境变量、Docker Compose 配置和 Web/Backend 镜像。
3. 恢复升级前备份的 SQLite 数据库。
4. 如已执行对象 key 迁移，按迁移脚本回滚说明恢复旧 key 或恢复 MinIO 备份。
5. 重新执行登录、媒体读取和管理端首页 smoke 验证。
